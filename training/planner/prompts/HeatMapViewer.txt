## 1. 🔬 Tool Purpose and Scientific Applications

### Overview
**HeatMapViewer** is a browser-based, interactive visualization module within the GenePattern genomic analysis platform. It renders gene expression matrices as color-coded heat maps where intensity represents expression magnitude. Its core scientific purpose is enabling researchers to rapidly identify patterns, clusters, and outliers in multi-dimensional array-based genomic data without requiring programming expertise.

### Scientific Applications

| Application Domain | Use Case | Key Benefit |
|---|---|---|
| **Microarray Gene Expression** | Visualize expression across phenotypic classes | Pattern detection across thousands of genes |
| **Differential Expression** | Display top-ranked differentially expressed genes | Confirm ComparativeMarkerSelection results |
| **RNA-seq (post-normalization)** | Explore normalized count matrices | Cross-condition comparisons |
| **Biomarker Discovery** | Heatmap of candidate marker genes | Visual validation of statistical results |
| **Quality Control** | Exploratory data analysis of raw/preprocessed data | Sample outlier detection |
| **Class Distinction** | Visualize class separation in expression data | Confirm biological hypotheses |
| **Epigenomics/Proteomics** | Any array-based numeric matrix | Generalizable beyond transcriptomics |

### Color Encoding Model
The default diverging color scheme encodes:
- 🔴 **Red (Hot)** → Largest/highest expression values
- 🔵 **Blue (Cool)** → Smallest/lowest expression values
- ⚪ **Light colors** → Intermediate/near-mean values

This red-blue diverging scale is biologically intuitive — upregulation appears as warm red tones, downregulation as cool blue tones — and is standard across the bioinformatics community.

## 2. 📁 Input/Output Formats and Requirements

### Input File Formats

#### Primary Format: GCT (Gene Cluster Text)
```
#1.2
[num_rows]   [num_cols]
Name    Description    Sample1    Sample2    Sample3 ...
GeneA   Description1   1.23       4.56       7.89   ...
GeneB   Description2   2.34       5.67       8.90   ...
```
- **Extension:** `.gct`
- **Structure:** Tab-delimited; version header `#1.2`, then dimension row, then data matrix
- **Missing Values:** ✅ Supported (blank cells)
- **Use Case:** Standard GenePattern expression format; most commonly used input
- **Source:** Any expression analysis pipeline, preprocessing modules, RNA-seq normalization

#### Secondary Format: RES (Affymetrix Expression)
```
Description    Accession    Sample1   C1   Sample2   C2 ...
[Gene descriptions and Affy call data]
```
- **Extension:** `.res`
- **Structure:** Tab-delimited; contains **absent (A) / present (P) / marginal (M)** calls per gene per sample
- **Missing Values:** ❌ NOT supported
- **Use Case:** Affymetrix-specific microarray data with call information
- **Note:** RES format is Affymetrix-chip specific; most modern workflows use GCT

#### Tertiary Format: ODF (Output Data Format)
- **Extension:** `.odf`
- **Structure:** GenePattern-internal pipeline interchange format
- **Use Case:** Direct output from `ExtractComparativeMarkerResults`, passed into HeatMapViewer within pipeline
- **Key Pipeline Use:** `ComparativeMarkerSelection → ExtractComparativeMarkerResults (.odf/.gct) → HeatMapViewer`

#### Supporting Format: CLS (Class Labels)
- **Extension:** `.cls`
- **Required For:** Sort-by-differential-expression feature (T-Test / Signal-to-Noise sorting)
- **Structure:** Defines phenotypic class assignment for each sample column
- **Note:** Not a direct input parameter to the module launch, but used interactively within the viewer

### Output

| Output Type | Format | Limitation |
|---|---|---|
| **Interactive Display** | Browser-rendered HTML/Canvas | Primary output; requires modern browser |
| **Saved Image** | PNG / SVG | ⚠️ **Gene annotations NOT included** in exported image (confirmed GenePattern FAQ) |
| **No computational output files** | N/A | This is a visualizer only; no downstream analysis outputs |

### File Format Compatibility Matrix

```
Input Source Module          → Compatible Input Format → HeatMapViewer
─────────────────────────────────────────────────────────────────────
PreprocessDataset             → .gct, .res               ✅ Direct
ComparativeMarkerSelection    → .odf                     ✅ Direct
ExtractComparativeMarkerRes.  → .gct, .odf               ✅ Direct
HierarchicalClustering        → .gct                     ✅ Direct
RNA-seq normalization tools   → .gct                     ✅ Direct
Amazon S3 cloud storage       → .gct, .res (URL)         ✅ (v14 bug fix)
Raw Excel/CSV data            → Must convert to .gct     ⚠️ Conversion needed
```

## 3. ⚙️ Parameter Analysis and Usage Patterns

### Parameter Taxonomy

#### Group 1: Required Input Parameters
| Parameter | Type | Required | Description |
|---|---|---|---|
| `dataset` | File | ✅ Yes | Gene expression dataset (.gct, .res, .odf). The sole mandatory parameter at launch. |

#### Group 2: Color & Normalization Controls (Interactive)
| Parameter | Type | Default | Options | Biological Significance |
|---|---|---|---|---|
| `color_scheme` | Enum | `Row Relative` | `Row Relative`, `Global` | **Row Relative**: Per-gene normalization; reveals relative expression patterns within each gene. **Global**: Absolute scale across entire dataset; preserves cross-gene magnitude relationships |
| `color_gradient` | Config | Red-Blue diverging | Custom user-defined | Maps numeric range to color spectrum |
| `row_normalization` | Boolean/Enum | Off | On/Off | Normalize each row to zero-mean unit-variance before coloring |

> 🔬 **Biological Significance of Color Scheme Choice:**
> - **Row Relative** is ideal when genes have vastly different absolute expression levels and you want to highlight *patterns* (up/down relative to a gene's own mean)
> - **Global** is ideal when *absolute magnitude comparisons across genes* matter (e.g., comparing highly vs. lowly expressed gene clusters)

#### Group 3: Display Geometry Controls (Interactive)
| Parameter | Type | Default | Options |
|---|---|---|---|
| `row_size` | Integer/Enum | Medium | Small, Medium, Large (zoom control for gene rows) |
| `column_size` | Integer/Enum | Medium | Small, Medium, Large (zoom control for sample columns) |
| `grid_lines` | Boolean | On | On / Off |

#### Group 4: Annotation & Labeling (Interactive)
| Parameter | Type | Default | Description |
|---|---|---|---|
| `gene_annotations` | Boolean | On | Display gene/feature names on row axis |
| `sample_annotations` | Boolean | On | Display sample names on column axis |

#### Group 5: Sorting and Analysis Features (Interactive)
| Parameter | Type | Options | Requirement |
|---|---|---|---|
| `sort_features` | Enum | `T-Test`, `Signal To Noise` | Requires associated CLS file for class labels |

> 🔬 **Scientific Significance:**
> - **T-Test sorting**: Ranks genes by statistical significance of expression difference between classes (two-sample t-statistic)
> - **Signal-to-Noise (S2N)**: Ranks by `(µ₁ - µ₂) / (σ₁ + σ₂)` — a robust SNR metric used prominently in GSEA methodology; less sensitive to outliers than raw t-test

## 4. 🛠️ Installation and Dependency Requirements

### Architecture: Browser-Based Visualizer
HeatMapViewer is fundamentally different from computational analysis modules — it is a **client-side HTML/jQuery visualizer** that renders entirely in the user's web browser.

### Technology Stack
```
Frontend:
├── HTML5                    (page structure, Canvas rendering)
├── jQuery                   (DOM manipulation, event handling)
├── CSS3                     (styling, layout)
└── JavaScript (ES5/ES6)     (data processing, color mapping)

Backend/Server:
├── GenePattern Server       (serves the module, handles file I/O)
├── Amazon S3 (optional)     (cloud file storage, v14 bug fix)
└── No R/Python/Java runtime required for visualization

Browser Requirements:
├── Modern web browser       (Chrome, Firefox, Safari, Edge)
├── JavaScript enabled       ✅ Required
├── Canvas API support       ✅ Required (HTML5)
└── Java plugin              ❌ NOT required (key improvement over older versions)
```

### Installation Within GenePattern
- **Module Type:** `Visualizer` (distinct from `Analysis` or `Pipeline` types)
- **Packaging:** ZIP archive containing HTML, JS, CSS, manifest
- **Deployment:** Install via GenePattern Module Manager (`.zip` upload)
- **No Docker container required** for the current HTML/jQuery implementation
- **No external binary dependencies** — pure web technology

### Historical Dependency Note
Prior to the HTML/jQuery rewrite, the module required a **Java applet** (JVM). In June 2014, GenePattern released updated visualizers (including HeatMapViewer) to maintain compatibility with Java 7. The current HTML/jQuery version eliminates this Java dependency entirely — a significant modernization and cross-browser compatibility improvement.

### S3 Integration (v14 Bug Fix)
The current version adds support for **Amazon S3 file URLs**, enabling:
- Files stored in AWS S3 buckets to be directly referenced as input
- Cloud-native genomics workflows (e.g., Terra, cloud GenePattern instances)
- No need to download/re-upload files when data is already in cloud storage

## 5. 🔄 Common Workflows and Use Cases

### Workflow 1: Differential Expression Visualization (Primary Use Case)
```
Input Files: all_aml_train.gct + all_aml_train.cls
      │
      ▼
┌─────────────────────────┐
│  ComparativeMarkerSel.  │  ← Computes per-gene class separation statistics
│  (T-Test, S2N metrics)  │
└──────────┬──────────────┘
           │ .odf output
           ▼
┌─────────────────────────────────┐
│  ExtractComparativeMarkerResult │  ← Filters top N differentially expressed genes
│  (output format: GCT)          │
└──────────┬──────────────────────┘
           │ .gct output
           ▼
┌──────────────────┐
│  HeatMapViewer   │  ← Visualize top markers; sort by T-Test or S2N
│  (color: row rel)│
└──────────────────┘
```
📚 *Documented in GenePattern Differential Expression Tutorial and PMC3893799*

### Workflow 2: Exploratory Data Analysis / Quality Control
```
Raw expression data (.cel files, FASTQ)
      │
      ▼ [external normalization]
┌──────────────────────┐
│  PreprocessDataset   │  ← Floor, ceiling, variation filter, log transform
└──────────┬───────────┘
           │ .gct
           ▼
┌──────────────────┐
│  HeatMapViewer   │  ← Visually assess preprocessing effects, sample outliers
└──────────────────┘
```

### Workflow 3: Hierarchical Clustering Visualization
```
expression_data.gct
      │
      ▼
┌──────────────────────────┐
│  HierarchicalClustering  │  ← Cluster genes and/or samples
└──────────┬───────────────┘
           │ .gct (+ optional .cdt/.atr/.gtr)
           ▼
┌──────────────────┐        ┌──────────────────────────────┐
│  HeatMapViewer   │   OR   │  HierarchicalClusteringViewer │
│  (no dendrograms)│        │  (with dendrograms)           │
└──────────────────┘        └──────────────────────────────┘
```
> **Decision rule:** Use `HierarchicalClusteringViewer` when dendrograms are needed. Use `HeatMapViewer` when a clean matrix view without tree structure is preferred.

### Workflow 4: Pipeline Terminal Step
```
[Any upstream analysis producing .gct output]
      │
      ▼
┌──────────────────┐
│  HeatMapViewer   │  ← Final pipeline visualization step (human review)
└──────────────────┘
```
In GenePattern pipelines, HeatMapViewer typically serves as a **terminal visualization node** — the last step where results are presented to the researcher for interpretation.

### Workflow 5: Cloud-Native Analysis (v14 Feature)
```
S3://my-bucket/expression-data.gct

┌──────────────────┐
```

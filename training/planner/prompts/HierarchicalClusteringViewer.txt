## 1. 🔬 Tool Purpose and Scientific Applications

### Core Purpose
`HierarchicalClusteringViewer` is a **browser-based interactive visualization module** within the GenePattern ecosystem that renders the output of hierarchical clustering algorithms. It serves as the dedicated downstream viewer for the `HierarchicalClustering` module, producing simultaneous **dendrogram** (cluster tree) and **heat map** representations of high-dimensional genomic data.

### Scientific Rationale
Hierarchical clustering is one of the oldest and most widely adopted unsupervised learning methods in genomics. It groups genes (rows) and/or samples (columns) by similarity, building a nested binary tree (dendrogram) whose structure reveals:

| Biological Insight | Visualization Feature |
|---|---|
| Sample subtype discovery (e.g., AML vs ALL) | Column dendrogram (ATR-driven) |
| Co-expressed gene modules / gene programs | Row dendrogram (GTR-driven) |
| Expression level trends across groups | Color gradient heat map |
| Cluster distances and cohesion | Dendrogram branch lengths |
| Outlier samples or genes | Long-branch nodes, isolated clusters |

### Primary Scientific Applications
1. **Cancer subtype discovery** — Identifying transcriptomic subtypes of tumors from microarray or RNA-seq data (historically used with the AML/ALL Golub et al. 1999 dataset as the canonical example)
2. **Differential expression visualization** — Displaying co-regulated gene sets across experimental conditions
3. **Quality control** — Visual inspection of replicate clustering, batch effects, or outlier samples
4. **Biomarker discovery** — Identifying gene signatures that distinguish clinical groups
5. **Pathway-level expression analysis** — Clustering genes within or across pathways
6. **Multi-omics data exploration** — Any tabular matrix data that has been hierarchically clustered

### Disciplinary Context
The tool is principally applied in:
- **Transcriptomics** (microarray, RNA-seq)
- **Proteomics / Metabolomics** (any expression matrix)
- **Epigenomics** (methylation or chromatin accessibility matrices)
- **Single-cell analysis** (when used with pre-clustered matrices)

## 2. 📂 Input/Output Formats and Requirements

### Input Files

The module consumes **three specialized Stanford-format cluster files**, all produced by the upstream `HierarchicalClustering` GenePattern module:

#### 🗂️ CDT File (Clustered Data Table) — **REQUIRED**
| Property | Details |
|---|---|
| Extension | `.cdt` |
| Format origin | Stanford Hierarchical Clustering program |
| Structure | Tab-delimited text; original expression data reordered to reflect clustering |
| Special columns | `GID` column added if genes were clustered (links to GTR) |
| Special rows | `AID` row added if arrays were clustered (links to ATR) |
| Content | Original numeric expression values in cluster-sorted order |
| Compatibility | **Cannot** be directly used as input to other GenePattern modules (must be converted to GCT first) |

**CDT Header Structure:**
```
GID    YORF    NAME    GWEIGHT    ARRAY1    ARRAY2    ...
AID                              AID1      AID2      ...
EWEIGHT                          1.0       1.0       ...
GENE1  YAL001  TFC3    1.0       0.83      -0.21     ...
GENE2  YAL002  VPS8    1.0       -1.12     0.44      ...
```

#### 🌿 GTR File (Gene Tree) — **Optional**
| Property | Details |
|---|---|
| Extension | `.gtr` |
| Description | Gene Tree — records the order in which **rows (genes)** were joined during hierarchical clustering |
| Content | Node identifiers and pairwise distance measures between gene clusters |
| When present | Created when clustering was performed on genes/rows |
| Drives | Row-side dendrogram in the viewer |

#### 🌿 ATR File (Array Tree) — **Optional**
| Property | Details |
|---|---|
| Extension | `.atr` |
| Description | Array Tree — records the order in which **columns (samples/arrays)** were joined during hierarchical clustering |
| Content | Node identifiers and pairwise distance measures between sample clusters |
| When present | Created when clustering was performed on samples/columns |
| Drives | Column-side dendrogram in the viewer |

### Input Flexibility Rules
```
Valid Input Combinations:
  [1] CDT + GTR          → Gene dendrogram + heatmap (no sample dendrogram)
  [2] CDT + ATR          → Sample dendrogram + heatmap (no gene dendrogram)
  [3] CDT + GTR + ATR    → Full bidirectional dendrogram + heatmap (most common)
  [4] CDT only           → Heatmap only (no dendrograms displayed)
```

### Output
As a **Visualizer-type module**, `HierarchicalClusteringViewer` produces **no computational output files**. It is a pure rendering engine:
- Renders an interactive visual in the user's web browser
- No result files are written to the GenePattern job results directory
- Export functionality (if any) is handled client-side via browser mechanisms

## 3. ⚙️ Parameter Analysis and Usage Patterns

### Module Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `cdt.file` | File (`.cdt`) | ✅ Required | Clustered Data Table from HierarchicalClustering |
| `gtr.file` | File (`.gtr`) | ⬜ Optional | Gene Tree file (row dendrogram data) |
| `atr.file` | File (`.atr`) | ⬜ Optional | Array Tree file (column dendrogram data) |

### Interactive UI Parameters (Post-Launch Controls)

Once the viewer launches in the browser, users can interact with these controls:

| Control | Type | Behavior |
|---|---|---|
| **Color Scheme** | Radio button | Toggle between **Row Relative** and **Global** color mapping |
| **Row Relative** mode | Color algorithm | Uses per-row mean, min, max to assign colors — highlights intra-gene variation |
| **Global** mode | Color algorithm | Uses dataset-wide min/max — allows cross-gene intensity comparison |
| **Feature Labels** | Button | `Add/Edit Labels` → `Edit > Feature Labels` — assign colors to gene sets |
| **Dendrogram interaction** | Mouse events | Click and inspect cluster nodes (exact behavior depends on version) |

### Color Scheme Analysis

```
ROW RELATIVE (default):
  Each row independently normalized → color = f(row_mean, row_min, row_max)
  ✓ Best for: Comparing patterns across genes with different expression scales
  ✗ Limitation: Cannot compare absolute intensities between genes

GLOBAL:
  Single scale across entire matrix → color = f(dataset_min, dataset_max)
  ✓ Best for: Comparing absolute expression levels across the entire heatmap
  ✗ Limitation: Low-variance genes may appear uniform if high-variance genes dominate
```

### Usage Pattern: Launching from HierarchicalClustering Output

```
Typical Workflow:
1. Complete HierarchicalClustering job
2. In Recent Jobs list, click icon next to .atr, .cdt, or .gtr result file
3. From context menu → select "HierarchicalClusteringViewer"
4. GenePattern auto-populates all matching input file parameters
5. Click "Run" to launch viewer in browser
```

### Key GenePattern Integration Points

4. **File Format Exclusivity**: `.cdt`, `.gtr`, `.atr` formats are exclusively consumed by this viewer within GenePattern — no other standard module accepts them directly.

5. **No Docker Container Required**: Unlike analysis modules, this visualizer does not need containerization. The module files (HTML/JS/CSS) are served directly from the GenePattern module store to the client browser.

6. **Browser Security Considerations**:
   - Cross-origin requests to S3 require correct CORS headers on S3 buckets
   - Modern browsers block mixed content (HTTPS page + HTTP S3) — S3 URLs must be HTTPS
   - Large CDT files (hundreds of MB for large datasets) may cause browser memory limits to be reached

## 9. ⚠️ Known Limitations and Caveats

1. **Browser Memory Constraints**: Very large CDT files (thousands of genes × hundreds of samples) may cause rendering slowdowns or browser crashes
2. **No Output Files**: Being a visualizer, it produces no downloadable result files from clustering trees — users cannot programmatically extract cluster assignments from within this module
3. **Format Lock-in**: Exclusively accepts Stanford CDT/GTR/ATR format — external clustering results must be converted to this format
4. **Static Interaction Model**: jQuery-based UI is less interactive than modern D3.js-based alternatives (limited zoom, pan, or export capabilities compared to NG-CHM or Clustergrammer)
5. **S3 Bug (Now Fixed)**: Prior versions could not correctly load files hosted on Amazon S3 — this was the critical bug motivating the current version
6. **No Standalone Execution**: Cannot be run independently — requires a GenePattern server for file parameter resolution and module hosting
7. **Limited Customization**: Color scheme options are binary (Row Relative vs Global); no continuous gradient customization, no custom color palette support in base version

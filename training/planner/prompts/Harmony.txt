# 🧬 Comprehensive Research Report: Harmony — GenePattern Module

## 1. 🔬 Tool Purpose & Scientific Background

### Overview
**Harmony** is a fast, sensitive, and accurate algorithm for **batch effect correction and multi-dataset integration** in single-cell genomics. It was originally published by Korsunsky et al. in *Nature Methods* (2019, DOI: [10.1038/s41592-019-0619-0](https://www.nature.com/articles/s41592-019-0619-0)) and has become one of the most widely adopted tools for single-cell RNA-seq (scRNA-seq) data integration, accumulating **4,000+ citations**.

The GenePattern Harmony module wraps the Harmony algorithm as an accessible, user-friendly pipeline producing **before/after PCA scatterplots** alongside corrected embedding outputs for downstream analysis.

### The Problem It Solves
In single-cell genomics, data collected across **different experiments, donors, batches, labs, or technologies** contains unwanted technical variation ("batch effects") that obscures true biological signal. Harmony removes these effects while preserving biologically meaningful variation, enabling joint analysis of datasets from disparate sources.

### The Algorithm (Iterative Correction in PCA Space)
Harmony operates on **pre-computed low-dimensional embeddings** (typically PCA) rather than raw gene expression values:

| Step | Action |
|------|--------|
| **1. Initialize** | Accept PCA embeddings (cells × PCs) and batch metadata |
| **2. Cluster** | Perform soft k-means clustering in PCA space |
| **3. Assess** | Compute batch composition within each cluster vs. global expectation |
| **4. Correct** | Apply linear correction factors (`W`) to remove batch-specific offsets per cluster |
| **5. Iterate** | Repeat steps 2–4 until convergence (objective function minimization) |

**Key Outputs of the Object:**
- `Z_corr` — Corrected cell embeddings *(cells × dims)* — the primary output used downstream
- `Z_orig` — Original PCA embeddings *(cells × dims)*
- `R` — Soft cluster assignment matrix
- `W` — Batch correction weights

## 2. 📁 Input/Output Formats & Requirements

### Required Inputs

| Input | Format | Description |
|-------|--------|-------------|
| **PCA Embedding Matrix** | CSV/TSV, dense matrix | Cells as rows, principal components as columns (cells × PCs). Typically 20–50 PCs. |
| **Metadata Table** | CSV/TSV, DataFrame | Cells as rows, covariate columns (must include batch/group variable). Row order must match embedding matrix. |
| **Batch Variable Name** | String | Column name in metadata identifying batch, donor, dataset, etc. Can be a comma-separated list for multi-covariate correction. |

> **Note on Pre-processing:** Harmony expects **already reduced** data. Upstream steps: normalize counts → log-transform → highly variable gene selection → scale → PCA. Harmony does NOT operate on raw count matrices (unless using `HarmonyMatrix()` with `do_pca=TRUE`).

### GenePattern Module Outputs

| Output File | Format | Description |
|-------------|--------|-------------|
| `BeforeHarmonyPlot.png` | PNG image | Scatter plot of cells in PCA space (PC1 vs PC2 by default) **before** correction, colored by batch variable |
| `AfterHarmonyPlot.png` | PNG image | Same PCA dimensions **after** Harmony correction, showing batch mixing |
| `CombinedPlot.png` *(or similar)* | PNG image | Side-by-side comparison of before and after plots |
| **Corrected Embeddings** | CSV/TSV | Matrix of Harmony-corrected PCA coordinates (`Z_corr`) for downstream use |

> **Plot Axes:** The x/y axes represent the 1st and 2nd principal components by default (`dim1=1`, `dim2=2`), but the user can specify alternative PC dimensions.

## 3. ⚙️ Parameter Analysis & Usage Patterns

### Complete Parameter Catalog

#### 🗂️ Group 1: Required / Core Inputs

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `data_mat` | File/Matrix | — | PCA embeddings matrix (cells × PCs) |
| `meta_data` | File/DataFrame | — | Cell metadata table with batch labels |
| `vars_use` | String/List | — | Batch variable column name(s) in metadata |

#### 🔧 Group 2: Algorithm Behavior Parameters

| Parameter | Type | Default | Range/Notes | Biological Significance |
|-----------|------|---------|-------------|------------------------|
| `theta` | Float or List | `2` (R) / `NULL`→2 (py) | ≥ 0 | **Diversity penalty strength.** Higher values enforce more aggressive batch mixing. Use lower values (0–1) when biological groups are confounded with batch to avoid over-correction. |
| `lambda` / `lamb` | Float or List | `1` | > 0 | **Ridge regression regularization.** Prevents over-fitting the correction to small batches. Higher values = more conservative correction. |
| `sigma` | Float or Array | `0.1` | > 0 | **Soft k-means cluster width.** Controls fuzziness of cluster assignment. Higher sigma = smoother, less sharp clusters. |
| `nclust` | Integer | Auto: `min(N/30, 100)` | ≥ 1 | **Number of integration clusters.** Auto-calculated from dataset size. Increase for very large or heterogeneous datasets. |
| `tau` | Integer | `0` | ≥ 0 | **Protection against over-clustering small datasets.** Sets a minimum expected cells per cluster. Useful when batch sizes are highly unequal. |

#### 🔄 Group 3: Convergence & Iteration Parameters

| Parameter | Type | Default | Notes |
|-----------|------|---------|-------|
| `max_iter_harmony` | Integer | `10` | Max Harmony outer-loop iterations. Increase to `20–50` for complex or large datasets. |
| `max_iter_kmeans` / `max_iter_cluster` | Integer | `20` | Max k-means inner iterations per Harmony step. |
| `epsilon_cluster` | Float | `1e-5` | Convergence tolerance for k-means step. |
| `epsilon_harmony` | Float | `1e-4` | Convergence tolerance for full Harmony loop. |
| `early_stop` | Boolean | `TRUE` | Stop iterations early if converged. |
| `block_size` | Float | `0.05` | Fraction of cells updated per round (mini-batch). Lower values = slower but more memory-efficient. |

#### 🎨 Group 4: Output / Visualization Parameters

| Parameter | Type | Default | Notes |
|-----------|------|---------|-------|
| `dim1` | Integer | `1` | PC dimension for x-axis of scatter plots |
| `dim2` | Integer | `2` | PC dimension for y-axis of scatter plots |
| `color_by` | String | batch variable | Column in metadata used for cell coloring |
| `plot_convergence` | Boolean | `FALSE` | Plot the Harmony objective function over iterations |
| `verbose` | Boolean | `TRUE` | Print iteration progress to console/log |

#### 🔁 Group 5: Reproducibility & Compute

| Parameter | Type | Default | Notes |
|-----------|------|---------|-------|
| `random_state` | Integer | `0` | Random seed for reproducible k-means initialization |
| `ncores` | Integer | `1` | Parallel CPU cores (R package; OPENBLAS threading caveat applies) |
| `reference_values` | List | `NULL` | Specify reference batch(es) to anchor correction |
| `cluster_prior` | Matrix | `NULL` | Prior probability distribution over clusters |

### ⚠️ Parameter Interaction Warnings
- **`theta` + `vars_use`**: When integrating over multiple covariates, provide a **list of theta values** (one per covariate), e.g., `theta=[2, 1]` for `vars_use=["batch", "donor"]`
- **`lambda` + `tau`**: Both protect against over-correction of small batches; `tau` acts as a soft floor on cluster size while `lambda` regularizes correction magnitude
- **`nclust` + dataset size**: With very large datasets (>500K cells), consider explicitly setting `nclust=100–200` to control memory usage
- **Default "blank" values** *(GenePattern-specific)*: The GenePattern module version notes that "default values have been changed to be blank," meaning the module likely passes `NULL`/`None` for optional parameters and lets the underlying Harmony library determine optimal values dynamically

### Workflow 2: Multi-Covariate Correction

Useful when data has **multiple confounds** (e.g., both sequencing batch AND donor):
```python
ho = hm.run_harmony(pcs, meta, 
                    vars_use=["sequencing_batch", "donor_id"],
                    theta=[2, 1],  # Strong batch correction, softer donor correction
                    lamb=[1, 1])
```

### Workflow 3: scATAC-seq Integration (LSI Embeddings)

Harmony is also applied to **LSI (Latent Semantic Indexing)** embeddings from scATAC-seq, treating LSI components analogously to PCs:
```r
harmony_embeddings <- RunHarmony(lsi_embeddings, meta_data, 
                                  vars_use = "library",
                                  do_pca = FALSE)  # LSI already computed
```

### Workflow 4: Reference-Anchored Correction

When one batch should be treated as a **reference** (e.g., a well-characterized control dataset):
```r
RunHarmony(embeddings, meta, vars_use="dataset",
           reference_values="reference_dataset_label")
```

### Typical Parameter Tuning Scenarios

| Scenario | Recommended Settings |
|----------|---------------------|
| Simple 2-batch correction | `theta=2`, `lambda=1`, defaults |
| Many small batches | `tau=5–20`, `lambda=2–5` |
| Biologically heterogeneous datasets | `theta=0.5–1` (reduce aggressiveness) |
| Large datasets (>100K cells) | `nclust=100`, `block_size=0.05` |
| Slow convergence | `max_iter_harmony=20–50` |
| Dataset size imbalance | `tau=10`, `lambda=3` |

### Key Differentiators of Harmony

**✅ Advantages:**
1. **Fastest runtime** among all integration methods — scales linearly with cell number; benchmarked on 1M+ cells
2. **Minimal over-correction** — preserves differential expression signal better than BBKNN, MNN, LIGER
3. **Multi-covariate support** — natively handles multiple batch variables with independent penalty weights
4. **Framework agnostic** — works with Seurat, Scanpy, SingleCellExperiment, raw matrices
5. **Operates on embeddings** — no need to share gene space across datasets
6. **Top-ranked** in Tran et al. 2020 benchmark (rank sum across all 4 metrics)

**⚠️ Limitations & Caveats:**
1. **Over-correction risk** when biological groups are fully confounded with batch (e.g., disease status = batch); reduce `theta` or use `reference_values` to mitigate
2. **Requires pre-computed embeddings** — cannot be applied to raw counts without a PCA step
3. **Linear correction only** — may not fully resolve non-linear batch effects (scVI more powerful here)
4. **Not recommended for DE testing** — corrected embeddings should feed clustering/visualization, not differential expression analysis
5. **OpenBLAS thread contention** — can cause silent hangs in Python without thread control

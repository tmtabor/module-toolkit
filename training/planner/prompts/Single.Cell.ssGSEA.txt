# 🧬 Comprehensive Research Report: `Single.Cell.ssGSEA`

---

## 📋 Executive Summary

**Single.Cell.ssGSEA** is a GenePattern-native bioinformatics module designed to perform Single-Sample Gene Set Enrichment Analysis (ssGSEA) specifically adapted for the unique statistical challenges of single-cell RNA-seq data — most critically, **transcriptomic sparsity**. It is the officially recommended replacement for the standard `ssGSEA` GenePattern module when analyzing scRNA-seq data, and is designed to slot directly into the GenePattern single-cell pipeline downstream of `Seurat.Clustering`.

> 📌 **From the official ssGSEA GenePattern documentation:**
> *"For single-cell data, use the [Single.Cell.ssGSEA](https://genepattern.github.io/Single.Cell.ssGSEA/documentation) module instead."*

## 1. 🎯 Tool Purpose & Scientific Applications

### 1.1 Scientific Motivation

Standard ssGSEA was developed for bulk RNA-seq data (Barbie et al., 2009), where each "sample" represents thousands of pooled cells. When applied naïvely to single-cell RNA-seq data, the extreme **sparsity** (dropout events, zero-inflation) introduces substantial instability in enrichment score estimation — leading to high coefficients of variation (CoV) and unreliable biological interpretations.

The key scientific problem being solved:
- **scRNA-seq dropout**: Genes with low expression are frequently not detected in any given cell, appearing as zeros.
- **Per-cell scoring instability**: When scoring each cell individually, sparse gene expression profiles make it impossible to reliably rank genes, causing KS-statistic-based enrichment to vary dramatically across near-identical cells.
- **Biological signal obscured by technical noise**: A cell type's characteristic pathway activity cannot be reliably detected with unadapted bulk methods.

### 1.2 Technical Innovation

Based on the benchmarking study **"Profiling ranked list enrichment scoring in sparse data elucidates algorithmic tradeoffs"** (Wenzel, Jun, Liefeld, Tamayo, Mesirov; *bioRxiv* 2024; DOI: 10.1101/2024.06.03.597180), the tool implements two key adaptations:

| Adaptation | Bulk ssGSEA | Single.Cell.ssGSEA |
|---|---|---|
| **Normalization** | Rank / log-rank / none | **Centered Log-Ratio (CLR) across cells** |
| **Scoring unit** | Per-sample | **Per-cluster (pseudo-bulk aggregation)** |
| **Scoring statistic** | KS-like statistic | **KS statistic** (retained, now stabilized by CLR) |
| **Data model** | GCT expression matrix | **Seurat RDS object with cluster metadata** |

> 🔑 **Key Finding:** The combination of KS scoring + CLR normalization achieves the most stable scoring across all cell type × gene set combinations, yielding a CoV consistently below **10⁻⁵** — a dramatic improvement over applying standard ssGSEA to sparse data.

### 1.3 Primary Scientific Applications

1. **Pathway activity profiling by cell cluster**: Characterize which biological pathways are active in each cell population identified by Seurat clustering
2. **Cell type functional annotation**: Annotate unknown clusters by their enrichment for known cell-type signature gene sets
3. **Tumor microenvironment analysis**: Dissect cellular states and functional heterogeneity in cancer scRNA-seq data
4. **Treatment response characterization**: Compare pathway activities across treatment groups or time points at single-cluster resolution
5. **Novel subtype discovery**: Identify cellular subtypes based on coordinated activity of biological process pathways
6. **Immune cell profiling**: Characterize immune cell activation states, polarization, and subset-specific functions

### 3.3 Critical Parameter Decisions

#### `normalization_method` — The Most Important Parameter
```
Supported values:
  CLR       → Centered Log-Ratio across cells [RECOMMENDED for scRNA-seq]
  rank      → Rank transformation
  log.rank  → Log of rank transformation
  log       → Log transformation
  none      → No normalization
```
> ⚠️ **Scientific rationale for CLR default:** Benchmarking by Wenzel et al. (2024) demonstrated that CLR normalization before KS scoring eliminates the instability caused by scRNA-seq sparsity. This is the single most important algorithmic distinction from the bulk ssGSEA module.

#### `cluster_data_label` — Integration with Seurat.Clustering
```
Default: "seurat_clusters"
  → Automatically populated by Seurat.Clustering module
  → Integer cluster IDs (0, 1, 2, ...)
  
Custom: "cell_type", "annotation", "condition", etc.
  → Any categorical metadata column in the Seurat object
  → Enables analysis by annotated cell types, conditions, or custom groupings
```

#### `weighting_exponent` (alpha)
```
Range:  0.0 → 1.0 (typical)
Default: 0.75

  α = 0:    Unweighted — equal contribution of all gene positions in ranked list
  α = 0.75: Moderate weighting — standard GenePattern default
  α = 1.0:  Linear weighting — proportional to expression magnitude
```

## 4. 🔬 Algorithm Deep Dive

### 4.1 The scGSEA Algorithm Pipeline

```
Single-Cell Expression Matrix (sparse)
         │
         ▼
  [CLR Normalization]
  For each gene g: CLR(x_g) = log(x_g / geometric_mean(x_all))
  Applied across cells within each cluster
         │
         ▼
  [Cell Aggregation by Cluster]
  Cells grouped by seurat_clusters metadata
  Pseudo-bulk profiles computed per cluster
         │
         ▼
  [KS Enrichment Scoring] (per cluster × gene set)
  For each gene set S:
    - Rank genes by CLR-normalized expression
    - Compute running enrichment score (KS statistic)
    - ES = max deviation of gene set members vs. background
         │
         ▼
  [Output: Gene Set × Cluster Enrichment Matrix]
  GCT + CSV files
```

### 4.2 Why CLR + Cluster-Level Scoring Works

The two key innovations address orthogonal aspects of the sparsity problem:

1. **CLR normalization** addresses *within-cluster* sparsity by transforming zeros to be relative to the compositional context of the expressed genes — avoiding the zero-inflation artifact that corrupts ranking-based statistics.

2. **Cluster-level aggregation** addresses *single-cell dropout* by pooling information across cells in the same cluster (pseudo-bulk approach), dramatically increasing effective sequencing depth and reducing the impact of individual cell dropout events.

### 8.3 Known Limitations & Caveats

| Issue | Details | Mitigation |
|---|---|---|
| **Large Seurat objects** | Memory-intensive for >500K cells | Recommend RAM ≥ 16GB for large datasets |
| **Gene symbol matching** | GMT gene symbols must match Seurat feature names | Pre-processing step for Ensembl IDs |
| **Cluster aggregation loses per-cell resolution** | Cannot detect rare single-cell states within a cluster | Use with sufficiently granular Seurat clustering (high resolution) |
| **CLR sensitivity to zero inflation** | CLR may behave poorly with extreme sparsity | Filter low-quality cells upstream in Seurat.Clustering |
| **Gene set size trade-off** | Very small gene sets (<10 genes) are unstable | Use `min_gene_set_size ≥ 15` |
| **Species specificity** | GMT files are species-specific | Ensure GMT matches organism of scRNA-seq experiment |

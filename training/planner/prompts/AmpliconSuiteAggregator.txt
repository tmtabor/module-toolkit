# 🔬 Comprehensive Research Report: AmpliconSuiteAggregator

## Executive Summary

**AmpliconSuiteAggregator** is a post-processing bioinformatics utility that occupies a critical bridging role in the **AmpliconSuite ecosystem** — aggregating multi-sample outputs from individual AmpliconSuite-pipeline runs (which detect extrachromosomal DNA and other focal amplifications in cancer genomes) into a single, repository-ready archive. It is the final standardization step before data deposition to **AmpliconRepository.org**, the dedicated public database for focal amplification data. The tool is available as a GenePattern module (LSID: `urn:lsid:genepattern.org:module.analysis:00429:4.1`), a PyPI package, and a Docker image.

---

## 1. 🎯 Tool Purpose and Scientific Applications

### 1.1 Biological Context
AmpliconSuiteAggregator operates within the oncogenomics field of **focal DNA amplification analysis**, particularly the study of:

| Amplification Type | Biological Significance |
|---|---|
| **ecDNA (extrachromosomal DNA)** | Circular DNA elements outside chromosomes; associated with oncogene amplification, drug resistance, and poor prognosis |
| **BFB (Breakage-Fusion-Bridge) cycles** | Chromosomal rearrangement mechanism producing large inverted duplications |
| **Linear amplifications** | Tandem duplications or other localized amplifications within chromosomes |
| **Complex amplicons** | Mixed/non-cyclic focal amplifications with multiple genomic contributors |

The underlying science relies on **Whole Genome Sequencing (WGS)** data processed through the **AmpliconArchitect (AA)** and **AmpliconClassifier (AC)** tools. ecDNA in particular has emerged as a major driver of oncogene amplification in >14% of human cancers and is associated with intratumoral heterogeneity and treatment resistance (Kim *et al.*, Nat. Genet. 2020).

### 1.2 Direct Scientific Role of AmpliconSuiteAggregator
The tool itself does not perform primary analysis — rather, it solves a **data management and interoperability problem** that emerges in large-scale cancer genomics studies:

1. **Multi-sample aggregation**: Individual AmpliconSuite-pipeline runs produce per-sample output archives. When analyzing cohorts of 10s–1000s of tumor samples, these outputs must be merged into a single structured project archive.
2. **Format normalization**: It restructures the internal directory layout of the input archives to match the strict schema required by AmpliconRepository's web interface.
3. **Optional re-classification**: The tool can invoke AmpliconClassifier on the aggregated dataset to refresh classification predictions across all samples.
4. **Direct upload API**: It integrates with the AmpliconRepository REST API to enable one-command upload of aggregated results.
5. **License compliance enforcement**: The latest version includes an `--accepted_license` parameter, reflecting the commercial licensing requirement of the underlying AmpliconArchitect software (Stanford University / Grail Inc. license).

### 1.3 Scale of Deployment
The broader AmpliconSuite ecosystem has been deployed in **30+ large-scale studies**, processing **>45,000 tumor samples** and **>3 petabytes** of sequencing data (AACR 2024). AmpliconSuiteAggregator is the gateway through which these cohort-level results are shared in the public domain.

## 2. 📂 Input/Output Formats and Requirements

### 2.1 Input Formats

| Format | Description | Notes |
|---|---|---|
| `.zip` | Compressed archive of AmpliconSuite-pipeline output for one or more samples | Most common from GenePattern module |
| `.tar.gz` | Gzip-compressed tarball of AmpliconSuite-pipeline output | Most common from command-line |
| `Directory path` | Uncompressed directory of AmpliconSuite outputs | Local execution only |
| Text filelist (`--filelist`) | Plain text file, one archive path per line | For batch processing of many samples |
| Pre-aggregated `.tar.gz` | Already-aggregated archive for direct upload | Used with `--upload_only Yes` |

#### ⚠️ Critical Input Requirements:
- **MUST include** both **AmpliconArchitect (AA)** AND **AmpliconClassifier (AC)** output files
- **MUST NOT include** original BAM or FASTQ sequencing data files (to keep archives manageable)
- **MUST be structured** with the standard AmpliconSuite-pipeline output directory hierarchy

#### Expected Contents of Each Input Archive:
```
sample_output/
├── sample_name_amplicon1_cycles.txt          # AA cycles file (0-based closed coords)
├── sample_name_amplicon1_graph.txt           # AA breakpoint graph
├── sample_name_amplicon1.png                 # AA visualization
├── sample_name_amplicon2_cycles.txt
├── sample_name_amplicon2_graph.txt
├── sample_name_amplicon2.png
├── classification/
│   ├── sample_name_result_data.json          # AC structured results
│   ├── sample_name_result_data.tsv           # AC tabular results
│   ├── sample_name_classification_bed.bed    # AC genomic intervals
│   └── sample_name_amplicon_classification_*.txt
└── AUX_DIR/                                  # (optional) auxiliary files
    └── [any additional project files]
```

> 💡 **AUX_DIR Feature**: Any directory containing a file named `AUX_DIR` will have its contents included in the upload alongside the standard results, enabling inclusion of custom metadata or supplementary files.

### 2.2 Output Format

| Output | Description |
|---|---|
| `{output_prefix}.tar.gz` | **Primary output** — single aggregated, restructured archive containing all input samples, formatted for direct AmpliconRepository ingestion |

The output archive contains an internal directory structure reorganized by AmpliconSuiteAggregator to match the schema expected by the AmpliconRepository web platform. All per-sample data is preserved but re-keyed to the project namespace.

| Parameter | Type | Default | Choices | Description |

#### 🟡 Optional — Upload to AmpliconRepository

| Parameter | Type | Default | Choices | Description |
|---|---|---|---|---|

           ↓

### 6.3 Known Integration Considerations & Caveats

| Consideration | Detail |
|---|---|
| **License Requirement** | AmpliconArchitect (underlying tool) requires academic/commercial license from Stanford/Grail Inc. The `--accepted_license` parameter enforces acknowledgment. |
| **Multi-file input** | GenePattern must support multiple file uploads for the `--files` parameter, or use the `--filelist` alternative with a server-side generated text file. |
| **Network access** | Upload functionality (`--username`) requires outbound HTTPS to `ampliconrepository.org`; GenePattern servers must allow this. |
| **Protected Health Information** | The GenePattern team notes PHI data should not be processed on public cloud GenePattern instances — relevant when uploading clinical samples. |
| **Pre-aggregated input bypass** | `--upload_only Yes` enables an "upload-only" mode that bypasses aggregation, useful for datasets already processed locally. |
| **Output re-usability** | The output `.tar.gz` produced by this module is the direct input format for AmpliconRepository and can also serve as a project archive for sharing. |
| **Server target** | Default should be `prod` for real users; module should expose `dev` as an option for testing without polluting production data. |
| **Classifier version synchronization** | If the module includes a newer AmpliconClassifier than was originally used to classify the inputs, `--rerun_classifier Yes` ensures consistent classifications. |

### 7.1 AmpliconSuite Tool Ecosystem

| **ecSimulator** | ecDNA genome structure simulation | Upstream data generation for benchmarking |

### 7.2 Unique Role of AmpliconSuiteAggregator
Unlike all other tools in the ecosystem, AmpliconSuiteAggregator is the **only tool focused on multi-sample project assembly and repository-ready data packaging**. It uniquely addresses:
- **N-to-1 aggregation**: Combines N per-sample archives into 1 project archive
- **Format compliance**: Guarantees the output meets AmpliconRepository ingestion requirements
- **API integration**: Direct programmatic upload capability
- **License governance**: Embedded compliance checking

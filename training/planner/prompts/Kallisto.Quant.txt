## 1. 📖 TOOL PURPOSE & SCIENTIFIC BACKGROUND

### 1.1 Primary Purpose
**Kallisto.Quant** is a GenePattern module wrapping **kallisto**, a near-optimal probabilistic RNA-seq quantification tool. It performs *pseudoalignment* of RNA-seq reads against a reference transcriptome to produce transcript- and gene-level abundance estimates, without requiring full read alignment to a genome.

### 1.2 Foundational Algorithm — Pseudoalignment
Kallisto is built on the concept of **pseudoalignment**, introduced by Bray *et al.* in *Nature Biotechnology* (2016). Rather than finding exact positions where reads map in a genome, pseudoalignment determines the *compatibility* of reads with transcripts by matching k-mers (default k=31) to a **colored de Bruijn graph** of the transcriptome. This results in transcript *equivalence classes* — sets of transcripts compatible with each read — which are then used to solve for abundance via the Expectation-Maximization (EM) algorithm.

> **Key Insight**: Reads need not be precisely placed — only their *transcript-of-origin* must be determined. This makes kallisto **~100× faster** than alignment-based approaches with comparable accuracy for transcript and gene quantification.

### 1.3 Primary Citation
> Bray NL, Pimentel H, Melsted P, Pachter L. **Near-optimal probabilistic RNA-seq quantification.** *Nature Biotechnology* **34**, 525–527 (2016). DOI: [10.1038/nbt.3519](https://doi.org/10.1038/nbt.3519) | PMID: 27043002

This paper has received **>10,000 citations** as of 2024, reflecting kallisto's widespread adoption. A companion long-read extension (lr-kallisto) was published in *PNAS* (2024), confirming the algorithm's ongoing development.

### 1.4 Scientific Applications
| Application | Description |
|---|---|
| **Bulk RNA-seq quantification** | Primary use case — transcript/gene abundance from Illumina RNA-seq |
| **Differential expression analysis** | Upstream step for DESeq2, edgeR, limma-voom |
| **Isoform-level analysis** | Transcript-level estimates for sleuth or tximport |
| **Uncertainty quantification** | Bootstrap resampling for downstream sleuth inference |
| **Single-cell pre-processing** | With bustools/kb-python for scRNA-seq (BUS format) |
| **Fusion detection** | With Pizzly for gene fusion identification |
| **Metatranscriptomics** | Against custom transcriptome indexes |

## 3. ⚙️ PARAMETER ANALYSIS & USAGE PATTERNS

### 3.1 Complete Parameter Reference for `kallisto quant`

#### Required Parameters
| Parameter | Type | Default | Description |
|---|---|---|---|
| `-i`, `--index` | STRING | *none* | Path to kallisto transcriptome index (`.idx`). In GenePattern: **dynamic dropdown** for Human (Gencode 37/GRCh38.p13) or Mouse (Gencode M26/GRCm39) |
| `-o`, `--output-dir` | STRING | *none* | Output directory. GenePattern populates this automatically |

#### Read Mode Parameters *(Mutually Exclusive)*
| Parameter | Type | Default | Biological Significance |
|---|---|---|---|
| *(default)* | — | paired-end | Standard paired-end RNA-seq; fragment length estimated from data |
| `--single` | Boolean flag | FALSE | For single-end libraries; **requires `-l` and `-s`** |
| `--single-overhang` | Boolean flag | FALSE | Include reads predicted to extend beyond transcript boundaries; useful for short transcripts |

#### Fragment Length Parameters *(Single-End Only)*
| Parameter | Type | Default | Biological Significance |
|---|---|---|---|
| `-l`, `--fragment-length` | DOUBLE | *required* | Estimated mean insert size. Typically 180–250 bp for standard RNA-seq libraries. Can be obtained from library prep documentation or FastQC/Picard metrics |
| `-s`, `--sd` | DOUBLE | `-l × 0.01` | Standard deviation of fragment length distribution. Typically 20–50 bp; reflects library heterogeneity |

> ⚠️ **Critical**: Incorrect `-l`/`-s` values for single-end data significantly affect effective length calculations and consequently TPM normalization accuracy.

#### Strandedness Parameters *(Optional but Recommended)*
| Parameter | Type | Biological Use |
|---|---|---|
| `--fr-stranded` | Flag | First-read-forward (sense) stranded. Use for **dUTP-based libraries** (most Illumina TruSeq Stranded), **ligation-based strand-specific** methods |
| `--rf-stranded` | Flag | First-read-reverse (antisense) stranded. Use for certain directional library preparations |
| *(omit both)* | — | Unstranded library (default); appropriate for standard non-stranded RNA-seq |

> ℹ️ **Best Practice**: Always specify strandedness when known. Incorrect strandedness can reduce the number of pseudoaligned reads and introduce quantification bias.

#### Computational Parameters
| Parameter | Type | Default | Guidance |
|---|---|---|---|
| `-t`, `--threads` | INT | 1 | Number of CPU threads. Recommend 4–8 for typical server environments; GenePattern should expose this |
| `--bias` | Boolean flag | FALSE | Sequence-based bias correction (hexamer priming). **Recommended** for most libraries, especially polyA-selected |
| `-b`, `--bootstrap-samples` | INT | 0 | Bootstrap iterations for uncertainty quantification. Use **≥100** for sleuth downstream analysis; 0 is sufficient for DESeq2/edgeR |
| `--seed` | INT | 42 | Random seed for bootstrap reproducibility |

#### Output Format Parameters
| Parameter | Type | Default | Notes |
|---|---|---|---|
| `--plaintext` | Flag | FALSE | Output plaintext TSV instead of HDF5; NOTE: omits bootstrap estimates |
| `--genomebam` | Flag | FALSE | Project pseudoalignments to genome-coordinate BAM (requires `--gtf`) |
| `--gtf` | FILE | *none* | GTF annotation; required for `--genomebam` |
| `--chromosomes` | FILE | *none* | Chromosome sizes file; improves BAM header consistency |
| `--fusion` | Flag | FALSE | Output fusion candidates for Pizzly analysis |
| `--verbose` | Flag | FALSE | Print detailed run information to stderr |

### 3.2 Index Building Parameters (`kallisto index`)
| Parameter | Type | Default | Notes |
|---|---|---|---|
| `-i`, `--index` | STRING | *none* | Output index filename |
| `-k`, `--kmer-size` | INT (odd) | 31 | k-mer length. Max value 31. Smaller k (e.g., 21) improves sensitivity for short reads (<50bp) at cost of slightly reduced specificity |
| `--make-unique` | Flag | FALSE | Replace duplicate transcript names with unique identifiers |
| `-t`, `--threads` | INT | 1 | Threads for index building |

     │  Differential Expression
     ▼

### 5.2 Key Workflow Commands

```bash
# Step 1: (GenePattern handles) — Build or use pre-built index
kallisto index -i human_gencode37.idx Homo_sapiens.GRCh38.cdna.all.fa.gz

# Step 2: Paired-end quantification with bias correction
kallisto quant \
    -i human_gencode37.idx \
    -o sample1_output \
    --bias \
    -t 8 \
    sample1_R1.fastq.gz sample1_R2.fastq.gz

# Step 3: Single-end quantification
kallisto quant \
    -i human_gencode37.idx \
    -o sample1_output \
    --single -l 200 -s 30 \
    --bias \
    -t 8 \
    sample1.fastq.gz

# Step 4: With bootstrap (for sleuth)
kallisto quant \
    -i human_gencode37.idx \
    -o sample1_output \
    -b 100 \
    --bias \
    -t 8 \
    sample1_R1.fastq.gz sample1_R2.fastq.gz

# Step 5: Gene-level aggregation in R via tximport
library(tximport)
files <- c("sample1/abundance.h5", "sample2/abundance.h5")
names(files) <- c("sample1", "sample2")
tx2gene <- read.csv("tx2gene_human_gencode37.csv")
txi <- tximport(files, type = "kallisto", tx2gene = tx2gene)
# txi$counts = gene-level count matrix for DESeq2
# txi$abundance = gene-level TPM matrix
```

### 6.2 Kallisto vs. Salmon (Closest Competitor)
| Aspect | Kallisto | Salmon |
|---|---|---|
| Algorithm | Pseudoalignment (exact k-mer) | Selective alignment (quasi-mapping) |
| Speed | Marginally faster | Slightly slower |
| Bias Correction | Sequence-based (`--bias`) | GC-bias + sequence-bias + positional-bias |
| Output | H5/TSV | SF (native), also HDF5 |
| scRNA-seq | Yes (via bustools/BUS) | Yes (via alevin) |
| Accuracy correlation | Rv = 0.9999 vs Salmon | Reference standard |
| GenePattern availability | ✅ (Kallisto.Quant) | Separate module |

### 6.3 When to Choose Kallisto.Quant
✅ **Best choice when:**
- Fast turnaround on well-annotated genomes (human, mouse)
- Low memory computing environments
- Gene-level counts for DESeq2/edgeR are the end goal
- Processing many samples in parallel
- Single-cell pre-processing (bus mode)

⚠️ **Consider alternatives when:**
- Novel splice junction detection is required → use STAR
- Extensive bias correction needed → consider Salmon
- Downstream analysis requires true genomic BAM → use STAR or HISAT2
- Transcript-level differential expression with full bias modeling → use Salmon + tximport

### 7.4 Gene-Level Aggregation (GenePattern-Specific Feature)
The GenePattern module goes beyond the base kallisto binary by integrating **tximport** (Bioconductor) to automatically perform transcript-to-gene level aggregation:

```R
# Internal R code for gene aggregation in the module
library(tximport)
library(rhdf5)

# tx2gene mapping derived from Gencode GTF
tx2gene <- read.csv("/opt/module_support/tx2gene_human_gencode37.csv")

files <- list.files("RNASeq_quant", pattern="abundance.h5", full.names=TRUE)
names(files) <- output.filename

txi <- tximport(
    files,
    type = "kallisto",
    tx2gene = tx2gene,
    ignoreTxVersion = TRUE
)

# Output: gene-level counts matrix (for DESeq2/edgeR)
write.csv(txi$counts, "gene_counts.csv")
# Output: gene-level TPM matrix
write.csv(txi$abundance, "gene_tpm.csv")
```

### 7.5 Known Limitations & Caveats for Module Documentation
1. **Novel transcripts**: Cannot quantify transcripts absent from the reference index
2. **Pseudobam limitations**: `--genomebam` BAM is not equivalent to true genomic alignment; cannot be used for variant calling
3. **Fragment length sensitivity**: Incorrect `-l`/`-s` for single-end data materially affects TPM normalization
4. **Index version incompatibility**: kallisto v0.50.0 uses a new index format incompatible with older versions — GenePattern must track index-binary version pairing
5. **Low-expression sensitivity**: Marginally lower detection of very lowly expressed genes compared to alignment-based methods (HISAT2)
6. **Multi-mapping**: Reads equally compatible with multiple transcript families are probabilistically assigned — appropriate for most gene-level analyses but requires caution for paralog-heavy gene families

## 9. ✅ SUMMARY & RECOMMENDATIONS FOR GENEPATTERN MODULE DEVELOPMENT

### Critical Implementation Points
1. **Dynamic Index Dropdown**: Implement versioned index selection with clear organism/assembly/annotation labels (e.g., `Human - GRCh38.p13 - Gencode 37`). Update process should not require full module version bump.
2. **Single-end validation**: Enforce that `-l` and `-s` are provided when `--single` mode is selected; add sensible default suggestions (l=200, s=30).
3. **Gene-level aggregation**: The tximport integration is a key differentiator of this module vs. raw kallisto — ensure the tx2gene mapping files are version-matched to the reference indexes.
4. **HDF5 output**: Keep `abundance.h5` as primary output (not `--plaintext`) to preserve bootstrap data for sleuth compatibility.
5. **Version tracking**: Maintain strict kallisto binary ↔ index version compatibility (v0.50.0 index is incompatible with older binaries).
6. **Container versioning**: Pin specific kallisto version in Docker image; use `quay.io/biocontainers/kallisto` BioContainers as fallback.
7. **Resource defaults**: Set default threads to 4; document expected runtime (~3–10 min for 50M reads with 4 threads).
8. **Quality metric surfacing**: Parse `run_info.json` to display pseudoalignment rate in the module output summary — rates <60% may indicate index mismatch or poor-quality data.

### Recommended Module Test Data
- **Test dataset**: `SRR1515119_50k.fastq.gz` (already used in GenePattern module testing)
- **Expected output**: ~50,000 reads processed, ~192 reads pseudoaligned (low rate expected for 50k subset)
- **Full validation**: Use SEQC benchmark dataset or GEUVADIS RNA-seq data

# 🧬 Comprehensive Research Report: `Seurat.Preprocessing` (GenePattern Module v4)

---

## 1. 🔬 Tool Purpose & Scientific Applications

### Overview
`Seurat.Preprocessing` is a **GenePattern module** that encapsulates the core preprocessing pipeline of the **Seurat R package (v4.0.3)** — one of the most widely adopted frameworks for single-cell RNA sequencing (scRNA-seq) analysis. It functions as the **central transformation hub** in the GenePattern Seurat Suite, converting quality-controlled, filtered count data into a fully normalized, reduced-dimensionality Seurat object ready for downstream biological interpretation.

### Biological Significance
The module addresses three fundamental challenges in scRNA-seq data analysis:

| Challenge | Solution Implemented |
|---|---|
| **Technical noise** (sequencing depth variation) | Per-cell normalization (`NormalizeData`) |
| **Curse of dimensionality** (~20,000 genes per cell) | Highly Variable Gene (HVG) selection + PCA |
| **Visualization of high-dimensional cell states** | UMAP / tSNE non-linear embedding |
| **Cell community structure** | k-NN graph + Louvain clustering |
| **Confounding biological/technical variation** | Regression via `ScaleData(vars.to.regress=...)` |

### Scientific Applications
- **Cell type discovery**: Unsupervised identification of distinct cell populations in complex tissues (PBMC, tumor microenvironment, brain, gut, etc.)
- **Developmental biology**: Preparation for pseudotime/trajectory analysis (precedes Monocle3, STREAM)
- **Multi-modal integration**: Foundation for CITE-seq, ATAC+RNA, spatial transcriptomics preprocessing
- **Disease atlases**: Preprocessing of large-scale patient cohort datasets (e.g., COVID-19 PBMC, cancer immune infiltrates)
- **Comparative biology**: Preprocessing for cross-species or cross-condition comparisons
- **Cell cycle analysis**: Scoring and optionally regressing out cell cycle phases

## 2. 📁 Input/Output Formats & Requirements

### Input Files

| File | Format | Required | Description |
|---|---|---|---|
| `input.file` | `.rds` | ✅ Required | Serialized R Seurat object from `Seurat.QC` |
| *(alternative)* | 10x MEX (`.mtx` + `.tsv`) | Context-dependent | CellRanger output; typically loaded in `Seurat.QC` |

**Input `.rds` Contents (from `Seurat.QC`):**
- Raw count matrix (sparse `dgCMatrix` in `@assays$RNA@counts`)
- Per-cell metadata: `nCount_RNA`, `nFeature_RNA`, `percent.mt`, `orig.ident`
- Cells have been filtered for QC thresholds (minimum features, maximum mitochondrial %, doublets removed)

### Output Files

| File | Format | Description |
|---|---|---|
| `seurat_preprocessed_dataset.rds` | `.rds` | **Primary output**: Fully preprocessed Seurat object |
| `output_plots.pdf` | `.pdf` | Diagnostic visualization plots |
| *(optional)* `variable_features.txt` | `.txt` | List of top HVGs |

**Output `.rds` Object Slots Populated:**
```
@assays$RNA
  @counts       # Original raw counts (preserved)
  @data         # Log-normalized expression matrix
  @scale.data   # Z-scored, regression-adjusted matrix
  @var.features # Top nfeatures highly variable genes

@reductions
  $pca          # npcs-dimensional PCA embedding
  $umap         # 2D UMAP coordinates (if run.umap=TRUE)
  $tsne         # 2D tSNE coordinates (if run.tsne=TRUE)

@graphs
  $RNA_nn       # k-NN graph (if run.neighbors=TRUE)
  $RNA_snn      # Shared nearest-neighbor graph

@meta.data
  $seurat_clusters   # Cluster identities (if run.clustering=TRUE)
  + all QC columns from upstream
```

**Output PDF Plots:**
1. `VariableFeaturePlot()` — Top HVGs labeled in mean-variance space
2. `VizDimLoadings()` — Top gene loadings on PC1/PC2
3. `ElbowPlot()` — Variance explained per PC (critical for `dims` selection)
4. `DimPlot()` — UMAP/tSNE colored by cluster
5. `DimHeatmap()` — Heatmap of top PC gene loadings

## 3. ⚙️ Parameter Analysis & Usage Patterns

### Full Parameter Catalog

#### **Group A: Input/Output Parameters**

| Parameter | Type | Default | Required | Description |
|---|---|---|---|---|
| `input.file` | File (`.rds`) | — | ✅ Yes | Seurat object from `Seurat.QC` |
| `output.filename` | String | `"seurat_preprocessed_dataset"` | No | Base name for output files |

---

#### **Group B: Normalization Parameters**

| Parameter | Type | Default | Options | Biological Guidance |
|---|---|---|---|---|
| `normalization.method` | Choice | `"LogNormalize"` | `LogNormalize`, `CLR`, `RC` | LogNormalize is standard for gene expression; CLR recommended for protein (CITE-seq) |
| `scale.factor` | Integer | `10000` | 1,000–1,000,000 | 10,000 = CPM-like normalization (counts per 10k) |
| `use.sctransform` | Boolean | `FALSE` | TRUE/FALSE | Activates SCTransform workflow (replaces Steps B–D); recommended for high-depth-variation datasets |

---

#### **Group C: Highly Variable Gene (HVG) Selection**

| Parameter | Type | Default | Options | Notes |
|---|---|---|---|---|
| `nfeatures` | Integer | `2000` | 1,000–5,000 | Increase for highly heterogeneous tissues (e.g., brain: 3,000–5,000); decrease for homogeneous samples |
| `selection.method` | Choice | `"vst"` | `vst`, `mean.var.plot`, `dispersion` | VST is strongly preferred in v4; `mean.var.plot` is legacy Seurat v2 method |

**Method Comparison:**
| Method | Algorithm | Recommendation |
|---|---|---|
| `vst` | Fits loess to mean–variance; standardizes residuals | ✅ **Best for most datasets** |
| `mean.var.plot` | Bins genes by mean; selects high-dispersion per bin | Legacy; use for v2 reproducibility |
| `dispersion` | Raw dispersion (not mean-normalized) | Not recommended for large datasets |

---

#### **Group D: Data Scaling & Regression**

| Parameter | Type | Default | Notes |
|---|---|---|---|
| `vars.to.regress` | String (comma-sep) | `NULL` | Comma-separated list of metadata column names to regress |
| `do.scale` | Boolean | `TRUE` | Scale variance to 1 per gene |
| `do.center` | Boolean | `TRUE` | Center mean to 0 per gene |

**Common `vars.to.regress` values and when to use them:**
| Variable | When to Use |
|---|---|
| `percent.mt` | Mitochondrial content varies substantially across conditions |
| `nCount_RNA` | Residual depth effects after normalization |
| `S.Score` + `G2M.Score` | Cycling cells mask biological signal of interest |
| `nFeature_RNA` | Feature detection rate confounds clustering |

> ⚠️ **Caution**: Aggressive regression (e.g., `nCount_RNA`) can remove biologically meaningful variation. Regress only when confounding is confirmed.

---

#### **Group E: Dimensionality Reduction (PCA)**

| Parameter | Type | Default | Range | Notes |
|---|---|---|---|---|
| `npcs` | Integer | `50` | 20–200 | Number of PCs to compute; use `ElbowPlot()` to choose how many to USE downstream |
| `seed.use` | Integer | `42` | Any integer | Reproducibility seed for stochastic components |

---

#### **Group F: UMAP/tSNE Embedding**

| Parameter | Type | Default | Notes |
|---|---|---|---|
| `run.umap` | Boolean | `TRUE` | Standard visualization; recommended default ON |
| `umap.dims` | String | `"1:30"` | R range notation; use ElbowPlot to determine |
| `run.tsne` | Boolean | `FALSE` | Optional alternative; computationally expensive |
| `n.neighbors` | Integer | `30` | UMAP parameter; higher = more global structure |
| `min.dist` | Float | `0.3` | UMAP compactness; lower = tighter clusters |
| `spread` | Float | `1.0` | UMAP scale; interacts with min.dist |
| `metric` | String | `"cosine"` | Distance metric for UMAP graph |

---

#### **Group G: Graph Construction & Clustering**

| Parameter | Type | Default | Range | Notes |
|---|---|---|---|---|
| `run.neighbors` | Boolean | `TRUE` | — | Required for clustering; builds SNN graph |
| `nn.dims` | String | `"1:30"` | R range | PCs for NN graph; should match umap.dims |
| `run.clustering` | Boolean | `TRUE` | — | Louvain graph-based community detection |
| `resolution` | Float | `0.5` | 0.01–5.0 | **Most impactful parameter for biology**: 0.1–0.3 = broad cell types; 1.0–2.0 = fine subtypes |

**Resolution Guide:**
| Resolution | Expected Use Case |
|---|---|
| 0.1–0.3 | Major cell lineages (T cells, B cells, myeloid) |
| 0.4–0.8 | Standard subtype resolution (**default range**) |
| 1.0–1.5 | Fine T-cell subsets, DC subpopulations |
| 1.5–2.0 | Ultra-fine rare population discovery |

## 5. 🔄 Common Workflows & Use Cases

### Workflow A: Standard Single-Sample Preprocessing
```
Seurat.QC (filtered.rds)
    ↓
Seurat.Preprocessing
  [normalization.method = "LogNormalize"]
  [nfeatures = 2000]
  [vars.to.regress = "percent.mt"]
  [npcs = 50]
  [umap.dims = "1:30"]
  [resolution = 0.5]
    ↓
seurat_preprocessed_dataset.rds
    ↓
Seurat.Clustering / Seurat.DE
```

### Workflow B: Multi-Sample (Post-Batch Correction)
```
Seurat.QC (Sample1) ──┐
Seurat.QC (Sample2) ──┤
Seurat.QC (Sample3) ──┘
         ↓
Seurat.IntegrateData (CCA or Harmony)
         ↓
Seurat.Preprocessing
  [normalization.method = "LogNormalize"]
  [nfeatures = 3000]  ← increase for diversity
  [vars.to.regress = "percent.mt,nCount_RNA"]
  [umap.dims = "1:40"]  ← more PCs post-integration
         ↓
Seurat.Clustering
```

### Workflow C: SCTransform Pipeline (High-Depth-Variation Datasets)
```
Seurat.QC (filtered.rds)
    ↓
Seurat.Preprocessing
  [use.sctransform = TRUE]
  [vars.to.regress = "percent.mt"]  ← handled inside SCTransform
  [npcs = 50]
  [umap.dims = "1:30"]
  [resolution = 0.8]
    ↓
seurat_preprocessed_dataset.rds (SCT assay)
```

### Workflow D: Cell Cycle Regression (Cycling Tissue Datasets)
```R
# Pre-step: Cell cycle scoring (in Seurat.QC or separate step)
seurat_obj <- CellCycleScoring(seurat_obj,
  s.features = cc.genes$s.genes,
  g2m.features = cc.genes$g2m.genes)

# Seurat.Preprocessing parameters:
vars.to.regress = "S.Score,G2M.Score"
```

### Known Integration Considerations & Caveats

1. **Memory management**: Large datasets (>100k cells) may exceed 16GB RAM during `ScaleData`; consider using `features` parameter to scale only HVGs
2. **SCTransform flag interaction**: When `use.sctransform=TRUE`, the `normalization.method`, `scale.factor`, and `selection.method` parameters are effectively bypassed — document clearly in the UI
3. **`umap.dims` vs `nn.dims` consistency**: These should typically be set to the same range; mismatches can cause visualization/clustering inconsistencies
4. **RDS format versioning**: Seurat objects are version-sensitive; objects created with Seurat 3.x must be updated with `UpdateSeuratObject()` before input
5. **`vars.to.regress` parsing**: The R script must correctly split comma-separated strings into R character vectors
6. **UMAP reproducibility**: UMAP uses stochastic initialization; `seed.use` parameter must be set and passed to `RunUMAP(..., seed.use=42)` for reproducible outputs
7. **Parallelization in Docker**: `future` parallelization may require explicit `plan(multisession)` setup in containerized environments

## 9. ✅ Recommendations for GenePattern Module Development

1. **Parameter Defaults**: The defaults above (`LogNormalize`, `nfeatures=2000`, `vst`, `npcs=50`, `dims=1:30`, `resolution=0.5`) are scientifically validated and should be pre-populated
2. **Conditional UI Logic**: Hide `scale.factor`/`selection.method` parameters when `use.sctransform=TRUE` to reduce user confusion
3. **ElbowPlot output**: Always include in output PDF — it's the primary diagnostic for choosing `umap.dims`
4. **Input validation**: Check that the `.rds` contains a valid Seurat object with a populated `@assays$RNA@counts` slot before running
5. **Progress reporting**: Log each major step (Normalize → HVG → Scale → PCA → UMAP → Cluster) to GenePattern stdout for large-dataset monitoring
6. **Test dataset**: Include the 2,700-cell PBMC dataset (10x Genomics) as the canonical validation dataset for module testing
7. **Version pinning**: Pin `Seurat==4.0.3` and `SeuratObject==4.0.4` in the Docker image to prevent breaking API changes from v5

## 1. 🔬 Tool Purpose and Scientific Applications

### 1.1 Overview

`spatialGE.Preprocessing` is a GenePattern module implementing the entry-point preprocessing pipeline of the **spatialGE R package**, developed by Dr. Oscar E. Ospina and the Fridley Lab at **Moffitt Cancer Center**. It is the mandatory first module in the GenePattern spatialGE spatial transcriptomics analysis suite.

The module consolidates four sequential bioinformatics operations into a single workflow step:
1. **Data Ingestion** — Load raw count matrices and spatial coordinates into an STlist S4 object
2. **Filtering** — Remove low-quality spots/cells and lowly-expressed/unwanted genes
3. **Pseudo-bulk Aggregation** — Optionally aggregate spot-level counts to sample-level profiles
4. **Normalization & Transformation** — Apply library-size normalization and transformation

### 1.2 Scientific Applications

| Application Domain | Description |
|---|---|
| **Tumor Microenvironment (TME)** | Characterizing spatial organization of immune infiltrates, cancer cell heterogeneity, and stromal interactions within tumor tissue sections |
| **Tissue Domain Analysis** | Identifying distinct tissue compartments (tumor core, invasive margin, stroma, necrosis) through spatially-resolved expression |
| **Developmental Biology** | Mapping gene expression gradients across developing tissues with preserved spatial context |
| **Multi-sample Cohort Studies** | Harmonizing and normalizing data across multiple tissue sections, patients, or platforms for comparative spatial analysis |
| **Platform-Agnostic Analysis** | Providing a unified preprocessing entry point regardless of whether data originates from Visium, CosMx-SMI, GeoMx, or generic count matrices |

### 1.3 Supported Spatial Transcriptomics Platforms

| Platform | Technology | Resolution | Data Mode |
|---|---|---|---|
| **10x Genomics Visium** | Barcoded oligonucleotide capture array | ~55 µm spots | Sequencing-based |
| **NanoString CosMx-SMI** | In situ hybridization (single-molecule imaging) | Single-cell | Imaging-based |
| **NanoString GeoMx DSP** | UV-cleaved barcodes from ROI selection | ~200–600 µm ROIs | Sequencing-based |
| **Generic CSV/TSV** | Any platform producing count matrices + coordinates | User-defined | Any |
| **Seurat Objects** | Can ingest pre-existing Seurat objects with spatial data | Inherited | Any |

Option B - Separate Files:
  rnacounts: filtered_feature_bc_matrix.csv

```
Required:
  rnacounts: expression_matrix.tsv (genes × ROIs)

  samples: sample_annotation.csv (ROI metadata, patient info, etc.)
```

### 2.2 Output Files

| File | Format | Description |
|---|---|---|
| **`<prefix>.rds`** | RDS (binary R) | Primary output: serialized STlist S4 object |

#### STlist S4 Object Internal Structure

```
STlist
├── @counts          List of sparse matrices (genes × spots/cells) — one per sample; RAW counts
├── @norm_counts     List of normalized/transformed matrices — populated after transform_data()
├── @spatial_meta    List of data frames with spatial coordinates and spot-level metadata per sample
├── @sample_meta     Data frame with sample-level clinical/experimental metadata
├── @pseudobulk      Data frame with aggregated pseudo-bulk counts — populated after pseudobulk_samples()
└── @misc            Named list for miscellaneous results from downstream analyses (clustering, etc.)
```

> **⚠️ Key Note on RDS Naming**: The module version note explicitly addresses a bug fix: *"Strip .tar from output filenames if it's part of the input filename. Stop putting .rds on the rds file twice."* This means the output is a clean `<sample_name>.rds`, not `<sample_name>.tar.rds` or `<sample_name>.rds.rds`.

## 3. 🔧 Parameter Analysis and Usage Patterns

### 3.1 Complete Parameter Catalog

The module's parameters map to four underlying spatialGE R functions executed sequentially:

---

#### **GROUP 1: Data Ingestion Parameters** (`STlist()`)

| Parameter | R Argument | Type | Default | Description |
|---|---|---|---|---|
| `expression_files` | `rnacounts` | File/Path(s) | *Required* | Gene expression count matrix/matrices. Accepts: file path(s) to CSV/TSV/H5 files, Space Ranger directory paths, or CosMx expression matrix. Multi-sample: pass a vector of file paths |
| `coordinate_files` | `spotcoords` | File/Path(s) | NULL | Spatial coordinate file(s). NULL for Visium (auto-detected) and CosMx (from metadata). For generic: CSV/TSV with spot_id, x, y columns |
| `metadata_file` | `samples` | File | NULL | Sample-level metadata CSV/TSV. One row per sample. Required for CosMx. Enables downstream clinical correlations |
| `cores` | `cores` | Integer | NULL | Parallel processing cores. Uses fork-based parallelization. **Not available on Windows** (falls back to single-core) |
| `verbose` | `verbose` | Logical | TRUE | Print progress messages during ingestion |

---

#### **GROUP 2: Filtering Parameters** (`filter_data()`)

| Parameter | R Argument | Type | Default | Valid Range | Description |
|---|---|---|---|---|---|
| `spot_pct` | `spot_pct` | Numeric | 0.1 | 0–1 | Min fraction of spots where a gene must be detected to retain that gene. E.g., 0.1 = gene must appear in ≥10% of spots |
| `genes_pct` | `genes_pct` | Numeric | 0.1 | 0–1 | Min fraction of genes that must be expressed in a spot to retain that spot. E.g., 0.1 = spot must express ≥10% of all genes |
| `rm_genes` | `rm_genes` | Character | NULL | — | Gene names or regex patterns to exclude. Common uses: `"^MT-"` (human mitochondrial), `"^mt-"` (mouse mitochondrial), `"^RP[SL]"` (ribosomal), `"NegControlProbe"` (GeoMx negative controls) |
| `min_gene_exp` | `min_gene_exp` | Numeric | 0 | ≥0 | Expression threshold: counts ≤ this value are treated as "not expressed" for percentage calculations |
| `filter_samples` | `sample_names` | Character | NULL | — | Process only named samples; NULL = process all samples in STlist |

---

#### **GROUP 3: Pseudo-bulk Parameters** (`pseudobulk_samples()`)

| Parameter | R Argument | Type | Default | Options | Description |
|---|---|---|---|---|---|
| `pseudobulk_aggregate` | `aggregate` | Character | "sum" | "sum", "mean" | Aggregation method for collapsing all spots per sample into one pseudo-bulk profile. "sum" = total counts; "mean" = average counts |
| `run_pseudobulk` | — | Logical | FALSE | TRUE/FALSE | GenePattern-level flag to enable/disable the pseudobulk step (optional step in pipeline) |

---

#### **GROUP 4: Transformation Parameters** (`transform_data()`)

| Parameter | R Argument | Type | Default | Options | Description |
|---|---|---|---|---|---|
| `transform_method` | `method` | Character | "log" | "log", "sct" | **"log"**: log1p(counts/library_size × scale_f) — recommended general-purpose method. **"sct"**: SCTransform via Seurat — negative binomial regression-based variance stabilization; filters additional low-expression genes |
| `scale_f` | `scale_f` | Numeric | 10000 | Any positive number | Scaling factor applied before log transformation. Default 10,000 ≈ "counts per 10K" (CPM-like). Only applies when `method = "log"` |
| `transform_cores` | `cores` | Integer | NULL | ≥1 | Parallel cores for transformation (especially relevant for 'sct' method). Not available on Windows |

### 3.3 Order of Operations — Critical Constraint

```
┌──────────────────────────────────────────────────────────────────────────┐
│    REQUIRED OPERATION ORDER IN spatialGE.Preprocessing                   │
│                                                                          │
│  1. STlist()             ── Data ingestion                              │
│         ↓                                                                │
│  2. filter_data()        ── QC filtering (before normalization!)         │
│         ↓                                                                │
│  3. pseudobulk_samples() ── [OPTIONAL] Sample-level aggregation          │
│         ↓                                                                │
│  4. transform_data()     ── Normalization/transformation                 │
│         ↓                                                                │
│  Output: RDS (STlist)                                                   │
└──────────────────────────────────────────────────────────────────────────┘
```

> **⚠️ Important**: Filtering MUST be applied before transformation. Running `transform_data()` before `filter_data()` would normalize based on library sizes that include technical noise genes (e.g., mitochondrial genes), producing suboptimal results.

### 5.3 Quality Control Metrics

After preprocessing, the following QC metrics can be assessed:
- **Spots per sample**: Number of spots/cells retained after filtering
- **Genes per sample**: Number of genes detected after filtering
- **Library size distribution**: Per-spot total counts (via `distribution_plots()`)
- **Pseudo-bulk PCA**: Inter-sample variation and potential batch effects (via `pseudobulk_pca_plot()`)
- **Pseudo-bulk heatmap**: Gene expression clustering across samples (via `pseudobulk_heatmap()`)

### 6.3 Known Issues and Caveats

| Issue | Impact | Mitigation |
|---|---|---|
| **Windows parallelism** | `cores` parameter is silently ignored on Windows; only single-core execution | Document clearly; ensure fallback works correctly |
| **`method='sct'` gene reduction** | SCTransform removes additional low-expression genes beyond `filter_data` thresholds | Warn users that `sct` produces fewer genes in `@norm_counts` than `log` |
| **CosMx FOV separation** | STlist automatically splits CosMx data by FOV — one "sample" per FOV in STlist | Users expecting whole-slide analysis need to understand FOV-level structure |
| **HDF5 system dependency** | `hdf5r` requires system-level HDF5 library (`libhdf5-dev`) | Must be pre-installed in Docker container |
| **RDS file naming bug** | Version note indicates a fixed bug where `.tar` in input names propagated to output, and `.rds` was appended twice | Resolved in current version; document expected output filename format |
| **Memory for large datasets** | CosMx datasets with millions of cells can exhaust memory | Recommend ≥64 GB RAM; consider chunked processing |
| **Coordinate auto-detection** | Visium coordinate auto-detection requires Space Ranger standard directory structure | If directory structure is non-standard, explicit coordinate files are needed |

**Key Competitive Advantages of spatialGE.Preprocessing:**
1. **Only R-based tool with native GeoMx support** — critically important for DSP users
2. **Unified multi-platform input** — one function handles Visium/CosMx/GeoMx/Generic
3. **Built-in pseudo-bulk QC** — sample-level PCA immediately after ingestion
4. **Seamless GenePattern pipeline** — designed as entry point with clean RDS handoff
5. **Tumor microenvironment focus** — explicitly designed and validated for cancer research applications

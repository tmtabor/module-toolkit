## 1. 🎯 Tool Purpose & Scientific Applications

### 1.1 Core Purpose
ComparativeMarkerSelection is a bioinformatics module within the GenePattern analysis platform designed to **identify differentially expressed genes (marker genes) that can discriminate between two distinct biological sample classes**. It operates on gene expression data, ranking every gene in a dataset by how well its expression profile separates the two phenotypic classes, and assigns rigorous statistical significance to each ranking.

The module implements the analytical layer of the *Comparative Gene Marker Selection Suite* — a three-module ecosystem that together covers analysis, visualization, and result extraction for differential expression workflows.

### 1.2 Scientific Background & Foundational Methods
The tool's roots trace to the landmark **Golub et al. (1999)** *Science* paper on molecular classification of cancer using gene expression. That work introduced the **Signal-to-Noise Ratio (SNR)** metric — one of ComparativeMarkerSelection's core test statistics — and established the canonical ALL/AML leukemia dataset still used as the example dataset for the module today.

The module bridges **permutation statistics** (Good, 2000) with modern **multiple hypothesis testing (MHT) correction** frameworks (Benjamini-Hochberg 1995; Westfall & Young 1993; Storey & Tibshirani 2003), making it one of the most statistically comprehensive single-gene differential expression tools available within GenePattern.

### 1.3 Key Scientific Applications

| Application Domain | Description |
|---|---|
| **Cancer Subtype Discovery** | Identify genes distinguishing tumor subtypes (e.g., ALL vs. AML leukemia) |
| **Biomarker Discovery** | Find candidate genes for clinical diagnostics or prognosis |
| **Phenotype-Expression Correlation** | Link molecular signatures to clinical phenotypes |
| **Class Prediction Preprocessing** | Generate ranked gene lists as input for classifiers (e.g., k-NN, SVM) |
| **Pathway Analysis Input** | Feed ranked marker lists into GSEA or other pathway-enrichment analyses |
| **Treatment Response Studies** | Compare treated vs. untreated or responder vs. non-responder profiles |
| **Microarray Quality Validation** | Verify separation of biological classes in expression data |

### 1.4 What Makes v11 Different
The critical upgrade in v11 is the **containerization in a dedicated R 2.5 Docker environment**, which resolves a key prior limitation: the `qvalue` R package (Storey/Bioconductor) is now fully bundled within the container. This enables **Q-value computation** — a statistically more powerful FDR estimation method than BH because it explicitly estimates π₀ (the fraction of truly null hypotheses), thereby increasing sensitivity for detecting true positives in large genomic screens.

## 2. 📁 Input/Output Formats & Requirements

### 2.1 Input Files

#### Primary Expression File (Required)
The module accepts two gene expression matrix formats:

**GCT Format (`.gct`)** — most common
```
#1.2
<num_genes>\t<num_samples>
Name\tDescription\tSample1\tSample2\t...
GENE_ID_1\tDescription1\t5.23\t3.11\t...
GENE_ID_2\tDescription2\t1.04\t7.82\t...
```
- Tab-delimited text
- First line: `#1.2` (format version)
- Second line: `nrows` and `ncols`
- Third line: column headers (Name, Description, then sample names)
- Subsequent lines: one gene per row

**RES Format (`.res`)** — Affymetrix-specific
- Similar to GCT but includes additional per-sample **"call" columns** (A/P/M: Absent/Present/Marginal)
- Used with older Affymetrix microarray workflows

#### Class Labels File (Required)

**CLS Format (`.cls`)** — space-delimited (NOT tab-delimited)
```
38 2 1
# ALL AML
0 0 0 1 1 1 0 0 ...
```
- Line 1: `<total_samples> <num_classes> 1`
- Line 2: `# <class0_name> <class1_name>` (comment line with class names)
- Line 3: space-delimited integer labels (0 or 1) for each sample in order

> ⚠️ **Critical Note:** CLS files must be **space-delimited**, not tab-delimited. Tab-delimited CLS files are the most common cause of input errors.

### 2.2 Input Data Requirements & Constraints

| Constraint | Details |
|---|---|
| **Minimum samples per class** | **3 samples per class** (hard minimum; module will fail with fewer) |
| **Recommended samples per class** | ≥10 per class for reliable p-values with 1000 permutations |
| **Data type** | Continuous expression values (microarray intensities, RPKM/FPKM/TPM for RNA-seq) |
| **Log transformation** | Module assumes **non-log-transformed** data by default; if log-transformed, set `log.transformed.data=yes` |
| **Two-class only** | Strictly designed for binary class comparisons; multi-class not supported |
| **Data scale** | No strict requirements, but SNR statistic is sensitive to the `min.std` threshold |

### 2.3 Output Files

**Primary Output: `.comp.marker.odf`**

The output file uses GenePattern's proprietary **ODF (Output Description Format)** — a structured tab-delimited text format with a metadata header block:

```
ODF 1.0
HeaderLines=N
Model=ComparativeMarkerSelection
DataLines=<num_genes>
...header key-value pairs...
<column headers>
<data rows>
```

Each row (one per gene) contains the following columns:

| Column | Type | Description |
|---|---|---|
| `Rank` | Integer | Gene rank (1 = most discriminatory) |
| `Score` | Float | Value of the test statistic (t-stat or SNR) |
| `Feature P` | Float | Permutation-based nominal p-value |
| `Feature P Low` | Float | Estimated lower bound for feature p-value |
| `Feature P High` | Float | Estimated upper bound for feature p-value |
| `FDR (BH)` | Float | Benjamini-Hochberg FDR estimate |
| `Q Value` | Float | Storey FDR estimate (estimates π₀) — **new in v11** |
| `Bonferroni` | Float | Bonferroni-corrected p-value |
| `maxT` | Float | Westfall-Young maxT adjusted p-value (FWER) |
| `FWER` | Float | Family-wise error rate |
| `Fold Change` | Float | Ratio of mean expression (class 1 / class 0) |

> **Output filename convention:** `<input_file_stem>.comp.marker.odf` (e.g., `all_aml_train.comp.marker.odf`)

## 3. ⚙️ Parameter Analysis & Usage Patterns

### 3.1 Complete Parameter Catalog

#### Group 1: Required Inputs

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `input.filename` | File (GCT/RES) | ✅ Yes | — | Gene expression dataset |
| `cls.filename` | File (CLS) | ✅ Yes | — | Sample class labels |

#### Group 2: Statistical Test Configuration

| Parameter | Type | Required | Default | Options | Description |
|---|---|---|---|---|---|
| `test.statistic` | Enum | No | `t-test` | `t-test`, `SNR`, `paired t-test` | Statistical test for differential expression scoring |
| `test.direction` | Enum | No | `2-sided` | `2-sided`, `class 0`, `class 1` | Direction of hypothesis test |
| `min.std` | Float | No | `0` | ≥0 | Min standard deviation per class; filters low-variance genes |

**Test Statistic Details:**
- **`t-test`** (default): Standard two-sample Student's t-test. Most widely applicable; assumes approximately normal distributions within each class.
- **`SNR` (Signal-to-Noise Ratio)**: `(μ₁ - μ₀) / (σ₁ + σ₀)` — from Golub et al. 1999. Directly measures the separation of class means in units of the combined standard deviations. Sensitive to `min.std` threshold.
- **`paired t-test`**: For matched/paired experimental designs (e.g., before/after treatment on same patients). Samples must be ordered such that pairs are consecutively aligned.

**Test Direction Details:**
- **`2-sided`**: Tests for differential expression in either direction — gene is a marker if overexpressed in *either* class.
- **`class 0`**: One-sided test; identifies genes preferentially upregulated in class 0.
- **`class 1`**: One-sided test; identifies genes preferentially upregulated in class 1.

#### Group 3: Permutation Configuration

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `number.of.permutations` | Integer | No | `1000` | Number of random permutations for null distribution |
| `complete` | Boolean | No | `no` | If `yes`, compute p-values from ALL possible permutations |
| `balanced` | Boolean | No | `no` | If `yes`, use balanced permutations (equal class sizes per permutation) |
| `random.seed` | Integer | No | `779948241` | Random seed for reproducibility |
| `smooth.p.values` | Boolean | No | `yes` | Apply smoothing to estimated p-values |

**Critical permutation guidance:**
- **Default (1000 random permutations):** Appropriate for datasets with ≥10 samples per class. Provides reliable p-value estimates.
- **`complete=yes`:** Overrides `number.of.permutations`. Computes the *exact* p-value from the complete permutation space. **Only use this when the total number of possible permutations is < 1000** (i.e., very small datasets). For n₀ samples in class 0 and n₁ in class 1, total permutations = C(n₀+n₁, n₀).
- **`balanced=yes`:** Ensures each permuted dataset maintains equal class representation, which can reduce variability in the null distribution estimate.
- **`random.seed`:** Setting a fixed seed ensures fully reproducible results. The default seed (779948241) is fixed across runs for consistency.

#### Group 4: Data Preprocessing Flags

| Parameter | Type | Required | Default | Options | Description |
|---|---|---|---|---|---|
| `log.transformed.data` | Boolean | No | `no` | `yes`, `no` | Flag indicating whether input data is log₂-transformed |

> ⚠️ **Most common error:** Providing log-transformed microarray data without setting this flag to `yes`. The fold-change calculation (linear ratio of means) will be **incorrect** if the data is already on a log scale and this parameter is not set. The test statistic itself (t-test/SNR) is computed on whatever scale is provided; only fold change is affected.

#### Group 5: Output Configuration

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `output.filename` | String | No | `<input_stem>.comp.marker.odf` | Name for the output ODF file |

### 3.3 Statistical Output Interpretation Guide

| Metric | Statistical Framework | Typical Threshold | Best Used When |
|---|---|---|---|
| **Feature P** | Permutation nominal p-value | <0.05 | Exploratory analysis, large datasets |
| **FDR (BH)** | Benjamini-Hochberg 1995 | <0.05 or <0.25 | Standard FDR control; conservative |
| **Q Value** | Storey & Tibshirani 2003 | <0.05 or <0.10 | When power is important; estimates π₀ |
| **Bonferroni** | Classical FWER correction | <0.05 | When false positives are very costly |
| **maxT** | Westfall-Young FWER | <0.05 | Strong FWER control with permutations |
| **FWER** | Family-wise error rate | <0.05 | Strict error rate control |
| **Fold Change** | Linear ratio of means | >2x or >1.5x | Biological significance screening |

> **BH vs. Q-value:** The Q-value method (Storey) provides more statistical power than BH because it estimates what fraction of genes are truly differentially expressed (1−π₀) and adjusts accordingly. BH is more conservative but makes no assumptions about π₀. For genomic screens with many true positives, Q-value is preferred. This is the key reason for the R container upgrade in v11.

### 6.4 Known Limitations & Integration Caveats

| Issue | Impact | Mitigation |
|---|---|---|
| **Two-class only** | Cannot handle multiclass designs | Document clearly; suggest preprocessing to create binary comparisons |
| **Not count-data aware** | Suboptimal for raw RNA-seq counts | Recommend DESeq2/edgeR for count data; CMS appropriate for TPM/FPKM |
| **Minimum 3 samples/class** | Will fail with tiny datasets | Input validation in module wrapper |
| **Log-transform flag** | Fold change errors if misset | Explicit warning in UI; mark as important parameter |
| **Complete permutations** | Computationally explosive for large n | Document usage limits; compute max feasible permutations |
| **Q-value in v11** | Requires R 2.5 container | This is why containerization is essential |
| **ODF format** | Proprietary format, not directly interoperable | Document conversion; provide downstream module connections |

### 6.5 Comparative Tool Positioning

| Tool | Best For | CMS Advantage |
|---|---|---|
| **DESeq2 / edgeR** | RNA-seq count data | CMS better for normalized continuous data; simpler interface |
| **limma** | Complex multi-factor designs | CMS simpler for straightforward 2-class; comprehensive MHT |
| **SAM** | Similar use case | CMS has deeper GenePattern integration + more MHT options |
| **GSEA** | Gene-set level analysis | CMS is gene-level; complementary (CMS output feeds GSEA) |

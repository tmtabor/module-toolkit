### Scientific Purpose
Seurat.IntegrateData implements the **anchor-based batch correction algorithm** from the Seurat v3/v4 framework. It solves one of the most critical challenges in single-cell RNA-seq (scRNA-seq) analysis: **technical batch effects** — systematic, non-biological differences introduced by experimental variables such as:

- Sequencing technology (10X Genomics vs. Smart-seq2 vs. Drop-seq)
- Processing laboratory or operator
- Sample collection date or batch
- Tissue dissociation protocol
- Experimental condition (disease vs. control)

Without correction, these technical effects dominate clustering analyses, causing cells from the same biological state across batches to cluster separately rather than together, masking true biological variation.

### Core Scientific Goal
> *"The module integrates (corrects for batch effects) multiple single-cell datasets and identifies shared cell states that are present across different datasets, regardless of their origin."*
> — GenePattern `Seurat.IntegrateData/doc.md`

### Research Applications
- **Multi-site studies**: Integrating datasets collected at different institutions
- **Technology harmonization**: Combining 10X, Smart-seq2, CEL-seq2, and other platforms
- **Longitudinal analyses**: Joining samples from multiple time points
- **Disease atlases**: Constructing unified reference maps from disease and control samples
- **Cross-species analyses**: Integrating orthologous gene expression data
- **Meta-analyses**: Re-analyzing multiple published datasets simultaneously

## 2. ⚙️ Underlying Algorithm

### Mathematical Framework — Anchor-Based CCA Integration

The algorithm (Stuart, Butler et al., *Cell* 2019) operates in **10 sequential steps**:

```
┌─────────────────────────────────────────────────────────────────┐
│  STEP 1: Normalize each dataset independently                   │
│          (LogNormalize OR SCTransform)                          │
│                         ↓                                       │
│  STEP 2: Identify highly variable features                      │
│          (anchor.features = 2000 per dataset)                   │
│                         ↓                                       │
│  STEP 3: Dimensional Reduction                                  │
│          CCA: Canonical Correlation Analysis (default)          │
│          RPCA: Reciprocal PCA (fast alternative)                │
│                         ↓                                       │
│  STEP 4: L2-Normalize CCA/PCA vectors                           │
│                         ↓                                       │
│  STEP 5: Anchor Identification                                  │
│          Find Mutual Nearest Neighbors in CCA space             │
│          (k.anchor = 5 neighbors per cell)                      │
│                         ↓                                       │
│  STEP 6: Anchor Filtering                                       │
│          Remove spurious anchors via high-dim neighbor check    │
│          (k.filter = 200 neighbors)                             │
│                         ↓                                       │
│  STEP 7: Anchor Scoring                                         │
│          Score pairs by neighborhood overlap                    │
│          (k.score = 30 neighbors)                               │
│                         ↓                                       │
│  STEP 8: Compute Correction Vectors                             │
│          Weighted combination of anchor pair differences        │
│                         ↓                                       │
│  STEP 9: Apply Gaussian Weighting                               │
│          Weight correction by cell proximity to anchors         │
│          (k.weight = 100, sd.weight = 1)                        │
│                         ↓                                       │
│  STEP 10: Output Integrated Assay                               │
│           Returns corrected expression in new "integrated"      │
│           assay within the Seurat object                        │
└─────────────────────────────────────────────────────────────────┘
```

### CCA vs. RPCA Reduction Methods

| Aspect | CCA (Default) | RPCA (Alternative) |
|---|---|---|
| **Method** | Canonical Correlation Analysis | Reciprocal PCA projection |
| **Correction strength** | Strong / Aggressive | Conservative |
| **Speed** | Moderate | Fast (3–5× faster) |
| **Best for** | ≤10 datasets; well-conserved cell types; max differences across experiments | >10 datasets; large datasets; partial cell type overlap |
| **Risk** | Over-integration of distinct populations | Under-correction of strong batch effects |
| **Memory** | Higher | Lower |
| **Parameter to increase alignment** | Increase `k.anchor` | Increase `k.anchor` |

## 3. 📁 Input/Output Specifications

### Input Files

| Format | Description | Notes |
|---|---|---|
| `.rds` | Serialized R Seurat object | **Primary format.** Must contain a list of preprocessed Seurat objects OR a merged object with batch metadata |
| `.tsv` | Tab-separated expression matrix | Genes as rows, cells as columns; header row required |
| `.txt` | Dense expression matrix | Same structure as TSV; used for inter-module compatibility in GenePattern Suite |

#### Input Object Requirements
The input Seurat object must:
- Contain **≥2 datasets/batches** to integrate
- Each batch should have **≥200 cells** (for default `k.filter=200`; adjust if smaller)
- Have been **normalized** (either `NormalizeData()` for LogNormalize or `SCTransform()` for SCT)
- For SCT workflow: `PrepSCTIntegration()` must have been run before `FindIntegrationAnchors()`
- Have **metadata** identifying batch membership (`orig.ident` or custom column)

### Output Files

| Output | Format | Description |
|---|---|---|
| `<output_name>_integrated.rds` | `.rds` | Seurat object with new `integrated` assay added |
| Pre-integration UMAP | `.pdf/.png` | Visualization of batch effects before correction |
| Post-integration UMAP | `.pdf/.png` | Visualization of corrected data |
| Pre-correction matrix | `.txt/.tsv` | Expression matrix before batch correction (added in March 2021 update) |
| Post-correction matrix | `.txt/.tsv` | Integrated expression matrix |

#### Output Assay Behavior by Normalization Method

| Normalization | Integrated Data Stored In | Downstream Use |
|---|---|---|
| `LogNormalize` | `data` slot of `integrated` assay | Switch: `DefaultAssay(obj) <- "integrated"` then run `ScaleData()`, `PCA()`, `UMAP()` |
| `SCT` | `scale.data` slot of `integrated` assay | Run `PCA()`, `UMAP()` directly; use `RNA` assay for DE analysis |

> ⚠️ **Critical**: The integrated assay should **only** be used for **dimensionality reduction, visualization, and clustering**. For differential expression analysis, always revert to the `RNA` (or `SCT`) assay.

| `nn.method` | String | `"annoy"` | `"annoy"`, `"rann"` | Approximate NN method. Annoy is faster for large datasets |
| `n.trees` | Integer | `50` | 10–100 | Trees for Annoy NN search. More trees = better accuracy, slower |

| Parameter | Type | Default | Range | Biological Significance |
|---|---|---|---|---|
| `new.assay.name` | String | `"integrated"` | Any string | Name of new batch-corrected assay in output Seurat object |

| `sd.weight` | Numeric | `1` | 0.1–5 | Bandwidth of Gaussian weighting kernel. Lower = more local correction |

### Complete Multi-Batch R Script Workflow

```r
library(Seurat)

# ── STEP 1: Load and split data ──────────────────────────────────
seurat.obj <- readRDS("input_merged.rds")
seurat.list <- SplitObject(seurat.obj, split.by = "orig.ident")

# ── STEP 2: Normalize each batch independently ───────────────────
# Option A: LogNormalize workflow
seurat.list <- lapply(X = seurat.list, FUN = function(x) {
  x <- NormalizeData(x, verbose = FALSE)
  x <- FindVariableFeatures(x, selection.method = "vst",
                            nfeatures = 2000, verbose = FALSE)
})

# Option B: SCTransform workflow (more accurate, slower)
seurat.list <- lapply(X = seurat.list, FUN = function(x) {
  SCTransform(x, method = "glmGamPoi", verbose = FALSE)
})
features <- SelectIntegrationFeatures(object.list = seurat.list, nfeatures = 3000)
seurat.list <- PrepSCTIntegration(object.list = seurat.list, anchor.features = features)

# ── STEP 3: Find integration anchors ─────────────────────────────
anchors <- FindIntegrationAnchors(
  object.list      = seurat.list,
  normalization.method = "LogNormalize",  # or "SCT"
  reduction        = "cca",               # or "rpca" for large datasets
  anchor.features  = 2000,
  dims             = 1:30,
  k.anchor         = 5,
  k.filter         = 200,
  k.score          = 30
)

# ── STEP 4: Integrate data ────────────────────────────────────────
integrated <- IntegrateData(
  anchorset            = anchors,
  normalization.method = "LogNormalize",
  new.assay.name       = "integrated",
  dims                 = 1:30,
  k.weight             = 100
)

# ── STEP 5: Standard downstream analysis ─────────────────────────
DefaultAssay(integrated) <- "integrated"
integrated <- ScaleData(integrated, verbose = FALSE)
integrated <- RunPCA(integrated, npcs = 30, verbose = FALSE)
integrated <- RunUMAP(integrated, reduction = "pca", dims = 1:30)
integrated <- FindNeighbors(integrated, reduction = "pca", dims = 1:30)
integrated <- FindClusters(integrated, resolution = 0.5)

# ── STEP 6: Save output ───────────────────────────────────────────
saveRDS(integrated, "seurat_integrated.rds")
```

## 8. ⚠️ Limitations & Caveats

### Biological Limitations

| Limitation | Detail | Mitigation |
|---|---|---|
| **Over-integration risk** | CCA can merge genuinely distinct populations that happen to share some gene expression patterns | Use RPCA; visually inspect pre/post UMAPs; validate with known markers |
| **Cell type conservation assumption** | Assumes shared cell types exist across batches; violating this gives poor results | Check for overlap with known marker genes before integrating |
| **Not for DE analysis** | The integrated matrix is NOT count data; it cannot be used with DESeq2, edgeR, or Wilcoxon count-based tests | Always revert to `RNA` assay for differential expression |
| **Feature restriction** | By default, only anchor features (~2000 genes) are in the integrated assay; other genes are excluded | Set `features.to.integrate` to include all genes if needed |

### Technical Limitations

| Limitation | Detail | Mitigation |
|---|---|---|
| **Memory scaling** | CCA computation is O(n²) with cells; becomes prohibitive >500k cells | Use RPCA; subset cells; use reference-based integration |
| **Small dataset failure** | `k.filter=200` requires ≥200 cells per batch | Set `k.filter=NA` for batches with <200 cells |
| **Multi-dataset complexity** | Not designed for >50 datasets simultaneously | Use `reference` parameter or hierarchical integration strategy |
| **SCT preprocessing cost** | `PrepSCTIntegration()` is memory- and compute-intensive | Pre-run as separate module step; use LogNormalize for large datasets |
| **Integration order dependency** | With >2 datasets, order matters (controlled by `sample.tree`) | Use `sample.tree` for complex experimental designs; default is adequate for most uses |

### When **NOT** to Use This Module

> ❌ When datasets have **completely non-overlapping cell populations** (use `FindTransferAnchors()` / label transfer instead)
> ❌ When **preserving biological differences between conditions** is the primary goal
> ❌ When working with **>100 datasets** without a reference strategy
> ❌ When **computational resources are severely limited** and only visualization is needed (use Harmony instead)
> ❌ When the downstream analysis requires **count-based statistics** on the corrected matrix

### Error Handling Recommendations

```r
# Recommended error handling within the R script
tryCatch({
  anchors <- FindIntegrationAnchors(...)
}, error = function(e) {
  if (grepl("k.filter", e$message)) {
    message("⚠️ k.filter too large for dataset size. Retrying with k.filter=NA...")
    anchors <- FindIntegrationAnchors(..., k.filter = NA)
  } else {
    stop(paste("FindIntegrationAnchors failed:", e$message))
  }
})
```

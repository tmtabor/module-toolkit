# 🧬 Comprehensive Research Report: `spatialGE.STgradient.Launcher`

---

## 1. EXECUTIVE SUMMARY

`spatialGE.STgradient.Launcher` is a **JavaScript-based interactive bridge module** in the GenePattern platform that occupies a critical position in the spatialGE spatial transcriptomics analysis pipeline. It sits between `spatialGE.STclust` (tissue niche detection) and `spatialGE.STgradient` (the core R-based gradient analysis engine). Its primary function is to **read the upstream clustering output, dynamically present the user with data-derived choices** (namely, which tissue niche/cluster to use as the spatial reference), and then **configure and launch** the downstream `spatialGE.STgradient` analysis module.

The underlying statistical methodology — implemented in the `spatialGE` R package (v1.2.0, Fridley Lab, Moffitt Cancer Center) — calculates **Spearman's rank correlation coefficients** between gene expression levels and the minimum Euclidean distance of each spot from a user-designated "reference" tissue niche, enabling detection of genes whose expression systematically increases or decreases as a function of spatial proximity to that reference domain.

## 2. SCIENTIFIC BACKGROUND & PURPOSE

### 2.1 Biological Problem Addressed

Tissues are spatially organized, and gene expression does not merely differ between discrete tissue compartments — it often **transitions continuously** along spatial axes between tissue niches. Traditional differential expression analysis compares groups (cluster A vs. cluster B), but fails to capture the **gradient nature** of many biological processes:

- Tumor-stroma interface gene expression gradients
- Inflammatory front infiltration patterns
- Developmental zone transitions (e.g., proliferative vs. post-mitotic zones)
- Hypoxia gradients emanating from necrotic cores
- Morphogen gradients in developmental biology

STgradient was specifically designed to ask: *"For a given reference tissue niche (e.g., tumor), which genes show systematic, distance-dependent expression changes across the surrounding tissue?"*

### 2.2 Statistical Framework

The algorithm follows these steps:

```
1. INPUT:  STlist with STclust-defined tissue niches
2. REFERENCE: User designates one niche (e.g., "tumor") as the reference
3. DISTANCE: For each non-reference spot, compute minimum Euclidean distance to 
             the nearest reference spot (or alternative distance summary metric)
4. CORRELATION: Compute Spearman's ρ between gene expression and distance vector
   - ρ < 0: Gene expression DECREASES as distance from reference INCREASES
             (i.e., higher expression closer to reference niche)
   - ρ > 0: Gene expression INCREASES as distance from reference INCREASES
             (i.e., lower expression closer to reference niche)
5. SIGNIFICANCE: Permutation testing to compute empirical p-values
6. CORRECTION: Multiple testing adjustment (FDR/BH method)
7. OUTPUT: Updated STlist + results table with ρ, p-value, adj.p-value per gene
```

### 2.3 Validated Applications

| Study | Application | Key Finding |
|---|---|---|
| PMC11228800 (Melanoma Leptomeningeal Disease) | Tumor-stroma spatial gradients | Identified NFKB-TNFA signaling enrichment near tumor boundary; results validated by in vivo mouse experiments |
| PMC11244876 (spatialGE Web App) | Tumor vs. non-tumor niche gradients in melanoma | Demonstrated complete STclust → STgradient → STenrich workflow |
| PMC11873723 (Cancer Research 2025) | Multiple cancer spatial transcriptomics datasets | Validated as "hypothesis-generative tool" for spatial biology |

### 3.2 The Launcher Design Pattern

The `Launcher` pattern in GenePattern is a specialized design where a **JavaScript UI module** acts as an intelligent intermediary:

| Characteristic | Description |
|---|---|
| **Language** | JavaScript (front-end/UI logic) |
| **Role** | Dynamic parameter collection, not statistical computation |
| **Key Capability** | Reads upstream RDS output to populate dropdowns dynamically |
| **What it launches** | `spatialGE.STgradient` (the R-based computational module) |
| **User interaction** | Presents a UI form where users select reference cluster from a list extracted from their actual clustering results |

This design is critical because the **reference cluster name is data-dependent** — it comes from what STclust labeled the clusters in the user's specific dataset (e.g., "cluster_1", "tumor", "niche_A"), and cannot be hardcoded in advance.

### 4.2 Parameters Collected by the Launcher UI

These are the parameters the Launcher presents to the user before submitting to `spatialGE.STgradient`:

| Parameter | R Argument | Type | Default | Description |

| **Reference Cluster** | `ref` | String (dropdown) | *(none)* | **[CRITICAL]** The tissue niche/domain to use as the spatial reference point. Distances are computed FROM all non-reference spots TO the nearest reference spot. Dynamically populated from STclust output. |
| **Top Genes** | `topgenes` | Integer | `2000` | Number of top variably expressed genes to test. Higher values are more comprehensive but computationally costlier. |
| **Samples** | `samples` | Multi-select | `NULL` (all) | Which samples in the multi-sample STlist to include. |
| **Annotation Column** | `annot` | String | `NULL` | Which metadata column holds cluster assignments (if multiple clustering results exist). |
| **Exclude Clusters** | `exclude` | Multi-select | `NULL` | Cluster names to exclude from distance calculations and testing. |
| **Remove Outliers** | `out_rm` | Boolean | `FALSE` | Whether to apply outlier spot removal prior to analysis. |
| **Distance Limit** | `limit` | Numeric | `NULL` | Maximum distance threshold; spots beyond this distance are excluded from correlation. |
| **Distance Summary** | `distsumm` | Select | `"min"` | How to summarize distance to reference (`"min"` = minimum distance to any reference spot). |
| **Min. Neighborhood** | `min_nb` | Integer | `3` | Minimum number of spots required in the reference niche (quality control). |
| **Robust Correlation** | `robust` | Boolean | `TRUE` | Use robust Spearman correlation implementation. |
| **Neighborhood Distance Threshold** | `nb_dist_thr` | Numeric | `NULL` | Distance threshold for defining the local neighborhood. |
| **Log-transform Distances** | `log_dist` | Boolean | `FALSE` | Apply log-transformation to distance values before computing correlations. |

## 11. KNOWN LIMITATIONS & CAVEATS

| Limitation | Details | Mitigation |
|---|---|---|
| **Requires prior clustering** | STgradient depends on STclust output; cluster quality directly affects gradient results | Ensure robust STclust parameters before proceeding |
| **Reference niche subjectivity** | User must manually identify which cluster represents the biologically meaningful reference | Provide biological annotation guidance in documentation |
| **Assumes continuous gradients** | Spearman correlation assumes monotonic expression-distance relationship; non-monotonic patterns are missed | Supplement with STenrich for non-linear spatial patterns |
| **Spot-level resolution** | Designed for Visium-like spot data; may need adaptation for single-cell resolution platforms | Verify spot size vs. biological gradient scale |
| **Sparsity sensitivity** | Lowly expressed genes may have spurious gradient signals | `topgenes` parameter helps by filtering to high-variance genes |
| **Distance metric** | Only Euclidean distance with `"min"` summary currently available | Future versions may support other distance metrics |
| **Multiple testing burden** | Testing 2000+ genes requires stringent FDR correction; may miss subtle gradients | Adjust `topgenes` and significance threshold based on study goals |

### 1.2 Primary Scientific Function

`tfsites.FindTfSitesAlteredBySequenceVariation` performs **exhaustive in silico single-nucleotide variant (SNV) scanning** of a user-provided DNA sequence against a scored motif database for **one transcription factor (TF)**. For every position in the input sequence, the tool tests all three possible nucleotide substitutions and classifies each resulting SNV into one of four biological effect categories:

| Effect Category | Biological Meaning | Research Relevance |
|---|---|---|
| **OPTIMIZING** | SNV *increases* affinity/score of an existing binding site | Gain-of-function, potential disease cause |
| **DECREASING** | SNV *reduces* affinity/score of an existing binding site | Partial loss of binding |
| **DELETING** | SNV *eliminates* a binding site entirely (score drops below threshold) | Loss-of-function, classical pathogenic variant |
| **CREATING** | SNV *creates* a new binding site where none existed | De novo binding, ectopic TF recruitment |

### 1.3 Scientific Rationale and Novelty

This tool is built upon a paradigm-shifting discovery from the Farley Lab: **enhancers deliberately use LOW-affinity TF binding sites** to achieve tissue-specific, precise, and time-controlled gene expression. This "suboptimization" principle (Farley et al., 2015, *Science*) means that:

- SNVs that **INCREASE** binding affinity (optimizing variants) are often **MORE pathogenic** than those that decrease it — the opposite of classical expectation
- Even tiny affinity gains (e.g., from score 0.15 to 0.25) can dramatically disrupt developmental gene expression
- These optimizing variants cause **gain-of-function (GOF)** expression patterns, not loss-of-function

> *"The prevalence of low-affinity binding sites within enhancers creates a vulnerability in genomes whereby single nucleotide variants that optimize binding affinity, even slightly, can be pathogenic."* — Farley Lab, Featured Papers

## 3. 📥📤 INPUT/OUTPUT FORMATS AND REQUIREMENTS

### 3.1 Input Files

| Input | Format | Description | Required? |
|---|---|---|---|
| **DNA Sequence File** | FASTA (`.fa`, `.fasta`) | The enhancer/regulatory sequence to scan | ✅ Required |
| **Motif Database File** | TSV/custom k-mer DB (`.db`, `.txt`, `.tsv`) | Pre-generated k-mer affinity scoring database for the TF of interest | ✅ Required (unless combined mode) |
| **JASPAR PFM File** | JASPAR format (`.jaspar`, `.pfm`, `.txt`) | Position Frequency Matrix — only needed in **combined mode** | 🔵 Optional (combined mode) |

#### DNA Sequence File Requirements
```
>MyEnhancer_sequence
ATCGGATCGATCGATCGATCGATCGATCG...
```
- Single or multi-FASTA accepted
- Typically 100–2,000 bp enhancer/regulatory sequences
- Standard IUPAC DNA alphabet (A, T, G, C)
- Both strands analyzed depending on parameter settings

#### Motif Database File
- Generated by companion module `tfsites.GenerateMotifDatabase` (also known as `tfsites.DefineTfBindingSitesFromPFM`)
- Contains all k-mer sequences and their normalized MFI (Median Fluorescence Intensity) affinity scores for a specific TF
- Scores are normalized 0.0–1.0 (1.0 = highest affinity k-mer conforming to IUPAC consensus)
- Derived from JASPAR PFMs, HT-SELEX data, or protein binding microarray (PBM) data

### 3.2 Output Files

| Output | Format | Description |
|---|---|---|
| **SNV Effects Table** | TSV (`.tsv`) | Main results table — one row per possible SNV |
| **Visualization Image** | PNG/SVG (`.png`, `.svg`) | Heatmap or annotated sequence plot showing SNV effect landscape |
| **Summary Statistics** | TXT/JSON | Counts of each SNV effect category |
| **Motif DB** *(combined mode)* | TSV/custom | Generated motif database (if combined mode enabled) |

#### SNV Effects Table Schema (Expected Columns)

| Column | Type | Description |
|---|---|---|
| `position` | Integer | 1-based position in the input sequence |
| `original_nucleotide` | Char | Reference/wild-type nucleotide |
| `mutant_nucleotide` | Char | Substituted nucleotide |
| `original_score` | Float | Affinity score at this position with wild-type sequence |
| `mutant_score` | Float | Affinity score at this position with the SNV |
| `score_delta` | Float | Change in score (mutant - original) |
| `effect_category` | String | OPTIMIZING / DECREASING / DELETING / CREATING |
| `site_coordinates` | String | Genomic coordinates of affected binding site window |

> **Scale note:** For a 500 bp sequence, the table contains ~1,500 rows (3 substitutions × 500 positions), enabling visualization as a comprehensive mutational heatmap.

### 4.2 Threshold Parameter Significance

The `binding.site.threshold` is biologically critical:
- **Lower thresholds** (e.g., 0.1) detect very low-affinity sites — important based on Farley lab research showing even 0.15-affinity sites are functional
- **Higher thresholds** (e.g., 0.5) focus only on canonical high-affinity sites
- **Recommended default of 0.25** is consistent with Farley lab findings that 0.25-affinity sites can be sufficient to drive expression and disrupt development when optimized

### 4.3 Combined Mode (New Feature)

The added "generate motif DB then run all in one module" feature is significant for usability:
- **Before:** User required two separate module runs: `tfsites.GenerateMotifDatabase` → `tfsites.FindTfSitesAlteredBySequenceVariation`
- **After (combined mode):** Single module run with JASPAR PFM as input; motif DB generated internally and then used immediately
- Ideal for **first-time users** or **one-off analyses**; pre-generating DB remains preferable for **repeated analyses** of the same TF across multiple sequences

### 10.3 Known Limitations and Caveats

1. **Single TF per run** — multi-TF analyses require multiple module executions; consider pipeline/workflow wrapping
2. **PHI Restriction** — module Terms and Conditions prohibit use with Protected Health Information
3. **Experimental affinity data dependency** — quality of results directly depends on availability and quality of the motif database (which depends on experimental TF binding data)
4. **In silico only** — all predictions require experimental validation; tool provides candidates, not definitive pathogenicity calls
5. **Sequence context not modeled** — k-mer scoring does not account for DNA shape, chromatin context, or cooperative TF binding (addressed in broader tfsites workflow)
6. **Sequence length scalability** — suitable for enhancer/regulatory region scale (100–5,000 bp); genome-scale scanning requires `tfsites.AnalyzeGenomeMPRA`

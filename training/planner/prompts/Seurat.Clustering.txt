# 🔬 Comprehensive Research Report: Seurat.Clustering (GenePattern Module)

## Executive Summary

**Seurat.Clustering** is a GenePattern module wrapping the Seurat v4.0.3 R package to perform **graph-based clustering, dimensionality reduction, and differential marker identification** on preprocessed single-cell RNA-seq (scRNA-seq) data. It occupies a critical position in the GenePattern Seurat Suite pipeline and provides a GUI-accessible entry point to one of the most widely adopted single-cell analysis frameworks in modern genomics (~10,000+ citations across Seurat publications).

---

## 1. 🎯 Tool Purpose & Scientific Applications

### Primary Purpose
Seurat.Clustering takes a preprocessed Seurat object (`.rds`) and executes the downstream analytical steps:
1. **KNN/SNN graph construction** — embeds cells in a shared nearest-neighbor graph
2. **Community detection (clustering)** — partitions the graph using Louvain or Leiden algorithms
3. **Dimensionality reduction for visualization** — UMAP and/or t-SNE
4. **Differential expression for marker genes** — identifies genes characterizing each cluster

### Scientific Applications

| Application Domain | Description |
|--------------------|-------------|
| **Cell Type Discovery** | Unsupervised identification of transcriptionally distinct cell populations |
| **Tumor Microenvironment Analysis** | Characterizing immune and stromal cell populations in cancer |
| **Developmental Biology** | Identifying progenitor and differentiated cell states |
| **Immunology** | Profiling T-cell, B-cell, NK cell, and myeloid subpopulations |
| **Drug Response Profiling** | Identifying treatment-responsive cell subpopulations |
| **Atlas Construction** | Building cell type references for organs/tissues |
| **Multi-modal Integration** | Clustering after CITE-seq, multiome, or batch-corrected data |

### Algorithmic Foundation
The clustering approach in Seurat is grounded in two landmark single-cell methodologies:
- **SNN-Cliq** (Xu & Su, *Bioinformatics*, 2015) — shared nearest-neighbor graph clustering
- **PhenoGraph** (Levine et al., *Cell*, 2015) — KNN graph partitioning for cytometry data

The graph-based strategy overcomes single-gene expression noise by using **PCA metafeatures** (each PC summarizes a correlated gene program) as the distance metric.

## 2. 📥📤 Input/Output Formats & Requirements

### Input Files

| File | Format | Source | Description |
|------|--------|--------|-------------|
| **Seurat Object** | `.rds` (R serialized) | `Seurat.Preprocessing` or `Seurat.IntegrateData` | **Required.** Must contain normalized data, scaled data, and PCA reduction. The `preserve.scale.data=TRUE` flag must have been set in Seurat.Preprocessing. |

#### Critical Input Requirements:
- ✅ `@reductions$pca` — PCA embeddings must be computed upstream
- ✅ `@assays$RNA@data` — Normalized expression matrix
- ✅ `@assays$RNA@scale.data` — Scaled data (required for PCA-based neighbor finding)
- ✅ `@meta.data` — Cell-level metadata (batch, sample, QC metrics)

### Output Files

| File | Format | Description |
|------|--------|-------------|
| `<output_base>_clustered.rds` | `.rds` | Updated Seurat object with cluster assignments, UMAP/tSNE, and all metadata |
| `<output_base>_markers.csv` | `.csv` | Differential marker genes per cluster (columns: gene, p_val, avg_log2FC, pct.1, pct.2, p_val_adj, cluster) |
| `<output_base>_umap.pdf/png` | PDF/PNG | UMAP colored by cluster identity |
| `<output_base>_tsne.pdf/png` | PDF/PNG | t-SNE plot colored by cluster identity |
| `<output_base>_violin.pdf/png` | PDF/PNG | Violin plots for top cluster markers |
| `<output_base>_feature.pdf/png` | PDF/PNG | Feature plots of top marker genes on UMAP |
| `<output_base>_dotplot.pdf/png` | PDF/PNG | Dot plot showing marker expression per cluster |

## 3. ⚙️ Parameter Analysis & Usage Patterns

### Parameter Groups for GenePattern Module

#### Group A: I/O Parameters
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `input.file` | File (.rds) | — | **Required.** Preprocessed Seurat object |
| `output.file.base` | String | `"seurat_clustered"` | Basename for all output files |

---

#### Group B: Graph Construction Parameters (`FindNeighbors`)
| Parameter | Type | Default | Range | Biological Significance |
|-----------|------|---------|-------|------------------------|
| `dims.use` | Integer range | `1:10` | `1:5`–`1:50` | PCA dimensions used for neighbor graph. Use ElbowPlot to guide selection; typically 10–30 PCs |
| `k.param` | Integer | `20` | 5–50 | Number of nearest neighbors. Higher k → coarser clusters; lower k → finer resolution |
| `prune.snn` | Numeric | `0.0667` (1/15) | 0–1 | Edge weight cutoff for SNN graph pruning. Lower = denser graph |
| `reduction` | String | `"pca"` | `"pca"`, `"harmony"`, `"integrated"` | Which reduction to use for graph; use `"harmony"` after Harmony batch correction |
| `nn.method` | String | `"rann"` | `"rann"`, `"annoy"` | KNN algorithm; `annoy` faster for large datasets |

---

#### Group C: Clustering Parameters (`FindClusters`)
| Parameter | Type | Default | Range | Biological Significance |
|-----------|------|---------|-------|------------------------|
| `resolution` | Numeric | `0.8` | `0.1`–`3.0` | **Most critical parameter.** Controls number of clusters. `0.4–1.2` for ~3K cells; scale up for larger datasets |
| `algorithm` | Integer | `1` | `1`–`4` | Clustering algorithm: 1=Louvain (default), 2=Louvain+multilevel, 3=SLM, 4=Leiden (requires `leiden` package) |
| `n.start` | Integer | `10` | 1–100 | Random starts for modularity optimization. Higher = more reproducible but slower |
| `n.iter` | Integer | `10` | 1–100 | Iterations per random start |
| `random.seed` | Integer | `0` | any | Reproducibility seed |
| `group.singletons` | Boolean | `TRUE` | — | Whether singleton cells join nearest cluster |

**Resolution Guidance:**
```
Dataset Size    Recommended Resolution
~1K cells       0.2 – 0.5
~3K cells       0.4 – 1.2   ← standard PBMC benchmark
~10K cells      0.5 – 1.5
~50K+ cells     0.8 – 2.5
```

---

#### Group D: UMAP Parameters (`RunUMAP`)
| Parameter | Type | Default | Range | Description |
|-----------|------|---------|-------|-------------|
| `umap.dims` | Integer range | `1:10` | matches `dims.use` | PCA dimensions for UMAP embedding |
| `n.neighbors` | Integer | `30` | 5–100 | Local neighborhood size. Lower=local structure; higher=global structure |
| `min.dist` | Numeric | `0.3` | 0.001–1.0 | Minimum distance between points. Lower=tighter clusters; higher=uniform spread |
| `umap.metric` | String | `"cosine"` | `"cosine"`, `"euclidean"`, `"manhattan"` | Distance metric for UMAP |
| `n.epochs` | Integer | `NULL` (auto) | 100–500 | Training iterations; more = smoother embedding |
| `umap.seed` | Integer | `42` | any | Reproducibility seed |
| `run.tsne` | Boolean | `TRUE` | — | Whether to also run t-SNE |

---

#### Group E: Marker Identification Parameters (`FindAllMarkers`)
| Parameter | Type | Default | Range | Description |
|-----------|------|---------|-------|-------------|
| `logfc.threshold` | Numeric | `0.25` | `0.0`–`5.0` | Minimum average log2 fold change threshold. Higher = fewer but stronger markers. `0.25` standard; `1.0` for strict markers |
| `min.pct` | Numeric | `0.10` | `0.0`–`1.0` | Minimum fraction of cells expressing the gene. Pre-filters non-expressed genes. `0.1` standard |
| `test.use` | String | `"wilcox"` | many options | Statistical test for DE. See table below |
| `only.pos` | Boolean | `FALSE` | — | Report only upregulated (positive) markers per cluster |
| `max.cells.per.ident` | Integer | `Inf` | 100–Inf | Downsample clusters to speed computation; `200` recommended for >10K cells |
| `min.diff.pct` | Numeric | `-Inf` | 0–1 | Minimum difference in % expressing between compared groups |
| `random.seed` | Integer | `1` | any | Seed for downsampling |

**Statistical Test Comparison for `test.use`:**
| Test | Best Use Case | Speed | Notes |
|------|--------------|-------|-------|
| `"wilcox"` | General purpose (default) | ⚡⚡⚡ | Non-parametric; robust to outliers |
| `"bimod"` | Bimodally distributed data | ⚡⚡ | McDavid et al. 2013; scRNA-specific |
| `"roc"` | Effect size estimation | ⚡ | Returns AUC; no p-values |
| `"t"` | Approximately normal data | ⚡⚡⚡ | Fast parametric test |
| `"negbinom"` | Count data with overdispersion | ⚡ | Requires `latent.vars` for batch |
| `"LR"` | Multi-variable models | ⚡ | Logistic regression; supports covariates |
| `"MAST"` | Complex experimental designs | ⚡ | Requires `MAST` package |
| `"DESeq2"` | Pseudo-bulk analysis | ⚡ | Requires `DESeq2` package; for bulk-like comparisons |

```r
# Core Seurat packages
install.packages("Seurat")        # 4.0.3
install.packages("SeuratObject")  # 4.0.x

install.packages(c(
  "ggplot2",    # Visualization
  "patchwork",  # Plot composition
  "dplyr",      # Data manipulation
  "Matrix",     # Sparse matrices
  "RANN",       # Fast KNN (rann method)
  "uwot",       # UMAP implementation
  "igraph",     # Graph algorithms
  "irlba",      # Truncated SVD for PCA
  "Rtsne",      # t-SNE implementation
  "scales",     # Color scales
  "RColorBrewer" # Color palettes
))

# Optional: For Leiden algorithm (algorithm=4)
install.packages("leiden")

# Optional: For advanced DE testing
```

#### 6.2 GenePattern-Specific Considerations

| **Run Time** | Expose `max.cells.per.ident` parameter; Leiden algorithm (4) faster than Louvain on large graphs |
| **Reproducibility** | Always set `random.seed` in FindClusters, RunUMAP, RunTSNE |
| **Output File Count** | Bundle plots into a single PDF for easier download; provide zip archive option |
| **Notebook Integration** | Module outputs compatible with Notebook #376 for interactive visualization |
| **Version Pinning** | Pin Seurat to exactly `4.0.3` in Docker image to avoid API changes |
| **Error Messaging** | Check for presence of PCA in Seurat object; throw informative error if scale.data missing |

## 8. ⚠️ Known Limitations & Caveats

| Limitation | Impact | Mitigation |
|------------|--------|------------|
| **Resolution sensitivity** | Cluster number highly dependent on `resolution` | Provide guidance; suggest running multiple resolutions |
| **No ground truth** | Clustering is unsupervised; "correct" clusters subjective | Document that biological validation is required |
| **Memory scaling** | Large datasets (>100K cells) require substantial RAM | Expose `max.cells.per.ident`; recommend Seurat v5 for very large data |
| **Marker test assumptions** | Different tests assume different data distributions | Document test.use options clearly |
| **UMAP non-determinism** | UMAP with seed still has minor run-to-run variation | Set seed; note visualization is stochastic |
| **Leiden requirement** | `algorithm=4` requires separate `leiden` Python/R package | Include in Docker; note in docs |
| **API changes v4→v5** | Some parameter names changed in Seurat v5 | Pin to v4.0.3 in container |
| **scale.data dependency** | Fails if upstream `preserve.scale.data=FALSE` | Validate input; provide clear error message |

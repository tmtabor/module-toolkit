## 1. 🎯 Tool Purpose and Scientific Applications

### Overview
`FeatureSummaryViewer` is a browser-based JavaScript visualization module within the GenePattern bioinformatics platform (Broad Institute) that renders the **feature summary results** (`*.feat.odf`) produced by the **KNNXValidation** module. It is the canonical downstream viewer for summarizing which genomic features (genes/probes) were selected by K-Nearest Neighbor (KNN) classifiers during cross-validation analyses of gene expression data.

### Scientific Applications
| Application | Description |
|---|---|
| **KNN Classifier Feature Inspection** | Displays every gene/probe that was used by at least one KNN classifier during cross-validation, along with the frequency of selection |
| **Biomarker Prioritization** | Identifies the most informative genes — those selected by *all or most* classifiers are scientifically the most robust |
| **Gene Expression Classification** | Part of the supervised machine learning pipeline for sample class prediction using microarray or expression data |
| **Exploratory Feature Selection** | Helps researchers understand which genomic features drive classifier decisions |

### Biological Context
The tool operates in the context of supervised gene expression class prediction — a core problem in cancer genomics (e.g., distinguishing AML vs. ALL subtypes, as in the canonical Golub et al. 1999 dataset). KNN cross-validation generates many classifiers; the FeatureSummaryViewer aggregates this information to reveal **which genes are consistently predictive** across iterations.

> 📖 **Key Citation:** Rifkin & Kim et al. *"Using GenePattern for Gene Expression Analysis"* PMC3893799 — explicitly describes FeatureSummaryViewer (v3) as part of the KNNXValidation → FeatureSummaryViewer → PredictionResultsViewer pipeline.

## 2. 📁 Input/Output Formats and Requirements

### Primary Input: `.feat.odf` (Feature Summary ODF File)

The sole input is the **GenePattern ODF (Output Description Format)** file, specifically the feature results file output from `KNNXValidation`.

> ⚠️ **Note:** GenePattern's ODF is a proprietary format — it is **NOT** the OASIS Open Document Format for Office Applications.

#### ODF File Format Specification

```
ODF 1.0                                          ← Line 1: Required format version declaration
HeaderLines=N                                    ← Line 2: Count of header lines that follow
Model=FeatureSummary                             ← Line 3: Model/type identifier
DataLines=N                                      ← Line 4: Number of data rows (genes/features)
COLUMN_TYPES:  String  int  [additional types]   ← Line 5: Data types per column (String/float/int/*)
COLUMN_NAMES:  Feature  NumClassifiers  [...]    ← Line 6: Tab-separated column headers
[Data rows: tab-delimited gene entries]          ← Lines N+1...: One row per gene/feature
```

#### Typical `*.feat.odf` Content Schema

| Column | Type | Description |
|---|---|---|
| `Feature` | String | Gene name / probe ID |
| `Count` (or `NumClassifiers`) | int | Number of KNN classifiers that selected this feature |
| Additional metadata columns | String/float | May include rank, score, or description fields |

**File naming convention:** `<input_prefix>.feat.odf`
**Example:** `all_aml_train.preprocessed.feat.odf`

### Input File Sources
- **Direct KNNXValidation output** — the `*.feat.odf` result file
- **GenePattern job results page** — accessible via the file icon context menu
- **AWS S3 URLs** — cloud.genepattern.org stores files on S3; the JavaScript viewer must fetch from presigned S3 URLs (this was the subject of the current bug fix release)

### Output
As a **pure visualizer module**, FeatureSummaryViewer produces **no persistent output files**. Its output is an interactive in-browser HTML rendering of the data table displayed within the GenePattern UI.

```

│        │                                                                 │

```

- **Source:** `cloud.genepattern.org/gp/pages/protocols/Prediction_knn_one.html`
- **Viewer Role:** Shows which genes contributed to classifiers across leave-one-out CV folds

### Use Case 2: Two-Dataset KNN Classification (Train/Test Split)
- **Goal:** Train KNN on one dataset, test on independent dataset
- **Source:** `cloud.genepattern.org/gp/pages/protocols/Prediction_knn_two.html`
- **Viewer Role:** Displays feature selection from the training phase KNNXValidation run

### Use Case 3: Exploratory Biomarker Discovery
- Open the viewer after classification
- Sort by the classifier count column (descending)
- Genes at the top are the most consistently selected biomarkers → highest research priority

# 🧬 Comprehensive Research Report: GeneListSignificanceViewer

---

## 📋 Executive Summary

**GeneListSignificanceViewer** is a GenePattern visualization module designed to display and interactively explore the ranked, statistically-scored output of marker analysis — primarily from the `ComparativeMarkerSelection` module. Originally implemented as a Java applet, it is now being rewritten in JavaScript (the repository at `https://github.com/genepattern/GeneListSignificanceViewer.git` represents this modern rewrite) following the permanent deprecation of all Java applet-based visualizers in all browsers (March 2017). It is a member of the **Comparative Marker Selection Suite** described in the landmark 2006 Bioinformatics publication by Gould *et al.*

---

## 1. 🔬 Tool Purpose & Scientific Applications

### 1.1 Core Scientific Function
GeneListSignificanceViewer serves as the **interactive significance inspection layer** in the GenePattern marker analysis pipeline. It visualizes ranked gene lists produced by differential expression analyses comparing exactly two biological classes (e.g., tumor vs. normal, treated vs. untreated), displaying the statistical evidence supporting each gene's designation as a class discriminator.

### 1.2 Scientific Context
In genomics, comparing two biological phenotypes across expression profiles of 10,000–50,000 genes creates an acute **multiple testing problem**. GeneListSignificanceViewer presents the full suite of corrected significance metrics, enabling researchers to:

- **Assess statistical robustness** of candidate biomarkers using multiple correction frameworks simultaneously
- **Apply biologically informed significance thresholds** (FDR, FWER, Bonferroni, raw p-value) interactively without re-running computation
- **Distinguish directionality** — genes up-regulated in Class 0 vs. Class 1 (positive vs. negative score values)
- **Explore the score–rank relationship** to assess score distribution and identify natural significance cutoffs

### 1.3 Suite Membership
The tool is the second of three modules in the **Comparative Gene Marker Selection Suite** (Gould *et al.*, 2006):

| Suite Position | Module | Role |
|:-:|---|---|
| **1** | `ComparativeMarkerSelection` | Compute ranked significance scores for all genes |
| **2** | **`GeneListSignificanceViewer`** | **Interactive visual inspection of results** |
| **3** | `ExtractComparativeMarkerResults` | Extract filtered gene sets/datasets for downstream use |

### 1.4 Primary Scientific Applications
| Application | Description |
|---|---|
| **Biomarker Discovery** | Identify candidate diagnostic or prognostic marker genes |
| **Class Prediction Feature Selection** | Select informative genes for classifier construction |
| **Differential Expression Review** | Validate and explore two-class expression differences |
| **Cancer Subtype Characterization** | Distinguish molecular subtypes (e.g., AML vs. ALL) |
| **Drug Response Signatures** | Find genes correlated with treatment response |
| **Two-Class Phenotype Association** | Any categorical phenotype linked to transcriptomic data |

## 2. 📁 Input/Output Formats & Requirements

### 2.1 Primary Input: ODF File (Output Description Format)

The primary input is a **GenePattern-specific ODF file** (`.odf` or `.comp.marker.odf`) produced by the `ComparativeMarkerSelection` module. This is **not** the OASIS Open Document Format.

#### ODF File Structure:
```
ODF 1.0
HeaderLines=N
Model=ComparativeMarkerSelection
DataLines=<number_of_genes>
COLUMN_NAMES=Rank	Feature	Description	Score (SNR)	Feature P Value	BH FDR	Q Value	FWER	Bonferroni	maxT p-value
<data rows...>
```

#### ODF Column Schema:
| Column Name | Data Type | Description |
|---|---|---|
| `Rank` | Integer | Gene rank by test statistic (1 = most significant) |
| `Feature` | String | Gene/probe identifier (e.g., Affymetrix probe ID) |
| `Description` | String | Gene annotation/description |
| `Score (SNR)` | Float | Test statistic value (positive = Class 1 marker, negative = Class 0 marker) |
| `Feature P Value` | Float [0,1] | Raw (nominal) p-value from permutation testing |
| `BH FDR` | Float [0,1] | Benjamini-Hochberg False Discovery Rate |
| `Q Value` | Float [0,1] | Storey's q-value (alternative FDR estimate) |
| `FWER` | Float [0,1] | Family-Wise Error Rate (permutation-based) |
| `Bonferroni` | Float [0,1] | Bonferroni-corrected p-value |
| `maxT p-value` | Float [0,1] | Westfall-Young maxT permutation statistic |

### 2.2 Optional Secondary Input: Expression Dataset
An optional expression dataset file (`.gct` or `.res`) may be provided to display expression profiles for individual selected genes alongside significance statistics.

#### GCT Format Reminder:
```
#1.2
<num_genes>   <num_samples>
Name    Description    Sample1    Sample2    ...
GeneID  Gene Desc      1.23       4.56       ...
```

### 2.3 Outputs
| Output Type | Format | Description |
|---|---|---|
| **Interactive Display** | Browser-rendered HTML/JS | Primary visualization in GenePattern UI |
| **Filtered Gene List** | `.grp` or `.txt` | Exportable list of genes passing significance filter |
| **Significance Table** | Tabular (sortable/filterable) | All genes with statistics |
| **(Optional) Expression Subset** | `.gct` | Expression data subset for significant genes |

## 3. ⚙️ Parameter Analysis & Usage Patterns

### 3.1 Complete Parameter Catalog

#### 🔴 Required Parameters
| Parameter | Type | Format | Description |
|---|---|---|---|
| `comparative.marker.selection.filename` | File | `.odf` | Output file from `ComparativeMarkerSelection` — the ranked, scored gene list |

#### 🟡 Core Optional Parameters
| Parameter | Type | Default | Description |
|---|---|---|---|
| `dataset.filename` | File | *(none)* | Original expression dataset (`.gct`/`.res`) for expression profile display |
| `significance.threshold` | Float [0,1] | `0.05` | Cutoff applied to selected significance method |
| `number.of.features` | Integer (≥1) | `50`–`100` | Top N features to display; limits rendering of large gene lists |
| `statistic.column` | Enum | `Score (SNR)` | Primary statistic used for ranking/highlighting |

#### 🟢 Filtering Parameters (Boolean Flags)
| Parameter | Type | Default | Biological Rationale |
|---|---|---|---|
| `filter.by.fdr` | Boolean | `true` | BH-FDR is the standard genomics filter (controls expected false discovery proportion) |
| `filter.by.fwer` | Boolean | `false` | Conservative; controls probability of *any* false positive |
| `filter.by.pvalue` | Boolean | `false` | Raw p-value (no correction; not recommended for genome-wide studies) |
| `filter.by.bonferroni` | Boolean | `false` | Most conservative correction; suitable for confirmatory settings |

#### 🔵 Display & Directionality Parameters
| Parameter | Type | Default | Description |
|---|---|---|---|
| `sort.by` | Enum | `score` | Column to sort gene list by (`score`, `p-value`, `fdr`, `rank`) |
| `show.class0.color` | Color | Blue | Color coding for Class 0 (down-regulated) genes |
| `show.class1.color` | Color | Red | Color coding for Class 1 (up-regulated) genes |
| `up.regulated.only` | Boolean | `false` | Restrict display to genes positively correlated with Class 1 |
| `down.regulated.only` | Boolean | `false` | Restrict display to genes negatively correlated with Class 1 |

### 3.3 Statistical Background for Parameters

The significance methods form a **stringency spectrum**:

```
MOST LENIENT ◄─────────────────────────────────► MOST CONSERVATIVE
   Raw p-value → BH FDR → Q-Value → FWER → Bonferroni → maxT
```

**SNR Score Interpretation:**
```
SNR = (μ₁ - μ₀) / (σ₁ + σ₀)

  Score > 0  →  Gene is a marker for Class 1 (higher expression in Class 1)
  Score < 0  →  Gene is a marker for Class 0 (higher expression in Class 0)
  |Score|    →  Magnitude of discriminating power
```

### 4.4 No Server-Side Computation
As a JavaScript visualizer, **GeneListSignificanceViewer requires zero server-side computation**. All parsing, filtering, sorting, and rendering occurs in the client browser. This means:
- No Docker container needed (unlike analysis modules)
- No Python/R/Java runtime dependencies
- No HPC or memory constraints beyond the user's browser
- Files are streamed from the GenePattern server to the browser

### 6.5 Key Integration Notes

1. **ODF Parser**: Must implement a robust GenePattern ODF parser in JavaScript. This is the most critical technical component. The ODF format uses a fixed header structure followed by tab-delimited data.

2. **Column Flexibility**: The columns present in the ODF may vary depending on which significance tests were run in `ComparativeMarkerSelection`. The JavaScript parser must handle optional columns gracefully.

3. **Large Dataset Handling**: For genome-wide datasets (~20,000–50,000 features), client-side rendering must use virtualization or pagination (e.g., DataTables with server-side pagination) to remain responsive.

4. **Bidirectional Ranking**: Genes ranked positively favor Class 1; genes ranked negatively favor Class 0. The viewer must clearly present both directions, potentially splitting the display into "Class 1 markers" and "Class 0 markers" sections.

5. **No GeneCruiser**: The JavaScript rewrite should explicitly **not** include GeneCruiser functionality (deprecated March 2016). The recommended path for probe annotation is the `CollapseDataset` module, which can be suggested via a "Send to Module" action.

6. **Export Functionality**: Should provide ability to export filtered gene list as a `.grp` file (GenePattern gene list format) for use with GSEA, ssGSEA, and other downstream modules.

## 8. ⚠️ Known Issues, Limitations & Caveats

| Issue | Description | Recommendation |
|---|---|---|
| **Java Applet Legacy** | Original version completely non-functional in modern browsers | JavaScript rewrite is the correct path |
| **GeneCruiser Removed** | Affymetrix probe annotation via GeneCruiser is deprecated | Direct users to `CollapseDataset` module |
| **ODF Column Variability** | Column set varies by test type; parser must be flexible | Implement defensive column detection |
| **Two-Class Only** | ComparativeMarkerSelection (and this viewer) is limited to exactly two-class comparisons | Multi-class users should use other modules |
| **Large Dataset Performance** | 50,000+ gene tables may be slow to render in browser without virtualization | Implement pagination or virtual scrolling |
| **Permutation Dependency** | Statistical validity depends on adequate permutation count in upstream CMS run | Document recommended permutation counts (≥1000) |
| **Score Sign Convention** | Positive score = Class 1 up-regulated; negative = Class 0 up-regulated — must be clearly labelled | Add clear class labels from ODF header |

## 📋 Executive Summary

**Mutect2** is the **industry-standard somatic short variant caller** developed and maintained by the Broad Institute as part of the **GATK4 toolkit**. It detects somatic single nucleotide variants (SNVs) and insertion/deletion variants (indels) in tumor samples through local de-novo haplotype assembly combined with a sophisticated Bayesian somatic genotyping model. It is the **primary somatic variant caller** recommended by GATK Best Practices, adopted by TCGA, GDC, and PCAWG consortia, and validated extensively in peer-reviewed literature (Benjamin et al., 2019, bioRxiv doi:10.1101/861054).

---

## 1. 🔬 Tool Purpose & Scientific Applications

### 1.1 Core Function
Mutect2 calls somatic short mutations by:
1. **Identifying active regions** — genomic windows with evidence of non-reference variation
2. **Local de-novo haplotype assembly** — reconstructing candidate haplotypes from reads using a De Bruijn graph
3. **PairHMM likelihood computation** — scoring each read against candidate haplotypes
4. **Bayesian somatic genotyping** — computing posterior probabilities that a variant is somatic (not germline or artifact), incorporating tumor/normal evidence, population allele frequencies, and panel-of-normals signal

### 1.2 Supported Variant Types
| Variant Type | Support Level |
|---|---|
| SNVs (Single Nucleotide Variants) | ✅ Full support |
| Small Indels (<50 bp) | ✅ Full support |
| MNPs (Multi-Nucleotide Polymorphisms) | ✅ With `--max-mnp-distance > 0` |
| Complex variants | ✅ Via local assembly |
| Structural variants | ❌ Not supported (use GATK-SV) |
| Copy number variants | ❌ Not supported (use GATK CNV) |

### 1.3 Operating Modes
| Mode | Flag | Use Case | AF Prior |
|---|---|---|---|
| **Tumor-Normal** | `-normal <name>` | Paired tumor + matched normal | `1e-6` (dynamic) |
| **Tumor-Only** | *(no -normal flag)* | No matched normal available | `5e-8` (dynamic) |
| **Mitochondrial** | `--mitochondria-mode` | High-depth mito variant calling | `4e-3` (dynamic) |

### 1.4 Scientific Applications
- **Cancer genomics**: Somatic mutation profiling in solid tumors and hematologic malignancies
- **TCGA/GDC pipelines**: Used in Cancer Genome Atlas somatic calling workflows
- **Tumor evolution studies**: Tracking mutational trajectories across tumor samples
- **ctDNA / liquid biopsy**: Ultra-low allele fraction detection in cell-free DNA
- **Mitochondrial heteroplasmy**: Detecting low-frequency mitochondrial variants
- **Somatic mosaicism**: Identifying post-zygotic mutations in non-cancer tissues
- **Clinical oncology**: FDA-recognized variant calling approach for clinical sequencing

## 2. 📁 Input/Output Formats & Requirements

### 2.1 Required Inputs

|---|---|

### 2.5 BAM Pre-processing Requirements
Mutect2 requires **GATK Best Practices pre-processed BAMs**:
1. **Alignment**: BWA-MEM to reference genome (GRCh38 / hg19 / hg38)
2. **Sorting**: `Picard SortSam` (coordinate sort)
3. **Duplicate Marking**: `Picard MarkDuplicates`
4. **Base Quality Score Recalibration (BQSR)**: `GATK BaseRecalibrator` + `ApplyBQSR`
5. **Indexing**: `samtools index` or `Picard BuildBamIndex`

## 5. 🔄 Common Workflows & Use Cases

### 5.1 GATK Somatic Short Variant Best Practices Pipeline (Full)

```
STAGE 1: PRE-PROCESSING
  FastQ → BWA-MEM → SortSam → MarkDuplicates → BQSR → Indexed BAM

STAGE 2: PoN CREATION (once per project/cohort)
  for each normal:
    Mutect2 (tumor-only, --max-mnp-distance 0) → normal.vcf.gz
  GenomicsDBImport → CreateSomaticPanelOfNormals → PoN.vcf.gz

STAGE 3: SOMATIC CALLING
  Mutect2 (tumor + normal + PoN + germline-resource + f1r2) → raw.vcf.gz + f1r2.tar.gz + .stats

STAGE 4: CONTAMINATION ESTIMATION
  GetPileupSummaries (tumor) + GetPileupSummaries (normal)
  CalculateContamination → contamination.table + segments.tsv

STAGE 5: ORIENTATION BIAS MODELING
  LearnReadOrientationModel (f1r2.tar.gz) → artifact_priors.tar.gz

STAGE 6: FILTERING
  FilterMutectCalls (raw.vcf.gz + contamination.table + artifact_priors) → filtered.vcf.gz

STAGE 7: ANNOTATION (Optional)
  Funcotator / VEP / ANNOVAR → annotated.vcf / MAF
```

### 7.4 Wrapper Script Logic (Pseudocode)

```bash
#!/bin/bash
# Build core command
CMD="gatk --java-options \"-Xmx${java_memory_gb}g\" Mutect2 \
  -R ${reference_fasta} \
  -I ${tumor_bam} \
  -O ${output_prefix}_unfiltered.vcf.gz"

# Conditional: Add normal if provided
[ -n "$normal_bam" ] && CMD="$CMD -I ${normal_bam} --normal-sample ${normal_sample_name}"

# Conditional: Add resources if provided
[ -n "$germline_resource" ] && CMD="$CMD --germline-resource ${germline_resource}"
[ -n "$panel_of_normals" ]  && CMD="$CMD --panel-of-normals ${panel_of_normals}"
[ -n "$intervals" ]          && CMD="$CMD -L ${intervals}"

# Conditional: Enable orientation model
[ "$run_orientation_model" == "true" ] && CMD="$CMD --f1r2-tar-gz ${output_prefix}_f1r2.tar.gz"

# Conditional: Mitochondrial mode
[ "$mitochondria_mode" == "true" ] && CMD="$CMD --mitochondria-mode"

# Add extra args if provided
[ -n "$extra_args" ] && CMD="$CMD ${extra_args}"

# Execute Mutect2
eval $CMD

# Run FilterMutectCalls if requested
if [ "$run_filter_mutect_calls" == "true" ]; then
    # Optionally run LearnReadOrientationModel
    if [ "$run_orientation_model" == "true" ]; then
        gatk LearnReadOrientationModel \
          -I ${output_prefix}_f1r2.tar.gz \
          -O ${output_prefix}_artifact_priors.tar.gz
    fi
    # Apply filters
    gatk FilterMutectCalls \
      -R ${reference_fasta} \
      -V ${output_prefix}_unfiltered.vcf.gz \
      ${artifact_priors_arg} \
      ${contamination_arg} \
      -O ${output_prefix}_filtered.vcf.gz
fi
```

### 7.5 Known Issues & Caveats for GenePattern

| Issue | Impact | Mitigation |
|---|---|---|
| Java version mismatch | Fatal error | Pin Docker image to specific GATK version |
| Missing `.bai` index | Fatal error | Validate BAM index presence in wrapper |
| Missing `.dict` / `.fai` | Fatal error | Validate reference indexes before run |
| SM tag mismatch in BAM header | Silent wrong results | Validate sample name against BAM header |
| Memory underallocation for WGS | OOM crash | Set `java.memory.gb` ≥ 16 for WGS |
| Large VCF.gz without .tbi index | Downstream tool failure | Auto-run `gatk IndexFeatureFile` on VCF outputs |
| gnomAD resource file size (~10GB) | Slow transfer | Cache in Docker or server path |
| FFPE artifacts | High FPR | Document `--pcr-indel-model NONE` for FFPE samples |
| Tumor-only mode high FPR | Misleading results | Strongly require PoN; document limitations |

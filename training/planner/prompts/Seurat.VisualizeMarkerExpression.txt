**Seurat.VisualizeMarkerExpression** is a GenePattern module that serves as the **fourth step in the GenePattern Seurat scRNA-seq analysis suite**, designed to visualize the expression of identified marker genes across cell clusters using two complementary plot types: **Violin Plots** (`VlnPlot()`) and **UMAP Feature Plots** (`FeaturePlot()`). ## 1. üéØ Tool Purpose and Scientific Applications

### 1.1 Scientific Context

Single-cell RNA sequencing (scRNA-seq) produces gene expression profiles for thousands of individual cells simultaneously. After unsupervised clustering (e.g., Louvain/Leiden graph-based clustering in Seurat), **each cluster must be biologically interpreted** by inspecting which marker genes are uniquely or highly expressed within it. This step is essential for:

- **Cell type annotation** (e.g., identifying T cells by *CD3D*/*CD8A*, B cells by *MS4A1*/*CD19*)
- **Biological validation** that clustering has separated biologically distinct populations
- **Quality control** of cluster results (detecting poor-quality or doublet clusters)
- **Hypothesis generation** for downstream experiments

### 1.2 Core Visualization Strategy

The module implements **two complementary visual approaches** to marker expression, both rooted in Seurat's standard visualization framework:

| Plot Type | Seurat Function | What It Shows | Biological Insight |
|---|---|---|---|
| **Violin Plot** | `VlnPlot()` | Distribution of expression per cluster | Expression levels and heterogeneity within each cluster |
| **UMAP Feature Plot** | `FeaturePlot()` | Expression overlaid on low-dimensional embedding | Spatial co-localization of expressing cells |

Together, these two plots provide a **comprehensive marker expression profile**: violin plots reveal *how much* and *how consistently* a marker is expressed across clusters, while UMAP feature plots show *where* expressing cells reside in transcriptomic space, confirming cluster specificity.

# Select top N markers per cluster
top_markers <- markers %>%
  filter(p_val_adj < 0.05, avg_log2FC > log2fc_threshold) %>%
  group_by(cluster) %>%
  top_n(n = num_markers, wt = avg_log2FC) %>%
  pull(gene) %>%
  unique()

  seurat_obj,
  features  = top_markers,
  pt.size   = pt_size,
  ncol      = ncol_vln,

feat_plot <- FeaturePlot(
  seurat_obj,
  features    = top_markers,
  reduction   = reduction,
  order       = order_cells,
  min.cutoff  = min_cutoff,
  max.cutoff  = max_cutoff,
  label       = label,
  repel       = repel,
  ncol        = ncol_feat
)

ggsave(paste0(output_prefix, "_violin_plots.pdf"),
       plot = vln_plot, width = plot_width, height = plot_height)
ggsave(paste0(output_prefix, "_umap_feature_plots.pdf"),
       plot = feat_plot, width = plot_width, height = plot_height)

## 9. ‚öÝÔ∏è Known Limitations and Caveats

| Limitation | Impact | Workaround |
|---|---|---|
| **Seurat v4/v5 RDS incompatibility** | Cannot mix module versions | Standardize entire pipeline to single Seurat version |
| **No interactive plots** | Static PDF/PNG only | For interactive exploration, use Loupe Browser or CellBrowser separately |
| **Memory scaling** | Very large datasets (>500K cells) may OOM | Use `raster=TRUE` and subsample with `Seurat::subset()` |

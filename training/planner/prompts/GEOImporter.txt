# 🧬 Comprehensive Research Report: GEOImporter (GenePattern Module)

---

## 📋 Executive Summary

**GEOImporter** is a GenePattern data ingestion module developed and maintained by the Broad Institute that serves as a critical bridge between the **NCBI Gene Expression Omnibus (GEO)** public data repository and the GenePattern analytical ecosystem. The module automates the retrieval and reformatting of GEO SOFT-format microarray expression data into GenePattern-native **GCT** and **CLS** file formats, enabling seamless downstream analyses such as differential expression, gene set enrichment, and clustering — all without requiring programming expertise. Version 6 (the current release) introduced updated NCBI FTP path handling, an important NA-value filtering capability, and migration to an R 4.0.3 Docker container for reproducibility.

---

## 1. 🎯 Tool Purpose and Scientific Applications

### 1.1 Primary Purpose

GEOImporter eliminates the manual data wrangling barrier that would otherwise prevent bench scientists from leveraging the world's largest public functional genomics repository within a GUI-based analytical platform. Its core workflow is:

```
NCBI GEO Repository (GSE accession)
         ↓  [Automatic SOFT download via FTP]
    GEOImporter (R 4.0.3, GEOquery backend)
         ↓  [Format conversion & NA filtering]
   GCT + CLS files (GenePattern native formats)
         ↓
   Full GenePattern analytical pipeline
```

### 1.2 Scientific Applications

| Application Domain | How GEOImporter Enables It |
|---|---|
| **Differential Gene Expression** | Imports GSE data → GCT + CLS → ComparativeMarkerSelection or DESeq2 |
| **Gene Set Enrichment Analysis (GSEA)** | Produces GSEA-ready GCT + CLS files; NA row removal critical here |
| **Unsupervised Clustering** | GCT output feeds directly into HierarchicalClustering or NMFConsensus |
| **Biomarker Discovery** | Enables re-analysis of published public microarray datasets |
| **Cross-Study Meta-Analysis** | Multiple GEO imports can be compared within GenePattern pipelines |
| **Reproducible Research** | Accession-based retrieval ensures data provenance is tracked |
| **Educational Bioinformatics** | Used in university courses as a zero-code entry point to expression analysis |

### 1.3 The GEO Ecosystem Context

GEO is the world's largest public functional genomics archive, currently hosting:
- **>5 million samples** across hundreds of thousands of studies
- **>85% NGS data** as of 2024 (though GEOImporter is optimized for the remaining array-based data)
- **>200,000 GSE Series records** spanning 20+ years of genomic research
- Data from virtually all major microarray platforms (Affymetrix, Illumina BeadArray, Agilent, etc.)

**GEO Data Organization Hierarchy:**

```
GEO
├── GPL (Platform)  — Array design, probe annotations
├── GSM (Sample)    — Individual hybridization data tables
├── GSE (Series)    — Complete study = multiple GSMs + GPLs  ← GEOImporter entry point
└── GDS (Dataset)  — Curated GEO staff-compiled cross-study summaries
```

## 2. 📁 Input/Output Formats and Requirements

### 2.1 Inputs

#### Input Method 1: GEO Accession Number (Primary — Recommended)
- **Format:** `GSE` followed by integer digits (e.g., `GSE5788`, `GSE41662`, `GSE123456`)
- **Resolution:** Module fetches data from NCBI FTP at runtime
- **NCBI FTP Paths (updated in v6):**
  - Series SOFT: `ftp://ftp.ncbi.nlm.nih.gov/geo/series/GSEnnn/GSExxx/soft/`
  - Series Matrix: `ftp://ftp.ncbi.nlm.nih.gov/geo/series/GSEnnn/GSExxx/matrix/`
- **Network Requirement:** Internet access to `ftp.ncbi.nlm.nih.gov` is **mandatory**

#### Input Method 2: Pre-Downloaded SOFT File (Alternative)
- **File Extensions:** `.soft`, `.soft.gz`
- **Use Cases:** Offline environments, version-controlled reproducibility, firewall-restricted servers
- **SOFT File Structure:**

```
^SERIES = GSE5788
!Series_title = "Expression profiling..."
!Series_geo_accession = GSE5788
...
^PLATFORM = GPL570
!Platform_title = "[HG-U133_Plus_2] Affymetrix Human Genome U133 Plus 2.0 Array"
...
^SAMPLE = GSM134739
!Sample_title = "Sample 1, normal tissue"
!Sample_geo_accession = GSM134739
...
!sample_table_begin
ID_REF    VALUE
1007_s_at 9.876
1053_at   7.234
...
!sample_table_end
```

### 2.2 Outputs

#### Output 1: GCT File (Gene Cluster Text) — Primary Output
Named `{accession}.gct` (e.g., `GSE5788.gct`)

```
#1.2                                    ← Version header (required)
12625    45                              ← #genes  #samples
NAME        Description    GSM134739  GSM134740  ...  GSM134783
1007_s_at   DDR1 // disc...  9.876      8.432      ...  7.123
1053_at     RFC2 // replic..  7.234      6.891      ...  8.456
...
```

| GCT Field | Content | Source in GEO |
|---|---|---|
| `NAME` | Probe ID / Gene accession | `ID_REF` column from GSM data table |
| `Description` | Gene annotation / probe description | GPL platform annotation table |
| `GSMxxxxxx` (columns) | Normalized expression values | `VALUE` column (or custom) from each GSM |

#### Output 2: CLS File (Class Labels) — Secondary Output
Named `{accession}.cls` (when sample metadata allows class inference)

```
45 2 1                    ← num_samples  num_classes  format_version
# tumor normal            ← # followed by space-delimited class names
0 0 0 1 1 1 0 0 1 1 ...  ← 0-indexed class assignment per sample
```

**Biological Significance of CLS:** Encodes experimental design (e.g., tumor vs. normal, treated vs. control) directly from GEO sample metadata (`!Sample_characteristics_ch1`), enabling immediate supervised analysis without manual annotation.

### 2.3 Data Type Compatibility Matrix

| GEO Data Type | Accession Example | GEOImporter Compatible? | Notes |
|---|---|---|---|
| Affymetrix microarray | GSE5788 | ✅ Yes | Optimal use case |
| Illumina BeadArray | GSE41662 | ✅ Yes | Works with VALUE column |
| Agilent two-color | GSE12345 | ✅ Usually | May need non-VALUE column |
| RNA-seq (NGS) | GSE123168 | ❌ No | Counts in supplementary files only |
| ChIP-seq / ATAC-seq | GSE200000 | ❌ No | Not expression data |
| Genome tiling arrays | GSE99999 | ⚠️ Limited | No standard VALUE column |
| GEO DataSet (GDS) | GDS507 | ⚠️ Indirect | Use underlying GSE instead |

## 3. ⚙️ Parameter Analysis and Usage Patterns

### 3.1 Full Parameter Catalog

| # | Parameter Name | Type | Default | Required | Biological/Technical Significance |
|---|---|---|---|---|---|
| 1 | `geo.accession.number` | String (GSExxxx) | *(none)* | Conditional* | Primary data source identifier; drives FTP download |
| 2 | `soft.file` | File (.soft/.soft.gz) | *(none)* | Conditional* | Alternative to online retrieval; enables offline reproducibility |
| 3 | `data.column.name` | String | `VALUE` | Optional | Selects which processed data column to extract; platform-dependent |
| 4 | `omit.rows.with.all.na.values` | Boolean (yes/no) | `yes` | Optional | Critical for GSEA; removes uninformative all-NA probe rows |
| 5 | `output.file.prefix` | String | GSE accession | Optional | Controls output filename prefix |

*Exactly one of `geo.accession.number` or `soft.file` must be provided.

### 3.2 Parameter Deep Dive

#### `data.column.name` — Critical Nuance
The GEO SOFT format requires each GSM to have a data table with at least an `ID_REF` column. The second column is typically the processed expression value:

| Common Column Names | Platform/Context |
|---|---|
| `VALUE` | Most microarray datasets (Affymetrix, Illumina) — **use this as default** |
| `PROCESSED` | Some custom processed datasets |
| `RMA` | RMA-normalized Affymetrix data (some submissions) |
| `MAS5` | MAS5-normalized Affymetrix data |
| `LOG2_RATIO` | Two-color array log2 ratios |

**Best Practice:** Always inspect the GEO dataset page for the sample data table structure before running, particularly for non-Affymetrix platforms.

#### `omit.rows.with.all.na.values` — v6 Addition (Critical for GSEA)
This parameter was added in v6 directly in response to user-reported failures in the GSEA GenePattern module. GEO datasets frequently contain probes with no expression values across all samples (common when a platform annotation includes probes not used in all samples of a multi-platform series). **Recommendation: Always set to `yes` for GSEA workflows.**

### 3.3 Usage Pattern Groups

```
┌──────────────────────────────────────────────────┐
│ GROUP 1: DATA SOURCE (Mutually Exclusive)        │
│  • geo.accession.number (online retrieval)       │
│  • soft.file (offline/local file)                │
└──────────────────────────────────────────────────┘
┌──────────────────────────────────────────────────┐
│ GROUP 2: DATA EXTRACTION                         │
│  • data.column.name → which expression metric    │
│  • omit.rows.with.all.na.values → data quality  │
└──────────────────────────────────────────────────┘
┌──────────────────────────────────────────────────┐
│ GROUP 3: OUTPUT CONFIGURATION                    │
│  • output.file.prefix → filename control        │
└──────────────────────────────────────────────────┘
```

## 5. 🔬 Common Workflows and Use Cases

### 5.1 Workflow 1: Classic Differential Expression Analysis

```
GEOImporter (GSExxxx)
    │
    ├──→ {accession}.gct ──→ PreprocessDataset ──→ {accession}.preprocessed.gct ──→ ComparativeMarkerSelection
    │                                                                                         │
    └──→ {accession}.cls ────────────────────────────────────────────────────────────────────┘
                                                                                              │
                                                                              ComparativeMarkerSelectionViewer
                                                                                     (ranked gene list)
```

**Use Case:** Re-analysis of published microarray studies to reproduce or extend findings; ideal for validating biomarker signatures across cohorts.

### 5.2 Workflow 2: Gene Set Enrichment Analysis (GSEA)

```
GEOImporter (GSExxxx, omit.rows.with.all.na.values=yes)
    │
    ├──→ {accession}.gct ──→ GSEA module ──→ Enrichment results
    │                              ↑
    ├──→ {accession}.cls ──────────┤
    │                              ↑
    └──────────────── MSigDB GMT gene sets (c2.all.v7.4.symbols.gmt)
```

**Critical Note:** The `omit.rows.with.all.na.values=yes` setting is essential here; GSEA will fail on rows that are entirely NA, and this was a documented community pain point resolved in v6.

### 5.3 Workflow 3: Unsupervised Clustering

```
GEOImporter (GSExxxx)
    │
    └──→ {accession}.gct ──→ PreprocessDataset ──→ HierarchicalClustering ──→ HeatMapViewer
                                                              OR
                                                    NMFConsensus ──→ Subtype Discovery
```

**Use Case:** Unsupervised discovery of patient subgroups or co-expressed gene modules within publicly available datasets.

### 5.4 Workflow 4: Multi-Study Pipeline (Advanced)

```
GEOImporter (GSE_Study1) ──→ GCT1
GEOImporter (GSE_Study2) ──→ GCT2  ──→ MergeHTSeqCounts or custom merger ──→ Meta-analysis
GEOImporter (GSE_Study3) ──→ GCT3
```

### 5.5 Educational/Course Use (Documented)

GEOImporter is explicitly used in university bioinformatics curricula (documented at Tufts University, CS/Bio 40):

Step 2: Enter GSE ID in GEOImporter (e.g., GSE5788)

### 6.3 Known Limitations and Mitigation Strategies

| Limitation | Impact | Mitigation |
|---|---|---|
| **RNA-seq datasets not supported** | Cannot import NGS-based GEO data (85% of new submissions) | Use DESeq2Importer or manually convert supplementary counts files to GCT; document clearly in module help |
| **All-NA rows in output** | Caused GSEA failures | ✅ Fixed in v6 with `omit.rows.with.all.na.values` parameter |
| **NCBI FTP path instability** | Module breaks on NCBI infrastructure changes | ✅ Updated in v6; consider HTTPS fallback in future versions |
| **Non-VALUE column names** | Default parameter fails for non-standard datasets | User must manually inspect GEO dataset; document common alternatives |
| **Multi-platform GSE series** | May generate multiple GCT files or require platform selection | Consider adding platform filter parameter in future versions |
| **Large GSE series (100+ samples)** | Slow FTP download, possible timeout | Consider progress indicator; server-side job timeout settings |
| **Partial-NA rows** | Retained even in v6; can cause issues in some analyses | Post-hoc: user must handle via PreprocessDataset filtering |

## 8. ⚠️ Known Issues and Caveats Summary

> **⚠️ CRITICAL:** GEOImporter **cannot import RNA-seq GEO datasets**. Approximately 85% of new GEO submissions (as of 2024) are NGS-based. Users attempting to import RNA-seq accessions will encounter errors or empty outputs. The module is **specifically designed for and validated against array-based microarray data** stored in GEO SOFT format.

> **⚠️ IMPORTANT:** Ensure `omit.rows.with.all.na.values = yes` when the downstream pipeline includes GSEA. This resolves a long-standing compatibility issue documented in the GenePattern community forums.

> **ℹ️ NOTE:** The `data.column.name` default of `VALUE` is correct for ~90% of GEO microarray datasets, but non-standard depositions may use alternative column names. Always verify the GEO dataset structure before running.

> **ℹ️ NOTE:** NCBI FTP paths are subject to change with NCBI infrastructure updates. Version 6 addressed the most recent FTP path change. Future path deprecations may require module updates.

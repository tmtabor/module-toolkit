## 1. 🔬 Tool Purpose and Scientific Applications

### What Is HISAT2?
**HISAT2** (Hierarchical Indexing for Spliced Alignment of Transcripts, version 2) is a fast, memory-efficient, and splice-aware alignment program for mapping next-generation sequencing (NGS) reads — both DNA and RNA — to a population of reference genomes. It was developed by **Daehwan Kim, Joseph Paggi, Chanhee Park, Christopher Bennett, and Steven L. Salzberg** at the Center for Computational Biology, Johns Hopkins University.

Unlike its predecessors (TopHat2, HISAT), HISAT2 employs a novel **Hierarchical Graph FM (HGFM) index** that represents not just a single reference genome, but a *population of genomes* by incorporating over **14.5 million SNPs and associated haplotypes** directly into its index structure. This allows HISAT2 to produce more accurate alignments for reads originating from individuals whose genomes differ from the reference.

### Core Scientific Applications

| Application Domain | Description |
|---|---|
| **RNA-seq Transcriptomics** | Primary use case — splice-aware alignment of mRNA-seq reads spanning exon-exon junctions |
| **Whole Genome Sequencing (WGS)** | DNA-seq alignment using `--no-spliced-alignment` mode |
| **Whole Exome Sequencing (WES)** | Exome capture data alignment |
| **Population Genomics** | SNP-aware alignment using HGFM index for population-level studies |
| **Differential Gene Expression** | Foundation step for DESeq2, edgeR, and limma pipelines |
| **Transcript Assembly** | Upstream step for StringTie/Cufflinks transcript assembly via `--dta` flag |
| **Isoform Discovery** | Novel splice site detection via `--novel-splicesite-outfile` |
| **ATAC-seq / ChIP-seq** | DNA chromatin accessibility and protein binding studies |
| **Variant Analysis** | SNP-level accuracy using graph genome index |

### Key Publications
- **Primary:** Kim D, Paggi JM, Park C et al. *Graph-based genome alignment and genotyping with HISAT2 and HISAT-genotype.* **Nature Biotechnology** 37, 907–915 (2019). DOI: [10.1038/s41587-019-0201-4](https://doi.org/10.1038/s41587-019-0201-4) — PMID: 31375807
- **Secondary:** Kim D, Langmead B, Salzberg SL. *HISAT: a fast spliced aligner with low memory requirements.* **Nature Methods** 12(4):357-60 (2015). DOI: [10.1038/nmeth.3317](https://doi.org/10.1038/nmeth.3317) — PMID: 25751142

## 2. 📁 Input/Output Formats and Requirements

### Input Files

| Input Type | Format | Flag | Notes |
|---|---|---|---|
| **Single-end reads** | FASTQ (.fq, .fastq) | `-U` | Default input format; gzip/bzip2 compressed supported |
| **Paired-end read 1** | FASTQ | `-1` | Comma-separated for multiple files |
| **Paired-end read 2** | FASTQ | `-2` | Must match `-1` file-for-file |
| **FASTA reads** | FASTA (.fa, .fasta) | `-f` | No quality score information |
| **Raw reads** | one-seq-per-line | `-r` | Minimal format |
| **Inline sequences** | Comma-separated strings | `-c` | For testing/small jobs |
| **Illumina QSEQ** | QSEQ | `--qseq` | Legacy Illumina format |
| **SRA Accession** | SRA | `--sra-acc` | Direct NCBI SRA download |
| **Genome Index** | HISAT2 `.ht2` files | `-x` | Basename pointing to `*.1.ht2 ... *.8.ht2` |
| **Known splice sites** | Tab-delimited `.ss` | `--known-splicesite-infile` | Generated by `hisat2_extract_splice_sites.py` |
| **Novel splice sites** | Tab-delimited | `--novel-splicesite-infile` | From a prior HISAT2 run |

### Quality Score Encoding

| Flag | Encoding | Standard |
|---|---|---|
| `--phred33` | Phred+33 (ASCII char `!`) | **Default** — modern Illumina (1.8+) |
| `--phred64` | Phred+64 | Legacy Illumina (1.3–1.7) |
| `--int-quals` | Space-delimited integers | Special use cases |

### Output Files

| Output | Format | Flag | Notes |
|---|---|---|---|
| **Alignment file** | SAM | `-S` | Primary output; use SAMtools for BAM conversion |
| **Alignment summary** | Text | `--summary-file` | Statistics on alignment rates |
| **New-format summary** | Text | `--new-summary` | Machine-parseable summary |
| **Novel splice junctions** | Tab-delimited | `--novel-splicesite-outfile` | Newly discovered splice sites |
| **Metrics file** | Text | `--met-file` | Internal alignment metrics |
| **Unaligned reads** | FASTQ | `--un` / `--un-conc` | Reads that failed to align |
| **Aligned reads** | FASTQ | `--al` / `--al-conc` | Reads that successfully aligned |

### Genome Index Files
```
<basename>.1.ht2    # Compressed BWT/FM index (small genomes: 8 files)
<basename>.2.ht2    # Up to .12.ht2 for HGFM (graph genome) indexes
...
<basename>.8.ht2
```
> **⚠️ Important:** Index must be built with HISAT2 ≥ 2.0. Not compatible with Bowtie/HISAT (v1) indexes.

## 3. 🎛️ Parameter Analysis and Usage Patterns

### Required Parameters

| Parameter | Type | Description |
|---|---|---|
| `-x <basename>` | String | **Genome index prefix** (path to `.ht2` files without the trailing `.N.ht2`) |
| `-U <file>` OR `-1 <file> -2 <file>` | File(s) | **Input reads**: single-end (`-U`) or paired-end (`-1` + `-2`) |
| `-S <file>` | File path | **Output SAM file** path |

### Parameter Groups (59 Total Flags Catalogued)

#### 🔷 Group 1: Input Format Selection
| Parameter | Default | Description |
|---|---|---|
| `-q` | ✅ Default | FASTQ format input |
| `-f` | — | FASTA format input |

| `--qseq` | — | Illumina QSEQ format |
| `--phred33` | ✅ Default | Phred+33 quality encoding |

| Parameter | Default | Description |
|---|---|---|

#### 🔷 Group 3: Spliced Alignment (RNA-seq Critical)

|---|---|---|

| `--max-intronlen <int>` | `500000` | Maximum intron size (bp) — cap for very long introns |

| `--pen-noncanintronlen <func>` | `G,-8,1` | Length penalty for long non-canonical introns |
| `--known-splicesite-infile` | — | Provide known splice sites to improve short-anchor alignment |
| `--novel-splicesite-outfile` | — | Report newly discovered splice junctions |
| `--novel-splicesite-infile` | — | Import junctions from prior HISAT2 run |
| `--no-temp-splicesite` | Off | Disable using splice sites discovered early in run to help later reads |
| `--no-spliced-alignment` | Off | **Disable splicing entirely (required for DNA-seq mode)** |

#### 🔷 Group 4: Strand Specificity
| `--rna-strandness` Value | Protocol | Typical Library Prep |

## 4. 🛠️ Installation and Dependency Requirements

### System Requirements

| Resource | Minimum | Recommended |
|---|---|---|
| **OS** | Linux x86_64 / macOS | Linux x86_64 |
| **RAM (HFM index, human)** | 4.3 GB | 8 GB |
| **RAM (HGFM index, human+SNP)** | 8–16 GB | 16 GB |
| **RAM (building HGFM index)** | **~200 GB** | HPC environment |
| **CPU** | 1 core | 8–16 cores |
| **Disk (index files)** | ~4 GB (HFM) | ~9 GB (HGFM) |

### Software Dependencies
```
- Python 2.7+ or 3.x (wrapper scripts: hisat2_extract_splice_sites.py, etc.)
- C++ compiler / pthreads (parallel threads)
- zlib (gzip compressed input support)
- TBB (Intel Threading Building Blocks) — optional, for performance
```

### Installation Methods
```bash
# Method 1: Bioconda (Recommended)
conda install -c bioconda hisat2

# Method 2: Mamba (faster conda alternative)
mamba install hisat2

# Method 3: Build from source
git clone https://github.com/DaehwanKimLab/hisat2.git
cd hisat2 && make

# Method 4: Docker (GenePattern container)
docker pull genepattern/docker-hisat2
```

## 5. 🔄 Common Workflows and Use Cases

### Workflow A: Standard RNA-seq → Differential Expression
```
Raw FASTQ
    ↓ FastQC (QC check)
    ↓ Trimmomatic (adapter trimming, optional)
    ↓ HISAT2 --dta (splice-aware alignment) → .sam
    ↓ SAMtools sort/index → .bam
    ↓ StringTie (transcript assembly + quantification) → .gtf + counts
    ↓ DESeq2 / edgeR / Ballgown (differential expression)
```

### Workflow B: RNA-seq → Gene-Level Counts
```
Raw FASTQ
    ↓ HISAT2 (alignment) → .sam
    ↓ SAMtools sort → sorted.bam
    ↓ featureCounts / HTSeq-count → count matrix
    ↓ DESeq2 / edgeR (DE analysis)
```

### Workflow C: DNA-seq Alignment (WGS/WES)
```
Raw FASTQ
    ↓ HISAT2 --no-spliced-alignment (no splice junctions) → .sam
    ↓ SAMtools sort/markdup → .bam
    ↓ GATK HaplotypeCaller (variant calling)
```

### Workflow D: GenePattern Pipeline
```
[HISAT2Indexer Module] (if custom genome)
         ↓
[HISAT2.aligner Module] ← .fastq.gz, genome index, strandness, threads
         ↓ .sam output
[SAMtools Module or downstream modules]
         ↓
[StringTie / featureCounts / DESeq2 Modules]
```

### Workflow E: nf-core/rnaseq Integration
HISAT2 is a supported aligner option in the **nf-core/rnaseq** pipeline:
```
nf-core/rnaseq --aligner hisat2 --genome GRCh38 ...
→ HISAT2 → SAMtools → StringTie → DESeq2 + MultiQC
```

### Critical Integration Notes

1. **Index Format Compatibility:** The module requires HISAT2 ≥ 2.0 index files (`.ht2`). The GenePattern HISAT2Indexer module should be recommended when using custom genomes. Prebuilt indexes for hg38, hg19, mm10, etc. are hosted at `https://datasets.genepattern.org`.

2. **Index Source Migration (v1.5 Change):** The key change in v1.5 was migrating the dynamic index source from `ftp://gpftp` to `https://datasets.genepattern.org`. This improves reliability and compatibility with environments that block FTP.

3. **Output Format:** HISAT2 natively outputs **SAM format**. For downstream modules that expect BAM, a SAMtools conversion step should be chained. Consider whether to add SAM→BAM sorting conversion within the module wrapper.

4. **Memory Management:** Users should be warned that the HGFM (graph) index requires 8–16 GB RAM at runtime. Pre-built HFM indexes for common genomes require only ~4.3 GB (human).

5. **`--dta` Flag:** For users intending to use StringTie or any GenePattern transcript assembly downstream module, the `--dta` parameter should be prominently exposed and documented. This flag changes anchor length requirements for splice site discovery and is **required** for optimal StringTie compatibility.

6. **Strandedness:** This is a **commonly misconfigured parameter** in RNA-seq workflows. The GenePattern module should provide clear documentation linking library prep protocols (e.g., Illumina TruSeq = RF) to parameter values. Including an auto-detection hint or link to `RSeQC infer_experiment.py` guidance would benefit users.

7. **Paired-End Handling:** The module must correctly pass `-1` and `-2` flags for paired-end reads. Users should be guided on proper paired-read file naming conventions.

8. **Read Groups (`--rg-id`, `--rg`):** Exposing read group parameters is important for users who intend to merge multiple BAM files downstream or use GATK for variant calling.

9. **Novel Splice Site Output:** Exposing `--novel-splicesite-outfile` as an optional output enables advanced users to chain multiple HISAT2 alignment runs using discovered junctions.

10. **Chromosome Naming Conflicts:** The `--remove-chrname` and `--add-chrname` flags should be available for users whose genome FASTA and annotation GTF files have mismatched `chr`-prefix conventions.

## 7. ⚖️ Comparative Analysis: HISAT2 vs. Alternative Aligners

| Feature | **HISAT2** | **STAR** | **TopHat2** | **Bowtie2** | **Salmon** |
|---|---|---|---|---|---|
| **Primary Use** | RNA + DNA | RNA-seq | RNA-seq | DNA-seq | RNA quantification |
| **Splice-aware** | ✅ Yes | ✅ Yes | ✅ Yes | ❌ No | Quasi-mapping |
| **RAM (human)** | **~4.3 GB (HFM)** | ~28–32 GB | ~3.5 GB | ~3 GB | ~8 GB (index) |
| **Speed** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **Population/SNP genome** | ✅ Yes (HGFM) | ❌ No | ❌ No | ❌ No | ❌ No |
| **Fusion detection** | ❌ No | ✅ Yes | Limited | ❌ No | ❌ No |
| **Active development** | ✅ Yes | ✅ Yes | ⚠️ Limited | ✅ Yes | ✅ Yes |
| **Alignment output** | SAM | BAM | BAM | SAM | None (counts only) |
| **Conda availability** | ✅ Bioconda | ✅ Bioconda | ✅ Bioconda | ✅ Bioconda | ✅ Bioconda |
| **Best for GenePattern** | ✅ **Recommended** | ⚠️ High-RAM | ❌ Outdated | ❌ Non-splice | ❌ No alignment |

> **Recommendation:** HISAT2 is the **optimal choice for GenePattern** due to its balance of speed, low memory requirements, splice-awareness, and population genome support — making it accessible across a wide range of computing environments including those with limited RAM.

# 🐟 Comprehensive Research Report: Salmon.Indexer (v0.5 / Salmon 1.6.0)

---

## 1. 🔬 Tool Purpose & Scientific Applications

### Overview
**Salmon.Indexer** is a GenePattern module wrapping the `salmon index` subcommand from the **Salmon** RNA-seq analysis suite (COMBINE-lab, Carnegie Mellon University). It performs the first of two required phases in Salmon's mapping-based quantification pipeline — constructing a **pufferfish-based transcriptome index** that is subsequently used by `Salmon.Quant` to estimate transcript-level and gene-level expression abundances from RNA-seq reads.

The indexing step is **reference-dependent but sample-independent**: it only needs to be run once per transcriptome version and can be reused for any number of samples sharing that reference.

### Scientific Applications
| Application | Details |
|---|---|
| **Bulk RNA-seq quantification** | Primary use case — transcript/gene expression from poly-A or total RNA-seq |
| **Differential expression analysis** | Prerequisite for DESeq2, edgeR, limma-voom pipelines |
| **Differential transcript usage** | Enables isoform-level analysis via `fishpond`/`DRIMSeq` |
| **Single-cell preprocessing** | Index supports `salmon alevin` for scRNA-seq (CEL-seq2, Drop-seq) |
| **Multi-organism studies** | Works with any FASTA transcriptome (human, mouse, plant, pathogen, de-novo assembled) |
| **Decoy-aware read mapping** | Reduces spurious transcript assignments from genomic loci via decoy sequences |

### Key Scientific Concepts
- **Pufferfish Index (v1.0+)**: Replaced the original quasi-mapping hash index. Built on a compacted colored de Bruijn graph (ccDBG) via TwoPaCo, then indexed using a minimal perfect hash function (MPHF) for sub-second k-mer lookups.
- **Selective Alignment**: Salmon ≥ v0.13 shifted from pure quasi-mapping to selective alignment — scoring candidate loci with a SIMD-accelerated DP algorithm (ksw2) using chaining from minimap2. The index is the enabler of this approach.
- **Decoy-Aware Indexing**: By including genomic sequences (whole genome or partial homologs) as "decoys" alongside the transcriptome, Salmon can identify reads that map better to non-transcribed genomic regions and exclude them from quantification, dramatically reducing false positive transcript assignments.

## 2. 📁 Input/Output Formats & Requirements

### Input Files

| Input | Parameter | Format | Required | Notes |
|---|---|---|---|---|
| **Transcriptome FASTA** | `-t / --transcripts` | `.fa`, `.fasta`, `.fa.gz`, `.fasta.gz` | ✅ Yes | Multi-FASTA of all target transcripts. For decoy-aware mode, this must be the **gentrome** (transcripts + genome concatenated) |
| **Decoys File** | `-d / --decoys` | Plain text `.txt`, one ID per line | ❌ Optional | Lists sequence IDs to treat as decoys (typically chromosome names). Enables decoy-aware index |
| **Output Directory Name** | `-i / --index` | String (directory path) | ✅ Yes | The target output directory. Created if it doesn't exist |

#### Constructing Input for Decoy-Aware Index
```bash
# Step 1: Concatenate transcriptome + genome into gentrome
cat Homo_sapiens.GRCh38.cdna.fa Homo_sapiens.GRCh38.dna.fa > gentrome.fa

# Step 2: Extract chromosome names as decoys
grep "^>" Homo_sapiens.GRCh38.dna.fa | cut -d " " -f 1 | sed 's/>//g' > decoys.txt

# Step 3: Build index
salmon index -t gentrome.fa -d decoys.txt -i salmon_index -p 8
```

#### Transcriptome-Only Index (v0.5 Module Feature)
```bash
# Simpler — no genome required, suitable for restricted environments
salmon index -t transcripts.fa -i salmon_index -k 31 -p 4
```

### Output Files (Index Directory)

| File | Type | Purpose |
|---|---|---|
| `complete_ref_lens.bin` | Binary | Complete reference sequence lengths |
| `ctable.bin` | Binary | Contig/color table for pufferfish traversal |
| `ctg_offsets.bin` | Binary | Contig position offsets |
| `duplicate_clusters.tsv` | Tab-separated text | Records of duplicate transcript sequences identified |
| `eqtable.bin` | Binary | Equivalence class table for quantification |
| `info.json` | JSON | **Critical** — index metadata (k-mer size, decoy status, sequence count, Salmon version). Used by `tximeta` for provenance tracking |

| `refAccumLengths.bin` | Binary | Accumulated reference lengths |
| `ref_indexing.log` | Text | Log from reference indexing phase |
| `reflengths.bin` | Binary | Per-reference sequence lengths |
| `refseq.bin` | Binary | Reference sequence data |
| `seq.bin` | Binary | Encoded sequence binary |
| `versionInfo.json` | JSON | Salmon version metadata for index compatibility checking |

## 3. ⚙️ Parameter Analysis & Usage Patterns

### Complete Parameter Catalog

#### Required Parameters

| Parameter | Short Flag | Type | Default | Description |
|---|---|---|---|---|
| `--transcripts` | `-t` | FILE | *Required* | Transcript FASTA (plain or gzip). In decoy-aware mode: concatenated gentrome FASTA |
| `--index` | `-i` | DIR | *Required* | Output directory path for the constructed index |

#### Optional Parameters

| Parameter | Short Flag | Type | Default | Biological Significance |
|---|---|---|---|---|
| `--kmerLen` | `-k` | INTEGER (7–31) | `31` | K-mer size for index. **Smaller k increases sensitivity** (especially for short reads ≤75 bp), increases index size and build time. Values of 23 or 25 recommended for short-read sensitivity improvements |
| `--decoys` | `-d` | FILE (text) | `None` | Decoy sequence ID list. **Critical for accuracy** — prevents spurious mapping of genomic reads to annotated transcripts |
| `--gencode` | — | FLAG | `false` | Parse GENCODE-format headers (splits at `\|`). **Required when using GENCODE FASTA files** (e.g., `gencode.v38.transcripts.fa`) |
| `--features` | — | FLAG | `false` | Expect TSV feature file format (non-standard reference input) |
| `--keepDuplicates` | — | FLAG | `false` | Retain duplicate transcript sequences. Useful when duplicate transcripts must be individually quantified |
| `--keepFixedFasta` | — | FLAG | `false` | Store the preprocessed FASTA in the index directory. Aids debugging and provenance |
| `--sparse` | — | FLAG | `false` | **Sparse index mode** — reduces memory from ~18 GB to ~14 GB for human genome at cost of slower mapping speed. Recommended for resource-limited compute environments |
| `--threads` | `-p` | INTEGER (1–N) | `2` | Parallel threads for index construction. Index building is CPU and I/O intensive; 4–16 threads typical |
| `--type` | — | STRING | `puff` | Index type. Only `puff` (pufferfish) is currently supported |
| `--tmpdir` | — | DIR | System temp | Temporary working directory during index construction |

Group 3: Memory & Performance

### Usage Pattern Decision Tree

```
Need to index transcriptome for Salmon quantification?
│
├── Do you have a genome FASTA available?
│   ├── YES (recommended for model organisms)
│   │   ├── Full genome → Concatenate (gentrome.fa) + extract decoys.txt → Use --decoys
│   │   └── Partial (MashMap2) → Use generateDecoyTranscriptome.sh → Use --decoys
│   └── NO → Transcriptome-only mode (no --decoys flag)
│
├── What annotation source are you using?
│   ├── GENCODE → Add --gencode flag
│   ├── Ensembl / NCBI / custom → No special flag needed
│   └── UCSC → Check header format
│
├── Are your reads short (< 75 bp)?
│   └── YES → Consider -k 23 or -k 25 (improved sensitivity)
│
└── Memory constraints?
    ├── < 16 GB RAM → Use --sparse flag
    └── ≥ 16 GB RAM → Default dense index (faster quantification)
```

## 5. 🔄 Common Workflows & Use Cases

### Workflow 1: Standard Human RNA-seq (Recommended)
```
1. Download reference:
   - GENCODE transcriptome:  gencode.v38.transcripts.fa.gz
   - GRCh38 genome:          GRCh38.primary_assembly.genome.fa.gz

2. [Salmon.Indexer] Build decoy-aware index:
   - transcriptome.fasta = gentrome.fa (concat)
   - decoys.file = decoys.txt
   - kmer.length = 31
   - gencode.flag = true
   - num.threads = 8
   → Output: salmon_index.tar.gz

3. [Salmon.Quant] Quantify each sample:
   - index = salmon_index.tar.gz
   - reads = sample_R1.fastq.gz, sample_R2.fastq.gz
   → Output: quant.sf (per sample)

4. [tximeta / tximport in R]:
   - Import quant.sf files
   - Summarize transcript → gene level
   → Output: DESeqDataSet or SummarizedExperiment

5. [DESeq2 / edgeR / limma-voom]:
   → Differential expression results
```

### Workflow 2: Non-Model Organism / De Novo Assembly
```
1. De-novo transcriptome assembly (Trinity or similar)
   → transcripts.fasta

2. [Salmon.Indexer]:
   - transcriptome.fasta = transcripts.fasta
   - No decoys file (genome typically unavailable)
   - kmer.length = 23 (for improved sensitivity)
   → Output: salmon_index.tar.gz

3. [Salmon.Quant] → quant.sf per sample
```

### Workflow 3: Memory-Limited Environment
```
[Salmon.Indexer]:
- transcriptome.fasta = transcripts.fa
- sparse.index = true
- num.threads = 4
→ Reduced memory footprint index
```

### Key Implementation Notes

1. **Output Packaging**: The index is a **directory** of binary files; it must be packaged as a `.tar.gz` archive for GenePattern file transfer and unpacked by `Salmon.Quant` before use.

2. **tximeta Compatibility**: The `info.json` file inside the index is used by the Bioconductor `tximeta` package for automatic metadata provenance. This file must be preserved intact in the archive.

3. **Version Pinning**: Salmon is strict about index compatibility — the index must have been built with the **same major Salmon version** used for quantification. Pin the Docker image to `salmon:1.6.0`.

4. **Input Validation**:
   - Validate FASTA format and non-empty sequence content
   - If `decoys.file` is provided, validate that all listed IDs exist in the FASTA headers
   - Warn if `--gencode` is not set but GENCODE-format headers (containing `|`) are detected

5. **Memory Management**: Default GenePattern job executor should request ≥ 8 GB RAM for transcriptome-only mode and ≥ 20 GB for decoy-aware (dense) mode.

6. **New in v0.5 (Salmon 1.6.0)**: Support for **transcriptome-only index** mode (no genome/decoys required) — important for non-model organisms and users without genome FASTA access.

## 9. ⚠️ Known Limitations & Caveats

1. **Index Version Locking**: Salmon indices are **not backward or forward compatible** across major versions. An index built with v1.6.0 cannot be used with v1.5.x or v1.7.x.
2. **No Windows Support**: Salmon binaries are Linux/macOS only; Docker is required for Windows GenePattern deployments.
3. **Memory for Full Genome Decoy**: Requires ~18 GB RAM for human/mouse, which may exceed default GenePattern cloud job allocations.
4. **Not Splice-Aware (Indexing)**: Salmon indexes transcripts, not the genome — splice junctions are implicitly handled through the transcript sequences. It cannot discover novel splice junctions (unlike STAR).
5. **Gentrome Preparation Complexity**: Building a proper decoy-aware gentrome requires external tools (GNU `cat`, `grep`, `sed`), which should ideally be automated within the GenePattern wrapper script.
6. **Transcriptome Quality Dependency**: Index quality is directly tied to the reference transcriptome completeness. Incomplete or redundant FASTA inputs will reduce quantification accuracy.
7. **k-mer Sensitivity Trade-off**: Default k=31 is optimal for long reads (>100 bp) but may reduce sensitivity for short reads; users should be guided to adjust this parameter appropriately.

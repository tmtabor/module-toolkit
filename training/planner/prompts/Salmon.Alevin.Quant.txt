# 🔬 Comprehensive Research Report: Salmon.Alevin.Quant

---

## 📋 Executive Summary

**Salmon.Alevin.Quant** is a GenePattern module (v0.6, beta) wrapping **Salmon's Alevin algorithm (v1.5.2)** — a mature, peer-reviewed, end-to-end pipeline for processing **3′ tagged-end droplet-based single-cell RNA sequencing (dscRNA-seq) data**. Developed by the COMBINE-lab (Rob Patro et al.) and integrated into GenePattern by Anthony S. Castanza (Mesirov Lab, UCSD), the module transforms raw FASTQ files into a cell-by-gene count matrix, the foundational input for virtually all downstream single-cell analyses. The underlying algorithm's key innovation is a **Bayesian UMI deduplication framework** that correctly handles gene-multimapping reads — a class of reads (14–23% of typical data) that competing tools entirely discard.

- **Source Publication:** Srivastava A, Malik L, Smith T, Sudbery I, Patro R. *Alevin efficiently estimates accurate gene abundances from dscRNA-seq data.* **Genome Biology (2019) 20:65.** DOI: [10.1186/s13059-019-1670-y](https://doi.org/10.1186/s13059-019-1670-y) — *284+ citations*
- **GenePattern LSID:** `urn:lsid:genepattern.org:module.analysis:00418:0.6`
- **Module Repository:** [github.com/genepattern/Salmon.Alevin.Quant/tree/v0.6](https://github.com/genepattern/Salmon.Alevin.Quant/tree/v0.6)
- **Upstream Tool:** [github.com/COMBINE-lab/salmon](https://github.com/COMBINE-lab/salmon)

## 1. Tool Purpose and Scientific Applications

### 1.1 Core Scientific Purpose

Alevin is an **integrated end-to-end scRNA-seq preprocessing pipeline** that performs six sequential operations in a single command:

| Step | Operation | Description |
|------|-----------|-------------|
| 1 | **CB Detection** | Scans reads for cell barcode (CB) sequences; identifies valid CBs using a knee-point detection algorithm on the CB frequency histogram |
| 2 | **Read Mapping** | Maps cDNA reads to the reference transcriptome using Salmon's quasi-mapping or selective alignment algorithm |
| 3 | **UMI Processing** | Applies graph-based UMI deduplication with PCR/sequencing error correction |
| 4 | **Gene Count Estimation** | Generates gene-level counts using transcript-level equivalence classes via a Bayesian framework |
| 5 | **Multi-mapping Resolution** | Resolves gene-ambiguous reads (unlike STARsolo/kallisto which discard them), accounting for the **14–23% of reads** in 3′ scRNA-seq data that map to multiple genes |
| 6 | **CB Whitelisting** | Post-quantification filtering to remove empty droplets and low-quality cells |

### 1.2 Target Data Types

The tool is specifically designed for **3′-biased, droplet-based** scRNA-seq protocols where:
- **Read 1 (R1):** Contains the **cell barcode + UMI** (short, fixed-length, no biological sequence)
- **Read 2 (R2):** Contains the actual **cDNA sequence** from the 3′ end of a transcript

### 1.3 Supported Sequencing Protocols

| Protocol Flag | Chemistry | CB Length | UMI Length | Notes |
|--------------|-----------|-----------|------------|-------|
| `--chromium` | 10x Chromium v1/v2 | 16 bp | 10 bp | Most common; 2018-present |
| `--chromiumV3` | 10x Chromium v3 | 16 bp | 12 bp | Current standard (2019+); deeper UMI saturation |
| `--dropseq` | Drop-seq | 10 bp | 8 bp | Academic droplet protocol (Macosko et al.) |
| `--gemcode` | GemCode | 14 bp | 10 bp | Legacy 10x Genomics platform |
| `--celseq` | CEL-seq | 8 bp | 4 bp | Plate-based; less common |
| `--celseq2` | CEL-seq2 | 6 bp | 6 bp | Improved CEL-seq |
| `--splitseqV1` | SPLiT-seq v1 | Combined barcodes | 10 bp | Split-pool ligation-based |
| `--splitseqV2` | SPLiT-seq v2 | Combined barcodes | 10 bp | Updated SPLiT-seq |
| `--quartzseq2` | Quartz-seq2 | 15 bp | 8 bp | Japanese high-throughput protocol |

### 1.4 Scientific Applications and Use Cases

- **Cell type identification and clustering** — count matrices used directly in Seurat, Scanpy, Monocle3
- **Differential gene expression** — across cell types or conditions (DESeq2, edgeR via tximeta)
- **RNA velocity analysis** — with alevin-fry's spliced/unspliced counting extension
- **Trajectory inference** — pseudotime analysis of developmental processes
- **Multi-modal integration** — CITE-seq (protein + RNA), feature barcoding experiments
- **Batch integration** — cross-sample and cross-experiment normalization studies
- **Uncertainty-aware analysis** — bootstrap replicates (`--numCellBootstraps`) enable fishpond differential expression with inferential uncertainty

## 2. Input/Output Formats and Requirements

### 2.1 Required Inputs

#### A. FASTQ Files (Mandatory)
```
-1: R1 FASTQ file(s)  → Cell Barcode + UMI reads
-2: R2 FASTQ file(s)  → cDNA sequence reads
```
- Compressed (`.fastq.gz`) or uncompressed (`.fastq`) accepted
- **Multi-lane experiments:** Space-separated file lists for `-1` and `-2`
- Files must be ordered so that R1[i] and R2[i] correspond to the same library
- R1 is typically **28 bp** (10x v2) or **28 bp** (10x v3), R2 is standard read length (75-150 bp)

#### B. Salmon Transcriptome Index (Mandatory)
```
-i: Directory containing pre-built Salmon index
```
Built with:
```bash
# Standard index (transcriptome only)
salmon index -t transcriptome.fa -i index_dir

# Recommended: Decoy-aware index (genome as decoy reduces spurious mappings)
cat transcriptome.fa genome.fa > gentrome.fa
grep "^>" genome.fa | cut -d " " -f 1 | sed 's/>//g' > decoys.txt
salmon index -t gentrome.fa -d decoys.txt -i index_dir --gencode
```
> ⚠️ **Important:** Using `--gencode` flag when building the index is recommended for GENCODE-formatted FASTA headers.

#### C. Transcript-to-Gene Mapping File (Mandatory)
```
--tgMap: TSV file, no header
```
Format:
```
ENST00000456328.2    ENSG00000223972.5
ENST00000450305.2    ENSG00000223972.5
ENST00000488147.1    ENSG00000227232.5
```
> **Column 1:** Transcript ID (must exactly match index sequences); **Column 2:** Gene ID

Generation from GTF:
```bash
zless -S genome.gtf | grep -v "#" | awk '$3=="transcript"' | cut -f9 | \
  tr -s ";" " " | awk '{print$6"."$8"\t"$2"."$4}' | sort | uniq | \
  sed 's/"//g' > txp2gene.tsv
```
Or via Bioconductor `tximeta`/`fishpond` helper functions.

#### D. Optional Cell Barcode Whitelist
```
--whitelist: Plain text file, one barcode per line (uncompressed)
```
- Can be the **technology-level** barcode list (e.g., 10x Genomics 3M whitelist)
- Or **experiment-level** barcodes (e.g., CellRanger `barcodes.tsv` output)

### 6.2 Key Differentiators of Alevin

1. **Multimapping-aware UMI deduplication**: The single most important technical advantage. Gene families (e.g., immunoglobulins, olfactory receptors) are significantly better quantified vs tools that discard multimapping reads.
2. **Inferential uncertainty output**: Bootstrap replicates enable downstream uncertainty-propagating differential expression, a unique capability in the open-source ecosystem.
3. **Transcript-level equivalence classes**: Internal use improves accuracy for genes with multiple isoforms.
4. **Open-source and bioconda-available**: Fully reproducible environments, unlike CellRanger.
5. **GenePattern integration**: Only major scRNA-seq quantifier available natively in GenePattern.

### 6.3 Known Limitations

- **Speed**: Slower than STARsolo/kallisto|bustools (~2–5× longer runtime); partially mitigated by `--sketch` mode
- **Version**: GenePattern module wraps v1.5.2; upstream Alevin has since been largely superseded by **alevin-fry** for new analyses
- **Memory**: Comparable to or slightly higher than STARsolo for large datasets
- **No built-in QC report**: Requires separate alevinQC package (can be bundled)
- **Transcriptome-only mapping**: Cannot distinguish reads from unannotated genomic loci without decoy index
- **No multi-modal native support**: CITE-seq ADT quantification requires a separate Alevin run

### 7.2 Critical Implementation Notes

1. **Index Handling**: Salmon indexes are directories, not single files. GenePattern should accept a `.zip` archive of the index directory and unpack it before running. Pre-built indexes for hg38/mm10 should be referenced or bundled.

2. **Multi-file FASTQ Input**: The `-1` and `-2` arguments accept space-separated file lists. GenePattern's multi-file input widgets must be configured to produce space-separated paths.

3. **Protocol Mutex**: The protocol flags (`--chromium`, `--chromiumV3`, `--dropseq`, etc.) are mutually exclusive. A dropdown selector in the GenePattern UI should map to the appropriate flag.

4. **tgMap Generation Guidance**: Users often struggle creating the tgMap file. The module documentation should include a recipe for generating this from a GTF file or point to Bioconductor helper functions.

5. **Output Packaging**: The output directory should be packaged as a ZIP file for downstream GenePattern modules to consume, maintaining the required directory structure (critical for alevinQC and tximeta compatibility).

6. **Memory Safeguards**: Implement a memory check or warning when genome size is large; alert users when estimated RAM usage approaches container limits.

7. **Bootstrap Dependency**: `--numCellBootstraps` **must** be paired with `--dumpFeatures` to generate `quants_boot_mat.gz`. Implement parameter dependency validation.

8. **Downstream Module Compatibility**: Output ZIP should be designed to feed directly into:
   - A GenePattern `alevinQC` module (HTML report generation)
   - Seurat/Scanpy preprocessing modules

### 7.3 Suggested Parameter Validation Rules

## 9. Summary and Recommendations for GenePattern Module Development

### ✅ Strengths to Emphasize
- Open-source, fully reproducible, bioconda-installable

- Bootstrap uncertainty quantification (unique feature)
- 9 supported protocols covering the vast majority of 3′ scRNA-seq experiments

## 1. TOOL IDENTITY & SCIENTIFIC PURPOSE

### 1.1 Overview
**spatialGE.STplot** is a GenePattern module wrapping the `STplot()` function from the **spatialGE R package** (v1.2.2, FridleyLab/Moffitt Cancer Center). It generates spatial "quilt plots" — scatter plot visualizations where each sequenced spot or cell is color-coded by gene expression level, tissue domain cluster membership, or sample metadata — rendered at the precise spatial coordinates recorded during tissue sectioning.

### 1.2 Scientific Applications
| Application Domain | Specific Use Case |
|---|---|
| Tumor Microenvironment (TME) | Mapping immune cell infiltration, tumor vs. stromal compartments |
| Spatial Heterogeneity | Visualizing intra-tissue gene expression gradients |
| Tissue Domain Discovery | Displaying STclust cluster assignments spatially |
| Biomarker Spatial Mapping | Visualizing hypoxia (NDRG1), B-cell (IGKC, MS4A1), collagen (COL1A1) markers |
| TLS Characterization | Mapping tertiary lymphoid structures in cancer tissue |
| Multi-sample Comparison | Side-by-side spatial plots across multiple tissue sections |
| QC Visualization | Checking spot-level expression after preprocessing/normalization |

### 1.3 Scientific Positioning
STplot is a **terminal visualization step** in the spatialGE analytical pipeline, designed to make spatial patterns interpretable by biologists who need to see where in tissue specific genes are active or which spatial domains contain which cell populations.

### 2.2 Core Data Structure: STlist
The **STlist (Spatial Transcriptomics List)** is an S4 R object that acts as the unified data container for all spatialGE operations:

```
STlist structure:
├── Raw counts matrix        → genes × spots/cells (sparse Matrix)
├── Transformed counts       → normalized + log-transformed expression
├── Spatial coordinates      → x, y positions per spot/cell
├── Clinical metadata        → sample-level variables (tissue type, patient ID, treatment, etc.)
├── STclust results          → cluster assignments per spot for various (k, w, deepSplit) settings
├── SThet results            → Moran's I / Geary's C spatial autocorrelation statistics
├── STenrich results         → gene set spatial enrichment results
└── STdiff results           → spatially-informed differential expression results
```

## 5. PARAMETER ANALYSIS (Complete)

### 5.1 Full Function Signature
```r
STplot(
  x,                  # Required: STlist object
  samples   = NULL,   # Character vector: sample names to plot
  genes     = NULL,   # Character vector: genes to visualize
  plot_meta = NULL,   # Character: metadata variable to plot
  ks        = "dtc",  # Integer or "dtc": cluster k value
  ws        = NULL,   # Numeric vector: spatial weights from STclust
  deepSplit = NULL,   # Integer (0-4): hierarchical clustering cut depth

)
```

> ⚠️ **Mutual Exclusivity Note:** `genes`, `plot_meta`, and `ks`/`ws`/`deepSplit` cluster parameters represent three distinct visualization modes. Gene expression mode and cluster mode cannot be used simultaneously — GenePattern module UI should enforce this via conditional parameter visibility.

## 7. COMMON WORKFLOWS & USE CASES

### 7.1 Standard spatialGE Analysis Pipeline
```
Step 1: spatialGE.Preprocessing
  Input:  Raw count matrices + spatial coordinate files
  Output: STlist.rds (normalized, QC-filtered)

Step 2: [Optional] spatialGE.STclust
  Input:  STlist.rds from Step 1
  Output: STlist.rds with cluster assignments (ks, ws, deepSplit)

Step 3: [Optional] spatialGE.STdiff / STenrich / STgradient
  Input:  STlist.rds from Step 2
  Output: STlist.rds with statistical results

Step 4: spatialGE.STplot  ← THIS MODULE
  Input:  STlist.rds from any of Steps 1–3
  Output: PNG/PDF spatial visualization plots
```

### 7.2 Key Use Case Scenarios

**Use Case A — Basic Exploratory Gene Expression**
> After preprocessing, a researcher wants to verify the spatial distribution of known marker genes (e.g., CD3E for T-cells, COL1A1 for fibroblasts) before deeper analysis.
- Parameters: `genes=c("CD3E","COL1A1")`, `data_type="tr"`, `color_pal="BuRd"`

**Use Case B — Spatial Domain Visualization After Clustering**
> A researcher has run STclust with multiple spatial weights (w=0, 0.05, 0.2) to detect tissue niches. They want to see how cluster assignments change across weight values.
- Parameters: `ws=c(0,0.05,0.2)`, `ks="dtc"`, `color_pal="highcontrast"`

**Use Case C — Multi-sample Publication Figure**
> A multi-patient study requires spatial expression plots for all 5 tissue sections for a key biomarker (e.g., PD-L1/CD274).
- Parameters: `genes="CD274"`, `samples=NULL` (all), `ptsize=0.5`, `color_pal="plasma"`

**Use Case D — Metadata Spatial Overlay**
> A user wants to visualize previously annotated pathology regions (tumor core, invasive margin, stroma) projected onto spatial coordinates.
- Parameters: `plot_meta="pathology_region"`, `color_pal="highcontrast"`

### 9.2 Suggested Module Manifest Notes
- **Module Category**: Visualization / Spatial Transcriptomics
- **Input file type**: `.rds` (R Data Serialization)
- **Output file types**: `.png`, `.pdf`
- **CPU**: 2 cores (STplot is not parallelized)
- **Memory**: 8–16 GB RAM (depends on dataset size)
- **Docker base image**: Should include R ≥ 4.2, spatialGE ≥ 1.2.0, ggplot2, ggpubr
- **Command line pattern**: `Rscript /path/to/run_STplot.R --input <stlist.rds> --genes <gene_list> --samples <sample_list> --color_pal <palette> ...`

### 9.3 Critical Implementation Notes
1. **Parameter validation**: Validate that gene names specified in `genes` exist in the STlist before calling STplot — provide informative error if gene not found
2. **Mode exclusivity**: If `genes` is provided, do not pass cluster parameters (`ws`, `ks`); enforce via R script logic with user-facing error message
3. **Launcher integration**: The `spatialGE.STplot.Launcher` companion module should pre-query the STlist (e.g., available genes, sample names, cluster results present) to guide parameter selection
4. **Plot output strategy**: Since STplot returns a *list* of ggplot objects, the R script should iterate and save each to a named file (e.g., `sample_094c_NDRG1.png`)
5. **NULL handling**: `ptsize=NULL` and `txsize=NULL` trigger spatialGE's auto-sizing; expose as optional parameters with "auto" option in the GenePattern UI

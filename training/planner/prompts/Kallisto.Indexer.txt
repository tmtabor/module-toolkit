## 1. Executive Summary

**Kallisto.Indexer** is a GenePattern module wrapper around the `kallisto index` subcommand from the Kallisto toolkit (Pachter Lab, UC Berkeley/Caltech). It builds a binary k-mer hash index from a FASTA-formatted transcriptome sequence file. This index is the mandatory prerequisite for all Kallisto-based RNA-seq quantification workflows — both bulk (`kallisto quant`) and single-cell (`kallisto bus`). The module occupies **Step 1** in a two-step workflow, with **Kallisto.Quant** consuming its output.

Kallisto is one of the most highly-cited bioinformatics tools of the past decade, described in:
> **Bray NL, Pimentel H, Melsted P, Pachter L.** "Near-optimal probabilistic RNA-seq quantification." *Nature Biotechnology* **34**, 525–527 (2016). DOI: [10.1038/nbt.3519](https://www.nature.com/articles/nbt.3519)

## 2. Scientific Background & Tool Purpose

### 2.1 The Pseudoalignment Paradigm

Kallisto revolutionized RNA-seq analysis by replacing traditional sequence alignment with **pseudoalignment** — a mathematical abstraction that determines which transcripts are *compatible* with a read without finding a precise genomic coordinate. This is achieved by:

1. **Index Construction** *(Kallisto.Indexer's role)*: The transcriptome FASTA is decomposed into overlapping k-mers (default k=31). These k-mers are stored in a **de Bruijn graph**, and the **colored compacted de Bruijn graph (ccDBG)** of the transcriptome is encoded into a binary hash table — the `.idx` file.

2. **Pseudoalignment** *(Kallisto.Quant's role)*: For each RNA-seq read, k-mers are extracted and looked up in the hash table. The **equivalence class** of compatible transcripts is determined in microseconds per read.

### 2.2 Scientific Applications

| Application | Description |
|---|---|
| **Bulk RNA-seq Quantification** | Transcript-level and gene-level abundance estimation from total RNA or mRNA-seq experiments |
| **Single-Cell RNA-seq** | Pre-processing via `kallisto bus` + bustools pipeline for UMI-based scRNA-seq (10x Genomics, inDrop, etc.) |
| **Multi-Species Studies** | Custom FASTA indexing for any organism with available transcript sequences |
| **Viral/Pathogen Quantification** | Custom FASTA index for viral transcripts or bacterial RNA |
| **Neoantigen/Fusion Detection** | Custom target sequence indexing for immunogenomics applications |
| **Proteomics (experimental)** | Amino acid FASTA indexing via `--aa` flag |
| **ERCC Spike-in Quantification** | Custom FASTA combining transcriptome + spike-in controls |

## 4. Input/Output Specifications

### 4.1 Inputs

| Parameter | Type | Required | Description |
|---|---|---|---|
| **Transcriptome FASTA** | File | ✅ Yes | FASTA file of target sequences — **MUST be transcriptome/cDNA, NOT genome FASTA** |
| Index output name | String | ✅ Yes | Output filename (e.g., `kallisto_index.idx`) |
| k-mer size (`-k`) | Integer | ❌ Optional | Odd integer ≤ 31; default = 31 |
| Make unique (`--make-unique`) | Boolean | ❌ Optional | Deduplicate repeated target names |
| Threads (`-t`) | Integer | ❌ Optional | CPU threads; default = 1 |
| Decoy list (`-d`) | File | ❌ Optional | FASTA of sequences to mask |
| EC max size (`-e`) | Integer | ❌ Optional | Max targets per equivalence class |

#### ⚠️ Critical Input Requirement
The transcriptome FASTA must contain **individual transcript sequences** (cDNA/mRNA). **Genome-level FASTA files will produce incorrect results.** Common valid sources:

- **Ensembl**: `Homo_sapiens.GRCh38.cdna.all.fa.gz`
- **GENCODE**: `gencode.v44.transcripts.fa.gz`
- **RefSeq**: `GCF_000001405.40_GRCh38.p14_rna.fna`
- **Custom**: Concatenated FASTA from specific gene sets, viral genomes transcriptomes, or synthetic sequences

### 4.2 Outputs

| File | Format | Description |
|---|---|---|
| `*.idx` / `*.kidx` | Binary (kallisto-specific) | The kallisto index: a k-mer hash table encoding the compacted colored de Bruijn graph of the transcriptome |

#### Index File Properties
- **Size**: ~2 GB for human transcriptome (GENCODE comprehensive)
- **Format**: Binary; not human-readable
- **Portability**: ⚠️ Index files are **NOT portable across major kallisto versions**; must be rebuilt after version upgrades
- **Reusability**: One index file serves **all samples** in a study using the same reference

## 5. Complete Parameter Reference

### 5.1 Parameter Details with Biological Significance

#### Required Parameter

**`-i / --index` (STRING)**
- **Purpose**: Specifies the output path/filename for the generated index
- **Recommended naming convention**: `{organism}_{assembly}_{release}.idx`
  - Example: `human_GRCh38_gencode44.idx`
- **GenePattern type**: `TEXT`
- **Default suggestion**: `kallisto_index.idx`

---

#### Core Optional Parameters

**`-k / --kmer-size` (INTEGER)**
- **Default**: 31 | **Max**: 31 | **Must be odd**
- **Biological rationale**: k-mer size controls the specificity vs. sensitivity tradeoff:
  - **k=31** (default): Optimal for reads ≥ 75 bp (standard Illumina short-read RNA-seq). High specificity, lower false-positive rate.
  - **k=21**: Better for reads 50–75 bp, or when sensitivity is paramount. Recommended for some legacy short-read datasets.
  - **k=15**: Short reads (<50 bp), ancient RNA, degraded samples. High sensitivity but more false positives.
  - **Lower k**: Increases mapping rate but risks multi-mapping inflation; useful for degraded RNA (FFPE samples) or very short reads
- **Community consensus**: k=31 is appropriate for ≥75 bp reads; k=21 for shorter reads
- **GenePattern type**: `INTEGER`

**`--make-unique` (FLAG)**
- **Purpose**: Replaces duplicate FASTA sequence identifiers with auto-generated unique names
- **When to use**: GENCODE FASTA files occasionally contain repeated transcript IDs across versions. This flag prevents index-build failures or silent errors caused by non-unique sequence headers.
- **GenePattern type**: `BOOLEAN`
- **Recommendation**: Enable as default for GENCODE-sourced FASTA files

**`-t / --threads` (INTEGER)**
- **Default**: 1
- **Biological impact**: None — threading only affects runtime, not results
- **Performance note**: Index building is primarily I/O-bound rather than CPU-bound; gains plateau above 4 threads
- **GenePattern type**: `INTEGER`
- **Recommended range**: 1–8 threads

---

#### Advanced Optional Parameters

**`-d / --d-list` (FILE)**
- **Purpose**: Provides a FASTA file of **decoy sequences** to mask from quantification. This implements decoy-aware pseudoalignment, reducing spurious assignment of reads from genomic decoy regions (e.g., introns, intergenic regions that share k-mer content with transcripts).
- **When to use**: Studies requiring maximum quantification accuracy, especially for genes with many pseudogenes, or when comparing with Salmon (which natively supports decoy-aware alignment)
- **Typical decoys**: Genome sequence of chroms used as decoys (similar to Salmon's selective alignment approach)
- **GenePattern type**: `FILE`

**`--aa` (FLAG)**
- **Purpose**: Build index from amino acid (protein) sequences instead of nucleotide sequences
- **When to use**: Proteomics-based workflows, ribosome profiling with peptide quantification, or experimental multi-omics applications
- **GenePattern type**: `BOOLEAN`

**`--distinguish` (FLAG)**
- **Purpose**: Forces each sequence to be distinguished by its name in the de Bruijn graph, even if sequences are identical
- **When to use**: Custom index construction where identical sequences from different sources must be tracked separately
- **GenePattern type**: `BOOLEAN`

**`-m / --min-size` (INTEGER)**
- **Purpose**: Specifies the minimizer length used in index construction
- **Default**: Automatically chosen by kallisto based on input
- **Advanced use**: Fine-tuning for non-standard transcriptomes or research into pseudoalignment behavior
- **GenePattern type**: `INTEGER`

**`-e / --ec-max-size` (INTEGER)**
- **Purpose**: Caps the maximum number of targets (transcripts) allowed in a single equivalence class
- **Default**: No maximum
- **When to use**: Very repetitive transcriptomes or when downstream tools struggle with large equivalence classes
- **GenePattern type**: `INTEGER`

## 7. Common Workflows & Use Cases

### 7.1 Standard Bulk RNA-seq Workflow

```
Step 1: [Kallisto.Indexer]
  Input:  Homo_sapiens.GRCh38.cdna.all.fa.gz
  Params: k=31, make-unique=true
  Output: human_GRCh38.idx

Step 2: [Kallisto.Quant] (for each sample)
  Input:  human_GRCh38.idx + sample_R1.fastq.gz + sample_R2.fastq.gz
  Output: abundance.tsv, abundance.h5, run_info.json

Step 3: [Sleuth / tximport + DESeq2]
  Input:  abundance.h5 files from all samples
  Output: Differential expression results
```

### 7.2 Custom/Non-Model Organism Workflow

```
Step 1: Download organism FASTA from Ensembl/NCBI
Step 2: [Kallisto.Indexer] with custom FASTA → custom.idx
Step 3: [Kallisto.Quant] with custom.idx
```

### 7.3 Spike-in + Transcriptome Combined Index

```bash
# Combine FASTA files before indexing
cat transcriptome.fa ERCC_spike_ins.fa > combined.fa
# Index the combined file
kallisto index -i combined.idx combined.fa
```
> This pattern should be documented in the GenePattern module — users can pass multiple FASTA files or a pre-concatenated FASTA.

### 7.4 Single-Cell RNA-seq (10x Genomics)

```
Step 1: [Kallisto.Indexer] → transcriptome.idx
Step 2: kallisto bus -i transcriptome.idx -o output/ -x 10xv3 *.fastq.gz
Step 3: bustools correct + sort + count → gene expression matrix
```

### 8.2 Kallisto vs. Salmon: Key Distinction

Both tools use alignment-free approaches and show near-identical quantification results (Pearson correlation Rv=0.9999 in comparative benchmarks). The key practical distinction for **Kallisto.Indexer** vs Salmon index:
- **Kallisto.Indexer**: Requires only a **transcriptome FASTA** — simplest input requirement
- **Salmon decoy-aware**: Requires both transcriptome FASTA + genome FASTA — more accurate but more complex setup

> **GenePattern positioning**: Kallisto.Indexer is the right choice for users wanting the simplest, fastest RNA-seq indexing workflow with no compromise in accuracy for standard experiments.

## 11. Known Issues, Caveats & Best Practices

| Issue | Detail | Resolution |
|---|---|---|
| **Version incompatibility** | Index built with kallisto 0.46.x is not compatible with 0.50.x | Always rebuild index when upgrading kallisto version; document version in index filename |
| **Genome vs. transcriptome confusion** | Using genome FASTA produces an invalid index with no error message | Document prominently; consider file-size heuristic warning in wrapper |
| **Duplicate sequence IDs** | GENCODE FASTA may contain duplicate IDs; causes errors or silent failures | Use `--make-unique` flag by default or when using GENCODE source |
| **k-mer size irreversibility** | k-mer size cannot be changed after index is built | Quantification reads must use index built with appropriate k for their length |
| **Large index files** | Human index ≈2 GB; storage can be an issue | Document expected sizes; warn users about disk space |
| **Non-portability** | `.idx` files are not inter-tool portable (only kallisto can use them) | Document that index is kallisto-specific binary format |
| **ARM architecture** | Some older builds may not support ARM (Apple Silicon) | Ensure Docker image is built for linux/amd64 and optionally linux/arm64 |

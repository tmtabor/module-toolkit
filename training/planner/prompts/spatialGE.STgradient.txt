## 1. 🔬 Tool Purpose & Scientific Applications

### Core Scientific Purpose
`spatialGE.STgradient` implements the `STgradient()` function from the `spatialGE` R package (Fridley Lab, Moffitt Cancer Center). It tests whether genes exhibit **spatially graded expression patterns** with respect to a user-defined **reference tissue domain/niche** — for example, genes that are highly expressed in spots/cells close to a tumor region but progressively decrease in expression as distance from the tumor increases.

> *"A gene showing a spatial gradient exhibits higher expression in spots/cells closer to a tissue domain and lower expression in spots/cells farther from the same domain."*
> — bioRxiv 2024 (Ospina et al.)

### Statistical Methodology
The core approach is elegantly non-parametric:
1. **Distance Calculation**: For each spot/cell, calculate its minimum (`distsumm="min"`) or mean (`distsumm="mean"`) Euclidean distance to **all spots in the reference tissue domain**
2. **Spearman Correlation**: Compute Spearman's rank correlation coefficient (ρ) between gene expression levels and these distances, across all spots in the tissue
3. **Linear Model Fitting**: Fits linear models alongside the Spearman correlation for inference
4. **Statistical Testing**: Derives p-values and FDR-adjusted p-values (BH method)
5. **Significance Criterion**: Genes are called as exhibiting spatial gradients at p < 0.05 AND |Spearman ρ| > 0.1

### Biological Applications
| Application Domain | Use Case | Evidence |
|---|---|---|
| **Cancer Biology** | Detect gene gradients radiating from tumor vs. normal tissue interface | Demonstrated in melanoma (10X Visium) |
| **Leptomeningeal Disease** | Spatial gradient analysis in leptomeningeal metastasis ST data | oospina/spatial_transcriptomics_leptomeningeal_disease |
| **Tissue Microenvironment (TME)** | Identify genes with expression patterns relative to immune/stromal niches | spatialGE Moffitt web app workflows |
| **Glioblastoma** | Identify invasion-associated spatial gradients | LSGI PMC paper (related methodology) |
| **General Spatial Biology** | Any tissue where domain-centric expression gradients are biologically meaningful | GenePattern module documentation |

## 2. 📁 Input/Output Formats & Requirements

### Primary Input
| Input Type | Format | Description |
|---|---|---|
| **STlist object** | `.RDS` (R serialized binary) | The core S4 data object from the spatialGE package containing: (1) raw gene expression counts per spot/cell, (2) normalized/transformed expression, (3) spatial coordinates (x/y), (4) sample metadata, (5) tissue domain annotations (output of `STclust`) |
| **Reference domain annotation** | Embedded in STlist metadata | Domain/cluster labels, typically the column name produced by `STclust` |

The STlist R object (class `STlist`) encapsulates:
- Multi-sample spatial transcriptomics data (multiple tissue sections)
- Spot/cell-level gene expression count matrices
- Spatial coordinates for each spot/cell
- Results from previous pipeline steps (normalization, clustering)

### Two Common Input Pathways
```
PATHWAY 1 (Recommended - via Launcher):
  spatialGE.STclust output (RDS) → spatialGE.STgradient.Launcher → spatialGE.STgradient
  
PATHWAY 2 (Direct / Advanced):
  Raw STlist RDS + user-specified annot/ref params → spatialGE.STgradient
```

### Primary Outputs
| Output Type | Format | Description |
|---|---|---|
| **Results table** | CSV/TSV | Per-gene Spearman correlation coefficient (ρ), p-value, FDR-adjusted p-value, distance type |
| **Updated STlist** | `.RDS` | STlist object with gradient results stored in designated slot |
| **Visualization plots** | PDF/PNG | Scatter plots of gene expression vs. distance, spatial gradient maps |

### Supported Spatial Transcriptomics Platforms
- **10X Visium** (spot-based, ~55 µm spot diameter, ~3,000-5,000 spots/section)
- **NanoString CosMx** (single-cell resolution)
- **Generic CSV/TSV** gene expression matrices paired with coordinate files

## 3. ⚙️ Parameter Analysis & Usage Patterns

### Complete Function Signature
```r
STgradient(
  x           = NULL,    # [REQUIRED] STlist object
  samples     = NULL,    # [OPTIONAL] Character vector of sample names
  topgenes    = 2000,   # [OPTIONAL] Number of most variable genes to test
  annot       = NULL,    # [REQUIRED] Column name with domain/cluster annotations
  ref         = NULL,    # [REQUIRED] Name of reference domain (e.g., "tumor")
  exclude     = NULL,    # [OPTIONAL] Domains to exclude from analysis
  out_rm      = FALSE,   # [OPTIONAL] Remove statistical outlier spots
  limit       = NULL,    # [OPTIONAL] Expression threshold limit
  distsumm    = "min",   # [OPTIONAL] Distance summarization: "min" or "mean"
  min_nb      = 3,       # [OPTIONAL] Minimum number of neighbors for distance calculation
  robust      = TRUE,    # [OPTIONAL] Use robust regression (huber/MM estimators)
  nb_dist_thr = NULL,    # [OPTIONAL] Neighbor distance threshold (spatial cutoff)
  log_dist    = TRUE,    # [OPTIONAL] Log-transform computed distances
  cores       = NULL     # [OPTIONAL] Number of parallel CPU cores (NULL = auto)
)
```

### Detailed Parameter Documentation

#### 🔴 Required Parameters
| Parameter | Type | Description | Biological Significance |
|---|---|---|---|
| `x` | STlist (S4 object) | Input spatial transcriptomics data container | Central data object holding expression + coordinates |
| `annot` | character | Column name in spot metadata containing domain/cluster labels | Links spatial positions to biologically annotated tissue regions |
| `ref` | character | The specific domain label to use as the reference "niche" | Defines the biological reference (e.g., `"tumor"`, `"stroma"`, `"cluster_1"`) |

#### 🟡 Key Analytical Parameters
| Parameter | Type | Default | Range/Options | Description |
|---|---|---|---|---|
| `topgenes` | integer | `2000` | 1 – total genes | Number of highest-variance genes to include in the gradient screen |
| `distsumm` | character | `"min"` | `"min"`, `"mean"` | How to summarize each spot's distance to the reference domain: **minimum** (closest edge contact) or **mean** (average) |
| `log_dist` | logical | `TRUE` | `TRUE`, `FALSE` | Whether to log-transform distances before correlation; log-scale linearizes exponential spatial decay patterns |
| `out_rm` | logical | `FALSE` | `TRUE`, `FALSE` | Remove statistical outliers from the distance/expression relationship before testing |
| `limit` | numeric | `NULL` | Any positive number | Expression threshold — spots/cells below this expression level may be filtered |
| `exclude` | character vector | `NULL` | Domain names | Specific domains to exclude from non-reference spots (e.g., exclude a domain of ambiguous identity) |

#### 🟢 Spatial/Statistical Tuning Parameters
| Parameter | Type | Default | Description |
|---|---|---|---|
| `min_nb` | integer | `3` | Minimum number of neighboring spots required for distance calculation; prevents unreliable estimates from isolated spots |
| `robust` | logical | `TRUE` | Use robust linear regression (Huber/MM estimators) to reduce influence of extreme expression values; recommended for sparse scRNA-seq-like data |
| `nb_dist_thr` | numeric | `NULL` | Spatial distance threshold for neighbor detection — defines maximum distance to be considered a "neighbor" in the spatial graph |
| `cores` | integer | `NULL` | Number of parallel CPU cores for gene-wise computation; `NULL` auto-detects available cores |
| `samples` | character | `NULL` | Subset of sample names to analyze; `NULL` runs on all samples in the STlist |

GROUP 1: Input Data

### Critical Integration Notes

1. **Launcher Relationship**: The GenePattern documentation explicitly states: *"If you just ran spatialGE.STclust, you probably want the spatialGE.STgradient.Launcher module instead."* This means `STgradient` can be called independently (e.g., for re-running with different reference domains), while the Launcher automates the handoff from `STclust`.

2. **RDS Input Format**: The module accepts RDS-serialized STlist objects as primary input — this is an R-native binary format. GenePattern must handle `.rds` file type association.

3. **Reference Domain Awareness**: The `annot` and `ref` parameters depend on what `STclust` produced. Users must know the exact column name and domain label from their clustering result.

4. **Memory Management**: For large CosMx datasets (100K+ cells), parallelization (`cores`) and memory monitoring are essential. Recommend setting job memory limits in GenePattern manifest.

5. **Pre-release Status**: The module is tagged "Fixing prerelease" — indicating active development. Monitor the GenePattern repository for parameter additions or breaking changes as the spatialGE package updates (currently v1.2.2 on CRAN).

6. **Test Data**: The spatialGE package ships with Thrane melanoma 10X Visium data (`system.file("extdata", package="spatialGE")`), which can serve as test data for the GenePattern module.

## 7. 🆚 Comparative Analysis

| Tool | Approach | Reference Domain Required | Platforms | Gradient Direction | Integration |
|---|---|---|---|---|---|
| **spatialGE.STgradient** | Spearman ρ (distance vs. expression) | ✅ Yes (from STclust) | Visium, CosMx, CSV | ✅ Directional | GenePattern, R, Web |
| **LSGI** | NMF program pairs | ❌ No | Visium | ✅ Directional | R package |
| **Spatial Gradient Screening** (Nat Comm 2024) | Histology-based, no prior grouping | ❌ No (uses histology) | Visium | Bidirectional | R code |
| **SpatialDE** | Gaussian process | ❌ No | Any ST | ❌ Not directional | Python/R |
| **SPARK-X** | Nonparametric covariance | ❌ No | Any ST | ❌ Not directional | R/Bioconductor |
| **nnSVG** | Nearest-neighbor GP | ❌ No | Any ST | ❌ Not directional | R/Bioconductor |
| **STDistance** | Cell-type distance | ✅ Yes (cell type) | Single-cell ST | Visualization only | R/CRAN |

**Key Differentiator**: `STgradient` is **uniquely domain-centric and directional** — it quantifies expression *with respect to a biologically defined tissue niche*, enabling discovery of microenvironment-driven expression programs. This is distinct from spatially variable gene (SVG) detection methods that only ask "does expression vary spatially?" without a biological reference point.

## 9. ⚠️ Known Issues, Limitations & Caveats

| Issue | Severity | Notes |
|---|---|---|
| **Prerelease Status** | 🟡 Medium | Module flagged "Fixing prerelease" — may have parameter changes |
| **Requires STclust Output** | 🟡 Medium | The `annot` and `ref` params require prior domain detection; not usable on raw count matrices alone |
| **Computational Intensity** | 🟡 Medium | Testing 2,000+ genes against distance vectors is parallelizable but memory-intensive for large CosMx datasets |
| **Single Reference Domain** | 🟠 Design constraint | `ref` accepts one domain at a time; multiple reference comparisons require multiple runs |
| **Spearman Limitation** | 🟢 Minor | Spearman ρ captures monotonic relationships; non-monotonic gradient patterns (e.g., bell-shaped) may be missed |
| **Distance Metric** | 🟢 Minor | Only Euclidean distance is implemented; tissue curvature or non-linear topology not accounted for |
| **Normalization Dependency** | 🟢 Minor | Results are sensitive to upstream normalization choice in `transform_data()` |

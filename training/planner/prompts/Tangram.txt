# 🔬 Comprehensive Research Report: Tangram — Spatial Alignment of Single-Cell Transcriptomic Data

---

## 1. 📋 EXECUTIVE SUMMARY

**Tangram** is a deep-learning-based computational framework published in *Nature Methods* (2021) for aligning single-cell (sc) or single-nucleus (sn) RNA-seq data to various forms of spatially resolved transcriptomics data. It bridges the gap between the transcriptomic resolution of scRNA-seq and the spatial context of spatial transcriptomics, enabling transcriptome-wide spatial gene expression mapping at single-cell resolution. The GenePattern module wraps the canonical `tangram-sc` Python package (Broad Institute/MIT) in a Docker container (v1.0), making it accessible through a GUI-driven, reproducible workflow.

---

## 2. 🧬 TOOL PURPOSE AND SCIENTIFIC BACKGROUND

### 2.1 Core Problem Statement
Spatial transcriptomics technologies (e.g., 10x Visium, MERFISH, STARmap, Slide-seq) capture gene expression with spatial context but face trade-offs:
- **Low-plex methods** (Visium): Measure thousands of genes but at ~10–100 cells/spot resolution
- **High-plex imaging** methods (MERFISH, smFISH): Single-cell resolution but limited gene panels (100–1,000 genes)
- **scRNA-seq**: Full transcriptome (~20,000 genes) but lacks spatial context

**Tangram's solution**: Learn an optimal probabilistic mapping matrix **M** (cells × spots) that places each single cell from scRNA-seq into its most probable spatial location, using a shared set of overlapping genes as training signal.

### 2.3 Key Scientific Capabilities
1. **Single-cell spatial deconvolution**: Resolve which individual cells occupy which spatial locations in tissue sections
2. **Transcriptome-wide gene expression imputation**: Predict the spatial expression of ~20,000 genes from only 100–1,000 spatially measured genes
3. **Cell type mapping**: Project scRNA-seq cell type annotations (e.g., Leiden clusters, curated cell types) onto spatial coordinates
4. **Multi-modal integration**: Map RNA-seq, ATAC-seq (via SHARE-seq), or other modalities onto spatial supports
5. **Histology integration**: Align to histological image landmarks from Allen Brain Atlas ISH images
6. **Cross-atlas mapping**: Used in Allen Brain Cell Atlas developmental mouse brain cell type mapping (*Nature* 2021)

## 4. ⚙️ PARAMETER ANALYSIS AND USAGE PATTERNS

### 4.1 Complete Parameter Reference for `map_cells_to_space()`

#### **Group A: Essential Inputs**
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `adata_sc` | AnnData | — | Pre-processed single-cell AnnData (after `pp_adatas`) |

#### **Group B: Mapping Mode Configuration**
| Parameter | Type | Default | Options / Notes |
|-----------|------|---------|----------------|
| `mode` | str | `'cells'` | `'cells'`: map individual cells; `'clusters'`: map cluster centroids (recommended when sc/sn from different sample); `'constrained'`: cell-level with voxel cell-count constraint |
| `cluster_label` | str | `None` | Column in `adata_sc.obs` containing cluster/cell-type labels — **required for `clusters` mode** |
| `target_count` | int | `None` | Desired total number of cells to map — used in `constrained` mode (typically `adata_sp.obs.cell_count.sum()`) |
| `density_prior` | str/array | `'rna_count_based'` | `'rna_count_based'`: weights by RNA counts; `'uniform'`: equal weights; or custom numpy array of per-voxel fractions (required for `constrained`) |

#### **Group C: Training/Optimization Parameters**
| Parameter | Type | Default | Biological Significance |
|-----------|------|---------|------------------------|
| `num_epochs` | int | `1000` | Training iterations; increase for complex datasets or until loss plateaus; typical range 500–3000 |
| `learning_rate` | float | `0.1` | Gradient descent step size; lower values → more stable but slower convergence |
| `scale` | bool | `True` | Normalize expression matrices before training; recommended `True` |
| `random_state` | int | `None` | Set integer seed for reproducibility |
| `device` | str | `'cpu'` | `'cpu'` or `'cuda'` (GPU); GPU dramatically accelerates large datasets |

#### **Group D: Loss Function Hyperparameters (Lambda Terms)**
| Parameter | Type | Default | Role in Objective Function |
|-----------|------|---------|---------------------------|
| `lambda_g1` | float | `1` | **Primary gene expression term** — cosine similarity between predicted and observed spatial gene expression; main learning signal |
| `lambda_g2` | float | `0` | Voxel-gene regularization term — encourages voxel-level consistency |
| `lambda_d` | float | `0` | Cell density KL divergence penalty — penalizes deviation from `density_prior` |
| `lambda_r` | float | `0` | General regularization term — reduces overfitting |
| `lambda_count` | float | `1` | Cell count regularization — used only in `constrained` mode to enforce per-voxel cell counts |
| `lambda_f_reg` | float | `1` | F-regularization — used in `constrained` mode |

> ⚠️ **Lambda Tuning Note**: `lambda_g1=1` is the core term and should rarely be changed. `lambda_d` can be increased (e.g., 0.1–1.0) when spatial density information is reliable. In `constrained` mode, `lambda_count` and `lambda_f_reg` work together to enforce cell count constraints.

#### **Group E: Preprocessing / Cross-Validation Parameters**
| Parameter | Type | Default | Notes |
|-----------|------|---------|-------|
| `cv_train_genes` | list/float | `None` | For cross-validation: list of genes held out from training to assess generalization, or float (fraction) for random holdout |
| `verbose` | bool | `True` | Print training progress (score, KL reg) every 100 epochs |

### 4.2 Preprocessing Function: `pp_adatas()`
```
tg.pp_adatas(adata_sc, adata_sp, genes=None)
```
| Parameter | Description |
|-----------|-------------|
| `adata_sc` | Single-cell AnnData |
| `adata_sp` | Spatial AnnData |
| `genes` | List of training genes (recommended: 100–1,000 cell-type marker DEGs). If `None`, all overlapping genes are used |

**Actions performed:**
1. Finds intersection of genes between sc and spatial datasets
2. Annotates `training_genes` in `.uns` of both AnnData objects
3. Removes all-zero genes from consideration
4. Computes `uniform_density` and `rna_count_based_density` in `adata_sp.obs`

### 4.3 Gene Selection Best Practices
- **Recommended**: 100–1,000 differentially expressed genes, stratified across cell types
- **Rationale**: Training genes should be informative markers; whole-transcriptome training can work but is slower and may overfit sparse data
- **Cross-validation**: Use `cv_train_genes` to hold out test genes and validate mapping quality on unseen genes
- **Source**: Compute DEGs using `scanpy.tl.rank_genes_groups()` prior to running Tangram

### 4.4 Mode Selection Guide
```
Is spatial data single-cell resolution (MERFISH, STARmap)?
  → YES: Use mode='cells'
  
Is spatial data low-resolution (Visium, old ST)?
  → Do you have per-spot cell count estimates?
     → YES: Use mode='constrained', provide target_count and density_prior
     → NO:  Use mode='cells' with density_prior='rna_count_based'

Is the sc/sn data from a DIFFERENT sample/donor than the spatial data?
  → YES: Use mode='clusters' with cluster_label='your_celltype_column'
```

### 6.1 Workflow 1: Cell-Type Deconvolution of Visium Data
**Goal**: Determine proportions of each cell type in low-resolution spatial spots

```python
import tangram as tg
import scanpy as sc
import numpy as np

# Load data
ad_sc = sc.read_h5ad("scrna_seq.h5ad")      # Annotated scRNA-seq
ad_sp = sc.read_h5ad("visium_spatial.h5ad") # 10x Visium data

# Select training genes (top 100 markers per cell type)
sc.tl.rank_genes_groups(ad_sc, "cell_type", method="wilcoxon")
markers = sc.get.rank_genes_groups_df(ad_sc, None)
markers = markers[markers["pvals_adj"] < 0.01].head(500)["names"].tolist()

# Preprocess
tg.pp_adatas(ad_sc, ad_sp, genes=markers)

# Map with constrained mode
ad_map = tg.map_cells_to_space(
    ad_sc, ad_sp,
    mode="constrained",
    target_count=ad_sp.obs.cell_count.sum(),
    density_prior=np.array(ad_sp.obs.cell_count) / ad_sp.obs.cell_count.sum(),
    num_epochs=1000, device="cpu"
)

# Project cell type annotations to space
tg.project_cell_annotations(ad_map, ad_sp, annotation="cell_type")
# → ad_sp.obs now has columns: 'cell_type_A', 'cell_type_B', ... (fractions per spot)
```

## 8. 📐 MATHEMATICAL FRAMEWORK

### 8.1 Optimization Objective
Tangram learns a mapping matrix **M** ∈ ℝ^(n_cells × n_spots) with constraints:
- Each row sums to 1 (softmax normalization): M represents probability distribution of a cell over space
- Each column sum represents relative cell density per voxel

**Loss function:**
```
L(M) = -λ_g1 · cosine_similarity(M^T · S_sc, S_sp)   # Gene expression alignment
      - λ_g2 · voxel_gene_term                         # Voxel-level consistency  
      + λ_d  · KL(M^T · 1 || density_prior)           # Density regularization
      + λ_r  · ||M||_F                                 # Frobenius norm regularization
      + λ_count · count_reg                            # Cell count constraint (constrained mode)
      + λ_f_reg · f_regularization                     # F-regularization (constrained mode)
```

Where:
- **S_sc**: single-cell expression matrix (cells × training_genes)
- **S_sp**: spatial expression matrix (spots × training_genes)
- **KL**: Kullback-Leibler divergence
- Optimization via PyTorch autograd (Adam optimizer)

### 8.2 Training Score Interpretation
```
Score: 0.9xx, KL reg: 0.00x   ← Good convergence
Score: 0.2xx, KL reg: 0.3xx   ← Early training / sparse data (can still be valid)
```
The score reflects cosine similarity on training genes; higher = better gene expression alignment. Low scores on sparse data (Visium) don't necessarily indicate failure — check held-out gene predictions.

## 9. ⚠️ KNOWN LIMITATIONS AND CAVEATS

### 9.1 Technical Limitations
1. **Memory scaling**: Mapping matrix M is (n_cells × n_spots); for 50K cells × 10K spots = 500M floats ≈ 2 GB RAM minimum
2. **Non-convex optimization**: Results may differ across runs without `random_state`; multiple runs recommended for robustness
3. **Gene sparsity sensitivity**: Performance degrades with highly sparse spatial data; gene selection becomes critical
4. **Cell count assumption**: In `cells` mode, each cell is assumed to be placed once; doesn't model multiple copies

### 9.2 Biological Caveats
1. **Assumes tissue similarity**: Best performance when sc and spatial data are from the same tissue type/region
2. **Cell type diversity**: If rare cell types are absent from spatial data, mapping may be forced and incorrect
3. **Cross-sample mapping**: `clusters` mode recommended when using sc atlas data mapped to spatial from different samples
4. **No spatial neighborhood model**: Base Tangram doesn't incorporate spatial autocorrelation (addressed in refinement strategies paper, Bioinformatics 2025)

### 9.3 Practical Issues
- GPU memory overflow with very large datasets → use `mode='clusters'` or batch processing
- Training instability with `lambda_d > 0` when density prior is poorly estimated
- Gene name mismatches between sc and spatial datasets will silently reduce training gene count

### 9.4 Recent Refinements (Bioinformatics 2025)
Stahl et al. identified and addressed key issues:
- Informative gene subset selection is critical
- Cell filtering of low-quality cells improves mapping
- Regularization tuning reduces inconsistent mappings
- Neighborhood information incorporation improves spatial coherence

| `{prefix}_mapping.h5ad` | AnnData | Core mapping matrix M (cells × spots) |
| `{prefix}_spatial_projected.h5ad` | AnnData | Projected gene expression (spots × all genes) |
| `{prefix}_cell_type_proportions.csv` | CSV | Per-spot cell type fraction table |
| `{prefix}_training_history.csv` | CSV | Per-epoch loss/score for QC |
| `{prefix}_training_genes.txt` | TXT | List of genes used for training |

### 10.5 Error Handling Recommendations
1. **Gene overlap check**: Warn if fewer than 50 training genes overlap between sc and spatial
2. **Memory estimate**: Pre-compute M matrix size and warn if > 4 GB
3. **Mode validation**: If `mode='constrained'` but no `target_count` provided, raise informative error
4. **Cell type label**: Validate `cluster_label` column exists in `adata_sc.obs`
5. **Device check**: If `device='cuda'` but no GPU available, fall back to CPU with warning

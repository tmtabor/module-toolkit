## üìå 1. Executive Summary

**Salmon.Quant** is a GenePattern module wrapping **Salmon v1.6.0**, a state-of-the-art RNA-seq transcript quantification tool developed by the COMBINE-lab. It performs **transcript-level abundance estimation** from RNA-seq reads using selective alignment (quasi-mapping), providing TPM and estimated read counts for each transcript without producing a traditional genome alignment (BAM) file. The GenePattern module (v0.9, beta) bundles pre-built **GENCODE** indices for human (release v39, GRCh38) and mouse (release vM28, GRCm39), making it immediately usable without any index-building step.

**Primary Citation:**
> Patro, R., Duggal, G., Love, M.I., Irizarry, R.A., & Kingsford, C. (2017). **Salmon provides fast and bias-aware quantification of transcript expression**. *Nature Methods*, 14, 417‚Äì419. DOI: [10.1038/nmeth.4197](https://doi.org/10.1038/nmeth.4197) | PMID: 28263959

## üéØ 2. Tool Purpose and Scientific Applications

### 2.1 Core Function
Salmon quantifies **transcript expression levels** from RNA-seq reads. Unlike alignment-based methods that map reads to a genome, Salmon uses a **two-phase inference procedure** combining:
1. **Selective alignment** (replacing earlier quasi-mapping from v1.0+): Maps reads to a compact index of the transcriptome using a SMEM-based algorithm with score validation
2. **Variational Bayesian EM (VBEM)**: Estimates transcript abundances while simultaneously modeling multi-mapping and various technical biases

### 2.2 Scientific Applications
| Application | Description |
|---|---|
| **Differential Expression** | Provides TPM + estimated counts as input to DESeq2, edgeR via tximport/tximeta |
| **Isoform-level analysis** | Transcript-resolution expression estimates, not collapsed to gene level |
| **Bias-corrected quantification** | Accounts for GC content, sequence-specific, and positional biases |
| **Uncertainty quantification** | Bootstrap/Gibbs sampling for expression uncertainty (used by sleuth, fishpond) |
| **Multi-condition comparisons** | Gene expression atlas studies, eQTL mapping, co-expression networks |
| **Single-cell preprocessing** | Alevin sub-command (not in this module) extends to single-cell RNA-seq |
| **Time-series / longitudinal** | Fast enough (~minutes/sample) to scale to large cohorts |

### 2.3 Key Scientific Innovations
- **First tool to correct for fragment-level GC bias** genome-wide (`--gcBias`)
- **Sequence-specific bias correction** models 5‚Ä≤/3‚Ä≤ primer biases (`--seqBias`)
- **Automatic library-type detection** (`-l A`) ‚Äî infers strandedness from data
- **Equivalence class dumping** (`--dumpEq`) for downstream use in compositional analysis
- **Effective length correction**: Corrects raw counts for transcript length √ó bias effects, improving differential expression accuracy when used with tximeta

### 4.2 Library Type Reference Table
```
Fragment Orientation:    I = Inward (‚Üí‚Üê)   O = Outward (‚Üê‚Üí)   M = Matching (‚Üí‚Üí)
Strandedness:           S = Stranded        U = Unstranded
Direction:              F = Forward         R = Reverse
```
| Code | Meaning | Typical Protocol |
|---|---|---|
| **A** | Auto-detect | **Recommended default** |
| **IU** | Inward, Unstranded | Standard unstranded PE (TruSeq unstranded) |
| **ISF** | Inward, Stranded, Forward | Ligation-based, ScriptSeq |
| **ISR** | Inward, Stranded, Reverse | **dUTP, TruSeq Stranded (most common)** |
| **U** | Unstranded (SE) | Single-end unstranded |
| **SF** | Stranded Forward (SE) | Single-end, forward strand |
| **SR** | Stranded Reverse (SE) | Single-end, reverse strand |

### 4.3 Recommended Usage Patterns

**Pattern 1: Standard Paired-End (Best Practice)**
```bash
salmon quant \
  -i /path/to/gencode_v39_index \
  -l A \
  -1 sample_R1.fastq.gz \
  -2 sample_R2.fastq.gz \
  --validateMappings \
  --gcBias \
  --seqBias \
  --gencode \
  -g gencode.v39.annotation.gtf \
  -p 12 \
  -o ./sample_quant
```

**Pattern 2: Single-End with Uncertainty Estimation**
```bash
salmon quant \
  -i /path/to/index \
  -l A \
  -r sample.fastq.gz \
  --validateMappings \
  --gcBias \
  --numBootstraps 30 \
  --fldMean 200 --fldSD 80 \
  -p 8 \
  -o ./sample_quant
```

**Pattern 3: Alignment-Based Mode (STAR BAM input)**
```bash
salmon quant \
  -t transcriptome.fa \
  -l A \
  -a STAR_Aligned.toTranscriptome.out.bam \
  --gcBias \
  -g annotation.gtf \
  -o ./sample_quant
```

## üîÑ 6. Common Workflows and Use Cases

### 6.1 Standard Bulk RNA-seq DE Workflow
```
Raw FASTQ ‚Üí [Quality Trim: fastp/Trimmomatic] ‚Üí Salmon.Quant ‚Üí tximeta/tximport (R) ‚Üí DESeq2/edgeR
```

1. **Salmon.Quant** (this module) ‚Üí produces per-sample `quant.sf` files
2. **tximeta** (R/Bioconductor) ‚Üí imports `quant.sf` files with metadata, creates `SummarizedExperiment`
3. **DESeq2 or edgeR** ‚Üí differential expression analysis using length-scaled counts from tximeta

### 6.2 Isoform-Level Analysis Workflow
```
Salmon.Quant ‚Üí IsoformSwitchAnalyzeR / DRIMSeq ‚Üí Differential Transcript Usage
```

### 6.3 Uncertainty-Aware Differential Expression
```
Salmon.Quant (--numBootstraps 30) ‚Üí sleuth ‚Üí LRT/Wald tests with uncertainty
Salmon.Quant (--numGibbsSamples 30) ‚Üí fishpond (swish test) ‚Üí DE with resampling
```

### 6.4 Multi-Sample Expression Atlas
```
[Batch of FASTQ pairs] ‚Üí Salmon.Quant (parallel runs) ‚Üí quantmerge ‚Üí Expression matrix
```
```bash
salmon quantmerge --quants sample1/ sample2/ sample3/ --names S1 S2 S3 -o merged_TPM.tsv
```

### 6.5 Downstream R Analysis (Canonical)
```r
library(tximeta)
library(DESeq2)

# Load sample table
coldata <- read.csv("sample_table.csv")
coldata$files <- file.path(coldata$names, "quant.sf")

# Import with tximeta (handles GENCODE metadata automatically)
se <- tximeta(coldata)
gse <- summarizeToGene(se)  # Collapse to gene level

# Differential expression
dds <- DESeqDataSet(gse, design = ~condition)
dds <- DESeq(dds)
res <- results(dds)
```

| **Best For** | Bulk RNA-seq DE pipelines | Ultra-fast exploratory | Strict reproducibility | Genome annotation studies |

### 7.2 Positioning of Salmon.Quant
**Salmon occupies the optimal balance** between speed and accuracy in the RNA-seq quantification landscape:
- **vs. kallisto**: Salmon is marginally slower but provides superior bias correction (especially `--gcBias`), which is critical for accurate isoform-level DE
- **vs. RSEM**: Salmon is 10‚Äì50x faster with comparable accuracy and no need for a full alignment step
- **vs. STAR+featureCounts**: Salmon provides transcript-level estimates and bias-corrected TPMs; STAR+featureCounts remains preferable for splice-junction discovery workflows

### 8.3 Critical Implementation Notes

1. **`--gencode` Flag**: **Must be auto-enabled** whenever a GENCODE index is selected; GENCODE transcript headers contain pipe-delimited fields (e.g., `ENST00000456328.2|ENSG00000223972.5|...`) that will cause quantification errors without this flag.

2. **Index Packaging**: Pre-built decoy-aware indices should be stored as compressed archives (`.tar.gz`) within the module container or a linked data repository; unpacking at runtime adds ~10‚Äì30 sec.

3. **Output Packaging**: For GenePattern, bundle the entire output directory as a `.tar.gz` or `.zip`; at minimum expose `quant.sf`, `quant.genes.sf`, and `aux_info/meta_info.json`.

4. **Paired-End Detection**: Validate that both R1 and R2 are provided for paired-end mode; flag file naming inconsistencies (mismatched read counts).

5. **Mapping Rate Check**: Parse `meta_info.json` to emit a warning if `percent_mapped < 50%` ‚Äî indicates possible species mismatch or wrong library type.

6. **Bootstrap/Gibbs Mutual Exclusion**: `--numBootstraps` and `--numGibbsSamples` are mutually exclusive; UI should enforce this.

7. **Single-End Fragment Length**: `--fldMean` and `--fldSD` are **only relevant for single-end** libraries; Salmon infers these from data for paired-end runs.

8. **Thread Safety**: The module should cap threads at the available container CPU allocation (typically 8 on shared GenePattern infrastructure).

### 8.4 Known Limitations and Caveats

| Issue | Impact | Mitigation |
|---|---|---|
| No Windows native support | N/A for Docker-based module | Docker handles this transparently |
| M1/ARM macOS incompatibility | Local development | Use x86_64 Docker image |
| Index must match genome build | Wrong results if mismatched | Clearly label index versions in UI |
| GENCODE pipe headers | Parse errors | Auto-enable `--gencode` for GENCODE indices |
| GC bias adds runtime | ~2‚Äì5 extra minutes | Enable by default; document in UI |
| Bootstraps increase disk usage | ~10x larger output | Only enable when needed for sleuth/fishpond |
| Transcript-only (no novel isoforms) | Cannot detect unannotated transcripts | Use StringTie for discovery workflows |
| Effective length < 1 warning | May affect rare very short transcripts | Salmon handles internally, low-concern |

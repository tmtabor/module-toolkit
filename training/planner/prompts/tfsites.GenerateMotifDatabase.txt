## 1. 📌 Tool Identity & Scientific Purpose

### 1.1 Overview

`tfsites.GenerateMotifDatabase` is a **bioinformatics data-preparation module** designed to serve as the **upstream foundational step** in the TFSites analysis pipeline. Its core function is to:

1. Accept a **JASPAR Position Frequency Matrix (PFM) database file** as input
2. **Extract PFM(s)** for one or more user-specified transcription factor(s) of interest
3. **Convert each PFM to its corresponding Position Weight Matrix (PWM)**, applying log-odds transformation with pseudocounting
4. Output a **PWM database file** that downstream tfsites modules use to score TF binding sites across genomic sequences

> **Key Insight:** This module is not a standalone analysis tool — it is an **essential data-bridge module** that translates the community-standard JASPAR open-access database format into the scoring matrices required by the entire tfsites computational toolkit.

### 1.2 Scientific Context

The module is part of the **TFSites computational toolkit**, developed in collaboration with the **Emma Farley laboratory at UC San Diego** (Institute for Genomic Medicine / School of Biological Sciences). The Farley lab focuses on:

- **Enhancer biology** and cis-regulatory element analysis
- **Transcription factor binding site grammar** in development and disease
- **High-throughput functional approaches** (MPRA, SELEX, in vivo embryonic assays) for deciphering how enhancers encode developmental instructions
- Identifying **enhancer mutations** associated with disease

The toolkit is described at [tfsites.org](https://tfsites.org) as:

> *"An easy-to-use computational toolkit to analyze enhancer sequences. Taking a variety of transcription factor (TF) reference data as input, TF Sites annotates and visualizes predicted sites within enhancers, reveals the effect of all possible mutations on TF-DNA interactions, and predicts which sites are altered across enhancer sequence variants."*

**Primary Citation:**
> Jindal, G. & Farley, E. (2021). *"Enhancer grammar in development, evolution, and disease – dependencies and interplay."* — foundational work underpinning the tfsites toolkit, widely cited in cis-regulatory biology.

## 2. 🔬 Scientific Background: PFM → PWM Conversion

Understanding the algorithm is critical for proper module parameterization:

### 2.1 Definitions

| Matrix Type | Abbreviation | Description |
|-------------|-------------|-------------|
| Position Frequency Matrix | **PFM** | Raw nucleotide counts at each position of a TF binding motif (from aligned binding sequences via ChIP-seq, SELEX, etc.) |
| Position Probability Matrix | **PPM** | Column-normalized PFM; each column sums to 1.0 |
| Position Weight Matrix | **PWM** | Log-odds of PPM vs. background; the scoring matrix for sequence scanning |
| Position-Specific Scoring Matrix | **PSSM** | Synonym for PWM |

### 2.2 Conversion Formula

```
Step 1 (Pseudocount): Adjusted_count[b,i] = PFM[b,i] + (pseudocount × bg[b])
Step 2 (PPM):         PPM[b,i] = Adjusted_count[b,i] / sum_b(Adjusted_count[b,i])
Step 3 (PWM):         PWM[b,i] = log2(PPM[b,i] / bg[b])
```

Where:
- `b` = nucleotide (A, C, G, T)
- `i` = position index within the motif
- `bg[b]` = background frequency of nucleotide `b` (default: 0.25 each for uniform/equal GC)
- `pseudocount` = small positive value to prevent log(0) errors (typically 0.1–0.5)

**Scoring interpretation:**
- **Positive PWM values** → nucleotide is enriched at that position (favored by the TF)
- **Negative PWM values** → nucleotide is depleted at that position
- The total **PWM score for a k-mer window** is the sum of position-specific scores and represents the log-odds of that k-mer being a real binding site

## 3. 🗂️ Input/Output Specifications

### 3.1 Primary Inputs

| Input | Format | Required | Description |
|-------|--------|----------|-------------|
| **JASPAR PFM Database File** | `.pfm` / `.txt` JASPAR format | ✅ Yes | A JASPAR-formatted file containing PFM matrices for one or more TFs |
| **Transcription Factor Name(s)** | String / comma-separated list | ✅ Yes | Names of TFs to extract (must match JASPAR identifiers, e.g., `CTCF`, `GATA1`, `SP1`, `SOX2`) |
| **Output Filename/Prefix** | String | Optional | Desired name for output PWM file |

### 3.2 JASPAR PFM File Format (Input)

JASPAR uses a FASTA-like structure. The **JASPAR 2016+ format** (the current standard) uses letter-prefixed rows with square brackets:

```
>MA0139.1 CTCF
A [  87  167  281   56    8  744   40  107  851    5  333   54 ]
C [ 291  145   49  466  182   66   81  325    0   12   78  708 ]
G [  76  414  449  118  246  137   54  167   11  744  333   54 ]
T [  72   14   59   97   92   31  281  228   18   17   74   54 ]

>MA0002.3 RUNX1
A [  10  12   4   1   2   2   0  164  0 ]
C [   2   2   7  21  22  52  55   38  0 ]
G [   3   5  16   2   2   42  9   27  0 ]
T [ 145  141 133 136 134   54  86  131 7 ]
```

**Key format details:**
- Each matrix is preceded by a `>` header line with JASPAR ID and TF name
- Rows correspond to A, C, G, T nucleotide counts respectively
- Multiple matrices can be concatenated in a single file
- Available for download from [jaspar.elixir.no/downloads](https://jaspar.elixir.no/downloads) by taxonomic group

### 3.3 Primary Output

| Output | Format | Description |
|--------|--------|-------------|
| **PWM Database File** | Tab-separated text / custom format | Contains computed log-odds weight matrices for each requested TF; directly consumed by downstream tfsites modules |

The output PWM file is the **key interoperability artifact** linking `GenerateMotifDatabase` to modules like `tfsites.AffinityScan`, which uses the PWM to scan genomic sequences and score all k-mer windows for TF binding affinity.

### 4.1 Parameter Catalog

| Parameter | Category | Type | Default | Biological Significance |
|-----------|----------|------|---------|------------------------|
| `jaspar_pfm_file` | Input | File | — | The JASPAR PFM flat file; typically downloaded from jaspar.elixir.no filtered by species (vertebrates, insects, plants, etc.) |
| `tf_names` | Input | String/List | — | Comma-separated or newline-separated list of TF names to extract (e.g., `CTCF,GATA1,SOX2`). Names are matched against JASPAR ID headers |
| `output_file` | Output | String | auto-generated | Filename for the resulting PWM database |
| `pseudocount` | Algorithm | Float | ~0.1–0.25 | Added to PFM counts before normalization to prevent log(0) and smooth low-count positions. **Biologically:** higher values reduce the weight of rare nucleotide observations; lower values amplify specificity signal |
| `background_frequencies` | Algorithm | Float[4] | [0.25, 0.25, 0.25, 0.25] | Expected background nucleotide frequencies for log-odds calculation. For GC-rich genomes (e.g., plants, some promoters), non-uniform backgrounds may be appropriate |
| `log_base` | Algorithm | Integer | 2 | Log base for PWM calculation. Log2 gives scores in bits; natural log gives nats |

## 9. ⚠️ Known Limitations & Caveats

| Limitation | Details | Mitigation |
|------------|---------|------------|
| **JASPAR-dependent** | Requires a pre-downloaded JASPAR PFM file; no built-in database query | Pre-bundle common JASPAR releases as bundled resources; document download process |
| **Crystallography code removed** | Prior version had crystallographic structure-based functionality; current version is PWM-only | Document scope clearly; not relevant for most genomic applications |
| **JASPAR version specificity** | Different JASPAR releases have different TF names and matrix IDs | Document tested JASPAR versions; recommend JASPAR 2022 or 2024 |
| **TF name matching** | Matching depends on TF name/ID string in JASPAR header — case sensitivity and aliases may cause failures | Implement fuzzy matching or name normalization option |
| **No deep learning motif models** | Only handles classical PFM/PWM; JASPAR 2024 TFFMs (deep learning models) are not supported | Document limitation; classical PWMs cover >95% of use cases |
| **Single species at a time** | Best practice is to use a species-specific JASPAR subset | Document species-specific download instructions |

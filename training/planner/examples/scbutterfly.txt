### Scientific Problem & Purpose

scButterfly addresses one of the central challenges in modern single-cell genomics: **cross-modality translation** — the computational prediction of one type of genomic measurement from another at single-cell resolution. Specifically:

- **Primary Problem:** Generating a complete multi-omic profile from a single-omic measurement. While technologies like 10x Multiome can simultaneously profile RNA and chromatin accessibility, they are expensive and technically demanding. Most existing datasets contain only one modality.
- **Core Task:** Given cells profiled for scRNA-seq only, *predict* what their scATAC-seq profiles would look like (and vice versa).
- **Biological Significance:** RNA-to-ATAC and ATAC-to-RNA translation illuminates the mechanistic relationships between **gene expression** and **chromatin accessibility**, forming a computational window into gene regulatory networks.

### Key Scientific Applications
1. **Bidirectional scRNA-seq ↔ scATAC-seq translation** (flagship use case)
2. **scRNA-seq ↔ ADT (protein abundance) translation** for CITE-seq data
3. **Single-cell perturbation response prediction** (predict post-perturbation cell states from control states)
4. **Integrative multi-omics analysis** of cells with only one measured modality
5. **Data enhancement** of poor-quality single-cell multi-omics data
6. **Automatic cell type annotation of scATAC-seq data** via RNA-space annotation transfer
7. **Consecutive/chained translation** across multiple modalities

## 2. 🏗️ TECHNICAL ARCHITECTURE

### Core Framework: Dual-Aligned Variational Autoencoder (DAVAE)

scButterfly is built on a novel **Dual-Aligned Variational Autoencoder** framework with **seven primary components**:

```
                     ┌─────────────────────────────────────────────────────┐
                     │              scButterfly Architecture               │
                     │                                                     │
  scRNA-seq (Xr) ──► │ [Enr: RNA Encoder] ──► μr, σr                     │
       ↑              │        │                  │                        │
       │              │        ▼                  ▼                        │
  [Der: RNA Decoder]  │  [Disr: RNA Discriminator]  ──── Alignment Loss   │
       │              │        │                                           │
       │              │   [Translator T]                                   │
       │              │  (samples from N(μ,σ) → latent z)                 │
  [Dea: ATAC Decoder] │        │                  ▲                        │
       ↑              │        ▼                  │                        │
  scATAC-seq (Xa) ──► │ [Ena: ATAC Encoder] ──► μa, σa                   │
                     │  [Disa: ATAC Discriminator]  ── Alignment Loss     │
                     └─────────────────────────────────────────────────────┘
```

| Component | Function |
|-----------|----------|
| **Enr** (RNA Encoder) | Maps scRNA-seq input → 128-dimensional mean vector; uses masking strategy to handle dropout noise |
| **Ena** (ATAC Encoder) | Maps scATAC-seq input → latent space; intra-chromosomal connections only (pruned inter-chromosomal links) |
| **T** (Translator) | Samples from multivariate Gaussian distribution; performs cross-modal translation |
| **Der** (RNA Decoder) | Reconstructs RNA profiles from latent embeddings (two FC blocks + LeakyReLU) |
| **Dea** (ATAC Decoder) | Reconstructs ATAC peak profiles; chromosome-aware architecture |
| **Disr** (RNA Discriminator) | Adversarial alignment of RNA latent space |
| **Disa** (ATAC Discriminator) | Adversarial alignment of ATAC latent space |

### Key Architectural Innovations
1. **Masking Strategy**: Randomly masks input features during encoding to simulate and handle the pervasive **dropout noise** in single-cell data.
2. **Intra-Chromosomal Pruning**: The ATAC encoder/decoder only maintains connections *within* each chromosome — reducing computational burden while biologically motivated (regulatory elements act locally within chromosomes).
3. **Dual Discriminators**: Two separate adversarial networks enforce alignment between the RNA and ATAC latent spaces, enabling high-fidelity cross-modal translation.
4. **Optimal Transport for Unpaired Data**: When paired training cells are unavailable, scButterfly-T uses optimal transport strategies to find cross-modal correspondences.

## 3. 📦 MODEL VARIANTS

scButterfly offers **three distinct model variants** controlled by the `aug_type` parameter:

| Variant | Name | `aug_type` value | Description | Best For |
|---------|------|------------------|-------------|----------|
| **scButterfly-B** | Basic | `None` | No data augmentation; trains directly on paired data | Clean paired datasets with sufficient cells |
| **scButterfly-T** | Type | `'cell_type_augmentation'` | Augments training data using cell-type label information from `adata.obs` | Datasets with available cell type annotations; also enables unpaired translation via optimal transport |
| **scButterfly-C** | Cluster | `'MultiVI_augmentation'` | First trains a MultiVI model for unsupervised clustering, then uses predicted clusters for augmentation | Datasets without cell type labels; requires MultiVI |

## 4. 📋 INPUT/OUTPUT FORMATS & REQUIREMENTS

### Input Data

| Input | Format | Description |
|-------|--------|-------------|
| **RNA Data** | `AnnData` (`.h5ad`) | Raw count matrix; shape: `n_cells × n_genes` |
| **ATAC Data** | `AnnData` (`.h5ad`) | Binary or count accessibility matrix; shape: `n_cells × n_peaks`; **peaks must be sorted by chromosome** |
| **Cell Split Indices** | `list` / `array` | Cell barcodes or integer indices for `train_id`, `test_id`, `validation_id` |
| **Cell Type Labels** *(optional)* | Column in `adata.obs` | String labels per cell; required for scButterfly-T |
| **Peaks per Chromosome** *(required)* | Python `list` of `int` | Number of peaks in each chromosome (sorted order); critical for intra-chromosomal architecture |

> ⚠️ **Critical Constraint**: ATAC peaks must be **sorted by chromosome** before input. The `peaks_count_per_chrom` parameter must match the chromosome order.

### Output Data

| Output | Format | Description |
|--------|--------|-------------|
| **Predicted Profiles** | `AnnData` (`.h5ad`) | Predicted modality profiles (RNA or ATAC) for test cells |
| **Model Checkpoints** | PyTorch `.pt` files | Saved model weights for subsequent inference |
| **Evaluation Metrics** | CSV / dict | AMI, ARI, HOM, NMI, Pearson correlation, AUROC scores |
| **Embeddings** | Stored in `adata.obsm` | Latent space representations for downstream analysis |
| **Visualizations** *(optional)* | PNG/PDF | t-SNE/UMAP plots of predicted vs. true profiles |

| `n_highly_var_genes` | `int` | `2000` | 500–20000 | Number of highly variable genes selected from RNA data |

## 7. 🔄 WORKFLOWS & USE CASES

### Primary Workflow: Paired scRNA-seq ↔ scATAC-seq Translation (scButterfly-B/T)

```python
import scanpy as sc
from scButterfly.butterfly import Butterfly

# --- Data Loading ---
rna_adata = sc.read_h5ad("rna_counts.h5ad")   # cells × genes (raw counts)
atac_adata = sc.read_h5ad("atac_counts.h5ad") # cells × peaks (sorted by chrom)

# --- Train/Test Split ---
import numpy as np
n_cells = rna_adata.n_obs
train_id = list(range(0, int(0.8 * n_cells)))
test_id  = list(range(int(0.8 * n_cells), int(0.9 * n_cells)))
val_id   = list(range(int(0.9 * n_cells), n_cells))

# --- Initialize & Load ---
butterfly = Butterfly()
butterfly.load_data(rna_adata, atac_adata, train_id, test_id, val_id)

# --- Preprocessing ---
butterfly.data_preprocessing(
    n_highly_var_genes=2000,
    n_top_peaks=30000
)

# --- Augmentation (scButterfly-T with cell type labels) ---
butterfly.augmentation(
    aug_type='cell_type_augmentation',
    cell_type_key='cell_type'    # column in atac_adata.obs or rna_adata.obs
)

# --- Construct Model (requires peaks per chromosome) ---
peaks_per_chrom = [500, 450, 800, ...]  # peaks on chr1, chr2, chr3, etc.
butterfly.construct_model(peaks_count_per_chrom=peaks_per_chrom)

# --- Train ---
butterfly.train_model(
    n_epochs=100,
    batch_size=64,
    learning_rate=1e-4
)

# --- Predict & Evaluate ---
predicted = butterfly.test_model(test_id=test_id)
# predicted: AnnData with predicted modality profiles
```

### Workflow 2: Unpaired Data Translation (scButterfly-T with OT)

Used when RNA and ATAC datasets are from different experiments (no paired cells). The tool uses **optimal transport** to construct pseudo-paired data before training.

## 9. ⚠️ KNOWN LIMITATIONS & CAVEATS

| Limitation | Impact | Mitigation |
|-----------|--------|------------|
| **Requires chromosome-sorted ATAC peaks** | Data preparation burden; unsorted peaks will produce incorrect chromosomal architecture | Enforce sorting in preprocessing wrapper |
| **Python 3.9 dependency** | May conflict with other tools requiring newer Python | Use dedicated conda environment |
| **MultiVI dependency for scButterfly-C** | Additional large dependency; potential version conflicts | Make scButterfly-C optional in module; fallback to -B or -T |
| **Paired training data needed (B & T)** | Can't be used directly on completely unpaired datasets for -B; -T mitigates via OT | Offer scButterfly-T as default when no paired data |
| **GPU strongly recommended** | CPU training is feasible but very slow for > 10K cells | Document expected runtimes; consider cloud GPU resources |
| **Memory scales with `n_top_peaks`** | Using all peaks (>100K) may exceed RAM | Provide guidance on `n_top_peaks` selection |
| **No direct CLI tool** | Python API only; requires scripting knowledge | Wrap in Python argparse CLI for GenePattern |
| **Version 0.0.9 still early-stage** | API may change in future versions | Pin version in Docker container |

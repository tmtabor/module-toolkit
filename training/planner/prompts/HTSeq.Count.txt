## 1. Tool Purpose & Scientific Applications

### Primary Purpose
HTSeq.Count is a **read-counting tool** for RNA-sequencing (RNA-seq) experiments. It takes pre-aligned reads (in SAM/BAM format) and a genome annotation file (in GTF/GFF format) and computes, for each annotated genomic feature (typically a gene), how many aligned reads overlap with that feature. The resulting count table is the fundamental input for most **differential gene expression (DGE) analyses**.

### Scientific Applications
| Application | Description |
|---|---|
| **Gene expression quantification** | Counting reads per gene as a proxy for transcript abundance |
| **Differential expression analysis** | Generating count matrices as input for DESeq2, edgeR, limma-voom |
| **RNA-seq QC** | The special counter rows (`__no_feature`, `__ambiguous`) serve as quality metrics |
| **ChIP-seq quantification** | Counting reads overlapping with binding peaks/regions |
| **Custom feature counting** | Any experiment requiring reads-per-feature statistics using GFF-defined intervals |
| **Exon-level analysis** | Counting at the exon level by changing `--type` parameter |

### Scientific Context
HTSeq.Count implements the critical **alignment → quantification** step in the canonical RNA-seq analysis pipeline. It converts alignment data into a **count matrix** — an integer matrix of samples × genes — which is the standard input for statistical DGE frameworks. The tool was specifically designed with DESeq2 and edgeR in mind, favoring conservative read assignment to ensure counts are statistically interpretable.

## 2. Algorithm & Counting Logic

### Core Algorithm: Set-Based Feature Assignment
For each aligned read (or read pair in PE mode), HTSeq.Count defines a **set S** using one of three modes and assigns the read accordingly:

```
For each position i in the read:
    S(i) = { all features overlapping position i }
```

Then, depending on mode, a composite set S is derived and the decision is made:
- **If |S| == 1** → Read counted for that one feature ✅
- **If |S| == 0** → Read counted as `__no_feature` ❌
- **If |S| > 1** → Governed by `--nonunique` option (default: counted as `__ambiguous`) ⚠️

### Three Counting Modes

```
UNION mode (Recommended)               INTERSECTION-STRICT            INTERSECTION-NONEMPTY
━━━━━━━━━━━━━━━━━━━━━━━━━━━            ━━━━━━━━━━━━━━━━━━━━━          ━━━━━━━━━━━━━━━━━━━━━
S = ∪ S(i) for all i                   S = ∩ S(i) for all i           S = ∩ of non-empty S(i)

Most permissive                        Most stringent                 Middle ground
Counts reads spanning                  Reads spanning introns         Better for overlapping
  intron-exon boundaries                 may be undercounted            genes
Recommended for standard               Can undercount in complex      Useful when genes have
  RNA-seq                                gene architectures              overlapping exons
```

### Strandedness Logic
```
--stranded=yes:    Read on forward strand → counted for forward-strand features only
--stranded=reverse: Read on forward strand → counted for REVERSE-strand features (dUTP protocols)
--stranded=no:     Strand ignored for assignment (unstranded protocols)
```
> ⚠️ **Critical**: Using `--stranded=yes` on unstranded data causes ~50% read loss. The GenePattern module defaults to `no`.

### Non-Unique Read Handling (`--nonunique`)
| Option | Behavior |
|---|---|
| `none` (default) | Multi-feature reads → `__ambiguous`, not counted |
| `all` | Read counted for ALL overlapping features (inflates counts) |
| `fraction` | Read contributes 1/N to each of N overlapping features |
| `random` | Read randomly assigned to one overlapping feature |

## 3. Input/Output Formats & Requirements

### Input Files

#### 3.1 Alignment Files (Required)
| Property | Details |
|---|---|
| **Formats** | SAM (text), BAM (binary), CRAM (newer versions) |
| **Multiple files** | Supported; GenePattern wrapper handles batch processing |

```

```

**Example SAM entry:**
```
read1  99  chr1  12345  255  100M  =  12500  255  ACTG...  ~~~~  NH:i:1
```

#### 3.2 Annotation File (Required)
| Property | Details |
|---|---|
| **Formats** | GTF (Gene Transfer Format), GFF2, GFF3 |
| **Recommended source** | Ensembl GTF (most compatible with defaults) |
| **Default in GenePattern v3.0** | Pre-loaded references from `datasets.genepattern.org` |
| **Previous source** | `ftp://gpftp.broadinstitutue.org` (changed in v3.0) |
| **Chromosome naming** | Must match BAM file exactly (chr1 vs 1 mismatch = common error) |

**Supported organisms via GenePattern default GTFs:**
- *Homo sapiens* (hg38/GRCh38, hg19/GRCh37)
- *Mus musculus* (mm10/GRCm38)
- *Rattus norvegicus*
- *Drosophila melanogaster*
- *Other organisms* via custom GTF file upload

**GTF file structure required by HTSeq:**
```
# Column 3 = feature type (default: exon)
# Column 9 = attributes including gene_id (default idattr)
chr1  Ensembl  exon  11869  12227  .  +  .  gene_id "ENSG00000223972"; transcript_id "ENST00000456328";
```

### Output Files

#### 3.3 Primary Count Table
```
# Format: Two-column tab-separated (single sample)
# Column 1: Feature ID (gene_id by default)
# Column 2: Integer read count

ENSG00000000003    1234
ENSG00000000005    567
ENSG00000000419    890
...
__no_feature       45231
__ambiguous        12045
__too_low_aQual    0
__not_aligned      8934

```

#### 3.4 Multi-Sample Combined Matrix (GenePattern extension)
```

ENSG00000000005    567      432       789
...
```

#### 3.5 Special Diagnostic Counters
| Counter | Meaning | Diagnostic Significance |
|---|---|---|
| `__no_feature` | Read not overlapping any feature | High value → wrong annotation, chr name mismatch, or wrong stranded setting |
| `__ambiguous` | Read overlapping multiple features | High value → overlapping genes; consider `--nonunique fraction` |
| `__too_low_aQual` | MAPQ below threshold (default 10) | High value → poor alignment quality |
| `__not_aligned` | Unmapped reads in BAM | Should be low after filtering |
| `__alignment_not_unique` | Multi-mapped reads (NH > 1) | High value → repetitive genome regions or short reads |

#### 3.6 Optional SAM Output (`-o`/`--samout`)
A SAM/BAM file with each alignment annotated with an `XF` tag indicating its assigned feature:
```
XF:Z:ENSG00000000003   # Read assigned to this gene
XF:Z:__ambiguous        # Read was ambiguous
XF:Z:__no_feature       # Read did not overlap any feature
```

```
┌──────────────┐    ┌──────────────┐    ┌─────────────────┐    ┌───────────────┐
```

**Concrete command in GenePattern context:**
```bash
# Step 1: Align with STAR (external)
STAR --runMode alignReads --genomeDir /ref/star_index \
     --readFilesIn sample1_R1.fastq.gz sample1_R2.fastq.gz \
     --outSAMtype BAM SortedByCoordinate \
     --outFileNamePrefix sample1_

# Step 2: Name-sort (GenePattern wrapper does this automatically)
samtools sort -n sample1_Aligned.sortedByCoord.out.bam -o sample1_namesorted.bam

# Step 3: HTSeq-count (via GenePattern module)
htseq-count -f bam -r name -s reverse \
            -t exon -i gene_id -m union \
            sample1_namesorted.bam Homo_sapiens.GRCh38.104.gtf \
            > sample1_counts.txt
```

## 9. Known Issues, Limitations & Best Practices

### 9.1 Known Issues

| Issue | Cause | Solution |
|---|---|---|
| `MemoryError` / buffer overflow | Large position-sorted paired-end BAM | Name-sort BAM before running (GP wrapper does this automatically) |
| All reads in `__no_feature` | Chromosome naming mismatch (chr1 vs 1) | Ensure BAM and GTF use same chromosome naming convention |
| High `__no_feature` rate | Wrong `--stranded` setting | Verify library prep protocol and set stranded correctly |
| High `__ambiguous` rate | Many overlapping genes | Consider `--nonunique fraction` or use featureCounts |
| `0 BAM alignments processed` | File format mismatch, BAM not indexed | Specify `-f bam` explicitly; check BAM integrity |
| Very slow performance | Python implementation | Expected behavior; use featureCounts for speed-critical applications |
| Wrong counts for PE data | BAM not name-sorted, reads split | Ensure name-sorted input |

### 9.2 Performance Benchmarks (Typical)

| Scenario | HTSeq 0.11.2 | featureCounts |
|---|---|---|
| 50M reads, SE, name-sorted | ~45–90 min | ~2–5 min |
| 50M reads, PE, name-sorted | ~60–120 min | ~3–8 min |
| 100M reads, PE, position-sorted | Risk of MemoryError | ~5–10 min |
| Memory usage | 1–8 GB | 200 MB–1 GB |

### 9.3 Best Practices Summary
1. ✅ **Always name-sort paired-end BAM files** (GenePattern does this automatically)
2. ✅ **Verify strandedness before running** — use RSeQC `infer_experiment.py` if unsure
3. ✅ **Match chromosome naming** between BAM and GTF (chr1 or 1, not both)
4. ✅ **Use union mode** for standard RNA-seq; only change for specific biological reasons
5. ✅ **Use Ensembl GTF files** for best compatibility with defaults
6. ✅ **Check diagnostic counters** — `__no_feature` > 30% suggests a problem
7. ✅ **Use `--stranded=no`** as safe default if library type is unknown
8. ✅ **Keep `--minaqual=10`** to exclude low-confidence multi-mapper assignments

# 🧬 Comprehensive Research Report: igv.js GenePattern Module

---

## 📋 Executive Summary

**igv.js** is a production-grade, embeddable JavaScript implementation of the Integrative Genomics Viewer (IGV), developed and maintained by the IGV team at the Broad Institute and UC San Diego. It is the definitive web-based genomic visualization library for embedding interactive genome browsing into web applications, with zero external dependencies. In the GenePattern context, the `genepattern/igv.js` module functions as an **HTML Visualizer** — a browser-launched interactive viewer that renders genomic data files directly in the user's web browser without any server-side compute requirements.

The most recent GenePattern-specific version note ("**Bug fix for S3 files**") directly addresses a known AWS S3 CORS/security issue ([igvteam/igv.js Issue #1740](https://github.com/igvteam/igv.js/issues/1740)) related to S3 bucket names containing dots (`.`).

> **Canonical Citation:** James T Robinson, Helga Thorvaldsdottir, Douglass Turner, Jill P Mesirov. *igv.js: an embeddable JavaScript implementation of the Integrative Genomics Viewer (IGV).* Bioinformatics, Volume 39, Issue 1, January 2023, btac830. https://doi.org/10.1093/bioinformatics/btac830 | PMID: 36562559 | PMC9825295

## 1. 🎯 Tool Purpose & Scientific Applications

### 1.1 Core Scientific Purpose

igv.js provides an **interactive, browser-native genomic visualization environment** capable of rendering virtually all standard bioinformatics file formats in a zero-installation, zero-dependency context. It is the web-embedded successor to the widely-adopted IGV Desktop application, sharing its rendering engine and track model while operating entirely as a client-side JavaScript library.

### 1.2 Scientific Application Domains

| Domain | Application | Track Type |
|--------|-------------|------------|
| **Cancer Genomics** | Somatic mutation visualization, structural variant review | Alignment, Variant, SEG, MUT |
| **Whole Genome Sequencing** | Read alignment inspection, de novo variant calling QC | Alignment (BAM/CRAM) |
| **Variant Analysis** | SNP/indel/CNV review, population genetics | VCF, SEG |
| **RNA-Seq** | Splice junction visualization, expression tracks | Alignment, WIG, bigWig, BED |
| **ChIP-Seq / ATAC-Seq** | Peak visualization, signal tracks, enhancer annotation | bigWig, narrowPeak, broadPeak |
| **Epigenomics** | Methylation, histone modification, ATAC signal | bedGraph, WIG, bigWig |
| **Gene Annotation** | Transcript models, regulatory elements | BED, GFF3, GTF, bigBed |
| **GWAS** | Genome-wide association study visualization | GWAS format |
| **3D Genomics** | Chromatin interaction arcs, Hi-C data | Interact (BEDPE) |
| **Comparative Genomics** | Multi-species alignment, synteny | MAF |
| **Clinical Genomics** | Variant review in clinical WGS/WES | VCF + Alignment |
| **Copy Number** | CNV segmentation, tumor purity | SEG, CNV Pytor |

### 1.3 Key Scientific Strengths (from Robinson et al., 2023)

- **Manual variant review**: Particularly validated for manual review of genome variants, both SNVs and structural variants
- **Cloud-native design**: Reads data in byte-range chunks from HTTP/HTTPS/S3 — no need to download full files
- **No backend requirement**: Runs entirely in the browser; no server-side processing of genomic data
- **Broad institute pedigree**: Same team that developed the desktop IGV, used in thousands of publications

## 2. 📁 Input/Output Formats & Requirements

### 2.1 Supported Input File Formats

#### Alignment Tracks
| Format | Extension | Index Required | Notes |
|--------|-----------|----------------|-------|
| BAM | `.bam` | `.bai` | Binary Sequence Alignment Map; most common |
| CRAM | `.cram` | `.crai` | Compressed Reference-based Alignment Map; smaller than BAM |
| SAM | `.sam` | None | Text-based; not recommended for large files |

#### Annotation Tracks
| Format | Extension | Index Required | Notes |
|--------|-----------|----------------|-------|
| BED | `.bed` / `.bed.gz` | `.tbi` (if gzipped) | Standard genomic intervals |
| bigBed | `.bb` | None (self-indexed) | Binary, indexed BED |
| GFF3 | `.gff3` / `.gff3.gz` | `.tbi` | Gene feature format v3 |
| GTF | `.gtf` / `.gtf.gz` | `.tbi` | Gene Transfer Format |
| genePred | `.gp` | None | UCSC gene prediction format |
| genePredExt | `.gpExt` | None | Extended genePred |
| PSL | `.psl` | None | BLAST-like alignment format |

#### Variant Tracks
| Format | Extension | Index Required | Notes |
|--------|-----------|----------------|-------|
| VCF | `.vcf.gz` | `.tbi` (tabix) | Requires bgzip + tabix; multi-sample supported |

#### Quantitative / Signal Tracks
| Format | Extension | Index Required | Notes |
|--------|-----------|----------------|-------|
| WIG | `.wig` | None | Fixed/variable step wiggle |
| bigWig | `.bw` | None (self-indexed) | Binary WIG; preferred for performance |
| bedGraph | `.bedgraph.gz` | `.tbi` | Tab-delimited coverage |
| TDF | `.tdf` | None | IGV tiled data format |
| narrowPeak | `.narrowPeak` | None | ENCODE peak format |
| broadPeak | `.broadPeak` | None | ENCODE broad peak format |

#### Copy Number & Mutation
| Format | Extension | Notes |
|--------|-----------|-------|
| SEG | `.seg` | Segmented copy number (CBS output, GATK) |
| MAF | `.maf` | Mutation Annotation Format (TCGA standard) |
| MUT | `.mut` | IGV mutation format |
| CNV Pytor | `.pytor` | CNV predictions from CNVPytor |

#### Interaction Tracks
| Format | Extension | Notes |
|--------|-----------|-------|
| BEDPE | `.bedpe` | Chromatin loops, structural variant arcs |

#### Other Formats
| Format | Notes |
|--------|-------|
| GWAS | Genome-wide association p-values |
| BP | RNA secondary structure format |
| FASTA | Reference sequence (with `.fai` index) |

### 2.2 Index File Requirements Summary

```
BAM    → .bam.bai   (samtools index)
CRAM   → .cram.crai (samtools index)
FASTA  → .fa.fai    (samtools faidx)
VCF    → .vcf.gz.tbi (bgzip + tabix)
BED    → .bed.gz.tbi (bgzip + tabix, optional)
GFF3   → .gff3.gz.tbi
GTF    → .gtf.gz.tbi
bedGraph → .bedgraph.gz.tbi
bigWig → self-indexed (no separate index needed)
bigBed → self-indexed (no separate index needed)
```

### 2.3 Output Format

> ⚠️ **Important architectural note**: igv.js is a **visualization-only module**. It produces **no file outputs**. The "output" is an interactive browser-rendered visualization session. Users interact with the genome browser in real time via the web UI.

The browser state (current view, loaded tracks, locus, settings) can be exported as a **JSON session file** via the `browser.toJSON()` API call, and the current view can be exported as an **SVG image** via `browser.getSVG()`.

## 3. 🔧 Parameter Analysis & Usage Patterns

### 3.1 GenePattern Module Parameters

The GenePattern `igv.js` module exposes these parameters via its command line:
```
igv_web.html ? <input.file> <index.file> <genome> <locus>
```

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `input.file` | File | ✅ Yes | — | Primary genomic data file to visualize |
| `index.file` | File | ❌ No | — | Index file (`.bai`, `.crai`, `.tbi`) |
| `genome` | String | ❌ No | `hg19` | Reference genome ID |
| `locus` | String | ❌ No | Full genome | Initial navigation locus |

### 3.2 Parameter Details

#### `input.file`
- **Accepted formats**: `bam`, `cram`, `vcf.gz`, `bed`, `bed.gz`, `gff3`, `gtf`, `wig`, `bw`, `bedgraph`, `bb`, `seg`, `maf`, `mut`, `bedpe`, `gwas`
- **URL handling**: GenePattern passes a URL to the file (can be local server path or S3 URL)
- **S3 support**: Files stored in S3 are accessed via byte-range HTTP requests; requires CORS configuration

#### `index.file`
- **When required**: Mandatory for BAM, CRAM; strongly recommended for bgzipped VCF, BED, GFF3
- **Without index**: igv.js will attempt to load the entire file (only feasible for small files)
- **Auto-detection**: igv.js can auto-detect index by appending `.bai` / `.tbi` to the file URL if not specified

#### `genome`
- **Hosted genome IDs** (resolved automatically by igv.js):

| Organism | Genome IDs |
|----------|------------|
| Human (Homo sapiens) | `hg38`, `hg19`, `hg18` |
| Mouse (Mus musculus) | `mm10`, `mm9` |
| Rat (Rattus norvegicus) | `rn6` |
| Zebrafish (Danio rerio) | `danRer11`, `danRer10` |
| Fruit fly (Drosophila) | `dm6` |
| C. elegans | `ce11` |
| Yeast (S. cerevisiae) | `sacCer3` |
| Arabidopsis | `tair10` |
| Chicken | `galGal6` |
| Gorilla | `gorGor4` |
| Chimpanzee | `panTro4`, `panPan2` |
| Pig | `susScr11` |
| Dog | `canFam3` |
| Cattle | `bosTau8` |

- **Custom genomes**: Can be specified as JSON objects with `fastaURL`, `indexURL`, `cytobandURL`

#### `locus`
- **Format options**:
  - Chromosome only: `chr1`
  - Coordinate range: `chr17:7,668,421-7,687,490`
  - Gene name: `TP53`, `BRCA1`
  - "All": `all` (whole-genome view)
  - Multiple loci (split view): `["chr1:100-200", "chr2:300-400"]`

## 4. 💻 Installation & Dependency Requirements

### 4.1 Upstream igv.js Library

```bash
# npm installation
npm install igv

# Current stable version
npm install igv@3.7.3

# CDN (no installation)
<script src="https://cdn.jsdelivr.net/npm/igv@3.7.3/dist/igv.min.js"></script>
```

**Dependencies**: **Zero** — igv.js has no external JavaScript dependencies. It is entirely self-contained.

### 4.2 GenePattern Module Infrastructure

Since this is an **HTML Visualizer module**, the infrastructure requirements are unusually minimal:

| Component | Requirement |
|-----------|-------------|
| **Docker** | ❌ Not required |
| **Python** | ❌ Not required |
| **Java** | ❌ Not required |
| **Server-side compute** | ❌ Not required |
| **Runtime environment** | Client web browser (Chrome ≥80, Firefox ≥78, Safari ≥14, Edge ≥80) |
| **GenePattern server** | Required only to serve the HTML/JS files and input file URLs |
| **Network access** | Required for hosted genome references (S3) and remote tracks |
| **CPU type** | `any` (specified in module manifest) |

### 4.3 File Serving Requirements

- GenePattern must serve the input files via **HTTPS URLs** (not local file paths)
- For **S3-hosted files**: Bucket must have CORS configured to allow `GET` and `HEAD` requests from the GenePattern server origin
- **S3 bucket naming**: Use virtual-hosted-style names (hyphens only, no dots) — e.g., `igv-genepattern-org` ✅ vs `igv.org.genomes` ⚠️

### 4.4 S3 Bug Fix Details

The version note "**Bug fix for S3 files**" resolves **igvteam/igv.js Issue #1740**:

```
Problem: S3 bucket names containing '.' (dots) cause CORS/security errors
because igv.js attempted to use virtual-hosted-style access 
(https://igv.org.genomes.s3.amazonaws.com/...) which is interpreted 
as a different domain by the browser security model.

Fix: Detect dotted bucket names and fall back to path-style access
(https://s3.amazonaws.com/igv.org.genomes/...) OR ensure GenePattern's 
S3 bucket is named without dots (e.g., 'igv-genepattern-org').
```

## 5. 🔬 Common Workflows & Use Cases

### 5.1 Typical GenePattern Workflows Leading to igv.js

```
RNA-Seq Pipeline:
  FASTQ → Alignment (STAR/HISAT2) → BAM → igv.js (alignment inspection)
                                         ↓
                              Cufflinks/DESeq2 → WIG/bigWig → igv.js (expression)

Cancer WGS/WES:
  BAM → GATK HaplotypeCaller → VCF → igv.js (variant review)
      → CNV caller → SEG file → igv.js (copy number)
      → MuTect2 → MAF → igv.js (somatic mutations)

ChIP-Seq:
  FASTQ → Alignment → BAM → igv.js (read inspection)
                          → MACS2 → narrowPeak → igv.js (peaks)
                          → deepTools → bigWig → igv.js (signal)

Annotation Review:
  Reference annotation → GFF3/GTF → igv.js (gene models)
  Custom intervals → BED → igv.js (regions of interest)
```

### 5.2 Common Use Case Scenarios

1. **Post-alignment QC**: After running STAR, BWA, or Bowtie2 aligners in GenePattern, users inspect reads at specific loci for mapping quality, coverage uniformity, and artifact detection

2. **Variant Validation**: After variant calling (GATK, MuTect), users visually verify candidate variants — particularly critical for indels and structural variants where automated callers may have lower confidence

3. **Copy Number Review**: After running CBS (Circular Binary Segmentation) or GATK CNV tools, users review SEG files overlaid with raw coverage

4. **Gene Expression Context**: After RNA-seq quantification, users view bigWig signal tracks alongside gene models (GTF) to understand expression at splice-isoform resolution

5. **Multi-sample Comparison**: Multiple BAM or VCF tracks loaded simultaneously for tumor/normal or case/control comparisons

6. **GWAS Visualization**: Viewing association p-values in genomic context alongside annotation tracks

### 5.3 Session Management Workflow

```javascript
// Save session state
const sessionJSON = await browser.toJSON();
localStorage.setItem('igv_session', JSON.stringify(sessionJSON));

// Restore session
const savedSession = JSON.parse(localStorage.getItem('igv_session'));
await browser.loadSession(savedSession);

// Export SVG snapshot
const svg = browser.getSVG();
```

### 7.3 S3 Integration Checklist

- [ ] GenePattern S3 bucket name uses hyphens only (no dots): e.g., `igv-genepattern-org`
- [ ] S3 bucket CORS policy allows `GET` and `HEAD` from GenePattern origin
- [ ] HTTPS URLs are used for all file references (S3 presigned URLs or public S3 URLs)
- [ ] Index files co-located with data files (same S3 prefix or named conventionally)
- [ ] igv.js version includes the S3 dot-bucket bug fix (confirmed in current GenePattern version)

### 7.4 Known Limitations & Caveats

| Limitation | Impact | Mitigation |
|------------|--------|------------|
| **No file output** | Cannot be used in analysis pipelines as an intermediate step | Use as terminal visualization step only |
| **Requires internet for hosted genomes** | Offline environments cannot resolve genome IDs | Pre-bundle genome JSON or use custom reference objects |
| **Large BAM files** | Without proper indexing, large files cause browser timeout | Always require `.bai` index for BAM inputs |
| **CORS restrictions** | Cross-origin file access requires CORS headers | Configure S3 bucket CORS or use presigned URLs |
| **Browser memory** | Very large datasets (whole-genome WGS at high zoom) can exhaust browser memory | Set appropriate `visibilityWindow` limits |
| **VCF multi-sample** | Very large multi-sample VCFs may render slowly | Recommend single-sample or filtered VCF inputs |
| **No offline mode** | Genome reference data fetched from S3 on demand | Cache busting and CDN configuration recommended |

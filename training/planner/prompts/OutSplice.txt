## 1. Executive Summary

**OutSplice** is a peer-reviewed Bioconductor R package (v1.10.0, Bioconductor ≥3.22) that identifies, quantifies, and classifies tumor-specific alternative splicing events from RNA-seq junction data. It is purpose-built for cancer genomics workflows—particularly TCGA cohort analyses—and employs a unique **rank-sum outlier statistical framework** followed by **Fisher's exact test with FDR correction** to detect junctions that are significantly over- or under-expressed in tumor samples relative to normal tissue controls. The tool further calculates **per-sample splice burden** (total count of significant aberrant splicing events) and classifies each event as a **skipping, insertion, or deletion** event using genomic annotation.

The GenePattern module wraps this Bioconductor package with a simplified interface: the current version described here **removes the low-level R object parameters** (`genome`, `TxDb`, `annotation`) and replaces them with a single user-friendly **`preset`** parameter (e.g., `hg19`, `hg38`), making the tool accessible to wet-lab biologists and non-R programmers through the GenePattern web interface.

## 2. Scientific Background & Purpose

### 2.1 Biological Motivation

Alternative splicing affects >95% of human multi-exon genes and is a fundamental mechanism of proteomic diversity. In cancer, splicing is frequently dysregulated—tumors exhibit 30% more alternative splicing events (ASEs) than matched normal tissues (TCGA PanCan Atlas, 2018). These cancer-specific splicing aberrations can:

- Create neoantigens (immunotherapy targets)
- Disrupt tumor suppressor function
- Activate oncogenic isoforms
- Alter drug target binding sites
- Serve as prognostic biomarkers (splice burden ↔ immune infiltration and survival)

### 2.2 The OutSplice Approach

Traditional differential splicing tools (rMATS, DEXSeq) assume **population-level consistency**—most samples in a group share the same splicing pattern. This is inappropriate for **heterogeneous tumor cohorts**, where a given splice event may occur in only 5–20% of tumors but be highly recurrent and biologically meaningful.

OutSplice addresses this by treating splicing as an **outlier problem**: rather than asking "is this junction more spliced on average in tumors?", it asks "does this junction represent an extreme outlier in a subset of tumor samples compared to the normal distribution?" This is analogous to the PARADIGM and mCLIP approaches used for somatic mutation burden analysis.

### 2.3 Key Novel Contributions (per published paper)

1. **Splice Burden Quantification**: First tool to explicitly calculate and compare splice burden (number of significant ASEs per sample) between tumor and normal cohorts
2. **Outlier-Based Detection**: Specifically designed for cancer heterogeneity; does not require the event to be shared across all tumors
3. **Event Type Classification**: Automatically assigns each outlier junction as skipping, insertion, or deletion using annotated exon/intron boundaries
4. **TCGA Integration**: Natively supports TCGA Firehose data formats via preset parameters
5. **Dual Analysis Modes**: Supports both user-provided data (`outspliceAnalysis`) and TCGA data (`outspliceTCGA`)

## 3. Statistical Methodology

### 3.1 Step-by-Step Algorithm

```
Step 1: NORMALIZATION
  ├── Compute junction RPM: raw_junction_counts / (total_reads × 10^6)
  └── Normalize by gene expression: junction_RPM / gene_TPM + offset
      → Produces junction-to-gene expression ratio per sample

Step 2: OUTLIER DETECTION (Rank-Sum Test)
  ├── For each junction: rank all tumor samples against normal distribution
  ├── Samples in the extreme tail of the normal distribution = outliers
  ├── Over-expressed outliers: junction ratio >> normal median
  └── Under-expressed outliers: junction ratio << normal median
      → Option: z-score cutoff (zscorecutoff) OR IQR-based method

Step 3: FISHER'S EXACT TEST
  ├── For each junction: contingency table of [tumor outliers, tumor non-outliers]
  │                                             [normal outliers, normal non-outliers]
  ├── Two-tailed Fisher test for over- AND under-expressed events
  └── Apply FDR correction (Benjamini-Hochberg or user-specified method)
      → Identifies junctions with significantly more outliers in tumors

Step 4: SPLICE BURDEN CALCULATION
  └── Per sample: count of FDR-significant junctions where that sample is an outlier
      → Outputs: over_expressed_burden, under_expressed_burden, total_burden

Step 5: EVENT TYPE CLASSIFICATION
  ├── Compare junction coordinates to annotated transcript database (TxDb)
  ├── Skipping: junction spans ≥2 annotated exons (exon skipping)
  ├── Insertion: junction is shorter than annotated intron (intron retention or new splice site)
  └── Deletion: junction is longer than annotated intron (extended intron usage)
```

### 3.2 Statistical Properties

| Property | Details |
|---|---|
| **Primary Test** | Two-tailed Fisher's Exact Test |
| **Multiple Testing Correction** | Benjamini-Hochberg FDR (default), configurable |
| **Default FDR Threshold** | 0.05 |
| **Outlier Method** | Rank-sum with IQR (default) or z-score cutoff |
| **Normalization** | Junction RPM ÷ Gene TPM (with offset) |
| **Design** | Unpaired tumor vs. normal cohort; replicates not required |

## 4. Input/Output Specifications

### 4.1 Input Files (All Required)

#### 🔵 Junction Count Matrix (`junc`)
```
Format:    Tab-delimited text (.txt)
Rows:      Junctions (format: chr:start-end, e.g., chr1:120000-125000)
Columns:   Sample identifiers (must match phenotype file)
Values:    Raw integer read counts spanning each junction
Source:    STAR aligner → SJ.out.tab → OutSplice Formatter
Example:
           Sample1  Sample2  Sample3  ...
chr1:1000-2000  45      0       12    ...
chr1:5000-7000   3     87        0    ...
```

#### 🔵 Gene Expression Matrix (`gene_expr`)
```
Format:    Tab-delimited text (.txt)
Rows:      Gene identifiers (Entrez IDs recommended)
Columns:   Sample identifiers (must match junction matrix)
Values:    TPM or normalized expression values (RSEM output)
Source:    RSEM rsem-calculate-expression → OutSplice Formatter
Example:
           Sample1  Sample2  Sample3  ...
7105        12.5    14.2     11.8    ...
10856       45.3     0.1     89.2   ...
```

#### 🔵 Raw Counts Matrix (`rawcounts`)
```
Format:    Tab-delimited text (.txt)
Rows:      Gene identifiers
Columns:   Sample identifiers
Values:    Raw integer gene read counts
Source:    RSEM or STAR ReadsPerGene.out.tab → OutSplice Formatter
```

#### 🔵 Phenotype File (`phenotype`)
```
Format:    CSV (.csv) with REQUIRED header row
Column 1:  Sample names (MUST exactly match matrix column headers)
Column 2:  T (tumor) or F (normal/control) designation
Example:
  sample_id,phenotype
  TCGA-BA-4077-01A,T
  TCGA-BA-4077-11A,F
  TCGA-BA-4074-01A,T
  ...
Note:      At least 2 normal (F) samples required for outlier statistics
```

### 4.2 Output Files

#### 🟢 Primary Output: RData File (`.RData`)
A named R list containing 8 data objects:

| Object | Type | Description |
|---|---|---|
| `junc.RPM` | Numeric matrix (junctions × samples) | Library-size normalized junction counts (reads per million) |
| `junc.RPM.norm` | Numeric matrix (junctions × samples) | Junction RPM normalized by gene expression |
| `Fisher.p` | Numeric matrix (junctions × 2) | Raw Fisher exact test p-values [over, under] |
| `FDR.p` | Numeric matrix (junctions × 2) | FDR-corrected p-values [over, under] |
| `outlier.matrix` | Binary matrix (junctions × samples) | 1 = outlier, 0 = non-outlier per sample |
| `splice_burden` | Numeric matrix (samples × 3) | [n_over_outliers, n_under_outliers, total_outliers] per sample |
| `splice_type` | Character vector (per junction) | Event type: `"skipping"`, `"insertion"`, or `"deletion"` |
| `NORM.gene_expr.norm` | Numeric vector (per junction) | Median normalized expression in normal samples (for plotting reference) |

#### 🟢 Secondary Output: Junction Expression Plots (`.pdf`)
- Box/scatter plots of junction expression (normalized ratio) for tumor vs. normal
- One panel per requested junction
- Color-coded by tumor (red, default) and normal (blue, default)
- Generated by `plotJunctionData()` function (also wrapped as the companion `OutSplice.plotJunctions` GenePattern module)

## 8. Comparative Analysis

### 8.1 Tool Comparison Matrix

| Feature | **OutSplice** | rMATS | FRASER | DEXSeq | LeafCutter |
|---|---|---|---|---|---|
| **Statistical Framework** | Rank-sum outlier + Fisher | Replicate-based likelihood | Autoencoder + z-score | Negative binomial GLM | Dirichlet-multinomial |
| **Design Assumption** | Outlier in heterogeneous cohort | Group-level difference | Single-sample outlier | Group-level difference | Population-level |
| **Event Types** | Skipping/Insertion/Deletion | SE, A5SS, A3SS, MXE, RI | Intron-centric (% spliced in) | Exon-level bins | Intron clusters |
| **Input** | Junction matrix + gene expr | BAM files | BAM files + annotation | BAM + GTF | BAM/junction files |
| **Splice Burden** | ✅ Per-sample score | ❌ | Limited | ❌ | ❌ |
| **TCGA Support** | ✅ Native preset | Manual | Manual | Manual | Manual |
| **Cancer Focus** | ✅ Designed for cancer | ✅ Applicable | ⚠️ Rare disease focus | ⚠️ General | ⚠️ General |
| **R/Bioconductor** | ✅ | ❌ (Python/C++) | ✅ | ✅ | ✅ |
| **Replicates Required** | ❌ (≥2 normals) | ✅ (multiple per group) | ❌ | ✅ | Preferred |
| **Annotation Required** | ✅ (TxDb/preset) | ✅ | ❌ (annotation-free) | ✅ | ❌ |
| **Publication** | 2023 BioMedInformatics | 2014 PNAS | 2021 Nature Comms | 2012 Genome Biology | 2018 eLife |

### 8.2 OutSplice Unique Strengths

1. **Cancer Heterogeneity Handling**: Unlike rMATS or DEXSeq, does not require the event to be shared across most tumors — ideal for rare, subclone-specific events
2. **Splice Burden Metric**: Unique per-sample quantification usable as a continuous clinical biomarker
3. **Integrated Event Classification**: Automatic skipping/insertion/deletion annotation without separate tools
4. **TCGA Presets**: Reduces barrier to entry for TCGA analysis
5. **Outlier vs. Average**: Captures events missed by mean-based tests in heterogeneous populations

### 8.3 Limitations vs. Competitors

1. **5-Event Typology**: rMATS offers more granular event classification (SE, A5SS, A3SS, MXE, RI) vs. OutSplice's 3-type scheme
2. **No PSI Values**: Does not calculate percent-spliced-in (PSI), a widely used quantitative metric
3. **Human-Centric**: Presets only for hg19/hg38; model organisms require manual TxDb/BSgenome setup
4. **Annotation Dependency**: Cannot detect novel junctions outside annotated genes as efficiently as LeafCutter
5. **Authors' Recommendation**: The original paper explicitly recommends **combining OutSplice with rMATS or FRASER** for cross-validation

The `.RData` output should be explicitly listed as a result file in the module manifest so it can be chained as input to `OutSplice.plotJunctions`.

### 9.5 Testing & Validation Considerations

- **Test dataset**: Use HNSCC TCGA subset (described in paper) as canonical test case
- **Edge cases**: Test with no significant junctions (empty output handling), all tumor samples, minimum normal sample count
- **Parameter validation**: Validate FDR ∈ [0,1], offset > 0, phenotype file format strict checking
- **Preset validation**: Confirm BSgenome + TxDb packages load correctly for both hg19 and hg38 in the Docker container

## 10. Known Limitations & Caveats

| Limitation | Impact | Mitigation |
|---|---|---|
| **Human-only presets** | Non-human organisms require manual TxDb objects; not exposed in simplified module | Document clearly; consider adding mouse mm10/mm39 presets in future versions |
| **Requires normal reference cohort** | Needs ≥2 normal samples for outlier statistics; cannot run tumor-only | Document minimum sample requirements in UI |
| **TCGA Firehose format dependency** | User data requires OutSplice Formatter preprocessing step before use | Link to OutSplice Formatter documentation; potentially create GP Formatter module |
| **No PSI values** | Missing widely-used metric; harder to compare with rMATS results | Document this clearly; recommend rMATS for complementary PSI analysis |
| **3-category event types** | Less granular than rMATS 5-category scheme | Acknowledge; recommend combined analysis |
| **Memory scaling** | Large cohorts (>500 samples, >1M junctions) require >16 GB RAM | Set appropriate GenePattern resource limits |
| **Gene-level normalization assumption** | Assumes junction-hosting gene is expressed; zero gene expression causes division issues (mitigated by offset) | Document offset parameter significance |
| **Sex chromosome handling** | Default `filterSex=TRUE` removes chrX/chrY — important consideration for sex-biased cancers | Expose as prominent parameter with clear documentation |
| **FDR vs. per-event significance** | FDR reported per event; splice burden counts rely on FDR cutoff — user-set FDR affects burden comparison | Emphasize that FDR threshold choice affects splice burden values |

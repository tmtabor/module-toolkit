## 1. 🔬 TOOL PURPOSE & SCIENTIFIC APPLICATIONS

### 1.1 Core Scientific Function

`spatialGE.STenrich` implements **spatial gene set enrichment analysis (Spatial GSEA)** — a method for identifying whether functionally related genes (pathways, gene sets) exhibit spatially restricted expression patterns (i.e., **hotspots**) within spatially-resolved transcriptomics data. Rather than asking *which genes* vary spatially, STenrich asks *which biological pathways or gene programs* are spatially concentrated in tissue.

**Key Biological Question Answered:**
> *"Are spots/cells that highly express a gene set (e.g., the Hallmark Hypoxia pathway) clustered together in tissue space, or is their expression randomly distributed across the tissue section?"*

### 1.2 Algorithm Summary

The STenrich algorithm operates in 5 steps:

```
Step 1: Score Computation
   → For each gene set, compute a per-spot/cell score
     (default: arithmetic mean of normalized expression across
     detected genes in the set)

Step 2: High-Expression Spot Identification
   → Spots with score > (mean + num_sds × SD) are labeled
     "high-expressing" for that gene set

Step 3: Observed Spatial Statistic
   → Compute the average pairwise Euclidean distance among
     all high-expression spots (small distance = spatial clustering)

Step 4: Permutation Test (null distribution)
   → Randomly sample the same number of spots 'reps' times
   → Compute average pairwise distance for each permutation
   → Empirical p-value = proportion of permuted distances ≤ observed

Step 5: Multiple Testing Correction
   → Adjust p-values across all gene sets tested
     (default: Benjamini-Hochberg FDR)
```

**Biological Interpretation:** A statistically significant result means the high-expressing spots are more spatially clustered than expected by chance — evidence of a spatial "hotspot" for that biological pathway.

### 1.3 Scientific Applications

| Application Domain | Use Case |
|---|---|
| **Tumor Microenvironment** | Detect spatially restricted immune infiltration, tumor hypoxia zones, or stromal activation regions |
| **Tissue Architecture** | Map biological pathway activity onto tissue morphology (e.g., cortex layers, crypt-villus axis) |
| **Cancer Biology** | Identify spatially restricted oncogenic pathway activation (e.g., EMT, proliferation hotspots) |
| **Immune Biology** | Map immune checkpoint pathways, T-cell exhaustion, or cytokine signaling to spatial tissue regions |
| **Developmental Biology** | Detect spatially patterned developmental programs in embryonic tissue sections |
| **Pathology** | Correlate histological features with spatial pathway activity |

## 2. 📁 INPUT/OUTPUT FORMATS & REQUIREMENTS

### 2.1 Primary Input: STlist Object (`.rds`)

The **STlist (Spatial Transcriptomics List)** is an S4 R object defined by the spatialGE package. It is the universal data container for all spatialGE analyses.

```
STlist Structure:
├── @counts          # Normalized gene expression matrices (sparse, genes × spots)
├── @spatial_meta    # Spot/cell metadata (x/y coordinates, tissue annotations, domains)
├── @sample_meta     # Sample-level metadata (patient ID, condition, etc.)
├── @misc            # Auxiliary data (images, histology, etc.)
└── @enrich          # POPULATED BY STenrich: enrichment results per sample
```

**How STlist Files Are Created (Upstream - spatialGE.Preprocessing):**

| Source Technology | Input Files |
|---|---|
| **10X Visium** | SpaceRanger output folder (`filtered_feature_bc_matrix/` + `spatial/`) |
| **NanoString CosMx SMI** | Cell × gene count matrix + X/Y coordinate file |
| **Generic CSV/TSV** | Gene expression matrix (genes as rows) + spot coordinate file |
| **GeoMx DSP** | Count matrix + metadata |
| **Seurat object** | Via conversion function |

### 2.2 Gene Sets Input

```
Format 1: Named R list (embedded in script or passed as RDS)
  gene_sets = list(
    "HALLMARK_HYPOXIA" = c("VEGFA", "LDHA", "HK1", ...),
    "HALLMARK_INFLAMMATORY_RESPONSE" = c("IL1B", "TNF", ...)
  )

Format 2: GMT file path (MSigDB Gene Matrix Transposed format)
  gene_sets = "/path/to/h.all.v2023.1.Hs.symbols.gmt"

Format 3: msigdbr R package (programmatic access to MSigDB)
  msigdbr(species="Homo sapiens", category="H")  # Hallmark gene sets
```

### 2.3 Outputs

| Output | Format | Description |
|---|---|---|
| **Updated STlist** | `.rds` | STlist with enrichment results in `@enrich` slot |
| **Results Table** | `.csv` / `.tsv` | Per-sample data frames with columns: `gene_set`, `n_genes_detected`, `obs_avg_distance`, `exp_avg_distance`, `pval`, `pval_adj` |
| **Visualizations** | `.png` / `.pdf` | Heatmaps of p-values across gene sets and samples; spatial maps of gene set scores (via STplot) |

### 3.2 Detailed Parameter Descriptions & Guidance

#### `x` — STlist Object *(REQUIRED)*
- The S4 STlist object containing preprocessed spatial transcriptomics data
- **GenePattern note:** Passed as a serialized `.rds` file path; the module reads it with `readRDS()`
- Must have been processed through `spatialGE.Preprocessing` (normalization essential)

#### `gene_sets` — Gene Set Collection *(REQUIRED)*
- Named list where each element is a character vector of gene symbols
- **Recommended sources:** MSigDB Hallmark (H), C2 (KEGG, Reactome), C5 (GO), custom lists
- **Important:** Gene names must match the gene names in the STlist count matrices
- Gene sets with fewer than `min_genes` detected genes are automatically skipped

#### `score_type` — Gene Set Scoring Method
- `"avg"`: Arithmetic mean of normalized expression across detected genes in the set *(default, recommended)*
- Defines how pathway-level activity is quantified per spot/cell before spatial analysis

#### `reps` — Permutation Iterations
- Controls statistical precision of empirical p-values
- **Minimum detectable p-value = 1/reps** (e.g., 1000 reps → min p-value = 0.001)
- **Trade-off:** 1000 reps is standard; 2000+ for publication; 500 for exploratory analysis
- **Compute impact:** Linear scaling — 2000 reps takes ~2× longer than 1000

#### `num_sds` — High-Expression Threshold
- Defines the cutoff for "high-expressing" spots: `score > mean(score) + num_sds × SD(score)`
- **Default `1`:** ~16% of spots classified as high-expressing (assuming normality)
- **Increasing to 2:** ~2.5% of spots classified; more stringent, fewer spots but more specific clustering signal
- **Biological guidance:** Use `1` for broad pathway screening; `2` for highly specific hotspots

#### `min_units` — Minimum Spot/Cell Count
- Prevents analysis on samples too small to yield reliable spatial statistics
- Default `20` is conservative; samples with fewer spots/cells are skipped

#### `min_genes` — Minimum Gene Set Coverage
- Ensures gene sets are adequately represented in the data
- Gene sets with < `min_genes` detected genes are excluded from testing
- Important for smaller targeted panels (e.g., CosMx 1000-gene panels vs. Visium whole transcriptome)

#### `domain` — Tissue Domain Restriction
- Restricts analysis to spots/cells annotated to a specific tissue domain or cell type
- Requires prior domain annotation in the STlist (from `STclust` or external annotation)
- **Use case:** Test for pathway hotspots specifically within the "tumor" domain, ignoring stromal regions

#### `pval_adj_method` — Multiple Testing Correction
- Options: `"BH"` (FDR, default), `"bonferroni"`, `"holm"`, `"hochberg"`, `"hommel"`, `"BY"`, `"fdr"`, `"none"`
- **Recommendation:** `"BH"` (Benjamini-Hochberg) is standard for gene set analyses; `"bonferroni"` for stringent control

#### `seed` — Random Seed
- **Critical for reproducibility** — always set explicitly in production analyses
- Default `12345` is sufficient; any integer works

#### `cores` — Parallel Processing
- `NULL`: Auto-detects available cores
- Set explicitly (e.g., `cores = 4`) for controlled resource usage in server environments
- **GenePattern note:** Set based on available module compute resources

### 6.3 Integration Considerations

| Consideration | Detail |
|---|---|
| **Input compatibility** | STlist `.rds` file must be produced by `spatialGE.Preprocessing` or compatible spatialGE workflow |
| **Gene set formats** | Support both GMT files (MSigDB-compatible) and R named list RDS files |
| **Memory requirements** | Large datasets (Visium: ~3,000 spots × 33,000 genes) + many gene sets (e.g., 50 Hallmarks × 1000 reps) may require 8–16 GB RAM |
| **Runtime** | 1000 reps × 50 gene sets × 1 sample ≈ 2–10 minutes (depends on spot count and gene set size) |
| **Parallelization** | `cores` parameter allows utilization of multi-core GenePattern server resources |
| **Reproducibility** | `seed` parameter must be set; results are deterministic given same seed |
| **Error handling** | Graceful handling when gene sets have < `min_genes` detected genes; skip with warning |
| **Output chain** | Output STlist RDS must be directly consumable by `spatialGE.STplot.Launcher` |

### 6.4 Known Limitations for GenePattern Context

1. **Minimum p-value resolution** is `1/reps` (0.001 at default 1000 reps) — communicate this to users
2. **Large gene set collections** (e.g., full MSigDB C2 with 6,000+ gene sets) can be very slow; recommend Hallmark (50) or curated subsets
3. **Gene name matching** — gene names in gene sets must exactly match gene names in the STlist; case-sensitive
4. **Spatial coordinate system** — analysis requires accurate spatial coordinates in the STlist; these should be validated upstream
5. **Single-cell resolution** (CosMx, Xenium) datasets with millions of cells may require sampling or subsetting for practical runtimes

## 1. 🎯 Tool Purpose & Scientific Applications

### What is GSEALeadingEdgeViewer?

GSEALeadingEdgeViewer is a **post-analysis visualization module** for Gene Set Enrichment Analysis (GSEA) that focuses specifically on **leading-edge subset analysis**. It serves as the downstream visual interpretation layer for GSEA results, transforming the statistical output of GSEA into four richly informative biological visualizations.

### Biological Foundation of Leading-Edge Analysis

The concept of the leading-edge subset is formally defined in the landmark GSEA paper:

> **Subramanian A, Tamayo P, et al. (2005).** *Gene set enrichment analysis: a knowledge-based approach for interpreting genome-wide expression profiles.* **PNAS 102(43):15545–50.** PMID: [16199517](https://pubmed.ncbi.nlm.nih.gov/16199517/)

**Definition:** The leading-edge subset consists of all genes in a gene set **S** that appear in the ranked gene list **before the point at which the running enrichment score (ES) achieves its maximum deviation from zero**. Mathematically, for a positively enriched gene set, this is the "head" of the ranked list up to the ES peak; for a negatively enriched gene set, it is the "tail."

```
Ranked Gene List (most correlated → least correlated)
|━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━|
 Gene1  Gene2  Gene3 [█ Set Gene] ...  [█ Set Gene] ...
        ↑ Running ES builds here
                              ↑ ES PEAK = leading edge boundary
|← — — — LEADING EDGE SUBSET — — — →|
(Core genes driving the enrichment signal)
```

### Scientific Applications

| Application Domain | How GSEALeadingEdgeViewer Is Used |
|---|---|
| **Pathway Biology** | Identify which specific genes within an enriched pathway are most active |
| **Mechanism Discovery** | Find shared gene members driving enrichment across multiple pathways |
| **Cross-Pathway Analysis** | Visualize overlap between signaling pathways, identify hub genes |
| **Transcriptomics QC** | Validate that enrichment is driven by biologically coherent core genes |
| **Comparative Genomics** | Compare leading edge subsets across conditions or experiments |
| **Cancer Biology** | Identify driver genes within enriched oncogenic/tumor suppressor pathways |
| **Immunology** | Dissect immune pathway responses to treatments or infections |
| **Drug Target Discovery** | Pinpoint high-priority target genes appearing across multiple enriched gene sets |

A published application is found in:
> *Leading edge analysis of transcriptomic changes during pseudorabies virus infection.* **PMC5066188** — demonstrates applied use of all four GSEALeadingEdgeViewer graph types in peer-reviewed virology/immunology research.

The addition of the leading edge analysis component to GSEA is documented in:
> *GSEA-P: a desktop application for Gene Set Enrichment Analysis.* **Bioinformatics 23(23):3251 (2007).** PMID: [17644558](https://pubmed.ncbi.nlm.nih.gov/17644558/)

## 2. 📁 Input/Output Formats & Requirements

### Input Requirements

#### Primary Input: GSEA Result ZIP Archive (`gsea.result.zip`)

This is the **only required input** — the complete `.zip` file produced by either the GSEA or GSEAPreranked GenePattern modules.

**ZIP Archive Internal Structure:**
```
my_analysis.GseaPreranked.XXXXXXXXXX.zip
│
├── edb/                          ← CRITICAL: EDB folder
│   ├── results.edb               ← XML enrichment database (primary data source)
│   └── gene_sets.gmt             ← GMT file filtered to expressed genes
│
├── index.html                    ← Main GSEA HTML report
├── gsea_report_for_na_pos.html   ← Positively enriched sets report
├── gsea_report_for_na_neg.html   ← Negatively enriched sets report
├── gsea_report_for_na_pos.tsv    ← TSV summary table (pos enriched)
├── gsea_report_for_na_neg.tsv    ← TSV summary table (neg enriched)
├── GENESET_NAME.html             ← Per-gene-set detail pages
├── GENESET_NAME.png              ← Enrichment score plots
└── *.gct                         ← Expression subset files for heatmaps
```

#### EDB File Format Deep Dive

The `results.edb` file is an **XML-based Enrichment DataBase** — the machine-readable backbone of GSEA results:

```xml
<EDB name="analysis_name" db="gene_sets.gmt">
  <DTG
    RANKED_LIST="my_ranked_list"
    GENESET="gene_sets.gmt#HALLMARK_OXIDATIVE_PHOSPHORYLATION"
    ES="0.742"
    NES="1.998"
    NP="0.0"
    FDR="0.0"
    FWER="0.0"
    HIT_INDICES="3 7 12 19 ..."       ← Positions of set members in ranked list
    RANK_SCORE="..."
    CORE_ENRICHMENT="Gene1|Gene2|..."  ← Leading edge genes
  />
  ...
</EDB>
```

Key fields extracted for leading edge analysis:
| XML Field | Description |
|---|---|
| `GENESET` | Gene set identifier linking to GMT |
| `ES` | Enrichment Score (raw) |
| `NES` | Normalized Enrichment Score |
| `NP` | Nominal p-value |
| `FDR` | False Discovery Rate q-value |
| `FWER` | Family-wise error rate p-value |
| `HIT_INDICES` | Ranked list positions of all set member genes |
| `CORE_ENRICHMENT` | Pipe-separated leading edge gene identifiers |

### Output Files

| Output | Format | Description |
|---|---|---|
| **HTML Leading Edge Report** | `.html` | Primary output — interactive report with all four visualization graphs |
| **Heat Map Image** | `.png` / embedded | Gene set × gene membership matrix visualization |
| **Set-to-Set Overlap Image** | `.png` / embedded | Pairwise overlap matrix between leading edge subsets |

## 3. 🔬 The Four Visualization Graphs (Core Scientific Output)

This is the heart of GSEALeadingEdgeViewer. When "Run leading edge analysis" is executed, four graphs are generated:

### Graph 1: Heat Map 🗺️
```
             Gene1  Gene2  Gene3  Gene4  Gene5  Gene6
HALLMARK_A    [1]    [1]    [0]    [1]    [1]    [0]
HALLMARK_B    [1]    [0]    [1]    [1]    [0]    [1]
HALLMARK_C    [0]    [1]    [1]    [0]    [1]    [1]
              (1 = gene IS in leading edge subset for that gene set)
```
- **Rows:** Gene sets
- **Columns:** Genes in leading edge subsets
- **Cell value:** Binary (1/0) — gene IS or IS NOT in the leading edge subset
- **Color coding:** Ranges from lowest expression/rank to highest
- **Clustering:** Gene sets are clustered by similarity of their leading edge subsets
- **Biological insight:** Dark vertical "stripes" reveal genes shared across many gene sets — these are the highest-priority biological drivers

### Graph 2: Set-to-Set Overlap 🔄
- Displays an **N × N pairwise overlap matrix** between all selected gene sets
- Cell value represents the count (or fraction) of genes shared between the leading edge subsets of two gene sets
- **Biological insight:** Reveals which pathways share mechanistic gene cores; dense overlap suggests co-regulation or convergent biology

### Graph 3: Gene-to-Set Membership 📊
- Displays **how many gene sets each leading-edge gene belongs to**
- Essentially a histogram/bar chart of gene membership frequency
- **Biological insight:** Genes appearing in many leading edge subsets simultaneously are potential **master regulators** or **hub genes** of biological interest — high-value targets for follow-up experiments

### Graph 4: Set Statistics Summary 📈
- Summarizes enrichment statistics (ES, NES, gene set size, leading edge size) for each selected gene set
- Bar chart format for quick comparison
- **Biological insight:** Enables rapid comparative ranking of gene set contributions

## 4. ⚙️ Parameter Analysis & Configuration

### Complete Parameter Catalog

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `gsea.result.zip` | File | ✅ YES | — | ZIP output from GSEA or GSEAPreranked module |
| `selected.gene.sets` | Text | ❌ No | *(all sets)* | Comma-separated gene set names to analyze |
| `gene.set.name.delimiter` | Text | ❌ No | `,` (comma) | Alternate delimiter for gene set name list |
| `output.file.name` | Text | ❌ No | Auto-generated | Base name for output HTML report |

### Parameter Usage Patterns

**Pattern 1 — Full Analysis (Analyze All Gene Sets):**
```
gsea.result.zip = my_gsea_results.zip
selected.gene.sets = [leave blank]
```

**Pattern 2 — Focused Subset Analysis (Specific Gene Sets):**
```
gsea.result.zip = my_gsea_results.zip
selected.gene.sets = HALLMARK_OXIDATIVE_PHOSPHORYLATION,HALLMARK_MYC_TARGETS_V1,HALLMARK_P53_PATHWAY
```

**Pattern 3 — Gene Sets with Commas in Names (Use Alternate Delimiter):**
```
gsea.result.zip = my_gsea_results.zip
selected.gene.sets = REACTOME_SIGNALING_BY_RAS;KEGG_MAPK_SIGNALING_PATHWAY
gene.set.name.delimiter = ;
```

### Parameter Validation Rules

| Validation Rule | Details |
|---|---|
| **Gene set names are case-sensitive** | Must match exactly as they appear in the GSEA results |
| **ZIP must contain EDB folder** | Without `edb/results.edb`, the module will fail |
| **Gene set names must exist in results** | Non-existent names will be silently skipped or cause errors |
| **Delimiter cannot appear in gene set names** | Choose delimiter carefully based on naming conventions |

### Security: The Log4J Removal Story

The v5 update is critically significant from a security perspective:

```
⚠️ CVE-2021-44228 ("Log4Shell") — Critical CVSS 10.0 vulnerability
   ├── Affected: Log4J 2.x versions < 2.17.0
   ├── Impact: Remote code execution via JNDI injection
   ├── GSEA 4.2.1: Updated to Log4J 2.17.0
   ├── GSEA 4.2.2: Updated to Log4J 2.17.1
   └── GSEA 4.2.3 (THIS VERSION): Log4J ENTIRELY REMOVED ✅
```

This makes GSEALeadingEdgeViewer v5 the **recommended secure version** for all institutional deployments.

**Additional Bug Fix in GSEA 4.2.3:**
The `weighted_p1.5` scoring mode contained a computation bug now corrected. Users who previously used this scoring mode in upstream GSEA analyses should re-run those analyses and then re-run leading edge analysis.

### Docker/Container Considerations

For GenePattern module packaging:
- Base image: Java 11 (OpenJDK 11-slim recommended)
- No external network calls required at runtime
- All GSEA library JARs bundled within module ZIP
- Memory requirements: ~2-4 GB heap depending on dataset size (gene set count × gene count)
- CPU: Single-threaded primary execution; no GPU requirement

### Specific Use Case Scenarios

**Use Case A: Pathway Crosstalk Discovery**
1. Run GSEA against Hallmark gene sets
2. Filter to top 10 positively enriched sets in GSEALeadingEdgeViewer
3. Use Set-to-Set Overlap graph to quantify pathway crosstalk
4. Identify mechanistically convergent pathways

**Use Case B: Drug Target Prioritization**
1. Run GSEA against C2 (curated gene sets) or C6 (oncogenic signatures)
2. Load all significant sets into GSEALeadingEdgeViewer
3. Use Gene-to-Set Membership graph to find genes in ≥3 leading edge subsets
4. These multi-pathway hub genes are prioritized drug target candidates

**Use Case C: Infection/Disease Transcriptomics** *(as in PMC5066188)*
1. Run GSEA on time-course or condition-contrast expression data
2. Select gene sets related to immune response pathways
3. Use Heat Map to visualize temporal/condition-specific core gene changes
4. Identify genes consistently in leading edge across multiple time points

**Use Case D: Comparative Analysis Across Experiments**
1. Run GSEALeadingEdgeViewer separately on two GSEA results (e.g., two cell lines)
2. Manually compare leading edge gene sets across outputs
3. Identify conserved vs. divergent biological mechanisms

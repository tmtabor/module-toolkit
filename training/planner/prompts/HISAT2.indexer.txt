# 🧬 Comprehensive Research Report: HISAT2.indexer

> **GenePattern Module Development Reference** | Version 1.9 | Research Date: 2025

## 📋 Executive Summary

**HISAT2.indexer** is a GenePattern module wrapping `hisat2-build`, the genome index construction engine of the HISAT2 suite. It transforms one or more reference FASTA files into a compressed, binary Hierarchical (Graph) FM index — a prerequisite for all HISAT2 alignment operations. The module bridges HISAT2's powerful graph-based alignment technology with GenePattern's cloud-native, reproducible workflow ecosystem, and is a direct upstream dependency of the `HISAT2.aligner` module.

---

## 1. 🔬 Tool Purpose & Scientific Applications

### 1.1 What is HISAT2.indexer?

`HISAT2.indexer` is a preprocessing/indexing tool that builds a **HISAT2 genome index** from reference DNA sequences. It implements `hisat2-build`, the index builder of the **HISAT2 (Hierarchical Indexing for Spliced Alignment of Transcripts 2)** package, a state-of-the-art splice-aware read aligner developed by Daehwan Kim, James Paggi, Chanhee Park, Christopher Bennett, and Steven Salzberg at Johns Hopkins University.

The tool supports construction of **two index architectures**:

| Index Type | Acronym | Description | Use Case |
|---|---|---|---|
| Hierarchical FM Index | **HFM** | Graph-based genome-only index | DNA-seq, simple alignment |
| Hierarchical Graph FM Index | **HGFM** | Genome + variant/splice graph | RNA-seq, population genomics |

### 1.2 Scientific Applications

- **RNA-seq Gene Expression Analysis**: The primary use case. Building a splice-site-aware HGFM index (with `--ss` and `--exon` from a GTF) dramatically improves read mapping accuracy across exon junctions.
- **Alternative Splicing Research**: The splice graph structure enables novel junction discovery beyond what is annotated in the reference.
- **Population Genomics / Variant-Aware Alignment**: The `--snp` and `--haplotype` options build a population genome graph, enabling alignment to represent genomic diversity rather than a single linear reference.
- **Whole Genome Sequencing (WGS / WES)**: Basic HFM indexing for DNA-seq applications.
- **Custom & Non-Model Organism Genomes**: Critical for organisms not covered by pre-hosted indexes.
- **Transcriptome Assembly Pipelines**: Upstream prerequisite for HISAT2 → StringTie → Ballgown/DESeq2 workflows.

### 1.3 Foundational Science

The algorithm rests on three mathematical foundations:
1. **Burrows-Wheeler Transform (BWT)**: Enables compact storage and rapid lookup of reference sequences.
2. **FM Index** (Ferragina & Manzini, 2000): Compressed full-text index supporting efficient substring searches.
3. **Karkkainen's Blockwise Algorithm**: Enables memory–speed trade-offs during index construction.

## 2. 📄 Input / Output Formats and Requirements

### 2.1 Inputs

| Input | Format | Required | Notes |
|---|---|---|---|
| Reference sequence(s) | FASTA (`.fa`, `.fasta`, `.fa.gz`) | **Yes** | Comma-separated list of multiple files supported (e.g., `chr1.fa,chr2.fa,chrX.fa`) |
| Splice sites file | 4-column TSV (HISAT2 format) | No | Used with `--ss`; generated by `hisat2_extract_splice_sites.py` from a GTF |
| Exons file | 3-column TSV (HISAT2 format) | No | Used with `--exon`; generated by `hisat2_extract_exons.py` from a GTF |
| SNP file | 5-column TSV (HISAT2 format) | No | Used with `--snp`; generated from UCSC SNP or VCF format |
| Haplotype file | HISAT2 haplotype format | No | Used with `--haplotype` alongside `--snp` to control index size |
| Structural Variant file | 5-column TSV (HISAT2 format) | No | Used with `--sv` |

**Splice site file format (4 columns, tab-separated):**
```
chr1    12227   12612   +
chr1    12720   13220   +
```

**Exon file format (3 columns, tab-separated):**
```
chr1    11873   12227
chr1    12612   12721
```

### 2.2 Outputs

| Output | Format | Description |
|---|---|---|
| `<prefix>.1.ht2` through `<prefix>.8.ht2` | Binary (HISAT2-specific) | Standard index (reference ≤ 4 billion nt) |
| `<prefix>.1.ht2l` through `<prefix>.8.ht2l` | Binary (HISAT2-specific) | Large index (reference > 4 billion nt) |
| Zipped archive (GenePattern) | `.zip` | GenePattern packages all 8 `.ht2` files into a single archive for downstream HISAT2.aligner input |

> ⚠️ **Important**: The index files are **binary and non-human-readable**. Once built, the original FASTA files are no longer needed for alignment. All 8 `.ht2` files are required together — they cannot be split.

### 2.3 Index Size Estimates

| Reference Genome | Index Type | RAM Required | Output Size |
|---|---|---|---|
| Human (hg38) | HFM (genome only) | ~8–10 GB | ~3.5 GB |
| Human (hg38) | HGFM + transcripts | ~160 GB | ~4.3 GB |
| Human (hg38) | HGFM + SNPs | ~200 GB | ~5+ GB |
| Mouse (mm10) | HFM | ~6–8 GB | ~2.5 GB |
| Small model organism | HFM | 1–4 GB | 100–500 MB |

## 3. ⚙️ Parameter Analysis and Usage Patterns

### 3.1 Required Arguments

| Parameter | Type | Description |
|---|---|---|
| `reference_in` | FASTA file(s) | Comma-separated list of reference FASTA files (e.g., `chr1.fa,chr2.fa`). Required. |
| `ht2_index_base` | String | Output prefix for the index files. Generates `PREFIX.1.ht2` … `PREFIX.8.ht2`. Required. |

### 3.2 Annotation / Graph Enhancement Parameters
*(for HGFM index; highly recommended for RNA-seq)*

| Parameter | Type | Default | Description | Biological Significance |
|---|---|---|---|---|
| `--ss <path>` | File path | None | Splice site file (from GTF via `hisat2_extract_splice_sites.py`) | **Critical for RNA-seq** — enables accurate mapping across known splice junctions |
| `--exon <path>` | File path | None | Exon annotation file (from GTF via `hisat2_extract_exons.py`) | Must be used together with `--ss`; defines exon boundaries in the graph |
| `--snp <path>` | File path | None | SNP list in HISAT2 format | Enables population-aware alignment across common variants |
| `--haplotype <path>` | File path | None | Haplotype combinations for SNPs | Prevents index size explosion; reduces index by encoding variant combinations |
| `--sv <path>` | File path | None | Structural variant list | Incorporates known SVs into alignment graph |

### 3.3 Performance & Memory Tuning Parameters

| Parameter | Type | Default | Valid Range | Description |
|---|---|---|---|---|
| `-p / --threads` | Integer | 1 | 1–N_CPU | Parallel build threads; reduces wall-clock time but **not** peak memory |
| `--bmax` | Integer | Auto | > 0 | Max suffixes per block; larger = faster build, more RAM. Overrides `--bmaxdivn` |
| `--bmaxdivn` | Integer | 4 | 1–∞ | Suffix block size as fraction of reference length; alternative to `--bmax` |
| `--dcv` | Integer | 1024 | Powers of 2, max 4096 | Difference-cover period; larger = less memory, potentially slower |
| `--packed` | Boolean | false | — | 2-bit/nt DNA packing; saves memory at cost of speed; **auto-enabled on OOM** |
| `-a / --noauto` | Boolean | false | — | Disables automatic memory management; user must set `--bmax`, `--dcv`, `--packed` manually |
| `--large-index` | Boolean | false | — | Force `.ht2l` large index format even for small references |
| `--offrate / -o` | Integer | 4 (5 for human) | 1–∞ | BWT row marking frequency (2^int); higher = less memory, slower coordinate lookups |
| `--ftabchars / -t` | Integer | 10 | 1–14 | FM-index lookup table characters; larger = faster queries, more memory (4^(n+1) bytes) |
| `--localoffrate` | Integer | 3 | 1–∞ | Local HGFM index annotation frequency |
| `--localftabchars` | Integer | 6 | 1–∞ | Local HGFM index lookup table size |
| `--seed` | Integer | 0 | Any | Pseudo-random number generator seed for reproducibility |
| `-q / --quiet` | Boolean | false | — | Suppress all non-error output |

### 3.4 Parameter Groupings for GenePattern UI

**Group 1 — Required Inputs (always visible):**
```
reference_in, ht2_index_base
```

**Group 2 — Transcript Annotation (recommended for RNA-seq):**
```
--ss, --exon
```

**Group 3 — Variant Incorporation (advanced, population genomics):**
```

```

**Group 4 — Performance Tuning (advanced users):**
```
-p/--threads, --bmax, --bmaxdivn, --dcv, --packed, -a/--noauto, --large-index
```

**Group 5 — Index Structure (expert):**
```

```

**Group 6 — Misc:**
```
--seed, --quiet
```

### 3.5 Canonical Usage Patterns

```bash
# Pattern 1: Basic HFM index (genome only — DNA-seq or quick RNA-seq)
hisat2-build genome.fa genome

# Pattern 2: HGFM index with transcript annotations (RECOMMENDED for RNA-seq)
hisat2_extract_splice_sites.py annotation.gtf > genome.ss
hisat2_extract_exons.py annotation.gtf > genome.exon
hisat2-build -p 8 --ss genome.ss --exon genome.exon genome.fa genome_tran

# Pattern 3: HGFM index with SNPs + transcripts (population genomics / variant calling)
hisat2-build -p 16 --snp genome.snp --haplotype genome.haplotype \
             --exon genome.exon --ss genome.ss genome.fa genome_snp_tran

# Pattern 4: Memory-constrained build (use packed representation)
hisat2-build --packed --bmaxdivn 8 genome.fa genome

# Pattern 5: Multiple FASTA files (chromosome-split reference)
hisat2-build chr1.fa,chr2.fa,chrX.fa,chrY.fa genome

# Pattern 6: Large index (reference > 4 billion nt)
hisat2-build --large-index genome.fa genome
```

## 5. 🔄 Common Workflows and Use Cases

### 5.1 The HISAT2 RNA-seq Standard Pipeline (Tuxedo2 Suite)

```
Reference FASTA + GTF Annotation
         │
         ▼
┌─────────────────────┐
│  HISAT2.indexer     │  ← Build splice-aware HGFM index
│  (hisat2-build)     │
└────────┬────────────┘
         │ .ht2 index files (zipped)
         ▼
┌─────────────────────┐
│  HISAT2.aligner     │  ← Map RNA-seq reads to genome
│  (hisat2)           │  ← Input: FASTQ + index
└────────┬────────────┘
         │ SAM/BAM
         ▼
┌─────────────────────┐
│  StringTie          │  ← Transcript assembly + quantification
└────────┬────────────┘
         │ Expression matrices
         ▼
┌─────────────────────┐
│  DESeq2/edgeR/      │  ← Differential expression analysis
│  Ballgown           │
└─────────────────────┘
```

### 5.2 Key Use Cases

| Use Case | Index Type | Critical Parameters | RAM Estimate |
|---|---|---|---|
| Standard RNA-seq (human) | HGFM + transcripts | `--ss`, `--exon` | 160 GB |
| Quick/small organism RNA-seq | HFM only | None additional | 2–8 GB |
| WGS / WES DNA alignment | HFM | `-p <threads>` | 8–10 GB |
| Population genomics | HGFM + SNPs | `--snp`, `--haplotype` | ~200 GB |
| Custom genome assembly | HFM | `--large-index` if > 4Gnt | Scales with genome size |
| Virus / plasmid alignment | HFM | Default settings | <1 GB |

### 5.3 One-Time vs. Repeated Indexing

> **Best practice**: Build the index **once** and reuse it for all alignment runs on the same genome version. Pre-built indexes for common genomes (hg38, hg19, mm10, rn6, etc.) are available for download from the HISAT2 website — `HISAT2.indexer` is primarily used when these pre-built indexes are insufficient.

### 6.3 Output Packaging Strategy

Since GenePattern requires single-file outputs, the standard approach is:
```bash
# After hisat2-build completes, package all 8 index files
zip -j ${output.index.name}.zip ${output.index.name}.*.ht2
# OR for large index
zip -j ${output.index.name}.zip ${output.index.name}.*.ht2l
```
The resulting `.zip` file is the module's output, which HISAT2.aligner decompresses before running.

### 6.4 Computational Resource Allocation

| Genome Size | Recommended RAM | Recommended CPU | Est. Wall Time |
|---|---|---|---|
| Small (< 100 Mb) | 4 GB | 4 cores | 2–10 min |
| Medium (100 Mb – 1 Gb) | 16 GB | 8 cores | 15–30 min |
| Large (human/mouse, no annotation) | 16 GB | 8–16 cores | 30–60 min |
| Large (human/mouse + transcripts) | 160–200 GB | 16 cores | 60–120 min |

> ⚠️ **GenePattern Cloud Warning**: HGFM indexes with SNPs (~200 GB RAM) may exceed standard cloud instance capabilities. Consider offering a warning or using pre-built hosted indexes for these cases.

### 6.5 Validation Rules

```
1. reference_in: Must be valid FASTA format; must not be empty
2. --ss and --exon: Must BOTH be provided or NEITHER (interdependent)
3. --haplotype: Should only be provided when --snp is also provided
4. --bmax and --bmaxdivn: Mutually exclusive (one overrides the other)
5. --dcv: Must be a power of 2 and ≤ 4096
6. output.index.name: Should not contain spaces or special characters
```

### 6.6 Known Limitations and Caveats

| Limitation | Impact | Mitigation |
|---|---|---|
| 160–200 GB RAM required for human HGFM | Cannot run on typical cloud instances | Use pre-built hosted indexes or HFM-only mode |
| No support for transcriptome-only index | Must always include genome | Expected — HISAT2 is genome-based |
| Output is binary, not inspectable | Hard to validate without running aligner | Use `hisat2-inspect` utility for post-build validation |
| Large index build is slow (1–2 hours) | Long-running jobs in GenePattern | Ensure job timeout settings are adequate (> 3 hrs) |
| Windows support limited | Cannot natively containerize on Windows hosts | Docker container sidesteps this completely |
| `.ht2` files are version-specific | Index built with HISAT2 2.x not usable with 1.x | Document HISAT2 version in module and output metadata |

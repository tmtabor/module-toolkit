# 🧬 Comprehensive Research Report: scVelo GenePattern Module

## 1. 📌 Executive Summary

**scVelo** is a landmark, peer-reviewed Python toolkit for **RNA velocity analysis** in single-cell RNA sequencing (scRNA-seq) data. Published in *Nature Biotechnology* (Bergen et al., 2020; DOI: 10.1038/s41587-020-0591-3), it is currently the **most widely adopted RNA velocity framework** in the single-cell genomics community. The GenePattern `scVelo` module (v1) wraps this tool to perform basic velocity analysis on datasets previously preprocessed by the `ScanpyUtilities` GenePattern module, enabling non-programmers to access RNA velocity workflows through a standardized GUI.

---

## 2. 🔬 Tool Purpose & Scientific Applications

### 2.1 Core Scientific Concept

RNA velocity exploits the kinetic relationship between **unspliced (nascent) pre-mRNA** and **spliced (mature) mRNA** to predict the **future transcriptional state** of each cell. Because unspliced mRNA reflects gene induction and spliced mRNA reflects steady-state expression, the ratio and dynamics between these two species encode the direction of transcriptional change — a "velocity vector" in gene expression space.

```
Kinetic Model (Dynamical):
du/dt = α(t) - β·u     [unspliced mRNA dynamics]
ds/dt = β·u - γ·s      [spliced mRNA dynamics]

where:  α = transcription rate (time-dependent)
        β = splicing rate
        γ = degradation rate
```

### 2.2 Three Velocity Models in scVelo

| Model | Description | When to Use | Computation |
|-------|-------------|-------------|-------------|
| **Deterministic** | Steady-state ratio; original La Manno approach | Quick exploration; high-quality data | Fast (seconds) |
| **Stochastic** *(default)* | Adds molecular fluctuation terms via 2nd-order moments | Standard analysis; balanced accuracy/speed | Moderate (minutes) |
| **Dynamical** | Solves full ODE system of kinetics per gene | Transient states; most biological accuracy | Slow (minutes to hours) |

### 2.3 Scientific Applications

- 🧫 **Developmental Biology**: Trace lineage commitment decisions (e.g., pancreatic endocrinogenesis, neurogenesis, hematopoiesis)
- 🔬 **Cell Differentiation**: Identify which progenitor states give rise to mature cell types
- 🧬 **Cancer Biology**: Characterize tumor cell state transitions and plasticity
- 🛡️ **Immunology**: Dynamics of immune cell activation, exhaustion, and differentiation
- 🫁 **Organoid Research**: Recapitulate and study developmental programs in vitro
- ⏱️ **Pseudotime Inference**: Direction-aware pseudotime ordering without requiring prior knowledge of start/end states
- 🗺️ **Fate Mapping**: Integration with CellRank for probabilistic fate predictions

### 2.4 Key Scientific Outputs
- **Velocity vectors** per cell in gene expression space
- **Velocity pseudotime**: RNA-velocity-guided temporal ordering of cells
- **Latent time** (dynamical mode): Absolute inferred cell-internal time
- **Terminal states**: Predicted root and endpoint cell populations
- **Velocity confidence**: Per-cell reliability of velocity estimates
- **Phase portraits**: Spliced vs. unspliced relationships per gene

## 3. 📂 Input/Output Formats & Requirements

### 3.1 Primary Input: AnnData Object (`.h5ad`)

The input must be an **AnnData-formatted HDF5 file** (`.h5ad`) produced by `ScanpyUtilities` or the `velocyto` pipeline. This is the central data structure of the Scanpy ecosystem.

**Required AnnData Components for scVelo:**

| Component | Key | Content | Status |
|-----------|-----|---------|--------|
| `adata.X` | — | Normalized, log-transformed gene expression matrix (cells × genes) | **Required** |
| `adata.layers['spliced']` | `spliced` | Spliced mRNA count matrix | **Required** |
| `adata.layers['unspliced']` | `unspliced` | Unspliced/pre-mRNA count matrix | **Required** |
| `adata.obsm['X_umap']` | `X_umap` | UMAP (or other 2D) embedding coordinates | **Required** (for plots) |
| `adata.obs` | — | Cell metadata: cluster labels, cell type annotations | **Recommended** |
| `adata.var` | — | Gene metadata: gene names, highly variable flags | **Recommended** |
| `adata.layers['ambiguous']` | `ambiguous` | Ambiguous read counts (from velocyto) | Optional |
| `adata.obsm['X_pca']` | `X_pca` | PCA coordinates | Optional (computed internally) |

> ⚠️ **Critical**: The `spliced` and `unspliced` layers are **non-negotiable** — scVelo cannot compute velocities without them. These must be generated upstream by `velocyto run`, `STARsolo`, `kallisto|bustools`, `alevin-fry`, or embedded in the ScanpyUtilities output.

### 3.2 Alternative Input: Loom Files (`.loom`)

```python

# Merge with existing AnnData

```

Barcode string matching is required for successful merging.

### 3.3 Primary Output: Modified AnnData (`.h5ad`)

After processing, scVelo adds the following data to the AnnData object:

**Added Layers:**
| Layer Key | Content |
|-----------|---------|
| `velocity` | RNA velocity vectors per gene per cell |
| `variance_velocity` | Variance of velocity (stochastic mode) |
| `Ms` | First-order moment (smoothed spliced counts) |
| `Mu` | First-order moment (smoothed unspliced counts) |

**Added Observation Annotations (`.obs`):**
| Key | Content |
|-----|---------|
| `velocity_pseudotime` | Velocity-derived pseudotime per cell |
| `velocity_confidence` | Mean cosine correlation with neighbors (0–1) |
| `velocity_length` | Speed/magnitude of velocity vector |
| `latent_time` | Inferred absolute cell time (dynamical mode only) |
| `root_cells` | Root cell probability scores |
| `end_points` | Terminal state probability scores |

**Added Variable Annotations (`.var`):**
| Key | Content |
|-----|---------|
| `velocity_genes` | Boolean mask of genes used for velocity |
| `fit_alpha` | Transcription rates (dynamical mode) |
| `fit_beta` | Splicing rates (dynamical mode) |
| `fit_gamma` | Degradation rates (dynamical mode) |
| `fit_t_` | Switch time points (dynamical mode) |
| `fit_likelihood` | Goodness-of-fit scores per gene |

**Added Unstructured Annotations (`.uns`):**
| Key | Content |
|-----|---------|
| `velocity_graph` | Sparse cell-cell transition probability matrix |
| `velocity_graph_neg` | Negative cosine correlations |
| `recover_dynamics` | Kinetic model parameters (dynamical mode) |

## 9. ⚠️ Known Limitations & Caveats

### 9.1 Biological Assumptions
- Assumes first-order transcriptional kinetics (simple ODE system)
- Requires genes to exhibit both induction and repression phases for dynamical model accuracy
- Results less reliable in highly synchronized or very slow-changing systems
- UMAP layout influences *visual* interpretation but not the underlying velocity calculation

### 9.2 Technical Limitations
- **Dynamical model** is computationally expensive (hours for >20k cells)
- **Sensitive to preprocessing**: spliced/unspliced quantification method significantly impacts results (see Soneson et al., PLOS Comp Bio 2021)
- **Version compatibility**: AnnData version mismatches can cause cryptic errors; pin dependency versions
- **Memory intensive**: Large datasets may require >32 GB RAM for dynamical mode
- **Barcode matching**: Merging loom files requires exact barcode string overlap

### 9.3 Active Development Notes
- scVelo 0.3.x (PyPI) vs 0.2.5 (bioconda) — significant API changes between versions; GenePattern module should pin to a specific version
- The tool is actively maintained by Theis Lab (Helmholtz Munich)
- Community transitioning toward integrated tools (veloVI, CellRank 2); scVelo remains foundational

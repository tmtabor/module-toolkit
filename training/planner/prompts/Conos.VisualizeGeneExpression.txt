## 📋 Executive Summary

`Conos.VisualizeGeneExpression` is the **third and final module** in GenePattern's three-part Conos single-cell RNA-seq analysis pipeline. It provides a user-friendly, GUI-accessible interface for visualizing gene expression on joint multi-sample embeddings generated by Conos (*Clustering On Network Of Samples*). The module wraps the Conos R6 class's `plotGraph()` and `plotPanel()` visualization methods, which internally delegate to `sccore::embeddingPlot()`, enabling researchers to overlay single or multiple gene expression patterns on UMAP/largeVis embeddings without writing any R code.

## 1. 🎯 Tool Purpose & Scientific Applications

### 1.1 Biological Context

Conos addresses a fundamental challenge in modern single-cell genomics: **integrating heterogeneous collections of scRNA-seq datasets** across samples, batches, tissues, and technologies. Standard per-sample clustering fails to identify *recurrent* cell populations that appear across multiple patient samples or conditions. Conos solves this by constructing a **joint graph** where cells from different samples are connected via inter-sample nearest-neighbor edges, forming a unified topology for downstream analysis.

`Conos.VisualizeGeneExpression` specifically enables researchers to **interrogate the biological meaning of clusters** by overlaying raw or diffusion-smoothed gene expression values on this joint embedding. This is one of the most critical steps in single-cell analysis — confirming that computationally-defined clusters correspond to biologically meaningful cell types.

### 1.2 Scientific Applications

| Application Domain | Use Case Example | Key Genes |
|---|---|---|
| **Immunology** | T/B/NK cell subtype identification | `CD3D`, `CD19`, `NCAM1`, `CD4`, `CD8A` |
| **Oncology** | Tumor microenvironment profiling | `PDCD1` (PD-1), `CD274` (PD-L1), `CTLA4` |
| **Hematopoiesis** | Lineage tracing and differentiation | `GATA1`, `GATA2`, `PAX5`, `SPI1` |
| **Neuroscience** | Neuronal/glial subtype delineation | `SNAP25`, `GFAP`, `MBP`, `MAP2` |
| **Developmental Biology** | Stem cell and progenitor markers | `SOX2`, `OCT4`, `NANOG`, `MKI67` |
| **Atlas Projects** | Cross-tissue homologous cell type validation | Tissue-specific markers |
| **QC / Batch Assessment** | Housekeeping gene uniformity check | `ACTB`, `GAPDH`, `B2M` |

### 1.3 Core Algorithm Background

```
Multi-sample scRNA-seq datasets
        │
        ▼
[Conos.Preprocess] → Pagoda2/Seurat per-sample objects
        │
        ▼
[Conos.Cluster] → Joint KNN graph + Community Detection (Leiden/Walktrap) + Embedding (UMAP/largeVis)
        │
        ▼
[Conos.VisualizeGeneExpression] → Gene expression overlay on joint 2D embedding
```

The joint graph is built via **pairwise inter-sample alignments** using mutual nearest neighbors in a shared low-dimensional space (PCA or CPCA). The resulting embedding preserves biological cell type structure while eliminating technical batch effects, making it ideal for gene expression visualization.

#### **Group 2: Visualization Mode Parameters** *(Key Functional Choices)*

| Parameter Name | Type | Default | Options | Description |
|---|---|---|---|---|
| `plot.type` | String | `"joint_graph"` | `"joint_graph"`, `"per_sample_panel"` | **`joint_graph`**: calls `con$plotGraph()` — all cells on joint UMAP/largeVis. **`per_sample_panel`**: calls `con$plotPanel()` — one sub-panel per sample. |

---

#### **Group 3: Diffusion Correction Parameters** *(Active when `use.diffusion.correction = TRUE`)*

| Parameter Name | Type | Default | Range | Description |
|---|---|---|---|---|
| `n.od.genes` | Integer | `500` | 50–5000 | Number of overdispersed genes used as the diffusion basis. More genes → more stable but slower. |
| `fading` | Float | `10.0` | 1–50 | Diffusion decay parameter `a` in equation `dv = exp(-a * (v + b))`. Controls rate of neighborhood smoothing. |
| `fading.const` | Float | `0.5` | 0–5 | Diffusion constant `b`. Offsets the expression baseline before smoothing. |
| `max.diffusion.iters` | Integer | `15` | 5–100 | Maximum diffusion iterations. Increase for convergence in sparse data. |
| `diffusion.tol` | Float | `0.005` | 1e-5–0.1 | Convergence tolerance. Lower = more precise but slower. |

---

#### **Group 4: Color Scale Parameters** *(Controlling Expression Color Mapping)*

| Parameter Name | Type | Default | Options/Range | Description |
|---|---|---|---|---|
| `color.palette` | String | `"default"` | `"default"`, `"viridis"`, `"plasma"`, `"RdBu"`, `"Blues"` | Color palette for continuous gene expression. `"default"` = yellow→dark red. `"viridis"` recommended for colorblind accessibility. |
| `gradient.range.quantile` | Float | `0.99` | 0.90–1.0 | Upper quantile cap for color scaling. `0.99` clips the top 1% of expression values to prevent outliers from compressing the color scale. **Biologically important**: set lower (e.g., `0.95`) when expression is highly skewed. |
| `color.range` | String | `"symmetric"` | `"symmetric"`, `"c(min,max)"` | Explicit numeric range for the color gradient. `"symmetric"` centers on 0 (useful for log-fold changes). Override with `c(0, 3)` to fix range across multiple genes for comparison. |

| Parameter Name | Type | Default | Range/Options | Description |

| `point.size` | Float | `0.8` | 0.1–5.0 | Point size. Reduce to 0.1–0.3 for large datasets; increase to 1.5–3.0 for small datasets (<5K cells). |

| `plot.title` | String | `(gene name)` | Any string | Override default title (gene name) with custom text. |
| `font.size` | Float | `7` | 4–20 | Base font size for title and legend text. |
| `min.cluster.size` | Integer | `0` | 0–500 | Minimum cell count required to label a cluster; prevents cluttering from small clusters. |
| `shuffle.colors` | Boolean | `FALSE` | `TRUE`/`FALSE` | Randomize color assignment order for discrete colorings (cluster/sample). Helps when default palette assigns similar colors to adjacent clusters. |

## 6. 🔌 GenePattern Integration Considerations

### 6.1 Module Implementation Architecture

```r
# Recommended R script structure for the GenePattern module:
# ============================================================

# 1. Parse command-line arguments (from module.xml)
args <- commandArgs(trailingOnly = TRUE)
rds.file      <- args[1]   # conos_cluster_output.rds path
gene.names    <- strsplit(args[2], ",")[[1]]  # Comma-separated genes
plot.type     <- args[3]   # "joint_graph" or "per_sample_panel"
# ... additional args

# 2. Load the Conos object
message("Loading Conos object from: ", rds.file)
saved_obj <- readRDS(rds.file)
con <- saved_obj$con
runleiden <- saved_obj$runleiden
runwalktrap <- saved_obj$runwalktrap

# 3. Auto-select clustering
if (clustering.to.use == "auto") {
  if (runleiden)    clustering.to.use <- "leiden"
  else if (runwalktrap) clustering.to.use <- "walktrap"
  else stop("No clustering found in input file.")
}

# 4. Validate gene names against joint count matrix
joint_genes <- rownames(con$getJointCountMatrix())
invalid_genes <- gene.names[!gene.names %in% joint_genes]
if (length(invalid_genes) > 0) {
  warning("Gene(s) not found in count matrix: ", paste(invalid_genes, collapse=", "))
  gene.names <- gene.names[gene.names %in% joint_genes]
}
if (length(gene.names) == 0) stop("No valid genes provided.")

# 5. Optional: diffusion correction
if (use.diffusion.correction) {
  con$correctGenes(genes = gene.names, n.od.genes = n.od.genes,
                   fading = fading, fading.const = fading.const)
}

# 6. Generate plots per gene
for (gene in gene.names) {
  outfile <- paste0("gene_expression_", gene, ".", output.format)
  
  if (plot.type == "joint_graph") {
    p <- con$plotGraph(
      gene = gene,
      clustering = clustering.to.use,
      show.legend = show.legend,
      alpha = point.alpha,
      size = point.size,
      title = ifelse(plot.title == "", gene, plot.title),
      gradient.range.quantile = gradient.range.quantile,
      raster = use.raster,
      raster.dpi = raster.dpi,
      font.size = c(font.size * 0.5, font.size)
    )
  } else {
    p <- con$plotPanel(
      gene = gene,
      use.common.embedding = use.common.embedding,
      plot.theme = ggplot2::theme_bw()
    )
  }
  
  ggplot2::ggsave(outfile, plot = p, width = output.width, 
                  height = output.height, dpi = raster.dpi)
  message("Saved: ", outfile)
}
```

### 6.4 Known Issues, Limitations & Caveats

| Issue | Severity | Mitigation |
|---|---|---|
| **Gene not found in joint matrix** | High | Validate gene names against `rownames(con$getJointCountMatrix())` before plotting; provide helpful error messages |
| **Embedding not present** | High | Check `con$embeddings` is non-NULL; if absent, embedding must be regenerated (requires re-running `Conos.Cluster`) |
| **Memory overflow for large atlases** | Medium | Enable `use.raster = TRUE`; consider cell downsampling via `subset` parameter |
| **Batch effects in raw expression** | Medium | Offer `use.diffusion.correction` option with appropriate defaults |
| **Deprecated largeVis embedding** | Low | largeVis is legacy; UMAP is preferred for new runs |
| **Conos Docker version lag** | Medium | `vpetukhov/conos:version-1.1.1` is ~6 years old (2019); may lack latest CRAN features (v1.5.2). Consider updating Docker image |
| **R6 class method access** | Low | Must use `$` accessor syntax carefully; ensure RDS loads as proper Conos R6 object |
| **plotPanel output dimensions** | Low | Panel plots require larger output dimensions; auto-scale based on number of samples |
| **Non-standard gene names** | Low | Handle ENSEMBL IDs vs gene symbols; add conversion utility if needed |

1. **Only GUI-accessible multi-sample Conos visualization tool** — makes the Conos workflow accessible to wet-lab biologists without R coding ability

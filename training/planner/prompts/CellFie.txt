# 🧬 Comprehensive Research Report: CellFie (Cellular Functions InferencE)

---

## 1. TOOL PURPOSE AND SCIENTIFIC BACKGROUND

### Overview
**CellFie** (Cellular Functions InferencE) is a computational framework developed by the **Lewis Lab at UC San Diego (LewisLabUCSD)** for quantifying the metabolic functions of cells from omics data. It bridges the gap between raw gene/protein expression data and interpretable biological function by leveraging **genome-scale metabolic models (GEMs)** to score the activity of hundreds of pre-defined **metabolic tasks** — functional units representing a cell's capacity to carry out specific biochemical conversions.

> **Core Question Answered**: *"What is this cell doing metabolically, and at what level of activity?"*

### Scientific Motivation
Traditional pathway enrichment tools (e.g., GSEA, hypergeometric tests) detect statistical overrepresentation of genes in gene sets but do not capture mechanistic dependencies between genes and metabolic reactions. CellFie's key innovation is that it uses **Gene-Protein-Reaction (GPR) logic** embedded in genome-scale metabolic models to propagate gene expression scores through enzymatic reactions and into measurable metabolic task scores, providing **mechanistic, model-based functional inference**.

### Primary Publication
> **Richelle A, Kellman BP, Wenzel AT, Chiang AWT, et al.** "Model-based assessment of mammalian cell metabolic functionalities using omics data." *Cell Reports Methods*, 1(3):100040, **July 26, 2021**. DOI: [10.1016/j.crmeth.2021.100040](https://doi.org/10.1016/j.crmeth.2021.100040)

## 2. ALGORITHM AND COMPUTATIONAL METHODOLOGY

### Step-by-Step Scoring Pipeline

```
Omics Data (Gene Expression)
         │
         ▼
  ┌─────────────────────┐
  │  1. GENE SCORING    │   GAL = 5 × log(1 + expression / threshold)
  │  (Gene Activity     │   → Generates continuous per-gene activity score
  │   Level, GAL)       │
  └─────────┬───────────┘
            │
            ▼
  ┌─────────────────────┐
  │  2. REACTION        │   GPR Logic:
  │  ACTIVITY SCORE     │   • AND (isoenzyme subunits) → MIN of gene scores
  │  (RAS)              │   • OR (isozymes/alternatives) → SUM of gene scores
  └─────────┬───────────┘
            │
            ▼
  ┌─────────────────────┐
  │  3. TASK SCORE      │   Aggregated from all reactions required by the task
  │  (Continuous +      │   • Continuous: activity magnitude
  │   Binary)           │   • Binary: 1 if task active, 0 if not
  └─────────────────────┘
```

### Threshold Definition Methods
CellFie provides multiple threshold strategies to determine if a gene is "expressed":

| Method | Parameter Value | Description |
|--------|----------------|-------------|
| **Local - minmaxmean** | `ThreshType='local'`, `LocalThresholdType='minmaxmean'` | Lower/upper bounds set per-gene using distribution (recommended) |
| **Local - mean** | `ThreshType='local'`, `LocalThresholdType='mean'` | Mean expression across samples used as local threshold |
| **Local - custom** | `ThreshType='local'`, `LocalThresholdType='custom'` | User-defined custom value per gene |
| **Global - value** | `ThreshType='global'`, `percentile_or_value='value'` | Single absolute expression cutoff across all genes |
| **Global - percentile** | `ThreshType='global'`, `percentile_or_value='percentile'` | Threshold set as percentile of entire expression distribution |

> ⭐ **Best Performing Configuration** (from benchmarking): `Local T2 + GM1 + Order 2` — local thresholding using the 25th percentile as lower bound and 75th percentile as upper bound across all genes and samples.

## 3. INPUT/OUTPUT FORMATS AND REQUIREMENTS

### Input Specifications

| Input | Format | Description | Requirements |
|-------|--------|-------------|--------------|
| **Gene Expression Matrix** | Tab-delimited / GCT / .mat | Genes (rows) × Samples (columns) | Genes must be in **Entrez ID** format |
| **Reference Metabolic Model** | MATLAB `.mat` file | Genome-scale metabolic model with GPR rules | Must match organism of study |
| **Threshold Parameters** | Command-line/config | ThreshType, bounds, method | See parameter table below |

#### Supported Omics Data Types
- ✅ **Microarray** (e.g., Affymetrix, Agilent)
- ✅ **Bulk RNA-seq** (raw counts, TPM, RPKM, FPKM)
- ✅ **Single-cell RNA-seq** (small datasets; use **scCellFie** for large atlas-scale data)
- ✅ **Proteomics / Mass spectrometry** abundance data
- ⚠️ Data units are flexible but must be **consistent across samples**; log-transformation may be appropriate

#### Supported Reference Genome-Scale Models

| Model | Organism | Description |
|-------|----------|-------------|
| **Recon2.2** (`MT_recon_2_2_entrez.mat`) | Human | Classic human metabolic model; most validated with CellFie |
| **iHsa** | Human | Human-specific model with expanded GPR annotations |
| **Human1 / Human-GEM** | Human | Latest comprehensive human GEM from Chalmers; includes updated reactions |
| **Recon3D** | Human | 3D structural metabolic model |
| Custom models | Any | Any COBRA-compatible GEM can be used |

> 🔄 **Version Note**: The current CellFie version includes an **update to reaction models**, reflecting integration with newer GEMs such as Human-GEM / Human1, providing more complete and curated GPR associations than Recon2.

### Output Files

| Output Variable | File/Format | Description |
|-----------------|-------------|-------------|
| `score` / **taskScore** | Matrix (tasks × samples), TSV/CSV | **Continuous** metabolic task activity scores per sample |
| `score_binary` | Matrix (tasks × samples), TSV/CSV | **Binary** task activity (1 = active, 0 = inactive) per sample |
| `taskInfos` | Table/TSV | Task metadata: names, subsystem, description, source model |
| `detailScoring` | Multi-column table | Reaction-level activity scores; **Column 6** = reaction scores for Escher map visualization |

#### The 7 Metabolic Systems and 195 Tasks
```
CellFie Metabolic Coverage:
├── 1. Energy Metabolism          (ATP production, oxidative phosphorylation, etc.)
├── 2. Nucleotide Metabolism      (purine/pyrimidine synthesis, salvage pathways)
├── 3. Carbohydrate Metabolism    (glycolysis, TCA cycle, gluconeogenesis, etc.)
├── 4. Amino Acid Metabolism      (synthesis/degradation of all 20 amino acids)
├── 5. Lipid Metabolism           (fatty acid synthesis/oxidation, phospholipids, etc.)
├── 6. Vitamin & Cofactor Metabolism (folate, B vitamins, CoA, etc.)
└── 7. Glycan Metabolism          (N-/O-glycosylation pathways, etc.)
    TOTAL: 195 curated tasks
```

## 4. PARAMETER ANALYSIS

### Complete Parameter Reference

| Parameter | Type | Default | Range/Options | Biological Significance |
|-----------|------|---------|---------------|------------------------|
| `ThreshType` | String | `'local'` | `'local'` or `'global'` | Local adapts per-gene; global applies single cutoff. Local is recommended for heterogeneous datasets |
| `LocalThresholdType` | String | `'minmaxmean'` | `'minmaxmean'`, `'mean'`, `'custom'` | Controls how local per-gene threshold is computed |
| `percentile_or_value` | String | `'value'` | `'value'` or `'percentile'` | Whether cutoff is absolute expression value or percentile of distribution |
| `value_low` | Numeric | `25` | 0–1000 (value) or 0–1 (percentile) | **Lower threshold bound** — genes below this are inactive. Default=25th percentile |
| `value_high` | Numeric | `75` | 0–1000 (value) or 0–1 (percentile) | **Upper threshold bound** — genes above this are fully active. Default=75th percentile |
| `SampleNumber` | Integer | (required) | ≥1 | Number of columns (samples) in the expression matrix |
| `ref` | String/Path | (required) | File path to `.mat` | Path to reference genome-scale metabolic model |

### Parameter Dependencies and Logic

```
if ThreshType == 'global':
    → use percentile_or_value + value_low only (single threshold)
    → LocalThresholdType is IGNORED

if ThreshType == 'local':
    → LocalThresholdType determines per-gene method
    → if LocalThresholdType == 'minmaxmean':
        → use BOTH value_low AND value_high (lower/upper bounds)
    → if LocalThresholdType == 'mean':
        → use value_low as fraction of mean expression
    → if LocalThresholdType == 'custom':
        → user provides custom threshold vector
```

## 6. COMMON WORKFLOWS AND USE CASES

### Workflow 1: Standard Bulk Transcriptomics Analysis
```
RNA-seq data (raw counts/TPM)
    → Normalize (TPM or similar)
    → Convert gene identifiers to Entrez IDs
    → Load into CellFie with local thresholding (25th/75th percentile)
    → Select reference model (Recon2.2 or Human-GEM)
    → Obtain 195-task score matrix
    → Cluster samples by metabolic phenotype
    → Visualize on Escher maps
```

### Workflow 2: Single-Cell Metabolic Profiling
```
scRNA-seq (10x Genomics, Smart-seq2)
    → Use scCellFie (Python) for atlas-scale data
    → Per-cell metabolic task scoring
    → UMAP embedding by metabolic features
    → Differential metabolic activity between cell types
```

### Workflow 3: Disease vs. Normal Comparison
```
Patient vs. healthy control transcriptomics
    → CellFie scoring for each group
    → Comparison of continuous + binary task scores
    → Identify up/down-regulated metabolic tasks
    → Link to phenotype or clinical variables
```

### CellFie's Unique Strengths
1. **Mechanistic foundation**: Uses biologically validated GPR logic from curated GEMs
2. **Functional ontology**: 195 task-level descriptions more interpretable than gene-level output
3. **Flexible omics**: Works with microarray, RNA-seq, or proteomics equally
4. **Dual output**: Both binary (active/inactive) and continuous (magnitude) scores
5. **GenePattern native**: Already deployed; active module with Docker support
6. **Extensible ecosystem**: secCellFie (secretory), scCellFie (single-cell), ImmCellFie (web)

### Key Integration Notes

1. **Gene ID Mapping**: Input data must use **Entrez IDs** as gene identifiers. A pre-processing step (e.g., BioMart conversion from gene symbols/Ensembl) should be documented or ideally provided as an upstream module.

2. **File Format**: GCT format (GenePattern native) is the natural input. The matrix should be **genes (Entrez IDs) × samples**.

3. **Updated Reaction Models**: The current version note references an "update to reaction models" — this likely reflects updated GPR data from **Human-GEM** / Human1, which is more complete than Recon2.2. The new module should expose model selection as a key user-facing parameter.

4. **Computational Resources**: 
   - Bulk RNA-seq (hundreds of samples): moderate RAM (~8–16 GB), no GPU needed
   - Single-cell data (thousands of cells): use scCellFie Python version instead
   - Runtime: typically minutes to tens of minutes depending on dataset size

5. **Docker**: The existing Docker image `genepatterndev/cellfie:v0.7.0` bundles the MATLAB runtime + COBRA Toolbox, eliminating the need for users to install MATLAB. **Verify this image is current with the reaction model update**.

6. **Output Compatibility**: Output TSV/CSV files should be clearly labeled:
   - `*_taskScore.tsv` — for downstream clustering or visualization
   - `*_binaryScore.tsv` — for presence/absence analysis
   - `*_detailScoring.tsv` — for Escher map visualization
   - `*_taskInfo.tsv` — for metadata about each task

## 9. KNOWN LIMITATIONS AND CAVEATS

| Limitation | Impact | Mitigation |
|-----------|--------|------------|
| **Requires Entrez ID input** | Incompatible with symbol/Ensembl-only data | Add upstream ID conversion step |
| **Human/mammalian-focused** | Limited models for non-human organisms | Custom GEM can be provided |
| **MATLAB runtime dependency** | Heavy container; proprietary language | Docker containerization resolves for GenePattern |
| **Single-cell scalability** | Large scRNA-seq datasets are slow | Redirect to scCellFie for single-cell |
| **Task list fixed at 195** | May not cover all pathways of interest | Tasks extensible with custom models |
| **Threshold sensitivity** | Results differ across threshold choices | Provide recommended defaults; document sensitivity |
| **No statistical significance** | Task scores are not p-values | Pair with permutation testing or group comparison tools |

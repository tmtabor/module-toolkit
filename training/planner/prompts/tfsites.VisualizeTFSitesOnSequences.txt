## 1. EXECUTIVE SUMMARY

`tfsites.VisualizeTFSitesOnSequences` is a GenePattern bioinformatics module designed for **multi-transcription factor (TF) binding site annotation and visualization across DNA sequences**. It is part of the broader `tfsites` GenePattern module suite and is containerized via Docker (v17.0.19) for reproducible execution. Its core capability is rendering a visual map of annotated transcription factor binding sites (TFBS) on a given DNA sequence, with optional integration of binding affinity/score data and an optional upstream motif database generation step. This module occupies the **visualization and reporting** layer of a TF regulatory analysis pipeline, sitting downstream of affinity scoring or motif database construction steps.

---

## 2. SCIENTIFIC BACKGROUND & PURPOSE

### 2.1 Biological Context

Transcription factor binding sites (TFBS) are short, conserved DNA sequences (~6–20 bp) within regulatory regions (promoters, enhancers, silencers, insulators) where specific TFs bind to control gene expression. The accurate identification, annotation, and **visualization** of TFBS is fundamental to:

- Understanding cis-regulatory logic governing gene expression
- Interpreting the functional impact of non-coding genetic variants (e.g., eQTLs, GWAS hits, SNPs)
- Characterizing enhancer/promoter architecture in developmental and disease biology
- Supporting CRISPR guide design in proximity to regulatory elements
- Investigating combinatorial TF occupancy patterns
- Generating testable hypotheses from ChIP-seq, ATAC-seq, and DNase-seq data

### 2.2 Tool Purpose

`tfsites.VisualizeTFSitesOnSequences` addresses the critical **visualization bottleneck** in TFBS analysis workflows. While tools like FIMO, HOMER, or JASPAR can predict and score binding sites computationally, they typically produce tabular output that lacks intuitive visual representation of how multiple TFs' binding sites are distributed, overlaid, and scored across a sequence of interest. This module fills that gap by:

1. **Annotating** a DNA sequence with TFBS locations from precomputed affinity files or motif database scans
2. **Labeling** each site with a unique binding site ID, genomic coordinates (start/end positions)
3. **Scoring** sites with affinity/score values when reference data is provided
4. **Color-coding** sites by affinity intensity — providing proportional visual weight to higher-confidence binding events
5. **Optionally generating** a motif database as a preprocessing step before visualization, enabling an end-to-end single-module workflow

### 2.3 Module Suite Context

This module is part of the **`tfsites` GenePattern suite**, a family of related modules that together provide a complete TF regulatory analysis pipeline:

| Likely Sister Module | Function |
|---|---|
| `tfsites.BuildMotifDatabase` | Constructs a TF motif database from PWMs |
| `tfsites.GenerateTFBindingAffinity` | Scores each position in a sequence for TF affinity |
| `tfsites.IntersectSNPsWithTFSites` | Intersects genetic variants with annotated TFBS |

- DNA sequence displayed linearly (5' → 3')

- Multiple TFs are color-differentiated (distinct hues per TF)
- Coordinate ticks/ruler optionally displayed below sequence
- Reverse complement strand sites may be displayed on opposite side

### 4.2 Parameter Dependency Logic

```
IF batch_motif_files PROVIDED AND motif_db_params PROVIDED:
    → First generate motif database
    → Then scan sequence using generated database
    → Then visualize
ELIF affinity_information_files PROVIDED:
    → Use precomputed affinity scores directly
    → Visualize with score-proportional coloring
ELIF batch_motif_files PROVIDED (no motif_db_params):
    → Use motif files directly for scanning
    → Visualize without score-proportional coloring (binary present/absent)
ELSE:
    → ERROR: At least one of affinity_information_files or batch_motif_files required

IF normalize_scores = TRUE AND min/max_affinity NOT SET:
    → Auto-detect min/max from input data
IF score_threshold > 0:
    → Filter out sites below threshold BEFORE visualization
```

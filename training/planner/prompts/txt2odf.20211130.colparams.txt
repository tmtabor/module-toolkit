## 1. 🔬 Tool Identity & Scientific Purpose

### 1.1 Module Overview

`txt2odf.20211130.colparams` is a **GenePattern utility/format-conversion module** developed under the GenePattern organization at the **Broad Institute**. Its singular, well-defined purpose is to convert generic **tab-delimited TXT files** into the GenePattern-native **ODF (Output Data Format)** — the primary data-exchange currency within GenePattern multi-module pipelines.

The module occupies a critical **bridging role** in GenePattern ecosystems: it allows researchers to import any structured tabular genomic data file — regardless of its origin — into GenePattern pipelines without requiring custom scripting or format-specific preprocessing.

### 1.2 Name Deconstruction

| Component | Meaning |
|-----------|---------|
| `txt2odf` | Core function: Text → ODF conversion |
| `20211130` | Development baseline date: November 30, 2021 |
| `colparams` | **Key differentiator:** Column parameter specification capability — users explicitly define column indices, types, and names |

The `colparams` suffix is architecturally significant. It distinguishes this module from a simpler `txt2odf` base variant that would rely solely on automatic column detection. The colparams variant gives the operator **explicit, declarative control** over how each column is typed and named in the output ODF.

## 2. 📁 Input/Output Format Specifications

### 2.1 Input Format: TXT (Tab-Delimited Text)

**Accepted extensions:** `.txt`, `.tsv`, `.tab`
**Expected structure:**

```
# Optional comment/metadata lines (skippable)
GeneID    Description    Sample1    Sample2    Sample3
BRCA1     Breast cancer gene 1    5.21    3.12    7.83
TP53      Tumor protein p53       2.44    9.23    1.14
EGFR      EGF receptor            8.31    4.55    6.72
```

**Supported TXT variants:**
| Variant | Description |
|---------|-------------|
| Expression matrix | Genes × samples, numeric float values |
| Differential expression results | Gene IDs + stats (p-value, FDR, logFC) |
| Feature / gene lists | One or more identifier columns |
| Clinical/phenotype data | Sample IDs + covariates |
| Annotation tables | IDs + free-text functional annotations |
| Ranked gene lists | For GSEA pre-ranked input preparation |

**Common input challenges handled by colparams:**
- Multiple header/comment rows requiring skipping
- Non-standard column ordering
- Mixed String/numeric column types
- Column names needing override or cleanup
- Subset column selection (not all columns needed downstream)

### 2.2 Output Format: ODF (Output Data Format)

ODF is GenePattern's **proprietary structured tabular format** developed at the Broad Institute. It extends plain tab-delimited data with a mandatory metadata header that declares file type, column types, column names, and data dimensions — enabling downstream GenePattern modules to programmatically validate, parse, and consume the file without user configuration.

**ODF General Structure:**

```
ODF 1.0
HeaderLines=5
Model=Dataset
DataLines=3
COLUMN_TYPES=String    String    float    float    float
Name    Description    Sample1    Sample2    Sample3
BRCA1    Breast cancer gene 1    5.21    3.12    7.83
TP53    Tumor protein p53    2.44    9.23    1.14
EGFR    Epidermal growth factor receptor    8.31    4.55    6.72
```

**Supported ODF Model Sub-types:**

| ODF Model | Use Case | Typical Downstream Consumer |
|-----------|----------|-----------------------------|
| `Dataset` | Expression matrices, numeric data grids | HeatMapViewer, HierarchicalClustering, NMFConsensus |
| `FeatureList` | Gene/feature identifier lists | GSEA, VennDiagram, FilterFeatures |
| `ClassVector` | Sample class/phenotype labels | ClassNeighbors, ComparativeMarkerSelection |
| `ComparativeMarkerSelectionResult` | Differential expression output | ComparativeMarkerSelectionViewer |
| `SOMClusterResult` | SOM clustering output | SOMViewer |
| `HierarchicalClusteringResult` | Hierarchical clustering output | HierarchicalClusteringViewer |

**ODF Column Data Types:**

| Type | Description | Example Values |
|------|-------------|----------------|
| `String` | Text/identifier data | "BRCA1", "chr17", "GO:0008150" |
| `float` | Single-precision floating point | 5.21, 0.003, -2.7 |
| `int` | Integer values | 1234, -5, 0 |
| `double` | Double-precision floating point | 1.234567890123456 |

### 3.2 Key `col.params` Format Specification

The `col.params` parameter is the **central innovation** of this module variant. Its expected format follows a colon-separated triplet per column, comma-delimited across columns:

```
<column_index>:<data_type>:<display_name>, ...
```

**Example:**
```
0:String:GeneID,1:String:Description,2:float:logFC,3:float:pvalue,4:float:FDR,5:int:rank
```

This single parameter effectively encodes:
- Which columns to include (by index)
- What data type each column holds
- What name each column receives in the ODF output

### 3.3 Parameter Dependency Graph

```
input.file ─────────────────────────────────────────────────────► ODF output
    │
    ├── skip.rows (removes comment/metadata lines first)
    ├── delimiter (parses field boundaries)
    ├── has.col.names (identifies header row)
    ├── header.lines (counts header rows)
    │
    └── COLUMN SPECIFICATION (one of three modes):
            Mode 1: col.params="0:String:Name,1:float:pvalue" [most explicit]
            Mode 2: col.types="String,float" + col.names="Name,pvalue" [split spec]
            Mode 3: include.columns="0,1" + auto-detect types [semi-auto]
            Mode 4: all auto-detected from header row [fully automatic]
    │
    └── data.type ─► ODF Model declaration (Dataset / FeatureList / ClassVector)
    └── row.id.col ─► Identifies primary key column for ODF indexing
    └── output.file ─► Names the output ODF
```

### 6.4 Critical Validation Rules

The following validation logic should be enforced in the module implementation:

```
1. input.file MUST exist and be readable
2. input.file MUST be parseable as tabular data with specified delimiter
3. If col.params provided → parse and validate format "idx:type:name"
   - idx must be ≥ 0 and < number of columns in file
   - type must be one of: String, float, int, double
4. If include.columns provided → all indices must be < column count
5. skip.rows + header.lines must be < total line count of file
6. data.type must be one of the supported ODF model types
7. Output: ODF header line count must match actual header lines written
8. Output: DataLines count must match actual data rows written
```

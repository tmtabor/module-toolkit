## 1. 📌 Tool Purpose and Scientific Applications

### 1.1 Core Scientific Role

`PreprocessVelocityTranscriptome` is the **critical first step** in an RNA velocity analysis pipeline. It automates the extraction of **spliced (exonic)** and **unspliced (intronic)** transcript reference sequences from a genome assembly and its annotation, producing the index files required by `alevin` (part of Salmon) to quantify both RNA species simultaneously from droplet-based single-cell RNA-seq data.

### 1.2 The Biological Rationale: RNA Velocity

**RNA Velocity** (La Manno et al., *Nature*, 2018; Bergen et al., *Nature Biotechnology*, 2020) predicts the **future transcriptional state of individual cells** by modeling the kinetic relationship between:

| RNA Species | Biological Meaning | Sequencing Signal |
|---|---|---|
| **Unspliced / Intronic** | Nascent/pre-mRNA; newly transcribed | Reads in intron regions |
| **Spliced / Exonic** | Mature processed mRNA | Reads in exon regions |

The ratio of unspliced-to-spliced counts, modeled with degradation rate (γ), splicing rate (β), and transcription rate (α), forms a velocity vector that indicates whether a gene is being **upregulated (positive velocity)** or **downregulated (negative velocity)** in each cell.

**From a quantification standpoint**, RNA velocity requires:
1. A reference FASTA containing **both** spliced transcript sequences AND intronic sequences
2. A **transcript-to-gene mapping** (t2g) that knows which features are spliced vs. unspliced
3. A **Salmon index** built from this combined FASTA (optionally with a genome decoy)

This module handles steps 1–3 before any read quantification occurs.

### 1.3 The `eisaR` Core Engine

The module wraps the **eisaR** (Exon-Intron Split Analysis in R) Bioconductor package, developed by **Charlotte Soneson** (Friedrich Miescher Institute / Biozentrum Basel), which is the methodologically validated gold-standard tool for this task. The package was validated in the landmark paper:

> Soneson C, Srivastava A, Patro R, Stadler MB. **"Preprocessing choices affect RNA velocity results for droplet scRNA-seq data."** *PLOS Computational Biology*, 17(1): e1008585, 2021. DOI: [10.1371/journal.pcbi.1008585](https://doi.org/10.1371/journal.pcbi.1008585)

## 2. 📂 Input/Output Formats and Requirements

### 2.1 Inputs

| Parameter | Format | Description | Requirements |
|---|---|---|---|
| **Genome FASTA** | `.fa` / `.fasta` / `.fa.gz` | Primary assembly genome sequence | Must match the GTF assembly version (e.g., GRCh38) |
| **Annotation GTF** | `.gtf` / `.gtf.gz` | Gene annotation with exon/intron coordinates | Compatible sources: GENCODE, Ensembl, RefSeq |

**Important Compatibility Notes:**
- The genome FASTA and GTF **must** be from the **same reference assembly version** (e.g., both GRCh38.p13)
- Chromosome naming conventions must match (`chr1` vs `1`) — GENCODE uses `chr` prefix, Ensembl does not
- **GENCODE** annotations are strongly recommended (have `--gencode` flag support in Salmon)
- For decoy-aware indexing, the full primary assembly genome (not just chromosomes) should be provided

**Common Reference Sources:**
- Human (GRCh38): [GENCODE](https://www.gencodegenes.org/human/), [Ensembl](https://ftp.ensembl.org/pub/release-104/fasta/homo_sapiens/)
- Mouse (GRCm38/mm10): [GENCODE](https://www.gencodegenes.org/mouse/), [Ensembl](https://ftp.ensembl.org/pub/release-104/fasta/mus_musculus/)

### 2.2 Outputs

| Output File | Format | Description | Downstream Use |
|---|---|---|---|
| `spliced_unspliced.fa` (or `splici.fa`) | FASTA | Combined spliced transcript + flanked intron sequences | `salmon index` input |
| `t2g_3col.tsv` | TSV (3-column) | Transcript ID → Gene ID mapping with spliced/unspliced annotation | `alevin --tgMap` |
| `t2g_spliced.tsv` | TSV (2-column) | Spliced-only transcript-to-gene mapping | Standard expression quantification reference |
| `chrnames.txt` | TXT | Chromosome/contig name list | `salmon index -d` (decoy parameter) |
| `salmon_index/` | Directory | Built Salmon index | `salmon alevin -i` |

```
ENST00000450305.2-I     ENSG00000223972.5       U
...
```
*Columns: transcript_id, gene_id, S (spliced) or U (unspliced)*

## 3. 🔧 Parameter Analysis and Usage Patterns

### 3.1 Core eisaR Parameters (via `getFeatureRanges()`)

| Parameter | Type | Default | Biological Impact | Recommendation |
|---|---|---|---|---|
| `featureType` | `character[]` | `c("spliced","intron")` | Which RNA species to extract | Always include both "spliced" and "intron" for velocity |
| `intronType` | `string` | `"separate"` | Per-transcript vs. per-gene intron definition | Use **`"separate"`** — more biologically precise |
| `flankLength` | `integer` | `90L` | Flanking nt added to each intron end for exon-intron boundary reads | Set to **`read_length - 1`** (e.g., 90 for 10x v3 reads) |
| `joinOverlappingIntrons` | `logical` | `FALSE` | Merges overlapping introns across different transcripts | Keep `FALSE` for per-transcript resolution |
| `collapseIntronsByGene` | `logical` | `FALSE` | Single intron entry per gene | Keep `FALSE` unless memory is severely limited |
| `verbose` | `logical` | `TRUE` | Progress reporting | Keep `TRUE` for GenePattern logging |

### 3.2 Parameter Deep Dive: The Critical Parameters

#### `intronType` — "separate" vs "collapse"

```
"separate" (RECOMMENDED):
  - Extracts introns individually for each transcript
  - Gene ENSG00000001 with 3 transcripts → 3 independent intron sets
  - More specific; allows detection of transcript-specific splicing dynamics
  - Produces more FASTA entries but better precision
  - Validated as more accurate by Soneson et al. 2021

"collapse":
  - Merges/reduces introns across all transcripts per gene
  - Less redundancy, smaller reference
  - May miss transcript-isoform-specific splicing signals
  - Use when: very large genomes, memory constraints
```

#### `flankLength` — The Boundary-Spanning Read Problem

```
WHY IT MATTERS:
  Reads that span exon-intron junctions would NOT map to either:
  - Pure exon sequences (spliced reference) → missed exonic read
  - Pure intron sequences (intron reference) → missed intronic read
  
  Adding flanking sequence EXTENDS each intron by N nucleotides on both sides,
  capturing reads that straddle the boundary.

RECOMMENDED VALUE:
  flankLength = read_length - 1
  
  For 10x Genomics v3 (98bp R2): flankLength = 97 ≈ 90L (typical value used)
  For 10x Genomics v2 (98bp R2): flankLength = 90L
  For older data (75bp reads): flankLength = 50L
  
  The Soneson et al. 2021 paper recommends:
  flankLength such that (read_length - flankLength - 1) ≥ minimum_seed_length
```

### 5.2 Typical Use Cases

1. **10x Chromium v3 (most common):** Human PBMC or tumor single-cell data; use `flankLength=90`, `intronType="separate"`
2. **10x Chromium v2:** Similar to v3 but older chemistry; same flank length recommended
3. **Single-nucleus RNA-seq (snRNA-seq):** Pre-mRNA quantification especially important; nuclei contain primarily unspliced RNA
4. **Developmental trajectory studies:** E.g., hematopoiesis, neurogenesis, embryogenesis where cellular state transitions are of interest
5. **Non-human model organisms:** Mouse (mm10/GRCm38), zebrafish (GRCz11), *Drosophila* (dm6) — any organism with genome + GTF

### 5.3 Full Reproducible Workflow Example

```r
# R Script: preprocess_velocity_transcriptome.R

library(eisaR)
library(Biostrings)

# Parameters (injected by GenePattern)
gtf_file       <- "Homo_sapiens.GRCh38.104.gtf"
genome_fa      <- "GRCh38.primary_assembly.genome.fa"
output_prefix  <- "velocity_ref"
intron_type    <- "separate"
flank_length   <- 90L
join_overlapping <- FALSE

# Step 1: Extract genomic feature ranges
grl <- getFeatureRanges(
  gtf            = gtf_file,
  featureType    = c("spliced", "intron"),
  intronType     = intron_type,
  flankLength    = flank_length,
  joinOverlappingIntrons = join_overlapping,
  verbose        = TRUE
)

# Step 2: Load genome and export combined FASTA
genome <- readDNAStringSet(genome_fa)
names(genome) <- sapply(strsplit(names(genome), " "), `[`, 1)  # Normalize chr names
exportToFile(grl, 
             filepath = paste0(output_prefix, "_spliced_unspliced.fa"), 
             genome = genome)

# Step 3: Generate transcript-to-gene mapping (3-column)
getTx2Gene(grl, filepath = paste0(output_prefix, "_t2g_3col.tsv"))

# Step 4: Get chromosome names for decoy-aware Salmon index
writeLines(names(genome), 
           con = paste0(output_prefix, "_chrnames.txt"))

cat("eisaR preprocessing complete.\n")
cat("Output files:\n")
cat(" - FASTA:", paste0(output_prefix, "_spliced_unspliced.fa"), "\n")
cat(" - t2g:  ", paste0(output_prefix, "_t2g_3col.tsv"), "\n")
```

### 7.4 Known Limitations and Caveats

1. **Memory constraint:** Loading full human genome into R via `readDNAStringSet()` requires ~20 GB RAM. Consider BSgenome packages as an alternative for common organisms.
2. **Chromosome name matching:** Users must ensure chromosome names are consistent between genome FASTA and GTF (e.g., GENCODE uses `chr1`, Ensembl uses `1`). Automated detection and warning is recommended.
3. **No genome FASTA compression support (direct):** `Biostrings::readDNAStringSet()` supports `.gz` but may be slow; recommend uncompressed for large genomes.
4. **Salmon `--gencode` flag:** Must be set when using GENCODE annotations (handles version suffixes in gene IDs like `.5`); autodetect based on `annotation.source` parameter.
5. **Run time:** This is a long-running job (30–90 minutes for human genome); GenePattern job timeout settings should be adjusted accordingly.
6. **Reference genome size:** The module is NOT appropriate for metagenomics or very large polyploid genomes without modification.

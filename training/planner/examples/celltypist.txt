## 1. 🔬 Tool Purpose & Scientific Applications

### Overview
**CellTypist** is an automated cell type annotation tool for single-cell RNA sequencing (scRNA-seq) datasets developed by the **Teichmann Lab at the Wellcome Sanger Institute**. It employs **logistic regression classifiers optimized by the stochastic gradient descent (SGD) algorithm** to rapidly and precisely assign cell type identities to individual cells in scRNA-seq experiments.

### Scientific Mission
CellTypist was originally developed to support the **Human Cell Atlas (HCA)** project for exploring tissue adaptation of cell types using semi-automatic annotations at scale. It has since evolved into a full annotation platform with:
- Built-in, curated pre-trained models across tissues and organisms
- A community working group curating ontologies and model updates
- An online portal for no-code analysis

### Primary Applications
| Application Domain | Description |
|---|---|
| **Immune Cell Profiling** | Annotate T cells, B cells, NK cells, monocytes, macrophages, dendritic cells, and all major immune subtypes |
| **Cross-Tissue Analysis** | Transfer cell type labels across tissues from reference to query datasets |
| **Disease Research** | COVID-19 immune landscape characterization, disease-specific immune infiltrates |
| **Developmental Biology** | Annotate fetal/embryonic cell populations (lung, brain, gut, pan-fetal) |
| **Large-Scale Atlases** | Process 100K–1M+ cell datasets via mini-batch training and inference |
| **Custom Annotation** | Train user-specific models for any organism, tissue, or novel nomenclature |

### Biological Validation & Scientific Impact
- **Primary Paper:** Domínguez Conde et al. (2022). *Cross-tissue immune cell analysis reveals tissue-specific features in humans.* **Science 376(6594)**. DOI: [10.1126/science.abl5197](https://doi.org/10.1126/science.abl5197)
  - CellTypist annotated **>330,000 immune cells** from **16 tissues** in **12 human donors**
  - Covered lymphoid, mucosal, and exocrine sites
- **v2.0 Paper:** Xu et al. (2023). *Automatic cell-type harmonization and integration across Human Cell Atlas datasets.* **Cell 186:5876–5891.e20**
  - Introduced CellHint — predictive clustering tree for cross-dataset harmonization
- **Community benchmarking:** Bioinformatics community comparisons consistently rank CellTypist among the most accurate tools alongside scANVI (scvi-tools), with its primary advantage being speed and ease of use in Python workflows.

> **ℹ️ Example data:** `demo_2000_cells.h5ad` — A 2,000-cell human immune cell dataset used in the official CellTypist tutorial, exactly the expected format.

### 2.2 Critical Input Preprocessing Requirements

> ⚠️ **This is one of the most important requirements for correct CellTypist operation.**

Input expression matrices **MUST** be:
1. **Normalized** to 10,000 counts per cell (CP10K):
   ```python
   sc.pp.normalize_total(adata, target_sum=1e4)
   ```
2. **Log1p-transformed:**
   ```python
   sc.pp.log1p(adata)
   ```

**Special cases:**
- `.csv` raw count files: CellTypist **automatically** applies normalization
- `.h5ad` files: Pre-normalized log-transformed matrix **expected** in `adata.X`
- `check_expression=True` (default) will warn if the data doesn't appear log-normalized

### 2.3 Output Files & Formats

| Output | Format | Description |
|---|---|---|
| **`predicted_labels.csv`** | CSV | Per-cell predicted labels + confidence scores; includes `majority_voting` column if enabled |
| **`probability_matrix.csv`** | CSV | Full probability matrix: rows=cells, columns=cell types |
| **`decision_matrix.csv`** | CSV | Raw logistic regression decision scores per cell type |
| **`annotated.h5ad`** | AnnData (.h5ad) | Input AnnData enriched with annotation columns in `.obs` |
| **UMAP plot** | PDF/PNG | UMAP colored by predicted cell type labels |
| **Probability plots** | PDF/PNG | Per-cell-type probability distribution plots (optional) |
| **Dotplot** | PDF/PNG | Marker gene × predicted cell type dotplot |

### 2.4 AnnotationResult Object Attributes

When `celltypist.annotate()` is called, it returns an `AnnotationResult` object with:

| Attribute | Type | Content |

| `probability_matrix` | `pandas.DataFrame` | Shape: cells × cell types, softmax probabilities |

### 8.2 Key Implementation Notes

1. **Model Handling Strategy:**
   - Support both named built-in models (string dropdown) AND uploaded custom `.pkl` files
   - Pre-bundle or cache built-in models in Docker container to avoid internet dependency at runtime
   - Model storage path: `~/.celltypist/data/models/`

2. **Preprocessing Gate:**
   - If input is raw counts (CSV), auto-apply `normalize_total(target_sum=1e4)` + `log1p`
   - If input is h5ad, assume pre-normalized (but offer `check_expression` validation)
   - Document preprocessing requirement prominently

3. **Output Files (standardized for GenePattern):**
   ```
   output/
   ├── {prefix}_predicted_labels.csv     # Per-cell annotations
   ├── {prefix}_probability_matrix.csv   # Full probability scores
   ├── {prefix}_annotated.h5ad           # Annotated AnnData object
   ├── {prefix}_umap.pdf                 # UMAP visualization
   └── {prefix}_dotplot.pdf              # Marker gene dotplot
   ```

4. **Majority Voting Dependencies:**
   - Requires `leidenalg` and `python-igraph` — must be in Docker container
   - If input h5ad already has neighbor graph computed, CellTypist uses it; otherwise auto-computed

5. **Memory / Compute Requirements:**
   - Standard datasets (1K–100K cells): 4–16 GB RAM, standard CPU
   - Large datasets (100K–1M cells): 32+ GB RAM, mini-batch mode recommended
   - GPU training optional but can be offered as advanced parameter

6. **Error Handling to Implement:**
   - Gene name mismatch between input and model (tool reports proportion matched)
   - Unnormalized input detection (`check_expression` warnings)
   - Missing majority voting dependencies (graceful fallback to simple prediction)
   - Custom model file format validation

7. **Python Script Template:**
```python
import scanpy as sc
import celltypist
from celltypist import models
import os, sys

def run_celltypist(input_file, model_name, majority_voting, mode,
                   p_thres, min_prop, output_prefix, ...):
    # Load data
    if input_file.endswith('.h5ad'):
        adata = sc.read_h5ad(input_file)
    else:  # CSV/TSV/MTX
        adata = sc.read_csv(input_file).T
        sc.pp.normalize_total(adata, target_sum=1e4)
        sc.pp.log1p(adata)

    # Download/load model
    if not model_name.endswith('.pkl') or not os.path.exists(model_name):
        models.download_models(model=model_name)

    # Run annotation
    predictions = celltypist.annotate(
        adata,
        model=model_name,
        majority_voting=majority_voting,
        mode=mode,
        p_thres=p_thres,
        min_prop=min_prop
    )

    # Export outputs
    predictions.to_table(folder='./', prefix=output_prefix)
    adata_out = predictions.to_adata(insert_labels=True, insert_conf=True, insert_prob=True)
    adata_out.write_h5ad(f'{output_prefix}_annotated.h5ad')
    predictions.to_plots(folder='./', prefix=output_prefix, format='pdf')
```

## 10. ⚠️ Known Limitations & Caveats

| Limitation | Impact | Mitigation |
|---|---|---|
| **Python-only** | No native R interface | Use reticulate in R; GenePattern abstracts this |
| **Pre-trained models focused on immune cells** | Limited for stromal, epithelial, or rare cells | Use tissue-specific models; train custom |
| **Requires log-normalized input for h5ad** | Common user error with raw count h5ad files | Add auto-detection + normalization in module |
| **Internet needed for model download** | Fails in air-gapped environments | Pre-bundle models in Docker container |
| **Majority voting needs neighbor graph** | Adds compute time; graph auto-built if absent | Pre-compute in preprocessing workflow |
| **Gene name matching** | Cross-species / alias genes may not match | Report matching statistics; use gene aliases |
| **Imbalanced training data** | Underrepresented cell types perform poorly | Use `balance_cell_type=True` in training |
| **No batch correction** | Cross-dataset transfer may suffer from batch effects | Apply harmony/scVI before CellTypist |

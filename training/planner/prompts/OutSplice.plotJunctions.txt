## 📋 EXECUTIVE SUMMARY

**OutSplice.plotJunctions** is a specialized post-processing visualization module in the GenePattern platform that generates bar plots and waterfall plots of RNA-seq junction expression data from tumor and normal tissue samples. It is a downstream companion to the **OutSplice** Bioconductor package's analysis modules, directly wrapping the `plotJunctionData()` R function to provide publication-quality figures. The module occupies a critical niche in cancer genomics: visualizing *outlier-based* aberrant splicing events discovered in heterogeneous tumor transcriptomes—a task not adequately served by mean-comparison tools like DEXSeq or rMATS.

> **Official LSID:** `urn:lsid:genepattern.org:module.analysis:00433`
> **Docker Image:** `genepattern/outsplice.plotjunctions:0.4`
> **Underlying R Package:** OutSplice v1.10.0 (Bioconductor 3.22+)

## 1. 🎯 TOOL PURPOSE AND SCIENTIFIC APPLICATIONS

### 1.1 Primary Function
OutSplice.plotJunctions is a visualization-only post-processing tool. It does NOT perform statistical analysis—it renders graphical representations of junction expression results that have already been computed by the companion **OutSplice** or **OutSplice.TCGA** GenePattern modules (LSID `00431`). The module calls the `plotJunctionData()` function from the OutSplice Bioconductor package.

### 1.2 Biological Problem Being Solved
Cancer transcriptomes are highly heterogeneous: different tumor samples may harbor distinct sets of aberrant splicing events even within the same cancer type. Classical differential splicing tools (rMATS, DEXSeq) compare **group means**, which can miss events that are extreme outliers in only a subset of tumors. OutSplice uses **outlier statistics** (modified Ghosh rank-sum method) to identify per-sample aberrant junctions. Once identified, OutSplice.plotJunctions makes these results interpretable by:

- **Bar plots**: Displaying raw junction expression (RPM or normalized RPM) per sample, with tumor and normal samples color-coded, allowing direct visual inspection of expression distributions.
- **Waterfall plots**: Sorting and ranking junction expression values across all samples to reveal the characteristic "outlier" shape — a few tumor samples with dramatically elevated or suppressed junction usage against a relatively flat normal baseline.

### 1.3 Scientific Applications
| Application | Description |
|---|---|
| **Cancer Biology** | Identifying splicing events unique to specific tumor subtypes or individual patients |
| **Biomarker Discovery** | Visualizing candidate junction biomarkers before downstream validation |
| **Splice Burden Analysis** | Confirming that high-burden samples show the expected outlier expression patterns |
| **Quality Control** | Visual verification of OutSplice statistical results for individual junctions |
| **Publication Figures** | Generating PDF figures directly suitable for journal submissions |
| **Exploratory Analysis** | Iterative inspection of top-ranked junctions at various p-value cutoffs |

### 1.4 Validated Scientific Contexts
The OutSplice framework has been validated and applied in peer-reviewed literature:
- **BioMedInformatics (2023):** Bendik et al. validated OutSplice on TCGA Head and Neck Squamous Cell Carcinoma (HNSC) data, identifying tumor-specific alternative splicing events using the complete OutSplice → plotJunctions workflow. DOI: [10.3390/biomedinformatics3040053](https://doi.org/10.3390/biomedinformatics3040053)
- **Head & Neck / PMC (2025–2026):** Kalavacherla et al. used OutSplice to define splice burden as **a novel biomarker** for immune infiltration and prognosis in HNSC (PMID: 40755385), demonstrating clinical relevance of the OutSplice workflow.
- **SINE Neoantigen study (MDPI IJMS):** OutSplice was used to identify significantly over-expressed splicing events in >50% of tumors for neoantigen pipeline integration.

## 2. 📁 INPUT/OUTPUT FORMATS AND REQUIREMENTS

### 2.1 Primary Input

| Parameter | Format | Description |
|---|---|---|
| `data_file` | `.RData` (R binary data) | **Required.** Output file from a prior OutSplice or OutSplice.TCGA GenePattern run. Contains all analysis objects as a saved R environment. |

The `.RData` file must contain the following R objects (produced by `outspliceAnalysis()` or `outspliceTCGA()`):

| Object | Type | Description |
|---|---|---|
| `TumorOverExpression` | `matrix` (logical) | Per-junction, per-sample logical matrix; `TRUE` = outlier over-expression in that tumor sample |
| `TumorUnderExpression` | `matrix` (logical) | Per-junction, per-sample logical matrix; `TRUE` = outlier under-expression in that tumor sample |
| `FisherP` | `matrix` (numeric) | FDR-corrected Fisher's exact test p-values per junction event |
| `junc.RPM` | `matrix` (numeric) | Raw junction counts normalized to Reads Per Million (RPM) by dividing by total per-sample read counts |
| `junc.RPM.norm` | `matrix` (numeric) | `junc.RPM` further normalized by gene-level expression (accounts for overall transcriptional activity) |
| `gene_expr` | `matrix` (numeric) | Gene expression values for each junction-associated gene |
| `splice_burden` | `matrix` (numeric) | Per-sample count of Fisher-P significant over-expressed, under-expressed, and total outlier junctions |

> ⚠️ **Critical Note:** This input format is tightly coupled to the OutSplice R package's internal output structure. The `.RData` file is not a generic format and must originate specifically from OutSplice analysis functions.

### 2.2 Optional Inputs
- `junctions`: A character vector of specific junction coordinates in the format `"chrN:start-end"` (e.g., `"chr1:150483674-150483956"` for the ECM1 gene junction used in the vignette example). This allows targeted visualization of pre-known or pre-selected junctions.

### 2.3 Primary Output

| Output | Format | Description |
|---|---|---|
| `pdffile` | `.pdf` | **Primary output.** Multi-page PDF containing bar plots and waterfall plots for all selected junctions |

Each junction of interest generates **two plot pages** in the PDF:
1. **Bar Plot** — Junction expression (RPM or normalized RPM) on the Y-axis, samples on the X-axis, tumor samples colored `tumcol` (default: red), normal samples colored `normcol` (default: blue).
2. **Waterfall Plot** — Samples ranked by junction expression value creating a descending waterfall curve, visually emphasizing the outlier distribution characteristic of tumor-specific events.

### 5.2 Common Use Cases

**Use Case 1: Discovery Mode — Explore Top Hits**
After running OutSplice analysis on a cancer cohort, researchers routinely visualize the top 10–20 most significant over-expressed junctions to identify candidates for follow-up validation. `NUMBER=20, tail="over", p_value=0.05`.

**Use Case 2: Hypothesis-Driven — Known Cancer Genes**
Researchers investigating specific cancer driver genes (e.g., TP53, EGFR, KRAS) use `GENE=TRUE, SYMBOL="TP53"` to visualize all significant junction events within a gene of interest regardless of global rank.

**Use Case 3: Publication Figure Generation**
Final validated junctions are plotted with specific coordinates (`junctions="chrN:start-end"`) and custom colors to generate consistent, publication-ready PDF figures. Color customization (`tumcol`, `normcol`) allows journal-specific styling.

**Use Case 4: TCGA Cohort Analysis**
When paired with OutSplice.TCGA module, the workflow retrieves junction and expression data directly from TCGA repositories, runs the analysis, then plotJunctions is used to visualize cohort-level outlier patterns in any of the 33 TCGA cancer types.

**Use Case 5: Comparative Splicing Survey**
Running plotJunctions with `tail=NULL` (both tails) allows researchers to compare over- vs. under-expressed junction patterns simultaneously — useful for identifying genes with complex bidirectional splicing deregulation.

### 6.5 Wrapper R Script Structure
```r
#!/usr/bin/env Rscript
# OutSplice.plotJunctions GenePattern Wrapper
# wraps OutSplice::plotJunctionData()

suppressPackageStartupMessages({
  library(OutSplice)
  library(optparse)
})

option_list <- list(
  make_option("--data_file",  type="character"),
  make_option("--NUMBER",     type="integer",   default=1),
  make_option("--junctions",  type="character", default=NULL),
  make_option("--tail",       type="character", default=NULL),
  make_option("--p_value",    type="double",    default=0.05),
  make_option("--GENE",       type="logical",   default=FALSE),
  make_option("--SYMBOL",     type="character", default=NULL),
  make_option("--pdffile",    type="character", default="junction_plots.pdf"),
  make_option("--tumcol",     type="character", default="red"),
  make_option("--normcol",    type="character", default="blue")
)

opt <- parse_args(OptionParser(option_list=option_list))

# Convert "both" tail selection to NULL for R function
if (!is.null(opt$tail) && opt$tail == "both") opt$tail <- NULL
if (!is.null(opt$junctions) && opt$junctions == "") opt$junctions <- NULL
if (!is.null(opt$SYMBOL) && opt$SYMBOL == "") opt$SYMBOL <- NULL

plotJunctionData(
  data_file = opt$data_file,
  NUMBER    = opt$NUMBER,
  junctions = opt$junctions,
  tail      = opt$tail,
  p_value   = opt$p_value,
  GENE      = opt$GENE,
  SYMBOL    = opt$SYMBOL,
  makepdf   = TRUE,
  pdffile   = opt$pdffile,
  tumcol    = opt$tumcol,
  normcol   = opt$normcol
)
```

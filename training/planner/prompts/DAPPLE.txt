# 🧬 DAPPLE — Comprehensive GenePattern Module Research Report

## Executive Summary

**DAPPLE** (Disease Association Protein-Protein Link Evaluator) is a network-biology algorithm developed at the **Broad Institute of MIT and Harvard** that evaluates whether proteins encoded by genes in disease-associated genomic loci (typically from GWAS studies) are more physically connected to each other—through protein-protein interactions (PPIs)—than would be expected by chance. The tool bridges GWAS discovery and functional biology by leveraging the InWeb3 high-confidence interactome. The GenePattern module (v0.19) provides containerized access to this Broad-hosted algorithm.

---

## Section 1: Tool Purpose & Scientific Background

### 1.1 Core Scientific Hypothesis

DAPPLE is built on a fundamental hypothesis in complex disease genetics:

> *"Causal genetic variation affecting complex diseases acts on a limited set of underlying biological mechanisms, and these mechanisms are detectable through patterns of physical protein-protein interaction."*

This hypothesis implies that if a GWAS identifies 30 independent genomic loci for a disease, the proteins encoded in those loci should interact with each other more than randomly selected proteins would — because they are wired into the same disease-relevant pathways.

### 1.2 The Biological Problem DAPPLE Solves

Post-GWAS analysis faces several key challenges that DAPPLE addresses:

| Challenge | DAPPLE's Solution |
|-----------|-------------------|
| Multiple genes per locus | Tests inter-locus (not intra-locus) connectivity to identify which gene is most likely causal |
| No functional annotation | Uses experimental PPI data as an unbiased molecular phenotype |
| Statistical inflation from hub proteins | Within-degree node-label permutation controls for highly-connected "hub" proteins |
| Interpreting genome-wide significant hits | Provides p-values for overall network connectivity AND per-gene candidate scores |

### 1.3 Primary Citation

> **Rossin EJ, Lage K, Raychaudhuri S, Xavier RJ, Tartar D, IIBDGC, Cotsapas C, Daly MJ.** (2011). "Proteins Encoded in Genomic Regions Associated with Immune-Mediated Disease Physically Interact and Suggest Underlying Biology." *PLoS Genetics* **7(1)**: e1001273. DOI: [10.1371/journal.pgen.1001273](https://doi.org/10.1371/journal.pgen.1001273)

> **Secondary Citation:** Lundby A & Rossin EJ et al. (2014). "Annotation of loci from genome-wide association studies using tissue-specific quantitative interaction proteomics." *Nature Methods* **11**: 868–874.

## Section 2: Algorithm & Methodology

### 2.1 Computational Workflow

```
INPUT: SNPs / Gene list / Genomic regions
         │
         ▼
[1. Seed Gene Definition]
  - SNP mode: Identify all genes within 'wingspan' kb of each SNP using HapMap/1KG LD data
  - Gene mode: Use provided genes with locus assignments
  - Region mode: Extract genes from genomic coordinates
         │
         ▼
[2. Network Construction using InWeb3 PPI Database]
  ┌──────────────────────────────────────┐
  │  Direct Network:                     │
  │  Seed Protein A ──────── Seed Protein B │
  │                                      │
  │  Indirect Network:                   │
  │  Seed Protein A ─[CI]─ Seed Protein B │
  │  (CI = Common Interactor)            │
  └──────────────────────────────────────┘
         │
         ▼
[3. Statistical Testing: Within-Degree Node-Label Permutation]
  - Shuffle protein labels while preserving each protein's degree (# of interactions)
  - Repeat N times (default: 1,000; literature studies use up to 10,000)
  - Build null distributions for each network statistic
         │
         ▼
[4. OUTPUT: p-values + Candidate Gene Rankings]
```

### 2.2 Network Statistics Computed

DAPPLE computes **four key network connectivity statistics** and their permutation-based p-values:

| Statistic | Description | Biological Interpretation |
|-----------|-------------|--------------------------|
| **Direct binding degree** | Total number of direct connections between seed proteins | How many seed protein pairs directly interact |
| **Avg. seed direct binding degree** | Mean connectivity of seed proteins in direct network | Average "hub-ness" of seed proteins toward each other |
| **Common interactor (CI) binding degree** | Number of seed proteins that common interactors connect | Depth of indirect connectivity |
| **Avg. CI binding degree** | Avg. number of seed proteins each CI connects to | Significance of bridging proteins |

### 2.3 The InWeb3 Interactome Database

| Property | Value |
|----------|-------|
| **Database** | InWeb3 (Lage et al., 2007) |
| **Total interactions** | 428,430 |
| **High-confidence, non-self interactions** | 169,810 |
| **Proteins covered** | 12,793 |
| **Organism** | Homo sapiens only |
| **Confidence filtering** | Bayesian integration of multiple PPI sources |

### 2.4 Permutation Strategy — Technical Detail

The **within-degree node-label permutation** is a critical methodological innovation:
- **Why it matters**: Highly connected hub proteins (e.g., TP53, BRCA1) would dominate networks by chance alone
- **How it works**: When shuffling protein labels, each protein is only swapped with another protein of the **same or similar interaction degree**
- **Alternative strategies tested** (original paper): SNP-matched permutation and edge permutation — both yielded consistent results, validating the primary approach
- **Caveat**: SNP permutation may not be robust in regions of extreme gene density

## Section 3: Input/Output Formats & Requirements

### 3.1 Input Modes — Three Supported Types

#### Mode 1: SNP List (Primary/Recommended Mode)
```
rs7903146
rs12255372
rs10811661
rs4506565
rs7901695
```
- **Format**: Plain text, one rsID per line
- **Requirement**: SNPs **must** exist in HapMap or 1000 Genomes Project databases
- **Why**: LD structure from these databases defines the "wingspan" gene inclusion region
- **Best practice**: Use the most significant SNP from each independent locus (tag SNPs)

#### Mode 2: Gene List (with Locus Assignment)
```
PTPN22 1
CTLA4 2
IL2RA 3
STAT4 4
CD247 5
TNFAIP3 6
```
- **Format**: Two-column, space or tab-delimited: `GENE_SYMBOL  LOCUS_ID`
- **Gene symbols**: HGNC/Hugo format
- **Locus ID**: Integer identifier grouping genes from the **same genomic region**
- ⚠️ **Critical**: DAPPLE tests connectivity **between** loci — genes from the same locus must share the same ID or results will be inflated

#### Mode 3: Genomic Regions
```
chr1:206,767,589-207,267,589
chr2:204,732,803-205,232,803
chr6:31,322,766-31,822,766
```
- **Format**: Chromosome:Start-End notation
- DAPPLE extracts all genes within each defined region

### 3.2 Output Files — Complete Specification

| File | Content | Use Case |
|------|---------|----------|
| `*_summary.txt` | Raw values for 4 network statistics (observed) | Quick inspection of network connectivity |
| `*_NetStats.txt` | **Permutation p-values** for all 4 network statistics | Primary result — is the overall network significant? |
| `*_SeedScores.txt` | Per-seed-protein p-values (individual connectivity scores) | **Candidate gene prioritization within loci** |
| `*_CIscores.txt` | P-values for common interactor proteins | Identify novel bridging proteins not in original GWAS |
| `*_directConnections.txt` | Edge list of direct seed-protein interactions | Network visualization / downstream analysis |
| `*_plot.pdf` | Network visualization (3-page PDF) | Publication figures / visual exploration |

**Plot Layout**:
- Page 1: Direct interaction network (seed–seed connections)
- Pages 2–3: Indirect interaction network (seed–CI–seed connections)

### 3.3 Output Interpretation Guide

```
Network is significantly connected if:
  NetStats p-value < 0.05 → Proteins share more interactions than expected by chance

To prioritize causal genes:
  SeedScores p-value < 0.05 → This gene's protein is significantly connected
  → Prefer the lowest p-value gene when multiple genes exist in a locus

Novel gene candidates:
  CIscores p-value < 0.05 → This interactor bridges multiple seed proteins
  → Common interactors not in GWAS hits may be novel disease genes
```

## Section 5: Scientific Applications & Validated Use Cases

### 5.1 Original Validation Studies (Rossin et al., 2011)

- **Rheumatoid Arthritis (RA)**: GWAS SNPs showed significant direct connectivity (p < 0.05) among encoded proteins vs. permuted null
- **Crohn's Disease (CD)**: Similarly significant PPI enrichment; cross-disease analysis revealed shared connectivity patterns between RA and CD proteins, suggesting overlapping immune pathways

### 5.2 Published Applications by Disease Area

| Disease Area | Key Finding | Citation |
|-------------|-------------|----------|
| **Multiple Sclerosis** | 67 GWAS SNPs → 75 proteins, 104 direct interactions (expected=61, p<0.0002) | *PLoS One* 2012 |
| **Schizophrenia** | Used alongside DEPICT, NetWAS, Prix Fixe for multi-method gene prioritization | *Translational Psychiatry* 2018 |
| **Juvenile Idiopathic Arthritis** | 348 JIA genes → highly connected interactome; 10,000 permutations | *Arthritis Res Therapy* 2014 |
| **Idiopathic Inflammatory Myopathies** | Systematic PPI + pathway analysis of disease-associated loci | *Arthritis Res Therapy* 2016 |
| **Schizophrenia (network analysis)** | Module gene connectivity evaluation in PPI context | *PLoS One* 2015 |
| **Psychiatric–Autoimmune comorbidity** | Cross-disorder PPI subnetwork construction | *Multiple journals* |
| **Blood pressure / Hypertension** | PPI network integration of GWAS findings | *Human Genomics* 2023 |

### 5.3 Typical Research Workflow

```
Stage 1: GWAS Analysis
  → Generate list of genome-wide significant SNPs (p < 5×10⁻⁸)
  → Perform LD-based clumping to identify independent loci
  
Stage 2: DAPPLE Analysis
  → Input: Independent tag SNPs (20+ loci recommended)
  → Configure wingspan based on LD structure of disease
  → Run 1,000–10,000 permutations
  
Stage 3: Results Interpretation
  → Assess overall network significance (NetStats p-values)
  → Rank seed proteins by SeedScores for candidate prioritization
  → Review CIscores for novel interacting candidates
  
Stage 4: Follow-up
  → Functional validation of top candidate genes
  → Overlap with eQTL, chromatin accessibility data
  → Cross-disease connectivity comparison
```

### 6.2 DAPPLE's Unique Strengths

1. **Purpose-built for GWAS loci**: Unlike general network tools, DAPPLE's statistical framework is specifically designed for the "multiple loci → one disease" topology
2. **Within-degree permutation**: Methodologically rigorous control for hub protein bias — more conservative and biologically sound than simple random permutation
3. **Two-level network analysis**: Both direct and indirect (common interactor) networks capture different biological signals
4. **Per-gene candidate scoring**: Directly actionable output for follow-up experiments
5. **GenePattern integration**: Accessible to non-computational biologists through a GUI interface

### 6.3 Key Limitations to Document

| Limitation | Impact | Mitigation |
|-----------|--------|------------|
| InWeb3 database (2007 vintage) | Missing ~15 years of new PPI discoveries | Acknowledge in results; consider supplementing with STRING |
| Human genome only | Cannot be used for model organism studies | Clearly document scope in module description |
| Binary input only | Cannot weight seed genes by GWAS p-value | Combine with p-value-based methods (e.g., DEPICT) for richer analysis |
| HapMap/1KG SNP requirement | Some newer or non-common variants not supported | Recommend using tag SNPs from standard GWAS catalogs |
| Minimum locus requirement | Underpowered with <10–15 loci | Warn users when input size is low |
| Broad server dependency | Module requires active Broad API; subject to downtime | Monitor API health; document fallback |

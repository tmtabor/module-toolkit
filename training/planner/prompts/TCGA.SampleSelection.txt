# 🧬 Comprehensive Research Report: TCGA.SampleSelection (v0.14)

---

## 1. Executive Summary

**TCGA.SampleSelection** is a GenePattern wrapper module developed by the Broad Institute that automates the complete data pipeline from TCGA data retrieval to GSEA-ready output generation. It queries the Broad FireBrowse (Firehose) REST API to retrieve RNA-seq expression data for any TCGA cancer cohort, stratifies samples by expression level of user-specified genes of interest, and produces GCT + CLS file pairs suitable for immediate use with GSEA, ssGSEA, and GSEAPreranked GenePattern modules. Version v0.14's principal change was updating the module to correctly handle non-float MSigDB version identifiers (e.g., `2022.1.Hs`, `2023.1.Hs`) that replaced the legacy floating-point versioning scheme.

---

## 2. Tool Purpose & Scientific Applications

### 2.1 Core Scientific Purpose

The module solves a critical and recurring bottleneck in cancer genomics research: **connecting large public TCGA datasets to gene pathway enrichment analysis**. The manual workflow — downloading raw TCGA data, parsing it, filtering by expression, building phenotype label files, and formatting for GSEA — is time-consuming, error-prone, and requires programming expertise. TCGA.SampleSelection eliminates all of these steps in a single GenePattern job.

### 2.2 Primary Research Questions It Addresses

| Research Question | How the Module Helps |
|---|---|
| What pathways are activated in tumors with high expression of oncogene X? | Stratifies TCGA samples by oncogene expression → GSEA reveals enriched pathways |
| Does suppressor gene Y loss associate with immune evasion signatures? | Selects low-expression samples → GSEA tests immune gene sets |
| What gene sets distinguish EGFR-high from EGFR-low lung adenocarcinomas? | Queries LUAD cohort, stratifies by EGFR, outputs GSEA-ready files |
| Is PIK3CA expression associated with hallmark gene set enrichment in breast cancer? | BRCA cohort + PIK3CA stratification → GSEA Hallmarks analysis |

### 2.3 Scientific Applications

- **Oncogene/tumor suppressor pathway analysis**: Understand downstream pathway consequences of high/low expression of cancer driver genes
- **Immune landscape characterization**: Correlate immune gene set enrichment with expression of checkpoint or immune-modulator genes
- **Cancer subtype analysis**: Identify molecular pathway differences between expression-defined subtypes within a cohort
- **Biomarker hypothesis generation**: Generate pathway enrichment evidence for novel candidate biomarkers across any of the 33 TCGA cohorts
- **Cross-cohort studies**: Replicate findings by running the same gene selection strategy across different cancer types

## 3. Data Source: FireBrowse (Broad GDAC Firehose)

### 3.1 FireBrowse API Overview

FireBrowse (`http://firebrowse.org`) exposes Broad GDAC (Genome Data Analysis Center) processed TCGA Level 3 data through a RESTful API. The critical endpoint for this module is:

```
GET http://firebrowse.org/api/v1/Samples/mRNASeq
```

**Key query parameters:**
| Parameter | Description | Example |
|---|---|---|
| `gene` | Hugo gene symbol(s) (**required**) | `EGFR`, `BRCA1,TP53` |
| `cohort` | TCGA cancer type code | `BRCA`, `LUAD`, `GBM` |
| `tcga_participant_barcode` | Specific patient barcode filter | `TCGA-XX-XXXX` |
| `sample_type` | Tumor/normal sample type | `TP` (Primary Tumor) |
| `protocol` | Normalization method | `RSEM` |
| `page` | Results page index | `1`, `2`, ... |
| `page_size` | Records per page (max 2000) | `250` (default) |
| `sort_by` | Sort column | `cohort`, `gene` |

**Response JSON fields per record:**
```json
{
  "cohort": "BRCA",
  "expression_log2": 7.062,
  "gene": "EGFR",
  "geneID": 1956,
  "protocol": "RSEM",
  "sample_type": "TP",
  "tcga_participant_barcode": "TCGA-AC-A7VB",
  "z-score": -0.599
}
```

### 3.2 Data Normalization

- **RSEM** (RNA-Seq by Expectation Maximization) is the primary quantification method
- Expression values are **log2-transformed**
- Upper quartile normalization sets the 75th percentile count to **1,000 (gene level)** or **300 (isoform level)**
- Z-scores are computed within-cohort, per gene, across all samples
- Data represents TCGA **Level 3** (processed, analysis-ready) mRNASeq data from the Broad Firehose pipeline

### 3.3 Important Caveat — FireBrowse Data Currency

> ⚠️ **FireBrowse hosts legacy TCGA data aligned to GRCh37 (hg19).** As of 2016, the Broad GDAC Firehose pipeline was frozen. Current harmonized GRCh38 TCGA data is served by the **NCI Genomic Data Commons (GDC)**. FireBrowse data remains scientifically valid and widely cited, but is not actively updated. For the most current TCGA data, the companion module **TCGAImporter** accesses the live GDC portal.

## 4. Input/Output Formats

### 4.1 Module Inputs (Parameters Only — No File Uploads Required)

The module takes no input files; all data is retrieved programmatically via the FireBrowse API. Users specify the following through the GenePattern UI:

| Input Parameter | Type | Description |
|---|---|---|
| `cancer_type` / `cohort` | String (CHOICE) | TCGA cohort code (e.g., `BRCA`, `LUAD`, `GBM`) |
| `gene_of_interest` | String (TEXT) | Hugo gene symbol(s) for stratification (e.g., `EGFR`) |
| `sample_type` | String (CHOICE) | `TP` (Primary Tumor), `NT` (Normal Tissue), etc. |
| `top_percentile` | Integer/Float | Expression cutoff for "High" group (e.g., 25 = top 25%) |
| `bottom_percentile` | Integer/Float | Expression cutoff for "Low" group (e.g., 25 = bottom 25%) |
| `msigdb_version` | String | MSigDB version for downstream GSEA compatibility |
| `high_label` | String | Label for high-expression phenotype (default: "High") |
| `low_label` | String | Label for low-expression phenotype (default: "Low") |
| `protocol` | String (CHOICE) | `RSEM` (default and primary supported option) |

### 4.2 Outputs

The module generates two primary output files designed as **direct inputs to GSEA/ssGSEA**:

#### Output 1: Gene Expression Matrix (GCT format)
```
#1.2
{nGenes}\t{nSamples}
Name\tDescription\tTCGA-XX-XXXX-01\tTCGA-YY-YYYY-01\t...
BRCA1\tBRCA1\t7.23\t4.11\t...
TP53\tTP53\t5.88\t6.44\t...
...
```
- Rows = all genes with expression data; Columns = selected samples
- Expression values are log2 RSEM values from FireBrowse
- Samples include BOTH High and Low expression groups
- Column order typically: all "High" samples first, then all "Low" samples

#### Output 2: Phenotype Label File (CLS format)
```
{nSamples}\t2\t1
# High\tLow
High\tHigh\tHigh\t...\tLow\tLow\tLow\t...
```
- Line 1: total samples, number of classes, `1`
- Line 2: `#` followed by class names (High, Low)
- Line 3: class label for each sample in same order as GCT columns

#### Possible Additional Outputs
- **Log/report file**: Summary of samples selected, expression thresholds applied
- **Sample manifest**: List of selected TCGA barcodes with expression values and group assignments

## 7. Version-Specific Change: Non-Float MSigDB Version Numbers

### 7.1 The Problem (Pre-v0.14)

MSigDB historically used **floating-point version numbers**: `6.0`, `6.1`, `6.2`, `7.0`, `7.1`, `7.2`, `7.3`, `7.4`, `7.5`. Module code that parsed or compared version numbers treating them as floats (e.g., `float("7.5") >= 7.0`) worked correctly.

### 7.2 The Breaking Change

Beginning with the 2022 release, MSigDB adopted a **date-based naming convention** with species suffixes:

| Version String | Release Date | Species |
|---|---|---|
| `2022.1.Hs` | 2022 | Human |
| `2023.1.Hs` | March 2023 | Human |
| `2023.2.Hs` | October 2023 | Human |
| `2024.1.Hs` | August 2024 | Human |
| `2023.1.Mm` | 2023 | Mouse |
| `2023.2.Mm` | 2023 | Mouse |

Attempting `float("2022.1.Hs")` raises a `ValueError` in Python. Similarly, string comparisons treating "2022.1.Hs" as a number fail.

### 7.3 The Fix in v0.14

The v0.14 update modified all version-handling logic to:
1. Treat MSigDB version as a **string** rather than numeric type
2. Handle the dropdown/choice list of available MSigDB versions properly in the module's manifest or wrapper code
3. Ensure URLs, file path construction, and API calls using the version string work correctly with the new naming format

### 7.4 MSigDB Collections Available (Current: 2024.1.Hs)

| Collection | # Gene Sets | Description |
|---|---|---|
| H (Hallmarks) | 50 | Oncogenic processes |
| C1 | ~300 | Positional (chromosomal bands) |
| C2 (curated) | ~6,000 | Canonical pathways (KEGG, Reactome, WikiPathways) |
| C3 (motif) | ~3,000 | Regulatory target gene sets |
| C4 (computational) | ~858 | Cancer-related gene signatures |
| C5 (ontology) | ~15,000 | GO terms (BP, MF, CC) |
| C6 (oncogenic) | ~189 | Oncogenic signatures |

```
    ┌──────────────────┐   ┌──────────────────────┐

    │  GenePattern     │   │   GenePattern         │
    │                  │   │                       │
    │ .gct + .cls +    │   │ .gct + gene set DB    │

```

### 9.2 Representative Use Cases

**Use Case 1 — Oncogene Pathway Analysis**
> *"What pathways are enriched in BRCA tumors with high EGFR expression?"*
- `cohort=BRCA`, `gene=EGFR`, `top_percentile=25`, `bottom_percentile=25`, `sample_type=TP`
- → GSEA using Hallmarks/C2 gene sets

**Use Case 2 — Tumor Suppressor Loss Characterization**
> *"What gene sets associate with PTEN loss in GBM?"*
- `cohort=GBM`, `gene=PTEN`, `top_percentile=25` (PTEN-high = WT), `bottom_percentile=25` (PTEN-low = loss)
- → GSEA using C6 oncogenic signatures

**Use Case 3 — Immune Evasion Profiling**
> *"Does PD-L1 (CD274) expression correlate with immunosuppressive gene sets in SKCM?"*
- `cohort=SKCM`, `gene=CD274`, `top_percentile=25`, `bottom_percentile=25`
- → GSEA using C7 immunologic gene sets

**Use Case 4 — Cross-Cohort Replication**
> *"Does PIK3CA-high expression enrich PI3K pathway gene sets across multiple cancers?"*
- Run independently for `cohort=BRCA`, `cohort=UCEC`, `cohort=HNSC` with `gene=PIK3CA`
- → Compare GSEA enrichment results across cancer types

**Use Case 5 — Tertile-Based Deep Stratification**
> *"Exclude the middle group to maximize signal in KRAS-high vs KRAS-low LUAD analysis"*
- `cohort=LUAD`, `gene=KRAS`, `top_percentile=33`, `bottom_percentile=33`
- → Strong contrast between extreme expression groups for GSEA

## 13. Known Limitations & Caveats

| Limitation | Impact | Mitigation |
|---|---|---|
| **FireBrowse data is frozen (2016)** | Uses GRCh37-aligned Firehose data, not latest GDC harmonized data | Acknowledge in docs; use TCGAImporter for GRCh38 |
| **Single-gene stratification** | Stratification by a single gene may not capture biological complexity | Users can apply external multi-gene scores before running |
| **Extreme percentile exclusion** | Middle-expression samples are discarded, reducing cohort size | Documented as intentional to maximize signal contrast |
| **FireBrowse API rate limits** | Large cohort queries may be slow or throttled | Implement retry logic and pagination |
| **RSEM-only normalization** | TPM or raw count alternatives not available via this module | Document clearly; use TCGAbiolinks for TPM data |
| **FFPE samples excluded** | Firehose excludes FFPE samples from standard analyses | Consistent with standard practice; note in documentation |
| **No clinical covariate adjustment** | No correction for age, stage, or other confounders | Users must consider this in biological interpretation |
| **Legacy species support** | Human TCGA only (no mouse) | By design; scope limited to human TCGA |

## 14. Literature Context & Scientific Validation

### 14.1 Methodological Foundation
- The expression-based stratification strategy (high vs. low expression → GSEA) is a **validated and widely published approach** with thousands of citations using TCGA data
- GSEA methodology (Subramanian et al., 2005, *PNAS*) is one of the most cited bioinformatics papers (~35,000 citations)
- The 25th/75th percentile stratification strategy is the **community-standard approach** for maximizing contrast in TCGA high-vs-low GSEA studies (Biostars community consensus)

### 14.2 Community Adoption
- The GenePattern platform has been used in hundreds of published studies including TCGA-related analyses
- Breast cancer TCGA GSEA workflows using GenePattern were documented in peer-reviewed literature (e.g., *Cell Genomics*, 2021)
- The specific FireBrowse API query approach is used by tools including `fbget` (Broad), `FirebrowseR` (R package), and multiple published bioinformatics pipelines

### 14.3 Data Quality
- FireBrowse/Firehose TCGA data has been extensively validated and is the gold-standard legacy processed TCGA dataset
- RSEM + upper quartile normalization is appropriate for cross-sample gene expression comparisons
- Log2 transformation is standard for downstream GSEA analysis
